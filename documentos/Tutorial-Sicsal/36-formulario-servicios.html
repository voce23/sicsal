1 Estructura de la clase y arrays estáticos

FormularioServicios es la página más grande del proyecto. Maneja 8 secciones del formulario 301 del SNIS usando propiedades públicas de tipo array para cada sección. Las constantes estáticas definen los "diccionarios" de tipos con sus labels.

¿Por qué arrays estáticos públicos?

Declararlos como public static permite accederlos desde la vista Blade con \App\Filament\Pages\FormularioServicios::$tiposReferencia. Así la vista puede iterar los mismos diccionarios sin duplicar las listas. Es el patrón "única fuente de verdad" (single source of truth).

2 cargarDatos() — keyBy() para 7 modelos

Al montar la página, se cargan los datos existentes de las 7 tablas prest_* + observaciones + causas. El patrón clave es ->get()->keyBy('campo') que convierte la colección en un diccionario indexado.

FormularioServicios.php — mount() + cargarDatos()
public function mount(): void
{
    $this->mountFormulario();  // FormularioMensualTrait — mes, anio, mesCerrado
    $this->cargarDatos(auth()->user()->centro_salud_id);
}

private function cargarDatos(int $centroId): void
{
    // ─── Consulta Externa: 10 grupos × 6 campos ───
    $gruposConsulta = ['menor_6m', '6m_menor_1', '1_4', '5_9',
        '10_14', '15_19', '20_39', '40_49', '50_59', 'mayor_60'];

    $rowsConsulta = PrestConsultaExterna::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)->where('anio', $this->anio)
        ->get()->keyBy('grupo_etareo');

    foreach ($gruposConsulta as $grupo) {
        $row = $rowsConsulta->get($grupo);
        $this->consultaExterna[$grupo] = [
            'primera_m'  => $row->primera_m  ?? 0,
            'primera_f'  => $row->primera_f  ?? 0,
            'nueva_m'    => $row->nueva_m    ?? 0,
            'nueva_f'    => $row->nueva_f    ?? 0,
            'repetida_m' => $row->repetida_m ?? 0,
            'repetida_f' => $row->repetida_f ?? 0,
        ];
    }

    // ─── Referencias: 7 tipos × (M, F) ───
    $rowsRef = PrestReferencia::where(...)->get()->keyBy('tipo');
    foreach (array_keys(self::$tiposReferencia) as $tipo) {
        $row = $rowsRef->get($tipo);
        $this->referencias[$tipo] = [
            'masculino' => $row->masculino ?? 0,
            'femenino'  => $row->femenino  ?? 0,
        ];
    }

    // ─── Odontología: 13 proc × 5 grupos × (M, F) ───
    $rowsOdonto = PrestOdontologia::where(...)
        ->get()->keyBy(fn($r) => "{$r->procedimiento}__{$r->grupo_etareo}");
    foreach (array_keys(self::$procedimientosOdonto) as $proc) {
        foreach (array_keys(self::$gruposOdonto) as $ge) {
            $key = "{$proc}__{$ge}";
            $row = $rowsOdonto->get($key);
            $this->odontologia[$key] = [
                'masculino' => $row->masculino ?? 0,
                'femenino'  => $row->femenino  ?? 0,
            ];
        }
    }
keyBy() con closure para claves compuestas

Odontología usa keyBy(fn($r) => "{$r->procedimiento}__{$r->grupo_etareo}") para crear claves como primera_consulta__menor_5. Al separar con __ (doble guion bajo), evitamos colisiones con nombres que contengan un solo _. El mismo patrón de clave se usa en wire:model="odontologia.primera_consulta__menor_5.masculino".

FormularioServicios.php — cargar Causas (10 posiciones × 10 grupos)
// Causas de Consulta Externa
$grupos = array_keys(CausaConsultaExterna::$grupos);
$rowsCausas = CausaConsultaExterna::where(...)
    ->get()->groupBy('posicion');

for ($pos = 1; $pos <= 10; $pos++) {
    $registros = $rowsCausas->get($pos, collect());
    $diagnostico = $registros->first()?->diagnostico ?? '';

    $grupoData = [];
    foreach ($grupos as $g) {
        $reg = $registros->firstWhere('grupo_etareo', $g);
        $grupoData[$g] = [
            'm' => $reg->masculino ?? 0,
            'f' => $reg->femenino  ?? 0,
        ];
    }
    $this->causasConsulta[$pos] = [
        'diagnostico' => $diagnostico,
        'grupos'      => $grupoData,
    ];
}
groupBy() vs keyBy()

keyBy('campo') produce un diccionario con UN registro por clave. groupBy('posicion') produce un diccionario donde cada clave tiene MÚLTIPLES registros (una colección). Causas usa groupBy porque cada posición tiene 10 registros (uno por grupo etáreo).

3 Los 8 métodos guardarX() — updateOrCreate

Cada sección tiene su propio botón "Guardar" que llama un método dedicado. Todos siguen el mismo patrón: verificar $mesCerrado, iterar el array público, y hacer updateOrCreate().

FormularioServicios.php — guardarConsultaExterna()
public function guardarConsultaExterna(): void
{
    if ($this->mesCerrado) return;  // Guard clause
    $centroId = auth()->user()->centro_salud_id;

    foreach ($this->consultaExterna as $grupo => $datos) {
        PrestConsultaExterna::updateOrCreate(
            // Clave compuesta — busca por estos 4 campos
            ['centro_salud_id' => $centroId,
             'mes' => $this->mes,
             'anio' => $this->anio,
             'grupo_etareo' => $grupo],
            // Valores a guardar/actualizar
            array_map('intval', $datos)
        );
    }

    Notification::make()->title('Consulta externa guardada')
        ->success()->send();
}
FormularioServicios.php — guardarOdontologia() (clave con 5 campos)
public function guardarOdontologia(): void
{
    if ($this->mesCerrado) return;
    $centroId = auth()->user()->centro_salud_id;

    foreach (array_keys(self::$procedimientosOdonto) as $proc) {
        foreach (array_keys(self::$gruposOdonto) as $ge) {
            $key = "{$proc}__{$ge}";
            $datos = $this->odontologia[$key] ?? [];

            PrestOdontologia::updateOrCreate(
                ['centro_salud_id' => $centroId,
                 'mes' => $this->mes,
                 'anio' => $this->anio,
                 'procedimiento' => $proc,
                 'grupo_etareo'  => $ge],
                ['masculino' => (int)($datos['masculino'] ?? 0),
                 'femenino'  => (int)($datos['femenino']  ?? 0)]
            );
        }
    }
}
array_map('intval', $datos) — sanitización simple

Los inputs HTML siempre envían strings. array_map('intval', $datos) convierte todos los valores a enteros de golpe. Para odontología, se usa (int)($datos['masculino'] ?? 0) porque hay que acceder a claves específicas dentro del array.

FormularioServicios.php — guardarCausasConsulta() (lógica de borrado)
public function guardarCausasConsulta(): void
{
    for ($pos = 1; $pos <= 10; $pos++) {
        $causa = $this->causasConsulta[$pos] ?? [];
        $diagnostico = trim($causa['diagnostico'] ?? '');

        foreach ($grupos as $g) {
            $m = (int)($causa['grupos'][$g]['m'] ?? 0);
            $f = (int)($causa['grupos'][$g]['f'] ?? 0);

            if ($diagnostico === '' && $m === 0 && $f === 0) {
                // Eliminar registros vacíos — no acumular ceros
                CausaConsultaExterna::where(...)->delete();
                continue;
            }

            CausaConsultaExterna::updateOrCreate(
                ['centro_salud_id' => $centroId, 'mes' => $this->mes,
                 'anio' => $this->anio, 'posicion' => $pos, 'grupo_etareo' => $g],
                ['diagnostico' => $diagnostico ?: "Causa $pos",
                 'masculino' => $m, 'femenino' => $f]
            );
        }
    }
}
Borrado condicional en Causas

Si el diagnóstico está vacío Y M=0 Y F=0, se eliminan esos registros. Esto evita acumular filas de ceros en la base. Si el usuario borra un diagnóstico, los registros se limpian automáticamente. Nota: si hay valores numéricos pero falta el diagnóstico, se usa un fallback "Causa $pos".

    </div>

  </div>
</x-filament-panels::page>
¿Por qué Alpine.js y no Livewire tabs?

Cambiar de tab con Alpine (x-data, x-show, x-on:click) es instantáneo — sin request al servidor. Los 8 tabs se renderizan en el HTML inicial pero se ocultan con CSS. Livewire solo interviene al hacer clic en "Guardar". Esto da una experiencia de usuario rápida incluso con las 8 secciones cargadas.

wire:model con arrays anidados

wire:model="consultaExterna.menor_6m.primera_m" enlaza directamente al valor $this->consultaExterna['menor_6m']['primera_m'] en PHP. Livewire 4 maneja la notación punto automáticamente para arrays anidados. El resultado: 10 grupos × 6 campos = 60 inputs en esta sola tabla, todos sincronizados.

10 × 10 × 2 + 10 = 210 inputs + 10 textos

Esta es la tabla más grande del sistema: 10 diagnósticos × 10 grupos × 2 sexos = 200 inputs numéricos + 10 campos de texto. Con los totales (3 columnas calculadas × 10 filas), la tabla tiene 23 columnas. El overflow-x-auto permite scroll horizontal en pantallas pequeñas. Los totales se calculan con collect($causa['grupos'])->sum('m').

9 Resumen

Lo que aprendimos en esta lección

  • Arrays estáticos públicos como diccionarios compartidos entre PHP y Blade
  • keyBy('campo') para indexar colecciones, y keyBy(fn()) para claves compuestas
  • groupBy() vs keyBy() — uno→muchos vs uno→uno
  • 8 tabs con Alpine.js (x-data, x-show, x-on:click, x-cloak) — sin roundtrip al servidor
  • wire:model con arrays anidados — notación punto hasta 3-4 niveles
  • updateOrCreate() con claves compuestas de 4-5 campos
  • array_map('intval', $datos) para sanitizar todos los valores
  • Borrado condicional de registros vacíos en causas de consulta
  • {{ $mesCerrado ? 'disabled' : '' }} para read-only cuando el mes está cerrado
  • @unless($mesCerrado) para ocultar botones en modo cerrado
  • collect()->sum('m') para totales calculados en Blade
