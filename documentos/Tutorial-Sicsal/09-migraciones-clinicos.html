1 Panorama: datos individuales vs. agregados

SICSAL tiene dos capas de datos:

Capa Tablas Propósito
Individual (esta lección) vacunas_ninos, embarazos, partos, etc. Registro detallado por persona, fecha, tipo
Agregado (lecciones 10-11) prest_vacunas, prest_partos, etc. Totales mensuales por centro para informes SNIS

Todas las tablas de esta lección siguen un patrón: tienen persona_id (o embarazo_id) como FK, campos clínicos específicos, y columnas de auditoría (registrado_por, mes, anio).

💡
Patrón recurrente: mes + anio
Muchas tablas incluyen mes (tinyInteger) y anio (year). Esto facilita las consultas de reportes mensuales sin tener que extraer mes/año de la fecha con funciones SQL.

2 vacunas_ninos — Inmunización infantil

Cada fila es una dosis aplicada a un niño. El enum tipo_vacuna contiene las 31 vacunas del esquema PAI boliviano.

database/migrations/XXXX_create_vacunas_ninos_table.php php 
Schema::create('vacunas_ninos', function (Blueprint $table) {
    $table->id();
    $table->foreignId('persona_id')->constrained('personas');
    $table->enum('tipo_vacuna', [
        'BCG', 'HepB_neonatal',
        'Pentavalente_1', 'Pentavalente_2', 'Pentavalente_3',
        'Pentavalente_4_refuerzo', 'Pentavalente_5_refuerzo',
        'IPV_1', 'bOPV_2', 'IPV_3', 'bOPV_4', 'bOPV_5',
        'Antirotavirica_1', 'Antirotavirica_2',
        'Antineumococica_1', 'Antineumococica_2', 'Antineumococica_3',
        'Influenza_1', 'Influenza_2', 'Influenza_unica',
        'SRP_1', 'SRP_2', 'Antiamarilica',
        'VPH_1', 'VPH_2',
        'dT_1', 'dT_2', 'dT_3', 'dT_4', 'dT_5', 'SR',
    ]);
    $table->date('fecha_aplicacion');
    $table->enum('dentro_fuera', ['dentro', 'fuera'])
          ->default('dentro');      // dentro/fuera del área
    $table->string('lote', 50)->nullable();
    $table->foreignId('aplicado_por')->nullable()->constrained('users');
    $table->tinyInteger('mes');
    $table->year('anio');
    $table->timestamps();
});
ℹ️
dentro / fuera
dentro = persona que vive en el área del centro. fuera = persona de otra área que fue atendida aquí. Esta distinción es importante para los informes SNIS.

3 crecimiento_infantil — Peso, talla, estado nutricional

Registra cada consulta de control de crecimiento de menores de 5 años.

database/migrations/XXXX_create_crecimiento_infantil_table.php php 
Schema::create('crecimiento_infantil', function (Blueprint $table) {
    $table->id();
    $table->foreignId('persona_id')->constrained('personas');
    $table->date('fecha');
    $table->decimal('peso_kg', 5, 2)->nullable();      // ej: 12.50
    $table->decimal('talla_cm', 5, 2)->nullable();     // ej: 85.30
    $table->decimal('perimetro_cefalico_cm', 5, 2)->nullable();
    $table->enum('clasificacion', [
        'normal', 'desnutricion_aguda', 'desnutricion_cronica',
        'desnutricion_global', 'sobrepeso', 'obesidad',
    ]);
    $table->enum('tipo_control', ['nuevo', 'repetido'])
          ->default('nuevo');
    $table->enum('dentro_fuera', ['dentro', 'fuera'])
          ->default('dentro');
    $table->tinyInteger('mes');
    $table->year('anio');
    $table->foreignId('registrado_por')->nullable()->constrained('users');
    $table->timestamps();
});

La clasificacion se determina según las curvas OMS de peso/talla que calcula un helper (lo veremos en lecciones futuras).

5 embarazos — Registro de gestación

Cada embarazo vinculado a una persona. De un embarazo se derivan los controles prenatales, el parto y el puerperio.

database/migrations/XXXX_create_embarazos_table.php php 
Schema::create('embarazos', function (Blueprint $table) {
    $table->id();
    $table->foreignId('persona_id')->constrained('personas');
    $table->date('fecha_inicio')->nullable();
    $table->date('fecha_probable_parto')->nullable();
    $table->tinyInteger('semanas_gestacion_ingreso')->nullable();
    $table->enum('estado', [
        'activa', 'migrada_temporal',
        'atendida_otro_centro', 'culminada', 'perdida',
    ])->default('activa');
    $table->text('observaciones')->nullable();
    $table->boolean('activo')->default(true);
    $table->timestamps();
});
ℹ️
Estados de un embarazo
  • activa — Embarazo en curso
  • migrada_temporal — La mujer viajó a otro lugar
  • atendida_otro_centro — Se atiende en otro centro
  • culminada — El parto ocurrió
  • perdida — Aborto o pérdida

6 controles_prenatales — Seguimiento prenatal

Cada visita prenatal se registra como un control vinculado a un embarazo.

database/migrations/XXXX_create_controles_prenatales_table.php php 
Schema::create('controles_prenatales', function (Blueprint $table) {
    $table->id();
    $table->foreignId('embarazo_id')->constrained('embarazos');
    $table->tinyInteger('numero_control');       // 1, 2, 3, 4…
    $table->date('fecha');
    $table->tinyInteger('semanas_gestacion')->nullable();
    $table->enum('dentro_fuera', ['dentro', 'fuera'])
          ->default('dentro');
    $table->string('grupo_etareo', 20)->nullable();
    $table->foreignId('registrado_por')->nullable()
          ->constrained('users');
    $table->timestamps();
});

Nota: esta tabla apunta a embarazo_id, no a persona_id. A la persona se accede a través del embarazo: control → embarazo → persona.

7 partos — Atención del parto

database/migrations/XXXX_create_partos_table.php php 
Schema::create('partos', function (Blueprint $table) {
    $table->id();
    $table->foreignId('embarazo_id')->constrained('embarazos');
    $table->date('fecha_parto');
    $table->enum('tipo', ['vaginal', 'cesarea']);
    $table->enum('lugar', ['servicio', 'domicilio']);
    $table->enum('atendido_por', [
        'personal_calificado', 'partera_empirica',
        'partera_capacitada', 'articulacion', 'otros',
    ]);
    $table->string('grupo_etareo', 20)->nullable();
    $table->enum('resultado', ['nacido_vivo', 'nacido_muerto'])
          ->default('nacido_vivo');
    $table->decimal('peso_rn_kg', 4, 3)->nullable();  // ej: 3.250
    $table->foreignId('registrado_por')->nullable()
          ->constrained('users');
    $table->timestamps();
});
⚠️
peso_rn_kg: decimal(4, 3)
En el peso del recién nacido, decimal(4, 3) permite valores como 3.250 (1 dígito entero + 3 decimales). Máximo: 9.999 kg.

8 puerperios — Controles posparto

Registra las 4 visitas de control posparto: a las 48 horas, 7 días, 28 días y 42 días después del parto.

database/migrations/XXXX_create_puerperios_table.php php 
Schema::create('puerperios', function (Blueprint $table) {
    $table->id();
    $table->foreignId('parto_id')->constrained('partos');
    $table->date('control_48h')->nullable();    // 48 horas
    $table->date('control_7d')->nullable();     // 7 días
    $table->date('control_28d')->nullable();    // 28 días
    $table->date('control_42d')->nullable();    // 42 días
    $table->timestamps();
});

Es la migración más simple: solo 4 columnas de fecha nullable. Las columnas son null cuando el control no se ha realizado aún. La cadena es: puerperio → parto → embarazo → persona.
