2 · Diseño para DomPDF: reglas de estilo

DomPDF no soporta Tailwind CSS ni CSS moderno. Las vistas PDF son HTML autónomo con estilos inline:

<!-- Cada PDF es un documento HTML completo, no usa layouts -->
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <style>
        /* Fuente DejaVu Sans — la única que DomPDF renderiza bien */
        body { font-family: 'DejaVu Sans', sans-serif; }

        /* Tamaño de fuente MUY pequeño (7-9px) — el PDF es denso */
        body { font-size: 9px; }
        th, td { font-size: 8px; }

        /* Saltos de página controlados con CSS */
        .page-break { page-break-before: always; }

        /* Colores semáforo para indicadores */
        .green  { color: #059669; }
        .yellow { color: #d97706; }
        .red    { color: #dc2626; }
    </style>
</head>
Restricciones de DomPDF:
  • No soporta flexbox, grid ni calc().
  • Solo <table> para layout columnar.
  • Sin JavaScript — los gráficos Chart.js no funcionan en PDF.
  • Fuente recomendada: DejaVu Sans (incluida con DomPDF, soporta UTF-8).

6 · Cómo se generan los PDFs (Filament + Livewire)

Los PDFs se generan desde dos puntos del sistema:

// 1. Desde Filament → Página Reportes (admin)
use Barryvdh\DomPDF\Facade\Pdf;

$pdf = Pdf::loadView('pdf.informe-cai', ['datos' => $datos])
    ->setPaper('letter', 'landscape');

return response()
    ->streamDownload(
        fn() => print($pdf->output()),
        "informe-cai-{$centro}.pdf"
    );

// 2. Desde Livewire → Componentes públicos
// InformeCAI.php, ComunidadesPoblacion.php
$pdf = Pdf::loadView('pdf.comunidades-poblacion', ['datos' => $datos])
    ->setPaper('letter', 'landscape');

return response()->streamDownload(...);
Orientación landscape — Los tres PDFs usan setPaper('letter', 'landscape') porque tienen muchas columnas (24+ en causas de consulta).

8 · Mapa interactivo con Leaflet.js

El mapa muestra los centros de salud con coordenadas GPS. Leaflet se integra con un selector para centrar la vista:

Tailwind v4 + Leaflet — El preflight de Tailwind v4 aplica img { max-width: 100% } que rompe los tiles del mapa. El layout public.blade.php incluye un <style> con .leaflet-container img { max-width: none !important } para corregirlo. El contenedor del mapa usa lg:h-[460px] (valor explícito) en lugar de lg:h-115 para máxima compatibilidad. 
// Datos de centros pasados desde el controlador
const centrosMapas = @json($centrosMapas);

const map = L.map('mapa').setView(centroInicial, 14);

L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
    attribution: '&copy; OpenStreetMap',
    maxZoom: 19,
}).addTo(map);

// Marcadores con ícono circular personalizado
centrosMapas.forEach(function(c) {
    const marker = L.marker([c.lat, c.lng], {
        icon: L.divIcon({
            html: '<div style="background:#14b8a6;border-radius:50%..."></div>',
        }),
    }).addTo(map);

    marker.bindPopup('<strong>' + c.nombre + '</strong><br>SNIS: ' + c.snis);
    marker.bindTooltip(c.nombre);
});

// Ajustar zoom para mostrar todos los centros
if (centrosMapas.length > 1) {
    const bounds = L.latLngBounds(centrosMapas.map(c => [c.lat, c.lng]));
    map.fitBounds(bounds, { padding: [40, 40] });
}
Selector de centro — Un <select> permite ver un centro específico. La función centrarMapa(lat, lng, nombre) anima el mapa al centro seleccionado y abre el popup correspondiente.

9 · Pirámides poblacionales con Chart.js

Cada centro muestra 2 pirámides lado a lado: INE (azul/rosa) vs Real (teal/ámbar):

function crearPiramide(canvasId, labels, dataM, dataF, colorM, colorF) {
    new Chart(ctx, {
        type: 'bar',
        data: {
            labels: labels,
            datasets: [
                {
                    label: 'Masculino',
                    data: dataM.map(v => -v),  // ← Negativo = izquierda
                    backgroundColor: colorM,
                },
                {
                    label: 'Femenino',
                    data: dataF,               // ← Positivo = derecha
                    backgroundColor: colorF,
                },
            ],
        },
        options: {
            indexAxis: 'y',        // Barras horizontales
            scales: {
                x: {
                    stacked: true,
                    ticks: {
                        callback: v => Math.abs(v),  // Valor absoluto en eje
                    },
                },
                y: { stacked: true, reverse: true },
            },
        },
    });
}

// INE: azul/rosa
crearPiramide('pir-ine-'+id, labels, ineM, ineF,
    'rgba(59,130,246,0.7)', 'rgba(236,72,153,0.7)');

// Real: teal/ámbar (solo si tiene datos)
if (pir.tieneReal) {
    crearPiramide('pir-real-'+id, labels, realM, realF,
        'rgba(20,184,166,0.7)', 'rgba(245,158,11,0.7)');
}
Truco de pirámide — Los datos masculinos se pasan como negativos (dataM.map(v => -v)) para que las barras crezcan a la izquierda. El tooltip muestra el valor absoluto con Math.abs().

@foreach ($modulos as $mod)
    @php $esProtegido = $mod['auth'] && !auth()->check(); @endphp
    <a href="{{ $esProtegido ? route('login') : $mod['href'] }}">

Acceso controlado — Los módulos con auth: true redirigen a login si el usuario no está autenticado. Solo el Blog es accesible sin login.
