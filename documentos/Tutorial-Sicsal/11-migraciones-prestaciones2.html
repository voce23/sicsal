ℹ️
Cobertura = (prestación / meta INE) × 100
La fórmula de cobertura usa estas metas como denominador. Si la meta INE para menores de 1 año es 50 y se vacunaron 45, la cobertura es 90%.

8 UNIQUE constraints en tablas clínicas

Esta migración agrega constraints de unicidad a las tablas clínicas individuales para prevenir datos duplicados:

database/migrations/XXXX_add_unique_constraints_person_level.php php 
public function up(): void
{
    // Misma persona no puede tener misma vacuna el mismo día
    Schema::table('vacunas_ninos', function (Blueprint $table) {
        $table->unique(
            ['persona_id', 'tipo_vacuna', 'fecha_aplicacion'],
            'vacunas_persona_tipo_fecha_unique'
        );
    });

    // Un control de crecimiento por persona por día
    Schema::table('crecimiento_infantil', function (Blueprint $table) {
        $table->unique(
            ['persona_id', 'fecha'],
            'crecimiento_persona_fecha_unique'
        );
    });

    // No duplicar micronutriente para la misma persona y fecha
    Schema::table('micronutrientes_ninos', function (Blueprint $table) {
        $table->unique(
            ['persona_id', 'tipo', 'fecha'],
            'micronutrientes_persona_tipo_fecha_unique'
        );
    });

    // Un método por persona por fecha
    Schema::table('anticoncepcion', function (Blueprint $table) {
        $table->unique(
            ['persona_id', 'metodo', 'fecha'],
            'anticoncepcion_persona_metodo_fecha_unique'
        );
    });

    // No duplicar número de control prenatal
    Schema::table('controles_prenatales', function (Blueprint $table) {
        $table->unique(
            ['embarazo_id', 'numero_control'],
            'controles_embarazo_numero_unique'
        );
    });

    // Un solo parto por embarazo
    Schema::table('partos', function (Blueprint $table) {
        $table->unique(['embarazo_id'], 'partos_embarazo_unique');
    });

    // Un solo puerperio por parto
    Schema::table('puerperios', function (Blueprint $table) {
        $table->unique(['parto_id'], 'puerperios_parto_unique');
    });
}
⚠️
Seguridad de datos
Los UNIQUE constraints son una capa de protección a nivel de base de datos. Aunque el código PHP valide duplicados, si hay un bug o un request concurrente, MySQL siempre rechazará el duplicado. Defensa en profundidad.

9 Índices de rendimiento

Agregamos índices en columnas que se usan frecuentemente en filtros WHERE:

database/migrations/XXXX_add_performance_indexes.php php 
public function up(): void
{
    Schema::table('personas', function (Blueprint $table) {
        $table->index('estado');    // filtrar residentes vs migrados
    });

    Schema::table('centros_salud', function (Blueprint $table) {
        $table->index('codigo_snis'); // búsqueda por código SNIS
    });
}
💡
¿Cuándo agregar un índice?
Agrega un índice cuando: (1) la columna se usa en WHERE frecuentemente, (2) la tabla tendrá muchas filas, (3) la columna tiene buena selectividad (muchos valores distintos). No abuses: cada índice ocupa espacio y ralentiza los INSERT.
