1 — ¿Qué son los archivos .ves y .mdb?

El SNIS (Sistema Nacional de Información en Salud de Bolivia) distribuye software basado en Access 97 que genera archivos .ves. Estos archivos son en realidad bases de datos Microsoft Access formato Jet 3.x (archivos .mdb renombrados).

Cada archivo .ves contiene:

  • Tablas TMP — Plantillas de formularios (301G01_TMP, 301G03_TMP, etc.)
  • Tablas DAT — Datos mensuales (301G01_DAT01 a DAT12 por cada grupo)
  • Tabla ESTABL — Datos del establecimiento de salud
  • Tabla 301_CAB — Cabecera con metadatos del archivo

SICSAL necesita leer estos archivos para importar datos históricos del SNIS al padrón digital.

Formato obsoleto
Access 97 (Jet 3.x) es un formato de 1997. Los drivers modernos de ODBC no siempre lo soportan. Por eso necesitamos DAO.DBEngine.36 en Windows (COM de 32-bit) o mdbtools en Linux.

3 — Constructor y constantes

app/Services/VesMdbReader.php — configuración
namespace App\Services;

class VesMdbReader
{
    private string $mdbPath;

    /** Ruta al cscript.exe de 32-bit (necesario para DAO) */
    private const CSCRIPT32 = 'C:\\Windows\\SysWow64\\cscript.exe';

    /** Ruta al VBScript lector, relativa a base_path() */
    private const VBS_SCRIPT = 'resources/scripts/leer_ves.vbs';

    /** Nombres de tabla permitidos (whitelist) */
    private const TABLAS_PERMITIDAS = [
        '301_CAB', '301G01_TMP', '301G02_TMP',
        '301G03_TMP', '301G04_TMP', '301G05_TMP',
        // ... 301G06 a 301G38 ...
        'ESTABL', 'CONSOLIDADO',
        // Tablas DAT mensuales (48 tablas)
        '301G01_DAT01', '301G01_DAT02', // ... hasta DAT12
        '301G03_DAT01', // ... grupos 03, 37, 38 × 12 meses
    ];

    public function __construct(string $mdbPath)
    {
        $this->mdbPath = $mdbPath;
    }
SysWow64 — no System32
En Windows 64-bit, SysWow64 contiene los binarios de 32-bit. Necesitamos la versión de 32-bit de cscript.exe porque DAO.DBEngine.36 es un COM de 32-bit que no puede cargarse en un proceso de 64-bit.

4 — El método tabla() — punto de entrada

tabla() — API pública
public function tabla(string $tabla, string $filtroWhere = ''): array
{
    // 1. Validar contra whitelist — previene inyección SQL
    if (! $this->tablaPermitida($tabla)) {
        throw new \InvalidArgumentException(
            "Tabla '{$tabla}' no está en la lista de tablas permitidas."
        );
    }

    // 2. Elegir estrategia por sistema operativo
    if (PHP_OS_FAMILY === 'Windows') {
        return $this->leerConVbscript($tabla, $filtroWhere);
    }

    return $this->leerConMdbtools($tabla, $filtroWhere);
}

Uso típico:

Ejemplo de uso
$reader = new VesMdbReader('C:\ruta\transfer.mdb');

// Leer tabla completa
$rows = $reader->tabla('301G01_DAT05');

// Leer con filtro WHERE
$rows = $reader->tabla('301G01_DAT05', "corr_estabgest='26300183'");

5 — Whitelist de tablas — seguridad

Como el nombre de tabla se inyecta en una query SQL (dentro del VBScript), es crítico validarlo. Se usa una doble validación:

tablaPermitida() — doble validación
private function tablaPermitida(string $tabla): bool
{
    // 1. Búsqueda exacta en la whitelist
    if (in_array($tabla, self::TABLAS_PERMITIDAS, true)) {
        return true;
    }

    // 2. Patrón genérico para tablas DAT de cualquier grupo/mes
    return (bool) preg_match('/^30[1-9]G\d{2}_DAT\d{2}$/', $tabla);
}
Seguridad — inyección SQL
Sin esta whitelist, un atacante podría pasar un nombre como tabla]; DROP TABLE ... que se ejecutaría dentro del VBScript. La whitelist estricta + el regex solo permiten nombres alfanuméricos con guiones bajos.

6 — Windows: VBScript + DAO 3.6

leerConVbscript() — ejecución del script
private function leerConVbscript(string $tabla, string $filtroWhere): array
{
    if (! file_exists(self::CSCRIPT32)) {
        throw new \RuntimeException('No se encontró '
            . self::CSCRIPT32);
    }

    $vbsPath = base_path(self::VBS_SCRIPT);
    if (! file_exists($vbsPath)) {
        throw new \RuntimeException(
            'No se encontró el script VBScript: ' . $vbsPath
        );
    }

    $args = [
        escapeshellarg(self::CSCRIPT32),
        '//nologo',
        escapeshellarg($vbsPath),
        escapeshellarg($this->mdbPath),
        escapeshellarg($tabla),
    ];

    if ($filtroWhere !== '') {
        $args[] = escapeshellarg($filtroWhere);
    }

    $cmd    = implode(' ', $args);
    $output = shell_exec($cmd);

    if ($output === null || $output === '') {
        return [];
    }

    $data = json_decode(trim($output), true);

    if (json_last_error() !== JSON_ERROR_NONE) {
        throw new \RuntimeException(
            'JSON inválido del VBScript: ' . json_last_error_msg()
        );
    }

    return $data ?? [];
}

Todos los argumentos se protegen con escapeshellarg() para prevenir inyección de comandos del sistema operativo.

7 — El script leer_ves.vbs

Este VBScript actúa como puente entre PHP y el motor DAO de 32-bit:

resources/scripts/leer_ves.vbs
' Uso: cscript //nologo leer_ves.vbs "ruta.mdb" "NombreTabla" ["filtro"]
' Salida: JSON array en stdout / errores en stderr

Dim mdbPath, tableName, extraFilter
mdbPath     = args(0)
tableName   = args(1)
extraFilter = ""
If args.Count > 2 Then extraFilter = args(2)

' Abrir base de datos con DAO 3.6 (Jet 3.x / Access 97)
Dim dbe, db
Set dbe = CreateObject("DAO.DBEngine.36")
Set db  = dbe.OpenDatabase(mdbPath, False, True)  ' read-only

' Construir SQL y abrir recordset
Dim sql
sql = "SELECT * FROM [" & tableName & "]"
If extraFilter <> "" Then sql = sql & " WHERE " & extraFilter

Dim rs
Set rs = db.OpenRecordset(sql, 4)  ' dbOpenSnapshot

' Serializar campo a campo a JSON
Dim out
out = "["
Do While Not rs.EOF
    ' Construir {"campo":"valor",...} por cada registro
    For Each f In rs.Fields
        ' Null → "null", String → escapar, Bool → true/false
        ' Date → string, Number → literal
    Next
    rs.MoveNext
Loop
out = out & "]"

rs.Close
db.Close
WScript.Echo out

El script produce un JSON array válido en stdout. El tipo VBScript de cada campo se mapea:

VarTypeTipo VBJSON output
Null—null
8vbString"valor escapado"
11vbBooleantrue / false
7vbDate"fecha como string"
OtroNumérico123 (sin comillas)
dbOpenSnapshot (tipo 4)
Se usa snapshot en lugar de dynaset porque es solo lectura y más rápido. No necesitamos modificar los registros del .mdb.

8 — Linux: mdbtools

En Linux (servidores de producción), se usa mdb-json del paquete mdbtools:

leerConMdbtools() — alternativa Linux
private function leerConMdbtools(
    string $tabla, string $filtroWhere
): array {
    $cmd    = 'mdb-json ' . escapeshellarg($this->mdbPath)
            . ' ' . escapeshellarg($tabla);
    $output = shell_exec($cmd);

    if ($output === null || $output === '') {
        return [];
    }

    // mdb-json devuelve NDJSON (un objeto por línea)
    $rows = [];
    foreach (explode("\n", trim($output)) as $line) {
        $line = trim($line);
        if ($line === '') continue;
        $row = json_decode($line, true);
        if (is_array($row)) {
            $rows[] = $row;
        }
    }

    // mdbtools no soporta WHERE → filtrar en PHP
    if ($filtroWhere !== '' && ! empty($rows)) {
        $rows = $this->filtrarEnPhp($rows, $filtroWhere);
    }

    return $rows;
}
NDJSON ≠ JSON array
mdb-json emite un objeto JSON por línea (Newline Delimited JSON), no un array. Por eso se parsea línea por línea con explode("\n") en lugar de un solo json_decode().

9 — Filtrado en PHP para mdbtools

Como mdb-json no soporta cláusula WHERE, el filtro se aplica en PHP:

filtrarEnPhp() — WHERE simulado
private function filtrarEnPhp(array $rows, string $filtroWhere): array
{
    // Parsear: campo='valor' o campo=123
    preg_match_all(
        "/(\w+)\s*=\s*(?:'([^']*)'|(\d+))/",
        $filtroWhere,
        $matches,
        PREG_SET_ORDER
    );

    if (empty($matches)) {
        return $rows;
    }

    // Construir array de condiciones
    $condiciones = [];
    foreach ($matches as $m) {
        $condiciones[$m[1]] = $m[2] !== ''
            ? $m[2]           // string
            : (int) $m[3];    // número
    }

    // Filtrar: todas las condiciones deben cumplirse (AND)
    return array_filter($rows, function ($row) use ($condiciones) {
        foreach ($condiciones as $campo => $valor) {
            if (! isset($row[$campo])) return false;
            if (trim((string) $row[$campo]) !== (string) $valor)
                return false;
        }
        return true;
    });
}

Solo soporta igualdades simples con AND — suficiente para filtrar por código de establecimiento (corr_estabgest='26300183'), que es el uso principal.
