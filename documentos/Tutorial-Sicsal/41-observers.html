2 Patrón compartido: created(+1) / deleted(−1)

Todos los observers siguen exactamente la misma estructura:

class XxxObserver
{
    use SincronizaPrestacion;

    public function created(Modelo $registro): void
    {
        $this->sincronizar($registro, +1);
    }

    public function deleted(Modelo $registro): void
    {
        $this->sincronizar($registro, -1);
    }

    private function sincronizar(Modelo $registro, int $delta): void
    {
        // 1. Obtener persona/paciente
        // 2. Calcular grupo_etareo y campo usando el trait
        // 3. firstOrCreate la fila Prest*
        // 4. increment o decrement según $delta
    }
}
Delta pattern: Un solo método sincronizar() maneja ambos casos (creación y eliminación) usando un $delta de +1 o −1. No hay updated() — si cambian datos del registro, se debe borrar y recrear.

3 VacunaNinoObserver — vacunas infantiles

El observer más representativo — traduce un registro de vacuna individual al consolidado:

private function sincronizar(VacunaNino $vacuna, int $delta): void
{
    $persona = $vacuna->persona;
    if (!$persona) return;

    // Parsear fecha y calcular edad
    $fechaRef  = Carbon::parse($vacuna->fecha_aplicacion);
    $edadMeses = Carbon::parse($persona->fecha_nacimiento)
                    ->diffInMonths($fechaRef);

    // Usar el trait para clasificar
    $grupoEtareo = $this->grupoEtareoVacuna((int) $edadMeses);
    $campo       = $this->campoDentroFueraSexo(
                       $vacuna->dentro_fuera, $persona->sexo);

    // Buscar o crear la fila consolidada
    $row = PrestVacuna::firstOrCreate(
        ['centro_salud_id' => $persona->centro_salud_id,
         'mes' => $vacuna->mes, 'anio' => $vacuna->anio,
         'tipo_vacuna' => $vacuna->tipo_vacuna,
         'grupo_etareo' => $grupoEtareo],
        ['dentro_m' => 0, 'dentro_f' => 0,
         'fuera_m'  => 0, 'fuera_f'  => 0]
    );

    // Incrementar o decrementar el campo correcto
    if ($delta > 0) {
        $row->increment($campo);
    } elseif ($row->$campo > 0) {
        $row->decrement($campo);
    }
}
Guard clause: elseif ($row->$campo > 0) previene que el contador baje a negativo. Si ya es 0, no decrementa — protección contra inconsistencias.

4 PartoObserver — partos, recién nacidos y BCG automática

El observer más complejo — un solo evento de parto actualiza 3 tablas Prest*:

private function sincronizar(Parto $parto, int $delta): void
{
    // Navegar: Parto → Embarazo → Persona (madre)
    $embarazo = $parto->embarazo()->with('persona')->first();
    if (!$embarazo?->persona) return;

    $grupoEtareo = $parto->grupo_etareo
        ?: $this->grupoEtareoMaterno(...);

    // ─── 1. PrestParto (tipo × lugar × atendido_por × grupo) ───
    $rowParto = PrestParto::firstOrCreate(
        ['centro_salud_id' => $centroId, ...,
         'tipo' => $parto->tipo,
         'lugar' => $parto->lugar,
         'atendido_por' => $parto->atendido_por,
         'grupo_etareo' => $grupoEtareo],
        ['cantidad' => 0]
    );
    // increment/decrement cantidad

    // ─── 2. Si nacido_vivo → PrestRecienNacido ───
    if ($parto->resultado === 'nacido_vivo') {
        $indicadorRN = ($parto->lugar === 'servicio')
            ? 'nacidos_vivos_servicio'
            : 'nacidos_vivos_domicilio';
        // firstOrCreate + increment/decrement

        // ─── 3. BCG automático en PrestVacuna ───
        $rowBcg = PrestVacuna::firstOrCreate(
            [... 'tipo_vacuna' => 'BCG',
                 'grupo_etareo' => 'menor_1'],
            ['dentro_m' => 0, ...]
        );
        // increment dentro_m (convención al no conocer sexo RN)
    }
}
BCG automática: Todo nacido vivo recibe vacuna BCG. El observer lo registra automáticamente en el consolidado de vacunas usando dentro_m por convención (al momento del parto no siempre se conoce el sexo del recién nacido).
Navegación de relaciones: Parto → embarazo() → persona es una cadena de 2 relaciones. El observer usa with('persona') para evitar N+1. El nullsafe $embarazo?->persona protege si el embarazo no tiene persona.

5 ControlPrenatalObserver — prenatales por trimestre

private function sincronizar(ControlPrenatal $control, int $delta): void
{
    $embarazo = $control->embarazo()->with('persona')->first();

    // tipoPrenatal() del trait clasifica según número de control y semanas
    $tipoControl = $this->tipoPrenatal(
        (int) $control->numero_control,
        (int) ($control->semanas_gestacion ?? 0)
    );

    // Campo: 'dentro' o 'fuera'
    $campo = $control->dentro_fuera === 'dentro' ? 'dentro' : 'fuera';

    $row = PrestPrenatal::firstOrCreate(
        ['centro_salud_id' => ..., 'mes' => ..., 'anio' => ...,
         'tipo_control' => $tipoControl,
         'grupo_etareo' => $grupoEtareo],
        ['dentro' => 0, 'fuera' => 0]
    );
    // increment/decrement $campo
}

Prenatales usa 2 campos (dentro/fuera) en lugar de 4. El grupo etáreo puede venir del registro o calcularse con grupoEtareoMaterno() del trait.

6 CrecimientoInfantilObserver — crecimiento infantil

private function sincronizar(CrecimientoInfantil $registro, int $delta): void
{
    $edadMeses   = Carbon::parse($persona->fecha_nacimiento)
                      ->diffInMonths($fechaRef);
    $grupoEtareo = $this->grupoCrecimiento(
                      $edadMeses, $registro->dentro_fuera);

    // Campo: nuevos_m, nuevos_f, repetidos_m, repetidos_f
    $prefijo = $registro->tipo_control === 'nuevo'
        ? 'nuevos' : 'repetidos';
    $campo = $prefijo . '_' . strtolower($persona->sexo);

    $row = PrestCrecimiento::firstOrCreate(
        [... 'grupo_etareo' => $grupoEtareo],
        ['nuevos_m' => 0, 'nuevos_f' => 0,
         'repetidos_m' => 0, 'repetidos_f' => 0]
    );
}
4 campos: tipo_control (nuevo/repetido) × sexo (M/F) = 4 combinaciones. El grupo etáreo ya incluye dentro/fuera (vía grupoCrecimiento()), así que el campo solo necesita combinar tipo_control + sexo.

7 AnticoncepcionObserver — métodos anticonceptivos

private function sincronizar(Anticoncepcion $ac, int $delta): void
{
    $grupoEtareo = $this->grupoEtareoMaterno(
        Carbon::parse($persona->fecha_nacimiento),
        $fechaRef
    );

    $row = PrestAnticoncepcion::firstOrCreate(
        ['centro_salud_id' => ...,
         'metodo'       => $ac->metodo,
         'tipo_usuaria' => $ac->tipo_usuaria,
         'grupo_etareo' => $grupoEtareo],
        ['cantidad' => 0]
    );
    // increment/decrement cantidad
}

La clave compuesta incluye metodo + tipo_usuaria + grupo_etareo, que es la misma estructura que usa FormularioMaterna (lección 37) para la sección de anticoncepción.

8 MicronutrienteNinoObserver — el más simple

El único observer que no usa el trait SincronizaPrestacion:

class MicronutrienteNinoObserver
{
    // No usa SincronizaPrestacion — no necesita clasificar edad

    private function sincronizar(MicronutrienteNino $micro, int $delta): void
    {
        $persona = $micro->persona;
        if (!$persona) return;

        $row = PrestMicronutriente::firstOrCreate(
            ['centro_salud_id' => $persona->centro_salud_id,
             'mes'  => $micro->mes,
             'anio' => $micro->anio,
             'tipo' => $micro->tipo],
            ['cantidad' => 0]
        );

        if ($delta > 0) {
            $row->increment('cantidad');
        } elseif ($row->cantidad > 0) {
            $row->decrement('cantidad');
        }
    }
}
¿Por qué no necesita el trait? El tipo de micronutriente (hierro_menor_1, vitA_2_5_1ra, etc.) ya está codificado en el registro individual. No hay que calcular grupo etáreo — el tipo ya lo implica.

9 firstOrCreate + increment/decrement

El patrón central de todos los observers merece análisis:

// 1. Obtener o crear la fila con contadores en cero
$row = PrestVacuna::firstOrCreate(
    $claveBusqueda,  // WHERE conditions
    $valoresDefecto  // Solo si no existe: inicializar en 0
);

// 2. Modificar el campo específico atómicamente
if ($delta > 0) {
    $row->increment($campo);    // UPDATE ... SET campo = campo + 1
} elseif ($row->$campo > 0) {
    $row->decrement($campo);    // UPDATE ... SET campo = campo - 1
}
¿Por qué no updateOrCreate?
  • firstOrCreate solo crea la fila si no existe, sin modificar si ya existe
  • increment/decrement modifican atómicamente a nivel de SQL (campo = campo + 1), sin race conditions
  • updateOrCreate sobrescribiría el valor — imposible con +1/−1
Propiedad dinámica: $row->$campo (con $ doble) accede a la propiedad cuyo nombre está en la variable $campo. Si $campo = 'dentro_m', entonces $row->$campo es equivalente a $row->dentro_m. Esto permite un solo código para 4+ campos.

10 Cierre de Fase 8

Lo que aprendimos en esta lección

  • 5 observers + 1 trait: Patrón consistente de sincronización individual → consolidado mensual.
  • Delta pattern (+1/−1): Un solo método sincronizar() maneja tanto created como deleted.
  • firstOrCreate + increment/decrement: Creación atómica de fila + modificación atómica de campo, sin race conditions.
  • PartoObserver multicadena: Un parto actualiza 3 tablas Prest* y registra BCG automáticamente para nacidos vivos.
  • Guard contra negativos: elseif ($row->$campo > 0) previene contadores negativos.
  • Propiedad dinámica: $row->$campo permite código genérico para múltiples campos.
