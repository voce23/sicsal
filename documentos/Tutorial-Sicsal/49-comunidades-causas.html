PLANILLA_GRUPOS — 11 rangos más finos para campo

private const PLANILLA_GRUPOS = [
    'menor_1' => [
        'label'  => '<1 año',
        'filter' => 'fecha_nacimiento > DATE_SUB(CURDATE(), INTERVAL 1 YEAR)',
    ],
    '1_2'     => ['label' => '1-2',  'filter' => 'TIMESTAMPDIFF(YEAR, ...) BETWEEN 1 AND 2'],
    '3_5'     => ['label' => '3-5',  'filter' => '... BETWEEN 3 AND 5'],
    // ... hasta 'mayor_60'
];

RATIO_MASCULINIDAD — para split INE "ambos"

private const RATIO_MASCULINIDAD = [
    'menor_1' => 1.05,   // 105 varones por 100 mujeres
    '1_4'     => 1.05,
    '5_9'     => 1.04,
    '20_39'   => 0.96,   // Emigración masculina invierte ratio
    '50_59'   => 0.92,
    'mayor_60'=> 0.88,   // Mayor longevidad femenina
];
¿Por qué 2 sets de grupos? El INE (Instituto Nacional de Estadística) usa 9 rangos estándar para metas poblacionales. La planilla de campo necesita 11 rangos más granulares para el censo casa-por-casa (ej. separar 1-2 y 3-5 en lugar de agrupar como 1-4).

3 · getDatosProperty — 3 datasets en un computed

El computed más complejo del proyecto genera tres estructuras simultáneas en un solo recorrido de comunidades:

public function getDatosProperty(): array
{
    $centroId = $this->getCentroId();
    if ($centroId === 0) return $emptyState;

    $comunidades = Comunidad::where('centro_salud_id', $centroId)
        ->orderBy('nombre')->get();

    foreach ($comunidades as $com) {
        $base = Persona::where('centro_salud_id', $centroId)
            ->where('comunidad_id', $com->id)
            ->where('activo', true);

        $baseNoMig = (clone $base)->where('estado', '!=', 'migrado');

        // Dataset 1: Resumen (total/H/M + 9 grupos + migrantes)
        $total    = (clone $baseNoMig)->count();
        $hombres  = (clone $baseNoMig)->where('sexo', 'M')->count();
        $grupoVals['menor_1'] = (clone $baseNoMig)
            ->where('fecha_nacimiento', '>', now()->subYear())->count();
        // ... 8 más con whereRaw(TIMESTAMPDIFF)

        // Dataset 2: Detalle M/F por grupo por comunidad
        foreach ($grupos as $g) {
            $filtro = $this->getAgeFilter($g);
            $m = (clone $baseNoMig)->where('sexo', 'M')->whereRaw($filtro)->count();
            $f = (clone $baseNoMig)->where('sexo', 'F')->whereRaw($filtro)->count();
            $comDetalle[$g] = ['M' => $m, 'F' => $f];
        }
    }

    // Dataset 3: Consolidado INE vs Real
    $metaIneRows = MetaIne::where('centro_salud_id', $centroId)
        ->where('anio', $anio)->get();

    foreach ($grupos as $g) {
        $meta = $metaIneRows->where('grupo_etareo', $g);
        // Si sexo='ambos', aplicar RATIO_MASCULINIDAD para split
        if ($metaAmbos) {
            $ratio  = self::RATIO_MASCULINIDAD[$g] ?? 1.0;
            $ineM   = (int) round($totalIne * $ratio / (1 + $ratio));
            $ineF   = $totalIne - $ineM;
        }
        $dif = $realTotal - $ineTotal;
        $pct = $ineTotal > 0 ? round($realTotal / $ineTotal * 100, 1) : 0;
    }
}
Patrón clone intensivo: Este método hace ~15 clones del query builder por comunidad. Con 20 comunidades son ~300 queries. Es un trade-off aceptable para datos públicos que se acceden con poca frecuencia, pero escalaría mal para miles de comunidades. En ese caso, se migraría a una sola query con selectRaw(SUM(CASE WHEN...)).

4 · getPlanillaProperty — la planilla de campo

Un segundo computed genera los datos para la planilla de censo con 11 grupos más granulares:

public function getPlanillaProperty(): array
{
    foreach ($comunidades as $com) {
        $base = Persona::where(...)->where('estado', '!=', 'migrado');

        foreach (self::PLANILLA_GRUPOS as $key => $cfg) {
            $m = (clone $base)->where('sexo', 'M')
                ->whereRaw($cfg['filter'])->count();
            $f = (clone $base)->where('sexo', 'F')
                ->whereRaw($cfg['filter'])->count();
        }

        // Dato especial: embarazadas (MEF 15-49 con estado='embarazada')
        $embarazadas = (clone $base)->where('sexo', 'F')
            ->whereRaw('TIMESTAMPDIFF(YEAR, ...) BETWEEN 15 AND 49')
            ->where('estado', 'embarazada')->count();
    }

    return ['filas', 'totales', 'centro', 'grupos' => self::PLANILLA_GRUPOS];
}

Las descargas aceptan ambos datasets:

// PDF: usa vista específica en landscape-legal
public function generarPlanillaPdf()
{
    $pdf = Pdf::loadView('pdf.planilla-censo', ['planilla' => $planilla])
        ->setPaper('legal', 'landscape')  // Legal landscape = más columnas

Legal landscape: Con 11 grupos × 2 columnas (M/F) + extras = ~25 columnas. El papel Legal (8.5×14") en landscape da más espacio horizontal que A4.

7 · CausasConsulta — lean delegator (46 líneas)

El componente más compacto del proyecto, con solo 46 líneas:

class CausasConsulta extends Component
{
    use HasCentroSelector;

    #[Url]
    public int $anio = 0;

    #[Url]
    public int $mes = 0;  // 0 = todo el año

    public function getDatosProperty(): ?array
    {
        if ($this->centroSaludId === 0) return null;
        return CausasConsultaHelper::getTop10(
            $this->centroSaludId,
            $this->anio,
            $this->mes
        );
    }

    public function getCentrosProperty(): array
    {
        return CentroSalud::where('activo', true)
            ->orderBy('nombre')
            ->pluck('nombre', 'id')
            ->toArray();
    }
}

Toda la lógica de agregación está en CausasConsultaHelper::getTop10() (ver Lección 43). El componente solo define: estado URL-bound, lista de centros, y computed de datos.

$mes = 0: Convención para "todo el año". El helper interpreta 0 como "sin filtro de mes", sumando todos los meses de la gestión.

Resumen de la lección

  • ComunidadesPoblacion (336 líneas) es el componente público más extenso — genera 3 datasets (resumen, detalle M/F, consolidado INE vs Real) + planilla de campo con 11 grupos.
  • Usa 2 constantes de grupos: GRUPOS (9, para INE) y PLANILLA_GRUPOS (11, para campo) + RATIO_MASCULINIDAD para split de "ambos".
  • El patrón clone intensivo ejecuta ~300 queries para 20 comunidades — aceptable para uso esporádico.
  • Las descargas usan Legal landscape para acomodar ~25 columnas.
  • CausasConsulta (46 líneas) es un delegator puro a CausasConsultaHelper, con filtro por mes (0 = gestión completa).
  • Su vista (264 líneas) usa identidad rose/red, tabla de 2-filas header con 10 grupos × M/F, filas vacías hasta 10, y barras horizontales de distribución.
  • Ambos componentes incluyen livewire:comentarios en la parte inferior.
