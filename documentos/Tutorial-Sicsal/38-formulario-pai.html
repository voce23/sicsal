1 Panorama general

FormularioPai captura el Programa Ampliado de Inmunización (PAI), micronutrientes y control de crecimiento infantil. Es el formulario más denso del sistema en cuanto a volumen de inputs:

SecciónModeloFilasCampos/filaTotal inputs
Vacunas menores 5PrestVacuna23 vacunas × 5 grupos4460
Vacunas otrasPrestVacuna28 vacunas × 6 grupos4672
MicronutrientesPrestMicronutriente21 tipos121
CrecimientoPrestCrecimiento6 grupos424
1.177 inputs potenciales — se organizan en 3 pestañas Alpine.js para no saturar al usuario. El tab de vacunas contiene a su vez 2 sub-secciones.

2 Arrays estáticos — catálogo de vacunas

El formulario define cuatro arrays public static que actúan como esquema del SNIS. Cada vacuna es un par [clave, etiqueta]:

// 23 vacunas para menores de 5 años
public static array $vacunasMenores5 = [
    ['BCG',               'BCG'],
    ['Pentavalente_1',     'Pentavalente 1ra dosis'],
    ['Pentavalente_2',     'Pentavalente 2da dosis'],
    // ... (23 ítems en total)
    ['Influenza_enf_cronicas_ninos', 'Influenza a niños con enf. crónicas'],

// 5 grupos etáreos para menores de 5
public static array $gruposMenores5 = [
    'menor_1'  => 'Menor de 1 año',
    '12_23m'   => '12 a 23 meses',
    '2_anios'  => '2 años',
    '3_anios'  => '3 años',
    '4_anios'  => '4 años',

Y el catálogo equivalente para mayores de 5:

3 allVacunasConfig() — combinación dinámica

Este método estático genera un array plano con todas las combinaciones vacuna×grupo, útil para cargarDatos() y guardarVacunas():

public static function allVacunasConfig(): array
{
    $all = [];

    // Menores de 5: 23 vacunas × 5 grupos = 115 combos
    foreach (self::$vacunasMenores5 as [$tipo, $label]) {
        foreach (self::$gruposMenores5 as $ge => $geLabel) {
            $all[] = [$tipo, $ge, $label];
        }
    }

    // Otras: 28 vacunas × 6 grupos = 168 combos
    foreach (self::$vacunasOtras as [$tipo, $label]) {
        foreach (self::$gruposOtrasVac as $ge => $geLabel) {
            $all[] = [$tipo, $ge, $label];
        }
    }

    return $all; // 283 combinaciones en total
}
Técnica: PHP array destructuring [$tipo, $label] permite extraer los dos valores del sub-array directamente en el foreach, sin usar índices numéricos.

4 cargarDatos() — keyBy compuesto y simple

Tres secciones, tres estrategias de indexación:

private function cargarDatos(int $centroId): void
{
    // ─── Vacunas: clave compuesta tipo__grupo ───
    $rowsVacunas = PrestVacuna::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)->where('anio', $this->anio)
        ->get()->keyBy(fn ($r) => "{$r->tipo_vacuna}__{$r->grupo_etareo}");

    foreach (self::allVacunasConfig() as [$tipoVacuna, $grupoEtareo, $label]) {
        $key = "{$tipoVacuna}__{$grupoEtareo}";
        $row = $rowsVacunas->get($key);
        $this->vacunas[$key] = [
            'dentro_m' => $row->dentro_m ?? 0,
            'dentro_f' => $row->dentro_f ?? 0,
            'fuera_m'  => $row->fuera_m  ?? 0,
            'fuera_f'  => $row->fuera_f  ?? 0,
        ];
    }

    // ─── Micronutrientes: keyBy('tipo') simple ───
    $rowsMicro = PrestMicronutriente::where(...)
        ->get()->keyBy('tipo');
    foreach ($tiposMicro as $tipo) {
        $this->micronutrientes[$tipo] = $row->cantidad ?? 0;
    }

    // ─── Crecimiento: keyBy('grupo_etareo') simple ───
    $rowsCrecimiento = PrestCrecimiento::where(...)
        ->get()->keyBy('grupo_etareo');
    foreach ($gruposCrecimiento as $ge) {
        $this->crecimiento[$ge] = [
            'nuevos_m'    => $row->nuevos_m    ?? 0,
            'nuevos_f'    => $row->nuevos_f    ?? 0,
            'repetidos_m' => $row->repetidos_m ?? 0,
            'repetidos_f' => $row->repetidos_f ?? 0,
        ];
    }
}
Tres niveles de keyBy:
  • Vacunas: keyBy(fn()) con closure → clave compuesta "BCG__menor_1"
  • Micronutrientes: keyBy('tipo') → columna directa (21 valores escalares)
  • Crecimiento: keyBy('grupo_etareo') → columna directa (6 sub-arrays)

5 Blade — pestañas Alpine.js (3 tabs)

La vista usa el mismo patrón Alpine de los otros formularios — un x-data con activeTab y botones que mutan el estado:

<div x-data="{ activeTab: 'vacunas' }" class="space-y-4">

    <!-- Navegación de tabs -->
    <nav class="flex gap-1 rounded-xl bg-gray-100 p-1">
        @foreach ([
            'vacunas'         => '1. Vacunas',
            'micronutrientes' => '2. Micronutrientes',
            'crecimiento'     => '3. Crecimiento',
        ] as $key => $label)
            <button
                x-on:click="activeTab = '{{ $key }}'"
                :class="activeTab === '{{ $key }}'
                    ? 'bg-white text-primary-600 shadow'
                    : 'text-gray-600 hover:text-gray-800'">
                {{ $label }}
            </button>
        @endforeach
    </nav>

    <!-- Contenido de cada tab con x-show + x-cloak -->
    <div x-show="activeTab === 'vacunas'" x-cloak> ... </div>
    <div x-show="activeTab === 'micronutrientes'" x-cloak> ... </div>
    <div x-show="activeTab === 'crecimiento'" x-cloak> ... </div>
</div>

El array inline en el @foreach es una técnica concisa de Blade para definir un diccionario directamente en la vista sin pasarlo desde el componente.

El cuerpo itera las vacunas con destructuring y construye la clave compuesta:

<tbody>
    @foreach (FormularioPai::$vacunasMenores5 as [$tipo, $label])
        <tr>
            <td>{{ $label }}</td>
            @foreach (array_keys(FormularioPai::$gruposMenores5) as $ge)

                        <input type="number" min="0"
                            wire:model="vacunas.{{ $key }}.{{ $campo }}"
                            {{ $mesCerrado ? 'disabled' : '' }}>
                    </td>
                @endforeach
            @endforeach
        </tr>
    @endforeach
</tbody>
Atención: Son 3 foreach anidados (vacunas × grupos × campos). Para "Menores de 5" eso genera 23 × 5 × 4 = 460 inputs individuales en el DOM. Los inputs tipo number min="0" y ancho fijo w-12 mantienen una grilla compacta.

9 guardarVacunas() — skip-empty optimization

Con 283 combinaciones posibles, muchas estarán en cero. El método aplica una optimización inteligente: solo persiste filas con al menos un valor > 0:

public function guardarVacunas(): void
{
    if ($this->mesCerrado) return;
    $centroId = auth()->user()->centro_salud_id;

    foreach (self::allVacunasConfig() as [$tipoVacuna, $grupoEtareo, $label]) {
        $key = "{$tipoVacuna}__{$grupoEtareo}";
        $datos = $this->vacunas[$key] ?? [];
        $values = array_map('intval', $datos);

        if (array_sum($values) > 0) {
            PrestVacuna::updateOrCreate(
                ['centro_salud_id' => $centroId,
                 'mes' => $this->mes,
                 'anio' => $this->anio,
                 'tipo_vacuna' => $tipoVacuna,
                 'grupo_etareo' => $grupoEtareo],
                $values
            );
        }
    }

    Notification::make()->title('Vacunas guardadas')->success()->send();
}
¿Por qué skip-empty? Sin esta optimización, se ejecutarían 283 queries por guardado. Con ella, un centro que solo aplica 10 tipos de vacunas ejecuta solo ~50 queries (10 tipos × 5 grupos). Es una optimización de rendimiento significativa.

Nota: array_map('intval', $datos) sanitiza los valores convirtiendo cualquier string vacío o null a 0 antes de persistir.

10 guardarMicronutrientes() y guardarCrecimiento()

Estos métodos son más directos — guardan todas las filas sin skip-empty (la cantidad total es pequeña: 21 y 6 respectivamente):

// Micronutrientes: valor escalar por tipo
public function guardarMicronutrientes(): void
{
    foreach ($this->micronutrientes as $tipo => $cantidad) {
        PrestMicronutriente::updateOrCreate(
            ['centro_salud_id' => $centroId,
             'mes' => $this->mes,
             'anio' => $this->anio,
             'tipo' => $tipo],
            ['cantidad' => (int) $cantidad]
        );
    }
    $this->verificarCeroMicronutrientes($centroId);
}

// Crecimiento: sub-array de 4 campos por grupo
public function guardarCrecimiento(): void
{
    foreach ($this->crecimiento as $ge => $datos) {
        PrestCrecimiento::updateOrCreate(
            ['centro_salud_id' => $centroId,
             'mes' => $this->mes,
             'anio' => $this->anio,
             'grupo_etareo' => $ge],
            array_map('intval', $datos)
        );
    }
    $this->verificarCeroCrecimiento($centroId);
}
Patrón updateOrCreate: Los 3 métodos comparten la misma llave (centro_salud_id + mes + anio + identificador de fila). Lo que cambia es el "identificador de fila": tipo_vacuna + grupo_etareo (compuesto), tipo (escalar) o grupo_etareo (escalar).

11 verificarCero — whereIn selectivo

Ambos métodos implementan el patrón de cero justificado, pero con un detalle importante: micronutrientes filtra solo los tipos infantiles.

// Crecimiento: suma los 4 campos de TODAS las filas
private function verificarCeroCrecimiento(int $centroId): void
{
    $total = PrestCrecimiento::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)->where('anio', $this->anio)
        ->sum(DB::raw('nuevos_m + nuevos_f + repetidos_m + repetidos_f'));

    if ($total == 0) {
        $this->dispatch('abrir-justificacion-cero',
            indicador: 'control_crecimiento');
    }
}

// Micronutrientes: solo suma los tipos INFANTILES
private function verificarCeroMicronutrientes(int $centroId): void
{
    $total = PrestMicronutriente::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)->where('anio', $this->anio)
        ->whereIn('tipo', [
            'hierro_menor_6m', 'hierro_menor_1',
            'hierro_1anio',    'hierro_2_5',
            'vitA_menor_1_unica', 'vitA_1anio_1ra',
            'vitA_1anio_2da',    'vitA_2_5_1ra',
            'vitA_2_5_2da',
            'zinc_menor_1', 'zinc_1anio',
        ])
        ->sum('cantidad');

    if ($total == 0) {
        $this->dispatch('abrir-justificacion-cero',
            indicador: 'micronutrientes_menor_5');
    }
}
¿Por qué whereIn selectivo? De los 21 tipos de micronutrientes, solo 11 corresponden a niños menores de 5 años. Los indicadores de embarazadas, puérperas, adultos mayores y nutrimamá no son relevantes para el indicador "micronutrientes_menor_5". Un centro podría tener cero en hierro infantil pero valores en hierro para embarazadas — y aún así debe justificar el cero infantil.
Contraste con crecimiento: verificarCeroCrecimiento usa DB::raw('nuevos_m + nuevos_f + ...') para sumar las 4 columnas de cada fila en una sola query. Micronutrientes tiene un solo campo (cantidad), así que usa sum('cantidad') directamente.

12 Resumen

Lo que aprendimos en esta lección

  • Arrays estáticos como esquema SNIS: 23 + 28 vacunas, 5 + 6 grupos, 21 micronutrientes, 6 grupos de crecimiento — todo declarado en arrays public static.
  • allVacunasConfig(): combinación de dos conjuntos (menores + otras) con doble destructuring, generando 283 combinaciones.
  • Tres niveles de keyBy: closure compuesta para vacunas, keyBy('tipo') para micro, keyBy('grupo_etareo') para crecimiento.
  • Cabecera triple: rowspan="3" + colspan=4 para organizar jerárquicamente grupo → dentro/fuera → M/F.
  • Skip-empty optimization: array_sum($values) > 0 antes de updateOrCreate para reducir queries en vacunas.
  • wire:model multinivel: vacunas.BCG__menor_1.dentro_m (3 niveles) vs micronutrientes.hierro_menor_6m (2 niveles) vs crecimiento.menor_1_dentro.nuevos_m (3 niveles).
  • whereIn selectivo: verificarCeroMicronutrientes filtra solo los 11 tipos infantiles de los 21 totales para el indicador correcto.
  • grid vs tabla: micronutrientes usa grid sm:grid-cols-2 (layout card) mientras que vacunas y crecimiento usan <table> por su naturaleza matricial.
