1 ¿Qué son las tablas prest_*?

El SNIS (Sistema Nacional de Información en Salud) de Bolivia requiere informes mensuales con totales agregados por centro de salud, grupo etáreo y sexo. Las tablas prest_* almacenan estos totales.

Capa Ejemplo Nivel de detalle
Individual (lección 09) María López, BCG, 15/01/2026 Cada persona, cada fecha
Agregado (esta lección) Centro Hornoma, enero 2026, BCG: 5 niños dentro, 2 fuera Totales mensuales por centro
💡
¿Por qué duplicar datos?
Los registros individuales existen para el seguimiento clínico de cada persona. Los totales prest_* existen para los informes SNIS. Se calculan automáticamente a partir de los individuales, pero se almacenan para poder generar reportes rápidamente sin recalcular miles de registros cada vez.

2 Patrón común: centro + mes + año + desglose

Todas las tablas prest_* comparten estas columnas base:

Patrón repetido en todas las prest_* php 
$table->id();
$table->foreignId('centro_salud_id')->constrained('centros_salud');
$table->tinyInteger('mes');         // 1-12
$table->year('anio');               // 2026
// ... columnas de desglose específicas ...
$table->timestamps();
$table->unique([...]);              // Constraint UNIQUE compuesto
⚠️
UNIQUE compuesto = una fila por combinación
Cada tabla tiene un UNIQUE compuesto que garantiza que no haya duplicados. Por ejemplo, en prest_consulta_externa: solo puede haber una fila por combinación de (centro, mes, año, grupo_etáreo). Si intentas insertar otra, MySQL dará error.

3 prest_consulta_externa

Totales mensuales de consultas médicas desglosadas por tipo (primera, nueva, repetida) y sexo.

database/migrations/XXXX_create_prest_consulta_externa_table.php php 
Schema::create('prest_consulta_externa', function (Blueprint $table) {
    $table->id();
    $table->foreignId('centro_salud_id')->constrained('centros_salud');
    $table->tinyInteger('mes');
    $table->year('anio');
    $table->enum('grupo_etareo', [
        'menor_6m', '6m_menor_1', '1_4', '5_9', '10_14',
        '15_19', '20_39', '40_49', '50_59', 'mayor_60',
    ]);
    $table->integer('primera_m')->default(0);   // Primera consulta, masc.
    $table->integer('primera_f')->default(0);   // Primera consulta, fem.
    $table->integer('nueva_m')->default(0);     // Nueva consulta, masc.
    $table->integer('nueva_f')->default(0);     // Nueva consulta, fem.
    $table->integer('repetida_m')->default(0);  // Repetida, masc.
    $table->integer('repetida_f')->default(0);  // Repetida, fem.
    $table->timestamps();
    $table->unique(
        ['centro_salud_id', 'mes', 'anio', 'grupo_etareo'],
        'pce_centro_mes_anio_grupo_unique'
    );
});
ℹ️
Nombre corto del constraint
MySQL limita el nombre de los índices a 64 caracteres. Usamos un alias corto como pce_centro_mes_anio_grupo_unique como segundo argumento de unique().

4 prest_vacunas

Totales de vacunas aplicadas, desglosados por dentro/fuera y sexo.

database/migrations/XXXX_create_prest_vacunas_table.php php 
Schema::create('prest_vacunas', function (Blueprint $table) {
    $table->id();
    $table->foreignId('centro_salud_id')->constrained('centros_salud');
    $table->tinyInteger('mes');
    $table->year('anio');
    $table->string('tipo_vacuna', 60);      // BCG, Pentavalente_1, etc.
    $table->string('grupo_etareo', 30);     // menor_1, 1_4, etc.
    $table->integer('dentro_m')->default(0);  // Dentro del área, masc.
    $table->integer('dentro_f')->default(0);  // Dentro del área, fem.
    $table->integer('fuera_m')->default(0);   // Fuera del área, masc.
    $table->integer('fuera_f')->default(0);   // Fuera del área, fem.
    $table->timestamps();
    $table->unique(
        ['centro_salud_id', 'mes', 'anio', 'tipo_vacuna', 'grupo_etareo'],
        'pv_centro_mes_anio_vacuna_grupo_unique'
    );
});

Nota: tipo_vacuna aquí es string en vez de enum. Esto da más flexibilidad para futuros tipos sin alterar la tabla.

6 prest_partos

Totales de partos mensuales por tipo, lugar, atendido_por y grupo etáreo.

database/migrations/XXXX_create_prest_partos_table.php php 
Schema::create('prest_partos', function (Blueprint $table) {
    $table->id();
    $table->foreignId('centro_salud_id')->constrained('centros_salud');
    $table->tinyInteger('mes');
    $table->year('anio');
    $table->enum('tipo', ['vaginal', 'cesarea']);
    $table->enum('lugar', ['servicio', 'domicilio']);
    $table->enum('atendido_por', [
        'personal_calificado', 'partera_empirica',
        'partera_capacitada', 'articulacion', 'otros',
    ]);
    $table->enum('grupo_etareo', [
        'menor_10', '10_14', '15_19',
        '20_34', '35_49', '50_mas',
    ]);
    $table->integer('cantidad')->default(0);
    $table->timestamps();
});

Esta tabla tiene más dimensiones (tipo × lugar × atendido_por × grupo_etáreo), por lo que una sola columna cantidad basta. Cada combinación única genera una fila.

7 prest_puerperio

Totales de controles puerperales: cuántos se hicieron a las 48h, 7 días, 28 días y 42 días.

database/migrations/XXXX_create_prest_puerperio_table.php php 
Schema::create('prest_puerperio', function (Blueprint $table) {
    $table->id();
    $table->foreignId('centro_salud_id')->constrained('centros_salud');
    $table->tinyInteger('mes');
    $table->year('anio');
    $table->enum('tipo_control', ['48h', '7dias', '28dias', '42dias']);
    $table->integer('cantidad')->default(0);
    $table->timestamps();
    $table->unique(
        ['centro_salud_id', 'mes', 'anio', 'tipo_control'],
        'ppuer_centro_mes_anio_control_unique'
    );
});

La más simple de las prest_*: solo 4 filas por centro/mes (una por cada tipo de control).

8 prest_crecimiento

Totales mensuales de controles de crecimiento infantil por grupo etáreo y dentro/fuera.

database/migrations/XXXX_create_prest_crecimiento_table.php php 
Schema::create('prest_crecimiento', function (Blueprint $table) {
    $table->id();
    $table->foreignId('centro_salud_id')->constrained('centros_salud');
    $table->tinyInteger('mes');
    $table->year('anio');
    $table->enum('grupo_etareo', [
        'menor_1_dentro', 'menor_1_fuera',
        '1_menor_2_dentro', '1_menor_2_fuera',
        '2_menor_5_dentro', '2_menor_5_fuera',
    ]);
    $table->integer('nuevos_m')->default(0);
    $table->integer('nuevos_f')->default(0);
    $table->integer('repetidos_m')->default(0);
    $table->integer('repetidos_f')->default(0);
    $table->timestamps();
    $table->unique(
        ['centro_salud_id', 'mes', 'anio', 'grupo_etareo'],
        'pcrec_centro_mes_anio_grupo_unique'
    );
});

El grupo_etareo aquí combina edad + dentro/fuera en un solo valor. Es un diseño compacto que simplifica las consultas SNIS.

✅ Resumen de la Lección 10

  • Las tablas prest_* almacenan totales mensuales agregados para informes SNIS
  • Patrón: centro_salud_id + mes + anio + dimensiones con constraint UNIQUE compuesto
  • prest_consulta_externa: por grupo etáreo, desglose primera/nueva/repetida × M/F
  • prest_vacunas: por tipo vacuna y grupo etáreo, desglose dentro/fuera × M/F
  • prest_prenatales: por tipo control y grupo etáreo, desglose dentro/fuera
  • prest_partos: por tipo/lugar/atendido_por/grupo, columna cantidad
  • prest_puerperio: por tipo de control (48h, 7d, 28d, 42d)
  • prest_crecimiento: por grupo etáreo+dentro/fuera, desglose nuevos/repetidos × M/F
  • Total de tablas: 20 (6 base + 8 clínicas + 6 prestaciones)
