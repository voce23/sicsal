2 Formulario con FileUpload, Select y Toggle

app/Filament/Pages/ImportarDesdeVes.php — form()
class ImportarDesdeVes extends Page
{
    protected static string|\UnitEnum|null $navigationGroup
        = 'Transferencias';
    protected static ?int $navigationSort = 10;

    public ?array $data = [];
    public ?array $resultado = null;
    public bool $importando = false;

    public function form(Schema $schema): Schema
    {
        return $schema->schema([
            FileUpload::make('archivo_ves')
                ->label('Archivo .ves del SNIS')
                ->maxSize(50 * 1024)  // 50 MB
                ->disk('local')
                ->directory('ves_uploads')
                ->visibility('private')
                ->required()
                ->columnSpanFull(),

            Select::make('solo_centro')
                ->label('Importar solo este centro (opcional)')
                ->options(
                    CentroSalud::whereNotNull('codigo_snis')
                        ->where('activo', true)
                        ->pluck('nombre', 'codigo_snis'))
                ->searchable()->nullable(),

            Toggle::make('limpiar_antes')
                ->label('Limpiar datos existentes del mes')
                ->default(false),
        ])->statePath('data');
    }
}
FileUpload con disco privado

disk('local') y visibility('private') almacenan el .ves en storage/app/ves_uploads/, NO accesible desde la web. Después de importar exitosamente, el archivo se elimina con Storage::disk('local')->delete(). Seguridad: los archivos .ves contienen datos sensibles de pacientes.

statePath('data')

Cuando el form de una Page tiene statePath('data'), todos los campos se almacenan dentro de $this->data['archivo_ves'], $this->data['solo_centro'], etc. Sin statePath, cada campo sería una propiedad pública directa.

3 ejecutarImportacion() — Artisan::call desde la UI

ImportarDesdeVes.php — lógica principal
public function ejecutarImportacion(): void
{
    $formData = $this->form->getState();
    $archivoRelativo = $formData['archivo_ves'];

    // Ruta absoluta en disco local
    $archivoAbsoluto = Storage::disk('local')
        ->path($archivoRelativo);

    $this->importando = true;

    try {
        $args = ['--archivo' => $archivoAbsoluto];
        if ($formData['limpiar_antes'])
            $args['--limpiar'] = true;
        if ($formData['solo_centro'])
            $args['--solo-centro'] = $formData['solo_centro'];

        $exitCode = \Artisan::call('snis:importar-ves', $args);
        $output = \Artisan::output();

        $stats = $this->leerStatsImportados();

        if ($exitCode === 0) {
            $this->resultado = [
                'exito' => true,
                'stats' => $stats,
                'total' => array_sum($stats),
            ];
            // Limpiar archivo subido
            Storage::disk('local')->delete($archivoRelativo);
            $this->form->fill();
        } else {
            $this->resultado = [
                'exito' => false,
                'output' => $output,
            ];
        }
    } catch (\Throwable $e) {
        $this->resultado = [
            'exito' => false,
            'output' => $e->getMessage(),
        ];
    } finally {
        $this->importando = false;
    }
}
Artisan::call() — ejecutar comando desde la web

\Artisan::call('snis:importar-ves', $args) ejecuta el mismo comando de consola (lección 55) pero de forma sincrónica dentro del request HTTP. Es un patrón útil para ofrecer la misma funcionalidad desde la terminal y desde la UI. El exitCode indica 0 = éxito.

private function leerStatsImportados(): array
{
    return [
        'consulta_externa' => \DB::table('prest_consulta_externa')->count(),
        'referencias'      => \DB::table('prest_referencias')->count(),
        'odontologia'      => \DB::table('prest_odontologia')->count(),
        // ... 9 tablas más ...
    ];
}

4 Vista Blade — formulario y loading states

importar-desde-ves.blade.php — info + form + botón
<x-filament-panels::page>
  <!-- Info box -->
  <div class="... border-blue-200 bg-blue-50 ...">
    <ol>
      <li>Centro genera archivo <strong>.ves</strong></li>
      <li>Envía por USB/correo/WhatsApp</li>
      <li>Suba el archivo y haga clic Importar</li>
      <li>Los datos se consolidan automáticamente</li>
    </ol>
  </div>

  <!-- Formulario Filament -->
  {{ $this->form }}

  <!-- Botón con loading state -->
  <x-filament::button
      wire:click="ejecutarImportacion"
      color="success"
      wire:loading.attr="disabled"
      wire:loading.class="opacity-50 cursor-not-allowed"
      wire:target="ejecutarImportacion">
    <span wire:loading.remove wire:target="ejecutarImportacion">
      Importar desde .ves
    </span>
    <span wire:loading wire:target="ejecutarImportacion">
      Importando, espere...
    </span>
  </x-filament::button>
wire:loading — UX durante la importación
  • wire:loading.attr="disabled" — desactiva el botón mientras se procesa
  • wire:loading.class="opacity-50" — lo hace semi-transparente
  • wire:loading.remove / wire:loading — alterna entre "Importar" y "Importando, espere..."
  • wire:target="ejecutarImportacion" — solo reacciona a esa acción específica

@forelse / @empty — lista con fallback

@forelse funciona como @foreach pero con un bloque @empty que se muestra cuando la colección está vacía. Aquí muestra los centros con código SNIS, y si no hay ninguno, muestra un mensaje con link a la sección de configuración.

6 Resumen y cierre de Fase 6

Lo que aprendimos en esta lección

  • FileUpload con disk('local') y visibility('private') — archivos sensibles fuera de la web
  • statePath('data') — agrupar campos del form en un array
  • Artisan::call() — ejecutar comandos de consola desde la UI web
  • Storage::disk('local')->delete() — limpiar archivos temporales
  • wire:loading + wire:target — UX durante operaciones largas
  • $this->form->fill() — resetear formulario después de éxito
  • Grid responsivo de estadísticas por tabla (grid-cols-5)
  • @forelse / @empty — lista con mensaje de fallback
