1 ¿Qué es un Trait en PHP?

Un trait es un mecanismo de reutilización de código. Es como copiar y pegar métodos en una clase, pero sin duplicar código. Se usa con use NombreDelTrait;.

Concepto de Trait php 
trait MiTrait {
    public function saludo() { return 'Hola'; }
}

class ClaseA { use MiTrait; } // ClaseA tiene saludo()
class ClaseB { use MiTrait; } // ClaseB también

2 HasCentroSelector — aislamiento de datos por centro

Este es el trait más importante del proyecto. Garantiza que cada usuario solo vea datos de su centro de salud.

app/Concerns/HasCentroSelector.php php 
use Livewire\Attributes\Url;

trait HasCentroSelector
{
    #[Url]
    public int $centroSaludId = 0;

    public function mountHasCentroSelector(): void
    {
        $user = auth()->user();

        if ($this->centroSaludId === 0) {
            $this->centroSaludId = $user?->centro_salud_id ?? 1;
        }

        // Forzar centro asignado para no-superadmins
        if ($user && ! $user->hasRole('superadmin') && $user->centro_salud_id) {
            $this->centroSaludId = (int) $user->centro_salud_id;
        }
    }
}
⚠️
#[Url] — sincronización con la URL
El atributo #[Url] de Livewire sincroniza $centroSaludId con el query string: ?centroSaludId=3. Pero el trait impide que un no-superadmin cambie este valor por URL — siempre se fuerza al centro asignado.

Protección al actualizar

Computed properties

HasCentroSelector — propiedades computadas php 
// Centros disponibles: solo superadmin ve la lista
public function getCentrosDisponiblesProperty(): array
{
    $user = auth()->user();
    if (! $user || ! $user->hasRole('superadmin')) {
        return [];
    }
    return CentroSalud::where('activo', true)
        ->orderBy('nombre')
        ->pluck('nombre', 'id')->toArray();
}

// Helper: ¿es superadmin?
public function getEsSuperadminProperty(): bool
{
    return auth()->user()?->hasRole('superadmin') ?? false;
}

// Acceso al ID actual desde cualquier componente
protected function getCentroId(): int
{
    return $this->centroSaludId;
}
💡
Uso en Blade
En la vista: $this->centrosDisponibles (nota: sin get/Property). Si el usuario es superadmin, renderiza un dropdown. Si no, no muestra nada.

3 PasswordValidationRules — reglas de password

app/Concerns/PasswordValidationRules.php php 
use Illuminate\Validation\Rules\Password;

trait PasswordValidationRules
{
    protected function passwordRules(): array
    {
        return ['required', 'string', Password::default(), 'confirmed'];
    }

    protected function currentPasswordRules(): array
    {
        return ['required', 'string', 'current_password'];
    }
}

Password::default() usa las reglas globales definidas en AppServiceProvider. En producción esto incluye: mínimo 8 caracteres, mixtas mayúsculas/minúsculas, números y símbolos.

Este trait se usa en:

  • CreateNewUser → $this->passwordRules()
  • ResetUserPassword → $this->passwordRules()

4 ProfileValidationRules — reglas de perfil

app/Concerns/ProfileValidationRules.php php 
trait ProfileValidationRules
{
    protected function profileRules(?int $userId = null): array
    {
        return [
            'name'  => $this->nameRules(),
            'email' => $this->emailRules($userId),
        ];
    }

    protected function nameRules(): array
    {
        return ['required', 'string', 'max:255'];
    }

    protected function emailRules(?int $userId = null): array
    {
        return [
            'required', 'string', 'email', 'max:255',
            $userId === null
                ? Rule::unique(User::class)
                : Rule::unique(User::class)->ignore($userId),
        ];
    }
}
💡
Rule::unique + ignore — edición sin conflicto
Al editar un perfil, el email del usuario actual debe ser válido aunque ya exista (es el suyo). Rule::unique()->ignore($userId) excluye al usuario actual de la verificación de unicidad.
