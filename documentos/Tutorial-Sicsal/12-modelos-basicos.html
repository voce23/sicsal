1 ¿Qué es un modelo Eloquent?

Un modelo es una clase PHP que representa una tabla de la base de datos. Con Eloquent (el ORM de Laravel):

  • Cada fila de la tabla es un objeto del modelo
  • Cada columna es una propiedad del objeto
  • Las relaciones entre tablas se definen como métodos
  • No escribes SQL — usas métodos como find(), create(), where()

ℹ️
Convención: nombre singular
La tabla se llama municipios (plural), el modelo Municipio (singular). Laravel asume automáticamente que el modelo Municipio usa la tabla municipios. Cuando los nombres no siguen esta convención (como centros_salud), debes especificarlo manualmente con $table.

3 Modelo Municipio

El más sencillo de los tres: nombre, departamento y activo.

app/Models/Municipio.php php 
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\HasMany;

class Municipio extends Model
{
    protected $fillable = ['nombre', 'departamento', 'activo'];

    protected function casts(): array
    {
        return ['activo' => 'boolean'];
    }

    public function centrosSalud(): HasMany
    {
        return $this->hasMany(CentroSalud::class);
    }
}

Desglosemos cada parte:

1
$fillable — Lista de columnas que se pueden asignar masivamente con create() o fill(). Es una protección contra mass assignment: solo estas columnas se aceptan.
2
casts() — En Laravel 12+, usamos un método en vez de la propiedad $casts. Aquí convertimos activo de 0/1 en la BD a true/false en PHP.
3
centrosSalud() — Relación HasMany: un municipio tiene muchos centros de salud. Laravel busca automáticamente la columna municipio_id en la tabla del modelo relacionado.
⚠️
¿Por qué no incluimos 'id' en $fillable?
El id es auto-incremental — Laravel lo genera automáticamente. Incluirlo en $fillable sería un riesgo de seguridad: un atacante podría asignar un ID específico.

4 Modelo CentroSalud

Este modelo es más complejo: tiene $table personalizado, múltiples casts, y 5 relaciones.

app/Models/CentroSalud.php php 
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;

class CentroSalud extends Model
{
    protected $table = 'centros_salud';

    protected $fillable = [
        'municipio_id', 'nombre', 'codigo_snis',
        'subsector', 'red_salud', 'poblacion_ine', 'activo',
        'latitud', 'longitud',
    ];

    protected function casts(): array
    {
        return [
            'activo'        => 'boolean',
            'poblacion_ine' => 'integer',
            'latitud'       => 'float',
            'longitud'      => 'float',
        ];
    }

    public function municipio(): BelongsTo
    {
        return $this->belongsTo(Municipio::class);
    }

    public function comunidades(): HasMany
    {
        return $this->hasMany(Comunidad::class);
    }

    public function personas(): HasMany
    {
        return $this->hasMany(Persona::class);
    }

    public function metasIne(): HasMany
    {
        return $this->hasMany(MetaIne::class);
    }

    public function usuarios(): HasMany
    {
        return $this->hasMany(User::class);
    }
}

Conceptos nuevos

1
$table = 'centros_salud' — Como el nombre de la tabla (centros_salud) no coincide con la convención automática (centro_saluds), debemos especificarlo manualmente.
2
BelongsTo — Es la inversa de HasMany. Un centro de salud pertenece a un municipio. Laravel busca la columna municipio_id en esta misma tabla.
3
Múltiples HasMany — Un centro tiene comunidades, personas, metas INE y usuarios. Cada relación es un método que devuelve HasMany.
💡
BelongsTo vs HasMany: ¿quién tiene la FK?
BelongsTo = "Yo tengo la columna municipio_id en mi tabla"
HasMany = "La otra tabla tiene mi centro_salud_id"
La FK siempre está en el lado de BelongsTo.

5 Modelo Comunidad

app/Models/Comunidad.php php 
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;

class Comunidad extends Model
{
    protected $table = 'comunidades';

    protected $fillable = [
        'centro_salud_id', 'nombre', 'distancia_km',
        'latitud', 'longitud', 'activo',
    ];

    protected function casts(): array
    {
        return [
            'activo'       => 'boolean',
            'distancia_km' => 'decimal:2',
            'latitud'      => 'decimal:7',
            'longitud'     => 'decimal:7',
        ];
    }

    public function centroSalud(): BelongsTo
    {
        return $this->belongsTo(CentroSalud::class);
    }

    public function personas(): HasMany
    {
        return $this->hasMany(Persona::class);
    }
}

Casts avanzados: decimal:N

A diferencia de CentroSalud que usa 'float', Comunidad usa 'decimal:7'. La diferencia:

Cast Tipo PHP Precisión Ejemplo
'float' float ~15 dígitos (IEEE 754) -17.78350067138672
'decimal:7' string Exactamente 7 decimales -17.7835000
ℹ️
¿Cuándo usar decimal vs float?
Usa decimal:N cuando necesitas precisión exacta (coordenadas GPS, dinero). Usa float cuando la aproximación es aceptable. En SICSAL, las coordenadas de comunidades son más precisas porque se usan para calcular distancias.

7 Probando con Tinker

Tinker es la consola interactiva de Laravel. Te permite ejecutar código PHP en el contexto de tu aplicación:

Terminal — Tinker bash 
$ php artisan tinker

// Crear un municipio
>>> $mun = App\Models\Municipio::create([
        'nombre' => 'Hornoma',
        'departamento' => 'La Paz',
        'activo' => true,
    ]);
=> App\Models\Municipio {id: 1, nombre: "Hornoma", ...}

// Crear un centro de salud DENTRO de ese municipio
>>> $centro = App\Models\CentroSalud::create([
        'municipio_id' => $mun->id,
        'nombre' => 'Centro de Salud Hornoma',
        'codigo_snis' => '020301',
        'activo' => true,
    ]);

// Acceder a la relación
>>> $mun->centrosSalud;
=> Collection {#items: [CentroSalud {id: 1, ...}]}

// Navegar en reversa
>>> $centro->municipio->nombre;
=> "Hornoma"

// Contar comunidades de un centro
>>> $centro->comunidades()->count();
=> 0
💡
$centro->comunidades vs $centro->comunidades()
Sin paréntesis: devuelve la colección de resultados (ejecuta la query).
Con paréntesis: devuelve el builder de la query (puedes agregar ->where(), ->count(), etc.).

✅ Resumen de la Lección 12

  • Modelo = clase PHP que representa una tabla de la BD
  • $fillable — protección contra mass assignment (lista de columnas permitidas)
  • casts() — convierte tipos de BD a tipos PHP (boolean, integer, float, decimal:N)
  • $table — necesario cuando el nombre de la tabla no sigue la convención plural
  • HasMany — "tengo muchos" (la FK está en la otra tabla)
  • BelongsTo — "pertenezco a" (la FK está en mi tabla)
  • Tinker — consola interactiva para probar modelos y relaciones
