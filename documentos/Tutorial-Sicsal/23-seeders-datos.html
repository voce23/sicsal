1 Propósito y diferencia con los seeders base

En la lección 22 vimos los 7 seeders que configuran la base del sistema. DatosPruebaSeeder es diferente:

DatosPruebaSeeder crea datos ficticios para poder probar reportes, gráficas y exportaciones sin esperar datos reales. Nunca lo ejecutes en producción.

4 Metas INE por porcentaje

El MetaIneSeeder de la lección 22 usa cantidades fijas (porque son datos oficiales de Hornoma). Pero para los 6 centros adicionales, DatosPruebaSeeder calcula las metas como porcentaje de la población:

    '15_19'   => 0.1030,   '20_39'   => 0.2793,
    '40_49'   => 0.0983,   '50_59'   => 0.0765,

Fórmula del cálculo

Para cada grupo: cantidad = max(1, round(poblacion_ine × porcentaje)). Ejemplo: Hospital Capinota (pop. 12,811) → menor_1 = round(12811 × 0.0187) = 240. Para HORNOMA (pop. 641) → round(641 × 0.0187) = 12 (coincide con el seeder base).

Borra y recrea

A diferencia de los seeders base que usan firstOrCreate, seedMetaIne() primero borra todas las metas del centro para el año, y luego las recrea con MetaIne::create(). Esto permite re-ejecutar el seeder con datos actualizados.

6 Lógica de prestaciones (15 tablas)

El método seedPrestaciones() calcula metas derivadas y luego invoca 15 sub-seeders para cada mes:

seedPrestaciones() — estructura
private function seedPrestaciones(CentroSalud $cs): void
{
    $cob        = self::$coberturaFactor[$cs->nombre] ?? 0.30;
    $pop        = $cs->poblacion_ine;
    $metaMenor1 = max(1, round($pop * 0.0187));  // ≈ meta < 1 año
    $metaMef    = max(1, round($pop * 0.2387));  // mujeres edad fértil
    $metaEmb    = max(1, round($pop * 0.0234));  // embarazos esperados
    $metaPartos = max(1, round($pop * 0.0187));  // partos esperados

    foreach ($this->meses as $mes) {
        $base = ['centro_salud_id' => $cs->id, 'anio' => 2026, 'mes' => $mes];

        $this->seedVacunas($base, $metaMenor1, $meta1anio, $cob);
        $this->seedConsultas($base, $pop, $cob);
        $this->seedMicronutrientes($base, $metaMenor1, $cob);
        $this->seedPrenatales($base, $metaEmb, $cob);
        $this->seedPartos($base, $metaPartos, $cob, $cs->nombre);
        $this->seedPuerperio($base, $metaPartos, $cob);
        $this->seedRecienNacidos($base, $metaPartos, $cob);
        $this->seedCrecimiento($base, $metaMenor1, $cob);
        $this->seedAnticoncepcion($base, $metaMef, $cob);
        $this->seedActividades($base, $cob);
        $this->seedReferencias($base, $pop, $cob);

        $this->seedCancer($base, $metaMef, $cob);

private function seedPartos(array $b, int $metaPartos, float $cob, string $centro): void

    $esHospital = str_contains($centro, 'Hospital');

}

Uso de DB::table() en vez de Eloquent

Nota que el seeder usa DB::table()->insert() en vez de modelos Eloquent. Es más rápido para inserciones masivas — evita instanciar modelos y disparar observers/events para miles de registros de prueba.

7 Variación aleatoria realista

Para que los datos no sean idénticos cada mes, se usa la función r():

Función r() — variación ±20%
private function r(float $base): int
{
    return max(0, (int) round($base * (0.8 + mt_rand(0, 40) / 100)));
}
Cómo funciona

mt_rand(0, 40) / 100 genera un decimal entre 0.00 y 0.40. Sumado a 0.8, da un multiplicador entre 0.80 y 1.20. Así, un valor base de 10 puede variar entre 8 y 12. El max(0, ...) garantiza que nunca sea negativo.

8 Método limpiar()

Para revertir todo lo que creó el seeder, hay un método estático limpiar():

DatosPruebaSeeder::limpiar()
public static function limpiar(): void
{
    $anio = 2026;
    $meses = [1, 2, 3, 4];

    $tablas = [
        'prest_vacunas', 'prest_consulta_externa', 'prest_micronutrientes',
        'prest_prenatales', 'prest_partos', 'prest_puerperio',
        'prest_recien_nacidos', 'prest_crecimiento', 'prest_anticoncepcion',
        'prest_actividades_comunidad', 'prest_referencias',
        'prest_odontologia', 'prest_enfermeria',
        'prest_internaciones', 'prest_cancer_prevencion',
    ];

    foreach ($tablas as $t) {
        $del = DB::table($t)->where('anio', $anio)
            ->whereIn('mes', $meses)->delete();
    }

    // Eliminar metas INE del 2026
    MetaIne::where('anio', $anio)->delete();

    // Eliminar centros creados (todos menos Hornoma)
    $codigos = ['300180', '300181', '300182', '300184', '300185', '300186'];
    CentroSalud::whereIn('codigo_snis', $codigos)->get()->each->delete();
}
No borra Hornoma

limpiar() elimina los 6 centros que creó el seeder (300180-300186 excepto 300183), sus prestaciones y metas. El centro HORNOMA (300183) y sus datos base se mantienen intactos porque fueron creados por los seeders base.

10 Resumen de Fase 5 completa

Lo que aprendimos en esta lección

  • DatosPruebaSeeder es un seeder separado, manual — no se ejecuta con db:seed
  • Crea 6 centros adicionales con sus metas calculadas por porcentaje de población
  • Usa un factor de cobertura diferente por centro (0.15 a 0.85)
  • Genera datos para 15 tablas prest_* × 4 meses × 7 centros
  • La función r() aplica variación aleatoria ±20% para datos realistas
  • Usa DB::table()->insert() por rendimiento (no Eloquent)
  • Incluye limpiar() para revertir todos los datos de prueba
  • Lógica condicional: partos institucionales vs. domiciliarios según tipo de centro
