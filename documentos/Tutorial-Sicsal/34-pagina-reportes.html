2 Filtros globales con wire:model.live

Reportes.php — propiedades + mount()
class Reportes extends Page
{
    protected string $view = 'filament.pages.reportes';
    protected static string|\UnitEnum|null $navigationGroup = 'Informes';
    protected static ?string $title = 'Reportes y Descargas';

    public int $centroSaludId = 0;
    public int $anio;
    public string $periodo = 'cai1';
    public int $mesCausas = 0; // 0 = todo el año

    public function mount(): void
    {
        $this->anio = (int) date('Y');
        $centros = $this->getCentros();
        if (count($centros) > 0) {
            $this->centroSaludId = (int) array_key_first($centros);
        }
    }

    public function getCentros(): array
    {
        $user = auth()->user();
        if ($user?->hasRole('super_admin')) {
            return CentroSalud::orderBy('nombre')
                ->pluck('nombre', 'id')->toArray();
        }
        // Admin/registrador → solo su centro
        $centro = CentroSalud::find($user->centro_salud_id);
        return $centro ? [$centro->id => $centro->nombre] : [];
    }
}

wire:model.live en selects nativos

No usamos componentes Filament para los filtros — son <select> HTML nativos con wire:model.live. Al cambiar cualquier selector, las Actions se re-evalúan. Si centroSaludId = 0, todos los botones quedan disabled.

3 Informe CAI — Excel, PDF y PPTX

Reportes.php — 3 Actions para Informe CAI
// ① Excel
public function descargarCaiExcelAction(): Action
{
    return Action::make('descargarCaiExcel')
        ->label('Excel')
        ->icon('heroicon-o-table-cells')
        ->color('success')
        ->disabled(fn () => !$this->centroSaludId)
        ->action(function () {
            $datos = CaiHelper::getDatosInforme(
                $this->centroSaludId,
                $this->periodo,
                $this->anio);
            return Excel::download(
                new InformeCAIExport($datos), $nombre);
        });
}

// ② PDF
public function descargarCaiPdfAction(): Action
{
    return Action::make('descargarCaiPdf')
        ->label('PDF')->color('danger')
        ->action(function () {
            $pdf = Pdf::loadView('pdf.informe-cai', [...])
                ->setPaper('a4', 'portrait');
            return response()->streamDownload(
                fn () => print($pdf->output()), $nombre);
        });
}

// ③ PowerPoint
public function descargarCaiPptxAction(): Action
{
    return Action::make('descargarCaiPptx')
        ->label('PowerPoint')->color('info')
        ->action(function () {
            $pptx = new PresentacionCAI(...);
            $tmpFile = $pptx->generate();
            return response()->streamDownload(
                function () use ($tmpFile) {
                    readfile($tmpFile);
                    @unlink($tmpFile);
                }, $nombre, [
                'Content-Type' => 'application/vnd.openxmlformats-...',
            ]);
        });
}
Tres patrones de descarga
  • Excel: Excel::download(new Export($datos), $nombre) — Maatwebsite genera el .xlsx
  • PDF: Pdf::loadView(...)->output() + streamDownload() — DomPDF genera el PDF
  • PPTX: clase custom PresentacionCAI->generate() → archivo temporal → readfile() + unlink()

4 Causas de Consulta — con filtro adicional de mes

Reportes.php — Actions de Causas
public function descargarCausasExcelAction(): Action
{
    return Action::make('descargarCausasExcel')
        ->label('Excel')->color('success')
        ->action(function () {
            $datos = CausasConsultaHelper::getTop10(
                $this->centroSaludId,
                $this->anio,
                $this->mesCausas  // 0 = año completo
            );
            return Excel::download(
                new CausasConsultaExternaExport($datos),
                $nombre);
        });
}
mesCausas = 0 → todo el año

La sección de Causas tiene un filtro de mes propio ($mesCausas). Cuando es 0, el helper calcula sobre la gestión completa. Esto permite descargar reportes mensuales o anuales desde la misma interfaz.

5 getDatosComunidades() — cálculos poblacionales

El método más complejo de la página (150+ líneas). Calcula población por comunidad, grupo etáreo, sexo y compara con metas INE.

Reportes.php — constantes y estructura
private const GRUPOS = [
    'menor_1' => '<1', '1_4' => '1-4', '5_9' => '5-9',
    '10_14' => '10-14', '15_19' => '15-19',
    '20_39' => '20-39', '40_49' => '40-49',
    '50_59' => '50-59', 'mayor_60' => '60+',
];

private const RATIO_MASCULINIDAD = [
    'menor_1' => 1.05, '1_4' => 1.05,
    '5_9' => 1.04, '10_14' => 1.04,
    '15_19' => 1.02, '20_39' => 0.96,
    '40_49' => 0.94, '50_59' => 0.92,
    'mayor_60' => 0.88,
];
RATIO_MASCULINIDAD — distribución por sexo del INE

Cuando la meta INE viene como "ambos sexos", se usa el ratio de masculinidad por grupo etáreo para estimar hombres y mujeres. Ejemplo: ratio 1.05 para menores de 1 año → 51.2% hombres, 48.8% mujeres.

// Consolidado: comparar real vs INE
foreach ($grupos as $g) {
    $consolidado[] = [
        'ine_total'  => $ineM + $ineF,
        'real_total' => $realM + $realF,
        'diferencia' => $realTotal - $ineTotal,

clone $base — reutilizar queries sin mutar

Se construye un $base query y se clona para cada operación. Sin clone, cada ->where() adicional modificaría el query original. Patrón necesario cuando se hacen múltiples conteos sobre la misma base de datos.

{{ $this->actionName }} — renderizar Actions en Blade

Las Actions definidas como descargarCaiExcelAction() se renderizan como {{ $this->descargarCaiExcelAction }} (sin paréntesis). Filament las convierte en botones automáticamente. El bloque <x-filament-actions::modals /> al final es obligatorio para los modales de confirmación.
