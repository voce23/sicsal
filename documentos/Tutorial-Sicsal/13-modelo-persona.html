1 Estructura general del modelo

El modelo Persona representa a los habitantes del padrón comunitario. Es el más largo del proyecto (~170 líneas) y el que mejor ilustra las capacidades de Eloquent.

app/Models/Persona.php — imports php 
<?php

namespace App\Models;

use Carbon\Carbon;
use Illuminate\Database\Eloquent\Casts\Attribute;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;
use Illuminate\Database\Eloquent\Relations\HasOne;

Nuevos imports respecto a los modelos anteriores:

  • Carbon — biblioteca para manejar fechas (ya incluida en Laravel)
  • Attribute — clase para definir accessors/mutators modernos
  • HasOne — relación "tiene uno" (para defunción)

2 $fillable y casts

app/Models/Persona.php php 
protected $fillable = [
    'centro_salud_id', 'comunidad_id', 'nombres', 'apellidos',
    'fecha_nacimiento', 'sexo', 'ci', 'tipo_seguro', 'estado',
    'destino_migracion', 'fecha_migracion', 'grupo_etareo',
    'fecha_registro', 'observaciones', 'activo', 'created_by',
    'verificado', 'fecha_verificacion', 'verificado_por',
];

protected function casts(): array
{
    return [
        'fecha_nacimiento'    => 'date',
        'fecha_migracion'     => 'date',
        'fecha_registro'      => 'date',
        'activo'              => 'boolean',
        'verificado'          => 'boolean',
        'fecha_verificacion'  => 'date',
    ];
}
ℹ️
Cast 'date' vs 'datetime'
'date' devuelve un objeto Carbon sin hora (solo Y-m-d). 'datetime' incluye hora. Usamos 'date' porque para las personas solo necesitamos la fecha, no el minuto exacto en que nacieron.

Nótese que hay 19 columnas en $fillable — muchas más que Municipio (3) o Comunidad (6). Esto refleja la complejidad del registro de una persona en el sistema de salud.

3 6 relaciones

app/Models/Persona.php — relaciones php 
// ─── BelongsTo (3) ───
public function centroSalud(): BelongsTo
{
    return $this->belongsTo(CentroSalud::class);
}

{
    return $this->belongsTo(Comunidad::class);

{
    return $this->belongsTo(User::class, 'created_by');

Conceptos nuevos

1
belongsTo(User::class, 'created_by') — Cuando la FK no sigue la convención (user_id), necesitas pasarla como segundo argumento. Aquí la columna se llama created_by en vez de user_id.
2
HasOne — Similar a HasMany, pero devuelve un solo objeto (o null). Una persona solo puede tener una defunción.

4 Mutators: normalizar nombres al guardar

Un mutator transforma el valor antes de guardarlo en la BD. En SICSAL, todos los nombres se guardan en formato Title Case:

app/Models/Persona.php — mutators php 
protected function nombres(): Attribute
{
    return Attribute::make(
        set: fn (string $value) =>
            mb_convert_case(mb_strtolower($value), MB_CASE_TITLE, 'UTF-8'),
    );
}

protected function apellidos(): Attribute
{
    return Attribute::make(
        set: fn (string $value) =>
            mb_convert_case(mb_strtolower($value), MB_CASE_TITLE, 'UTF-8'),
    );
}
"JUAN PÉREZ"
→ mb_strtolower →
"juan pérez"
→ MB_CASE_TITLE →
"Juan Pérez"
→ BD
ℹ️
¿Por qué mb_ y no strtolower/ucwords?
Las funciones mb_* manejan correctamente caracteres UTF-8 como tildes (á, é, í) y eñe (ñ). Las funciones nativas de PHP sin mb_ pueden corromper estos caracteres.

5 Accessors: propiedades computadas

Un accessor genera un valor al leer el modelo, sin almacenar nada en la BD. Es una propiedad virtual.

app/Models/Persona.php — accessors php 
// $persona->edad → 34
public function getEdadAttribute(): int
{
    return Carbon::parse($this->fecha_nacimiento)->age;
}

// $persona->edad_meses → 408
public function getEdadMesesAttribute(): int
{
    return Carbon::parse($this->fecha_nacimiento)->diffInMonths(now());
}

// $persona->nombre_completo → "Juan Pérez"
public function getNombreCompletoAttribute(): string
{
    return "{$this->nombres} {$this->apellidos}";
}

// $persona->grupo_etareo_calculado → "20_39"
public function getGrupoEtareoCalculadoAttribute(): string
{
    $meses = $this->edad_meses;
    $anios = $this->edad;

    if ($meses < 6)  return 'menor_6m';
    if ($meses < 12) return '6m_menor_1';
    if ($anios < 5)  return '1_4';
    if ($anios < 10) return '5_9';
    if ($anios < 15) return '10_14';
    if ($anios < 20) return '15_19';
    if ($anios < 40) return '20_39';
    if ($anios < 50) return '40_49';
    if ($anios < 60) return '50_59';

    return 'mayor_60';
}
💡
Convención de nombres
Los accessors legados usan el patrón get{NombreEnCamelCase}Attribute. Laravel convierte automáticamente:
  • getEdadAttribute → $persona->edad
  • getEdadMesesAttribute → $persona->edad_meses
  • getGrupoEtareoCalculadoAttribute → $persona->grupo_etareo_calculado

¿Por qué calcular el grupo etáreo?

La tabla personas tiene una columna grupo_etareo, pero la edad cambia con el tiempo. El accessor grupo_etareo_calculado siempre devuelve el grupo actual basado en la fecha de nacimiento. Los rangos coinciden exactamente con los del formulario 301 del SNIS.

6 Scopes: filtros reutilizables

Un scope es un método que agrega un WHERE a las queries de Eloquent. Se definen con el prefijo scope y se usan sin él:

app/Models/Persona.php — scopes php 
// Filtros simples ──────────────────────────────────
public function scopeActivos($query)
{
    return $query->where('activo', true);
}

public function scopeResidentes($query)
{
    return $query->where('estado', 'residente');
}

public function scopeVerificados($query)
{
    return $query->where('verificado', true);
}

// Scope compuesto ──────────────────────────────────
public function scopePadronActivo($query)
{
    return $query->where('activo', true)
                 ->whereIn('estado', ['residente', 'temporal'])
                 ->where('verificado', true);
}

// Scopes por grupo etáreo/sexo ─────────────────────
public function scopeMenoresDe5($query)
{
    return $query->where('fecha_nacimiento', '>=', now()->subYears(5));
}

public function scopeMujeresMef($query)
{
    return $query->where('sexo', 'F')
        ->where('fecha_nacimiento', '<=', now()->subYears(15))
        ->where('fecha_nacimiento', '>=', now()->subYears(49));
}

Uso encadenado

Ejemplo — combinando scopes php 
// Contar menores de 5 activos y verificados de un centro
$total = Persona::where('centro_salud_id', $centroId)
    ->padronActivo()     // activo + residente/temporal + verificado
    ->menoresDe5()       // nacidos hace menos de 5 años
    ->count();

// Listar mujeres MEF migradas
$migradas = Persona::migrados()
    ->mujeresMef()
    ->get();
⚠️
Scopes vs queries directas
Sin scopes: Persona::where('activo', true)->where('estado', 'residente')->where('verificado', true)
Con scopes: Persona::padronActivo()
Mismo resultado, pero el scope es reutilizable, legible y centraliza la lógica de negocio.

💡
Patrón: centralizar lógica en el modelo
En vez de repetir where('activo', true)->where('estado', 'residente') en 10 archivos diferentes, lo definimos una vez como scope en el modelo. Si la regla cambia, solo la modificamos en un lugar.
