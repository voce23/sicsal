1 Crear el proyecto con Composer

Abre la terminal de Laragon (clic derecho → Terminal) y navega a la carpeta de proyectos:

Terminal bash 
$ cd C:\laragon\www

# Crear proyecto Laravel 13 con el starter kit de Livewire
$ composer create-project laravel/livewire-starter-kit sicsal

Creating a "laravel/livewire-starter-kit" project at "./sicsal"
Installing laravel/livewire-starter-kit (v1.x.x)
  - Downloading laravel/livewire-starter-kit (v1.x.x)
  ...
Application key set successfully.

Esto creará la carpeta C:\laragon\www\sicsal\ con Laravel 13, Livewire 4 y Flux preinstalados.

ℹ️
¿Qué es un Starter Kit?
Laravel ofrece starter kits preconfigurados. El livewire-starter-kit incluye autenticación (login/register), Livewire 4 y Flux (componentes UI). Es la base perfecta para SICSAL porque usamos Livewire para todo lo interactivo.

Entra al proyecto y abre VS Code:

Terminal bash 
$ cd sicsal
$ code .

2 ¿Por qué Livewire Starter Kit?

SICSAL usa estas tecnologías del starter kit:

Componente Versión Uso en SICSAL
Laravel Framework 13.x Backend completo: rutas, modelos, migraciones
Livewire 4.x Componentes reactivos (InformeCAI, Poblacion, Blog)
Flux 2.x Componentes UI (modals, dropdowns, navigation)
Fortify 1.x Autenticación (login, register, 2FA, verificar email)

Además de lo que trae el starter kit, agregaremos estos paquetes manualmente en la sección 8:

Paquete Para qué
filament/filament Panel de administración con CRUD
spatie/laravel-permission Roles y permisos (superadmin, admin, registrador)
maatwebsite/excel Exportar/importar Excel
barryvdh/laravel-dompdf Generar PDFs
phpoffice/phppresentation Generar presentaciones PowerPoint

3 Configurar el archivo .env

El archivo .env contiene la configuración del entorno (base de datos, correo, etc.). Nunca subas este archivo a Git — contiene datos sensibles.

Abre .env en VS Code y modifica estas líneas:

.env env 
APP_NAME=SICSAL
APP_ENV=local
APP_KEY=base64:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
APP_DEBUG=true
APP_URL=http://sicsal.test

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=sicsal
DB_USERNAME=root
DB_PASSWORD=
⚠️
Seguridad
Nunca escribas contraseñas reales en código ni en archivos que suban al repositorio. El archivo .env ya está listado en .gitignore por defecto — no lo quites de ahí. En producción, usarás una contraseña fuerte para la base de datos.

¿Qué significa cada variable?

Variable Significado
APP_NAME Nombre de la aplicación. Aparece en títulos y correos.
APP_ENV local en desarrollo, production en servidor real.
APP_KEY Clave de encriptación. Se genera automáticamente.
APP_DEBUG true muestra errores detallados. Siempre false en producción.
APP_URL URL base. Laragon crea sicsal.test automáticamente.
DB_DATABASE Nombre de la base de datos que crearemos en el siguiente paso.

4 Generar APP_KEY

Si APP_KEY está vacío (no debería, el instalador lo genera), ejecuta:

Terminal bash 
$ php artisan key:generate
INFO  Application key set successfully.

Este comando genera una clave aleatoria de 32 bytes (codificada en base64) y la escribe en .env. Laravel la usa para encriptar cookies, sesiones y tokens CSRF.

🚨
No compartas tu APP_KEY
Cada instalación debe tener su propia clave. Nunca copies una APP_KEY entre proyectos ni la publiques.

5 Crear la base de datos

Necesitamos crear la base de datos sicsal que configuramos en el .env.

Opción A: Desde terminal

Terminal bash 
$ mysql -u root
mysql> CREATE DATABASE sicsal CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
Query OK, 1 row affected
mysql> exit

Opción B: Desde Laragon (más fácil)

1
En Laragon, clic derecho → MySQL → Create database...
2
Escribe sicsal como nombre y presiona OK.
💡
Laragon auto-crea bases de datos
Laragon puede crear automáticamente una base de datos con el mismo nombre del proyecto cuando usas "Quick app" o creas un proyecto desde el menú. Si ya lo hiciste así, la BD sicsal ya existe.

Verifica que la BD existe:

Terminal bash 
$ php artisan db:show
  MySQL ........................................ 8.0.30
  Database ..................................... sicsal
  Tables ....................................... 0

6 Probar el servidor local

Primero, instalemos las dependencias de npm y compilemos los assets:

Terminal bash 
$ npm install
added 22 packages in 8s

$ npm run build
vite v8.0.0 building for production...
✓ 50 modules transformed.
✓ built in 3.20s

Ahora levanta el servidor de desarrollo. Hay dos formas:

Opción A: Artisan serve (básico)

Terminal bash 
$ php artisan serve
INFO  Server running on [http://127.0.0.1:8000].

Opción B: Composer dev (recomendado)

Este comando levanta el servidor, la cola de trabajos y Vite simultáneamente:

Terminal bash 
$ composer dev
[server] INFO  Server running on [http://127.0.0.1:8000].
[queue]  INFO  Processing jobs from the [default] queue.
[vite]   VITE v8.0.0  ready in 300 ms

Abre tu navegador y visita http://127.0.0.1:8000. Deberías ver la página de bienvenida de Laravel.

💡
¿Artisan serve o Apache de Laragon?
Con Laragon puedes usar directamente http://sicsal.test (el virtual host de Apache) sin necesidad de artisan serve. Pero necesitas tener npm run dev corriendo en otra terminal para que Vite compile cambios en tiempo real. El comando composer dev lanza todo junto.

7 Virtual host con Laragon

Laragon crea automáticamente un virtual host para cada carpeta en C:\laragon\www\. Como nuestro proyecto está en www\sicsal, automáticamente tenemos:

http://sicsal.test → apunta a C:\laragon\www\sicsal\public\

Si no funciona, verifica:

1
Laragon debe estar con Apache corriendo (indicador verde).
2
Clic derecho en Laragon → Apache → sites-enabled → verifica que existe auto.sicsal.test.conf.
3
Si acabas de crear el proyecto, reinicia Laragon (Stop All → Start All) para que genere el virtual host.

A partir de ahora usaremos http://sicsal.test como la URL del proyecto.

8 Instalar paquetes del proyecto

El starter kit trae Laravel, Livewire y Fortify. Necesitamos agregar los paquetes adicionales que usa SICSAL:

Terminal bash 
# Panel de administración
$ composer require filament/filament "~5.0"

# Roles y permisos
$ composer require spatie/laravel-permission

# Exportar Excel
$ composer require maatwebsite/excel

# Generar PDFs
$ composer require barryvdh/laravel-dompdf

# Generar PowerPoint
$ composer require phpoffice/phppresentation

También necesitamos Chart.js para gráficos (pirámide poblacional, coberturas):

Terminal bash 
$ npm install chart.js

Después de instalar Filament, ejecuta la instalación del panel:

Terminal bash 
$ php artisan filament:install --panels
 What is the ID? (e.g. app, admin):
 > admin

INFO  Filament panel [admin] created successfully.
ℹ️
Panel ID: admin
Usamos admin como ID del panel. Esto significa que el panel administrativo será accesible en http://sicsal.test/admin. Filament creará el archivo app/Providers/Filament/AdminPanelProvider.php que configuraremos más adelante.

Publica la configuración de Spatie Permission:

Terminal bash 
$ php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
Copied File [.../permission.php] To [/config/permission.php]
Copied File [.../create_permission_tables.php] To [/database/migrations/...]
