1 · Visión general: dos fuentes, un destino

El SNIS (Sistema Nacional de Información en Salud) genera datos mensuales en dos formatos que SICSAL puede importar:

FuenteFormatoComandoLíneas
Backup mensual (.ves) Cabinet → Access 97 (Jet 3.x) snis:importar-ves 762
SNIS instalado (carpeta local) Access .mdb vía ODBC snis:importar 645

Ambos comandos comparten la misma lógica de mapeo codsubvar → campos SIMUES y escriben en las mismas 12+ tablas de prestaciones.

2 · El formato codsubvar del SNIS

Cada dato del formulario 301 se identifica por un código de 9 dígitos:

// Formato codsubvar: FFFF GG VV SS
//   FFFF = código formulario (3010, 3011, etc.)
//   GG   = grupo (G01..G38)
//   VV   = variable (fila del formulario)
//   SS   = subvariable (columna: grupo etáreo, tipo, dentro/fuera)

// Ejemplo: 301010812
//   3010 = formulario 301
//   01   = G01 (Consulta Externa)
//   08   = variable 08 (mayor_60)
//   12   = subvariable 12 (menor_6m)

Cada fila del MDB tiene columnas V (varones) y M (mujeres). Los importadores extraen VV y SS con substr() y los cruzan contra arrays de constantes.

// Extracción del codsubvar (ambos commands)
$cs  = trim($row['codsubvar'] ?? '');
$var = substr($cs, 5, 2);  // posiciones 5-6 → VV
$sub = substr($cs, 7, 2);  // posiciones 7-8 → SS

3 · ImportarVes — firma y opciones

class ImportarVes extends Command
{
    protected $signature = 'snis:importar-ves
        {--archivo=  : Ruta completa al archivo .ves}
        {--limpiar   : Eliminar datos existentes del mes antes de importar}
        {--solo-centro= : Importar solo un centro (código SNIS, ej: 300183)}';

    protected $description = 'Importa un archivo .ves del SNIS (backup mensual consolidado) al SIMUES';
}
¿Qué es un .ves? — Es un archivo Cabinet (.cab) de Windows que contiene auditoria.text (hash MD5) y transfer.sql (un archivo Access 97 / Jet 3.x renombrado). Se extrae con expand.exe en Windows o cabextract en Linux.

El comando usa tres opciones:

OpciónRequeridaDescripción
--archivoSíRuta al archivo .ves
--limpiarNoElimina datos previos del mismo mes/año antes de importar
--solo-centroNoFiltra importación a un solo centro por código SNIS

5 · Extracción del .ves (Cabinet → MDB)

El método extraerVes() desempaqueta el archivo Cabinet a un directorio temporal:

public function extraerVes(string $vesPath): string|false
{
    $tmpDir = sys_get_temp_dir() . DIRECTORY_SEPARATOR
            . 'sicsal_ves_' . uniqid();
    mkdir($tmpDir, 0700, true);
    $mdbDest = $tmpDir . DIRECTORY_SEPARATOR . 'transfer.mdb';

    if (PHP_OS_FAMILY === 'Windows') {
        // Windows: usa el binario nativo expand.exe
        $cmd = sprintf(
            '"%s" "%s" -F:* "%s"',
            'C:\\Windows\\System32\\expand.exe',
            $vesPath, $tmpDir
        );
        exec($cmd, $output, $code);
    } else {
        // Linux: cabextract (apt install cabextract)
        $cmd = sprintf('cabextract -d %s %s 2>&1',
            escapeshellarg($tmpDir),
            escapeshellarg($vesPath)
        );
        exec($cmd, $output, $code);
    }

    // El Cabinet contiene transfer.sql (realmente un .mdb)
    rename($tmpDir . '/transfer.sql', $mdbDest);

    return $mdbDest;
}
Seguridad — El directorio temporal usa permisos 0700 y se elimina en el bloque finally con @unlink($mdbPath). En Linux se usa escapeshellarg() para prevenir inyección de comandos.

6 · Metadatos y mapeo de centros

Una vez extraído el MDB, se leen los metadatos del archivo:

private function leerMetadatos(): array|false
{
    $r = self::SUFIJO_REF;  // '2005'

    // Municipio desde tabla de referencia
    $munis = $this->reader->tabla("municipio{$r}");
    $municipio = trim($munis[0]['nommunicip'] ?? 'Desconocido');

    // Año desde EstabGest2005 → idgestion
    $estab = $this->reader->tabla('EstabGest' . $r);
    $anio  = (int) ($estab[0]['idgestion'] ?? 0);

    // Si idgestion tiene formato YYYYMMDD, extraer solo año
    if ($anio > 9999) {
        $anio = (int) substr((string) $anio, 0, 4);
    }

    // Mes desde la cabecera del formulario 301
    $cab = $this->reader->tabla('301_CAB' . self::SUFIJO_FORM);
    $mes = (int) ($cab[0]['mes'] ?? 0);

    return compact('anio', 'mes', 'municipio');
}

El mapeo de centros cruza las tablas del .ves con los centros configurados en SIMUES:

private function mapearCentros(int $anio): array
{
    // Centros SIMUES con código SNIS configurado
    $centrosSicsal = CentroSalud::whereNotNull('codigo_snis')
        ->where('activo', true)
        ->pluck('id', 'codigo_snis')->toArray();

    // Establecimientos presentes en el .ves
    $establecimientos = $this->reader->tabla('EstabGest' . self::SUFIJO_REF);

    // El corr_estabgest = últimos 2 dígitos del año + código SNIS
    $prefijo = substr((string) $anio, -2);
    $map = [];
    foreach ($centrosSicsal as $codigoSnis => $centroId) {
        $corrEstabgest = $prefijo . $codigoSnis;
        if (isset($enVes[$codigoSnis])) {
            $map[$corrEstabgest] = $centroId;
        }
    }
    return $map;
}
Prefijo corr_estabgest — El SNIS usa un identificador compuesto: los últimos 2 dígitos del año + el código del establecimiento. Ejemplo: año 2026 + código 300183 → 26300183.

7 · Flujo principal y transacción

El método handle() sigue un flujo de 5 pasos envuelto en una transacción:

public function handle(): int
{
    // 1. Extraer .ves → MDB temporal
    $mdbPath = $this->extraerVes($archivoVes);

    // 2. Leer metadatos (año, mes, municipio)
    $this->reader = new VesMdbReader($mdbPath);
    $metadatos = $this->leerMetadatos();

    // 3. Mapear centros SNIS → SIMUES
    $centrosMap = $this->mapearCentros($metadatos['anio']);

    // 4. Limpiar datos previos (si --limpiar)
    if ($this->option('limpiar')) {
        $this->limpiarDatos(array_values($centrosMap), $anio, $mes);
    }

    // 5. Importar dentro de transacción
    DB::beginTransaction();
    try {
        foreach ($centrosMap as $snisCorr => $centroId) {
            $this->stats['consulta_externa']  += $this->importarG01($snisCorr, $mes, $centroId, $anio);
            $this->stats['referencias']       += $this->importarG02(...);
            $this->stats['odontologia']       += $this->importarG03(...);
            $this->stats['prenatales']        += $this->importarG04(...);
            $this->stats['anticoncepcion']    += $this->importarG05(...);
            $this->stats['crecimiento']       += $this->importarG06(...);
            $this->stats['enfermeria']        += $this->importarG07(...);
            $this->stats['micronutrientes']   += $this->importarG08(...);
            $this->stats['actividades']       += $this->importarG09(...);
            $this->stats['vacunas']           += $this->importarG37(...);
            $this->stats['vacunas']           += $this->importarG38(...);
            $this->stats['internaciones']     += $this->importarG12(...);
            $this->stats['recien_nacidos']    += $this->importarG18(...);
        }
        DB::commit();
    } catch (\Throwable $e) {
        DB::rollBack();
        $this->error('Error: ' . $e->getMessage());
        return self::FAILURE;
    } finally {
        @unlink($mdbPath);  // limpiar MDB temporal
    }

    // Mostrar resumen
    $this->table(['Tipo de dato', 'Registros'],
        collect($this->stats)->map(
            fn ($v, $k) => [str_replace('_', ' ', ucfirst($k)), $v]
        )->values()->toArray()
    );
    return self::SUCCESS;
}

El limpiarDatos() elimina de 14 tablas (prestaciones + causas + ILE) filtrando por centro_salud_id, año y mes:

private function limpiarDatos(array $centroIds, int $anio, int $mes): void
{
    $tablas = [
        'prest_consulta_externa', 'prest_referencias',
        'prest_odontologia',     'prest_prenatales',
        'prest_anticoncepcion',  'prest_crecimiento',
        'prest_enfermeria',      'prest_micronutrientes',
        'prest_actividades_comunidad', 'prest_vacunas',
        'prest_internaciones',  'prest_recien_nacidos',
        'causas_consulta_externa', 'prest_ile',
    ];

    foreach ($tablas as $tabla) {
        DB::table($tabla)
            ->whereIn('centro_salud_id', $centroIds)
            ->where('anio', $anio)->where('mes', $mes)
            ->delete();
    }
}

8 · Importadores por grupo (G01–G38)

Cada grupo sigue el mismo patrón: leer filas → parsear codsubvar → agrupar → updateOrInsert. Veamos los tres patrones principales:

Patrón 1 — Agrupación M/F simple (G01: Consulta Externa)

private function importarG01(string $corr, int $mes, int $centroId, int $anio): int
{
    $rows    = $this->queryGrupo('G01', $corr, $mes);
    $grouped = [];

    foreach ($rows as $row) {
        $cs  = trim($row['codsubvar'] ?? '');
        $var = substr($cs, 5, 2);
        $sub = substr($cs, 7, 2);
        $grp = self::CE_GRUPO[$var] ?? null;
        $tip = self::CE_TIPO[$sub]  ?? null;
        if (! $grp || ! $tip) continue;

        if (! isset($grouped[$grp])) {
            $grouped[$grp] = [
                'primera_m' => 0, 'primera_f' => 0,
                'nueva_m'   => 0, 'nueva_f'   => 0,
                'repetida_m'=> 0, 'repetida_f'=> 0,
            ];
        }
        $grouped[$grp]["{$tip}_m"] += $this->v($row, 'V');
        $grouped[$grp]["{$tip}_f"] += $this->v($row, 'M');
    }

    $count = 0;
    foreach ($grouped as $grp => $vals) {
        if (array_sum($vals) === 0) continue;
        DB::table('prest_consulta_externa')->updateOrInsert(
            ['centro_salud_id' => $centroId, 'mes' => $mes, 'anio' => $anio, 'grupo_etareo' => $grp],
            array_merge($vals, ['updated_at' => now()])
        );
        $count++;
    }
    return $count;
}

Patrón 2 — Dentro/Fuera (G04: Prenatales)

private function importarG04(...): int
{
    $grouped = [];
    foreach ($this->queryGrupo('G04', $corr, $mes) as $row) {
        $tipo = self::PRENATAL_TIPO[$var]  ?? null;
        $grp  = self::PRENATAL_GRUPO[$sub] ?? null;

        // Subvar impar = dentro, par = fuera
        $isDentro = ((int) $sub) % 2 === 1;
        $total    = $this->v($row, 'V') + $this->v($row, 'M');

        $key = "{$tipo}|{$grp}";
        $grouped[$key][$isDentro ? 'dentro' : 'fuera'] += $total;
    }
    // updateOrInsert con ['dentro' => X, 'fuera' => Y]
}

Patrón 3 — Vacunas con matriz dentro/fuera × M/F (G37)

private function importarGrupoVacunas(
    string $grupo,
    array  $tipoMap,
    array  $colMap, ...
): int {
    $grouped = [];
    foreach ($this->queryGrupo($grupo, $corr, $mes) as $row) {
        $tip = $tipoMap[$var] ?? null;
        $col = $colMap[$sub] ?? null;
        [$grp, $isDentro] = $col;

        $key = "{$tip}|{$grp}";
        if ($isDentro) {
            $grouped[$key]['dentro_m'] += $this->v($row, 'V');
            $grouped[$key]['dentro_f'] += $this->v($row, 'M');
        } else {
            $grouped[$key]['fuera_m']  += $this->v($row, 'V');
            $grouped[$key]['fuera_f']  += $this->v($row, 'M');
        }
    }
    // updateOrInsert a prest_vacunas con 4 columnas
}
G37 + G38 comparten el mismo método — importarGrupoVacunas() recibe los mapas por parámetro, permitiendo reutilizar la misma lógica para menores de 5 años y otras vacunaciones.

9 · ImportarSnis — conexión ODBC a Access

A diferencia de ImportarVes que extrae un Cabinet, ImportarSnis lee directamente los archivos .mdb del SNIS instalado en una carpeta local:

class ImportarSnis extends Command
{
    protected $signature = 'snis:importar
        {--ruta=C:\SNIS2026 : Ruta de instalación del SNIS}
        {--anio=2026 : Gestión SNIS a importar}
        {--anio-destino= : Año destino en SIMUES (default = mismo que --anio)}
        {--meses= : Meses a importar separados por coma}
        {--limpiar : Eliminar datos existentes del período}';
}

La conexión se establece con PDO vía el driver ODBC de Access:

private function connectMdb(string $path): PDO
{
    $dsn = "odbc:Driver={Microsoft Access Driver (*.mdb, *.accdb)}"
        . ";Dbq={$path}"
        . ";Pwd=" . self::dbPassword();

    return new PDO($dsn, '', '', [
        PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]);
}

private static function dbPassword(): string
{
    return env('SNIS_DB_PASSWORD', '');
}
Credenciales en .env — La contraseña del SNIS se lee de SNIS_DB_PASSWORD, nunca hardcodeada. El driver ODBC de Access requiere estar instalado en el sistema (incluido con Office o descargable por separado).

La detección de meses disponibles consulta la tabla 301_CAB directamente:

private function getMesesDisponibles(array $centrosMap): array
{
    $corrList = array_keys($centrosMap);
    $placeholders = implode(',', array_fill(0, count($corrList), '?'));

    $stmt = $this->pdoData->prepare(
        "SELECT DISTINCT mes FROM [301_CAB]
         WHERE corr_estabgest IN ({$placeholders}) AND mes > 0
         ORDER BY mes"
    );
    $stmt->execute($corrList);

    return array_column($stmt->fetchAll(), 'mes');
}

10 · Diferencias respecto a ImportarVes

AspectoImportarVesImportarSnis
Fuente de datos Archivo .ves (un mes, un municipio) Base .mdb instalada (múltiples meses)
Lector MDB VesMdbReader (servicio propio) PDO con ODBC directo
Tablas MDB Sufijo 05 (form) / 2005 (ref) Sin sufijo (301G01_DAT)
Grupos importados G01–G09, G12, G18, G37, G38 (13 métodos) G01–G09, G37, G38 (10 métodos)
Año/Mes Se detectan del propio .ves Se pasan como opciones del CLI
Opción especial --solo-centro --anio-destino (re-mapeo de año)
Limpieza 14 tablas (incluye causas, ILE) 10 tablas (solo prestaciones)
Consulta de filas queryGrupo('G01', corr, mes) vía reader queryGroup('301G01_DAT', corr, mes) vía PDO
¿Cuándo usar cada uno? — snis:importar-ves cuando se recibe el backup mensual (.ves) de otros municipios. snis:importar cuando se tiene el SNIS instalado localmente (la aplicación de escritorio del Ministerio de Salud).

11 · Patrón updateOrInsert idempotente

Todos los importadores usan DB::table()->updateOrInsert() que garantiza idempotencia:

// Si existe la combinación (centro_salud_id, mes, anio, tipo) → UPDATE
// Si no existe → INSERT
DB::table('prest_micronutrientes')->updateOrInsert(
    // Condición de búsqueda (clave compuesta)
    [
        'centro_salud_id' => $centroId,
        'mes'             => $mes,
        'anio'            => $anio,
        'tipo'            => $tipo,
    ],
    // Valores a actualizar/insertar
    [
        'cantidad'   => $cantidad,
        'updated_at' => now(),
    ]
);

Esto permite ejecutar el mismo comando múltiples veces sin duplicar datos — esencial para re-importaciones cuando se corrigen datos en el SNIS.

El helper v() asegura que valores nulos o vacíos se traten como cero:

private function v(array $row, string $campo): int
{
    return (int) ($row[$campo] ?? 0);
}

Resumen de la lección

  • Dos fuentes, un destino — snis:importar-ves (762 líneas) extrae archivos Cabinet (.ves), snis:importar (645 líneas) lee el SNIS instalado vía ODBC.
  • codsubvar — Código de 9 dígitos (FFFF GG VV SS) que identifica cada dato del formulario 301; se parsea con substr() en posiciones 5-6 y 7-8.
  • 15 mapas de constantes — Arrays private const que traducen códigos numéricos SNIS a enums de SIMUES, organizados por los 13 grupos del formulario.
  • 3 patrones de importación — Agrupación M/F simple (consulta externa, referencias), dentro/fuera (prenatales, vacunas), y acumulación escalar (micronutrientes, internaciones).
  • updateOrInsert — Garantiza idempotencia para re-importaciones sin duplicar datos.
  • Transacción completa — Todo-o-nada con DB::beginTransaction() + commit()/rollBack().
  • Limpieza temporal — El MDB extraído se elimina en finally; credenciales siempre desde .env.
