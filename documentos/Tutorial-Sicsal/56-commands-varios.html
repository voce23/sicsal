2 · ImportarCenso — Pacientes SOAPS → Personas SIMUES

class ImportarCenso extends Command
{
    protected $signature = 'soaps:importar-censo
        {--anio=2025 : Año base del censo}
        {--codestabl=300183 : Código de establecimiento SOAPS}
        {--centro-salud=1 : ID del centro de salud en SIMUES}
        {--limpiar : Eliminar personas existentes antes de importar}
        {--solo-conteo : Solo mostrar conteo sin importar}';
}

El flujo principal tiene 4 pasos:

  1. Consulta SOAPS — Pacientes con consulta en la gestión (incluye ene-mar del año siguiente).
  2. Excluir fallecidos — Cruza con Tbl_datosFallecido normalizando nombres para comparación.
  3. Resolver comunidad — Mapea zon_descripcion y HCL_DIRECC a comunidades SIMUES.
  4. Importar con dedup — Busca duplicados por CI o por nombre+fecha_nacimiento; actualiza o crea.
// Paso 4: Deduplicación inteligente
$existente = null;
if ($data['ci']) {
    $existente = Persona::where('centro_salud_id', $centroSaludId)
        ->where('ci', $data['ci'])->first();
}
if (! $existente) {
    $existente = Persona::where('centro_salud_id', $centroSaludId)
        ->where('apellidos', $data['apellidos'])
        ->where('nombres', $data['nombres'])
        ->where('fecha_nacimiento', $data['fecha_nacimiento'])
        ->first();
}

if ($existente) {
    $existente->update([
        'comunidad_id' => $data['comunidad_id'],
        'tipo_seguro'  => $data['tipo_seguro'],
    ]);
    $actualizados++;
} else {
    Persona::create($data);
    $nuevos++;
}
Período ampliado — El censo incluye enero-marzo del año siguiente ($anioSig), ya que la gestión boliviana del SNIS abarca hasta marzo del siguiente año calendario.

3 · Resolución de comunidades por zona y dirección

Tanto ImportarCenso como ImportarDefunciones usan un sistema de dos niveles para asignar comunidades:

// Nivel 1: Mapeo directo zona → comunidad
private const ZONA_COMUNIDAD = [
    'HORNOMA'      => 'Hornoma',
    'HUAYCHOMA'    => 'Huaychoma',
    'VILLCABAMBA'  => 'Villcabamba',
    'COCOMA'       => 'Cocoma',
    'TOCOHALLA'    => 'Tocohalla',
    'CHALLAVILQUE' => 'Challavilque',
    'CALACAJA'     => 'Calacaja',
    'SIQUIMIRANI'  => 'Siquimirani',
];

// Nivel 2: Búsqueda por palabras clave en dirección
private const DIRECC_COMUNIDAD = [
    'HORNOMA'       => 'Hornoma',
    'HHUAYCHOMA'    => 'Huaychoma',   // typo frecuente
    'VILLCABAM'     => 'Villcabamba',  // parcial
    'CHALLAVILLQUE' => 'Challavilque', // typo
    'CALCAJA'       => 'Calacaja',    // typo
    'CALA CAJA'     => 'Calacaja',    // con espacio
    'HIORNOMA'      => 'Hornoma',     // typo
];
Tolerancia a errores de tipeo — Los datos del SOAPS contienen errores de digitación frecuentes. El fallback a DIRECC_COMUNIDAD maneja variantes comunes. Los pacientes sin comunidad se asignan a Hornoma (centro de salud principal) y se reportan con --verbose.

4 · ImportarDefunciones — Mortalidad SOAPS → SIMUES

class ImportarDefunciones extends Command
{
    protected $signature = 'soaps:importar-defunciones
        {--desde=2024 : Año desde el cual importar}
        {--hasta=2026 : Año hasta el cual importar}
        {--codestabl=300183 : Código de establecimiento SOAPS}
        {--centro-salud=1 : ID del centro de salud en SIMUES}
        {--todas : Importar todas las gestiones}
        {--limpiar : Eliminar defunciones importadas antes}
        {--solo-conteo : Solo mostrar conteo}';
}

La consulta al SOAPS cruza CONSULTA_EXTERNA_2014 con SE_HC, filtrando por MORTALIDAD_TODAS IS NOT NULL:

$defunciones = $pdo->query("
    SELECT ce.Fecha, ce.HC, ce.NOMBRE, ce.DIAGNOS_DESCRIP, ce.CIE_I,
           ce.zona_res, h.HCL_APPAT, h.HCL_APMAT, h.HCL_NOMBRE,
           h.HCL_NUMCI, h.HCL_SEXO, h.HCL_FECNAC, h.HCL_DIRECC
    FROM CONSULTA_EXTERNA_2014 ce
    JOIN SE_HC h ON h.HCL_CODIGO = ce.HC
    WHERE ce.MORTALIDAD_TODAS IS NOT NULL
      AND ce.MORTALIDAD_TODAS <> '' AND ce.MORTALIDAD_TODAS <> '0'
      AND ce.codestabl = '{$codestabl}'
      {$filtroAnio}
    ORDER BY ce.Fecha DESC
");

Al crear la defunción, el modelo Defuncion auto-calcula grupo_etareo y desactiva la persona vinculada:

// Buscar persona en el censo para vincular
$persona = Persona::where('ci', $data['ci'])->first();
$data['persona_id'] = $persona?->id;

// El modelo auto-calcula grupo_etareo y desactiva persona
Defuncion::create($data);

5 · ImportarCausasConsulta — Top-N diagnósticos CIE-10

class ImportarCausasConsulta extends Command
{
    protected $signature = 'soaps:importar-causas
        {--servidor=.\\SNS : Instancia SQL Server del SOAPS}
        {--db=BDestadistica : Base de datos SOAPS}
        {--anio=2026 : Gestión a importar}
        {--top=10 : Cantidad de principales causas}
        {--limpiar : Eliminar datos existentes}';
}

El flujo por cada centro y mes:

  1. Cargar diccionario CIE-10 desde SE_CIE10.
  2. Consultar todas las atenciones del mes, calculando total_meses de vida para grupo etáreo.
  3. Acumular en PHP: $conteos[CIE][grupo][M/F].
  4. Ordenar por total descendente y tomar las Top-N causas.
  5. Limpiar + insertar en causas_consulta_externa.
// Grupo etáreo a partir de meses de vida
private function getGrupoFromMeses(int $totalMeses): string
{
    if ($totalMeses < 6)   return 'menor_6m';
    if ($totalMeses < 12)  return '6m_menor_1';
    if ($totalMeses < 60)  return '1_4';
    if ($totalMeses < 120) return '5_9';
    // ... hasta mayor_60
}

// Acumulación y ranking
$conteos = [];
foreach ($allRecords as $d) {
    $cie  = trim($d['cie']) ?: 'Z76.8';
    $g    = $this->getGrupoFromMeses((int) $d['total_meses']);
    $sexo = $d['Sexo'];
    $conteos[$cie][$g][$sexo === 'Masculino' ? 'm' : 'f']++;
}

arsort($totales);
$top10 = array_slice($totales, 0, $top, true);
Codificación Latin-1 → UTF-8 — SOAPS devuelve descripciones CIE-10 en Windows-1252. Se aplica mb_convert_encoding() para insertar correctamente en MySQL (UTF-8). 
// Conversión segura de encoding
$diag = mb_detect_encoding($rawDiag, ['UTF-8'], true)
    ? $rawDiag
    : mb_convert_encoding($rawDiag, 'UTF-8', 'Windows-1252');

6 · NormalizarNombres — Title Case masivo

El comando más breve (38 líneas) convierte nombres/apellidos de MAYÚSCULAS a Title Case:

class NormalizarNombres extends Command
{
    protected $signature   = 'sicsal:normalizar-nombres';
    protected $description = 'Convierte nombres a Title Case';

    public function handle(): int
    {
        $total = Persona::count();
        $bar = $this->output->createProgressBar($total);

        Persona::chunk(200, function ($personas) use (&$procesados, $bar) {
            foreach ($personas as $persona) {
                $nuevo = mb_convert_case(
                    mb_strtolower($persona->getRawOriginal('nombres')),
                    MB_CASE_TITLE, 'UTF-8'
                );

                if ($persona->getRawOriginal('nombres') !== $nuevo) {
                    $persona->timestamps = false;
                    $persona->updateQuietly([
                        'nombres' => $nuevo,
                    ]);
                }
                $bar->advance();
            }
        });
    }
}
Tres técnicas clave:
  • getRawOriginal() — Lee el valor sin accessors/mutators del modelo.
  • timestamps = false — No actualiza updated_at (cambio cosmético).
  • updateQuietly() — No dispara observers ni eventos (evita efectos secundarios).

7 · TransferirSoaps — Flujo inverso SOAPS → SNIS

El comando más grande (891 líneas) transfiere datos del SOAPS (SQL Server) al SNIS (Access .mdb) — es decir, en dirección opuesta a los comandos snis:importar-*:

class TransferirSoaps extends Command
{
    protected $signature = 'soaps:transferir-snis
        {--mes= : Mes a transferir (1-12)}
        {--anio=2025 : Año a transferir}
        {--ruta-snis=C:\SNIS2025 : Ruta del SNIS}
        {--codestabl=300183 : Código de establecimiento}
        {--limpiar : Eliminar datos antes de insertar}
        {--solo-consulta : Solo G01}
        {--solo-odontologia : Solo G03}
        {--solo-302 : Solo Vigilancia Semanal}
        {--solo-305 : Solo Vigilancia Mensual}
        {--formulario= : 301,302,305}';
}

Establece dos conexiones PDO simultáneas:

// SQL Server (SOAPS) — lectura
$this->pdoSoaps = $this->connectSoaps();

// Access .mdb (SNIS) — escritura
$dataFile = "{$rutaSnis}\\snis{$anio}.mdb";
$this->pdoSnis = $this->connectSnis($dataFile);

El flujo principal despacha a tres transferidores según el formulario seleccionado:

// 301 = Consulta Externa + Odontología
if (in_array('301', $formularios)) {
    $stats = $this->transferir301($corr, $anio, $mes, $limpiar);
}

// 302 = Vigilancia Epidemiológica Semanal (302A)
if (in_array('302', $formularios)) {
    $stats['302_semanal'] = $this->transferir302Semanal(...);
}

// 305 = Vigilancia Epidemiológica Mensual (305/302B)
if (in_array('305', $formularios)) {
    $stats['305_mensual'] = $this->transferir305Mensual(...);
}

8 · TransferirSoaps: el mapeo CIE-10 (302A y 305)

Los formularios de vigilancia epidemiológica usan patrones regex contra códigos CIE-10 para clasificar diagnósticos:

private const CIE_302_MAP = [
    // G06 - Enfermedades prevalentes
    ['grp' => '06', 'var' => '10', 'rx' => '/^A0[0-9]/'],       // EDA (A00-A09)
    ['grp' => '06', 'var' => '11', 'rx' => '/^J(0[0-6]|2[0-2])/'], // IRA sin neumonía
    ['grp' => '06', 'var' => '12', 'rx' => '/^J1[2-8]/'],       // Neumonía

    // G01 - Notificación inmediata (40+ patologías)
    ['grp' => '01', 'var' => '11', 'rx' => '/^B0[56]/'],       // Sarampión/Rubéola
    ['grp' => '01', 'var' => '33', 'rx' => '/^U07/'],         // COVID-19
    ['grp' => '01', 'var' => '36', 'rx' => '/^B04/'],         // Mpox
    ['grp' => '01', 'var' => '37', 'rx' => '/^A93\.0/'],      // Oropouche
    // ... G04 (Inmunoprevenibles), G05 (ITS), G07 (TB),
    // G08 (Violencia), G13 (Intoxicaciones), G17 (Tropicales)
];

private const CIE_305_MAP = [
    // G02 - Enfermedades crónicas no transmisibles
    ['grp' => '02', 'var' => '01', 'rx' => '/^E10/'],          // Diabetes Tipo 1
    ['grp' => '02', 'var' => '06', 'rx' => '/^I1[0-5]/'],     // HTA
    ['grp' => '02', 'var' => '13', 'rx' => '/^C/'],           // Otros cánceres

    // G03 - Salud Mental (12 categorías)
    ['grp' => '03', 'var' => '01', 'rx' => '/^F3[23]/'],      // Depresión
    ['grp' => '03', 'var' => '09', 'rx' => '/^F84/'],        // Autismo

    // G04 - Neurodegenerativas
    ['grp' => '04', 'var' => '01', 'rx' => '/^G30/'],        // Alzheimer
    ['grp' => '04', 'var' => '02', 'rx' => '/^G20/'],        // Parkinson
];

La función matchCie() aplica la primera coincidencia regex en orden de prioridad:

private function matchCie(array $map, string $cie): ?array
{
    foreach ($map as $entry) {
        if (preg_match($entry['rx'], $cie)) {
            return $entry;
        }
    }
    return null;
}

9 · TransferirSoaps: formularios 301 + 302 + 305

301 — Consulta Externa (G01)

Agrega datos de SE_DATOS con un GROUP BY enorme que clasifica edad y reconsulta directamente en SQL Server, generando los var_code y sub_code del codsubvar:

// La query SQL Server calcula el grupo etáreo con CASE WHEN
CASE
    WHEN DAT_FNANIO = 0 AND DAT_FNMES < 6  THEN '12'  -- menor_6m
    WHEN DAT_FNANIO = 0 AND DAT_FNMES >= 6 THEN '13'  -- 6m_menor_1
    WHEN DAT_FNANIO >= 60                   THEN '08'  -- mayor_60
    ...
END AS var_code

301 — Odontología (G03)

Lee individualmente de se_odontologia y acumula por codsubvar en PHP. Cada campo del SOAPS se traduce a un codvariabl del SNIS:

private const G03_CAMPOS_VAR = [
    'PRIMERA_CONSULTA'     => '01',
    'RESINA_FOTOCURABLE'   => '06',  // Restauraciones (agrupado)
    'RESINA_AUTOCURABLE'   => '06',
    'AMALGAMA'             => '06',
    'IONOMERO'             => '06',
    'PULPOTOMIA'           => '08',  // Endodoncia
    'FLUORINIZACION_DENTRO'=> '21',  // Medidas preventivas
    // ... 26 campos totales
];

302A — Vigilancia Epidemiológica Semanal

Agrupa por semana ISO (el jueves determina el mes SNIS). Evalúa CIE-10 en las 4 columnas diagnósticas (CIE_I, CIE_II, CIE_III, CAUSA_EXTERNA).

305 — Vigilancia Epidemiológica Mensual

Similar al 302 pero agrega por mes. Usa 20 subvars (10 edades × 2 sexos) para grupos G02-G04, donde V = caso nuevo y M = en control (Reconsulta).

Columna «Años» con ñ — La columna del SOAPS usa Años (chr 241 Latin-1). El código accede a ella con un hex literal: $aniosCol = "A\xf1os".

10 · Conexiones y seguridad

Todos los comandos SOAPS usan el mismo patrón de conexión con credenciales desde .env:

// Credenciales SOAPS (SQL Server)

private static function soapsUser(): string   { return env('SOAPS_USER', 'sa'); }

Flag --solo-conteo — Los comandos de censo y defunciones incluyen esta opción para verificar datos sin escribir nada. Útil para validación previa.

Resumen de la lección

  • ImportarCenso (277 líneas) — Importa pacientes del SOAPS al censo de SIMUES, excluyendo fallecidos y resolviendo comunidades por zona y dirección con tolerancia a typos.
  • ImportarDefunciones (260 líneas) — Importa mortalidad, vincula con personas del censo y las desactiva automáticamente vía el modelo Defuncion.
  • ImportarCausasConsulta (209 líneas) — Calcula las Top-N causas de consulta externa por mes y centro, con conversión Latin-1 → UTF-8 para diagnósticos CIE-10.
  • NormalizarNombres (38 líneas) — Convierte nombres de MAYÚSCULAS a Title Case con chunk(200), getRawOriginal() y updateQuietly().
  • TransferirSoaps (891 líneas) — Flujo inverso SOAPS → SNIS (.mdb) para formularios 301 (consulta + odontología), 302A (vigilancia semanal con regex CIE-10) y 305 (vigilancia mensual de enfermedades crónicas y salud mental). Conexión dual PDO (SQL Server + Access).
  • Seguridad — Todas las credenciales en .env; PDO con ERRMODE_EXCEPTION; escapeshellarg() donde hay exec.
