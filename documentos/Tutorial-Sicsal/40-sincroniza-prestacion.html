1 Contexto — registros individuales vs totales mensuales

SICSAL mantiene dos niveles de datos:

NivelEjemploTabla
IndividualVacuna aplicada a Juan Pérez el 15/marzovacunas_ninos
Consolidado mensualTotal BCG menores de 1 año, dentro, masculino = 12prest_vacunas

Los formularios mensuales (Fase 7) trabajan con las tablas prest_*. Pero cuando un registrador ingresa un registro individual, el total mensual debe actualizarse automáticamente. Ahí entran los Observers y este trait.

16 tablas Prest* consolidan los totales: PrestVacuna, PrestMicronutriente, PrestCrecimiento, PrestParto, PrestPrenatal, PrestAnticoncepcion, PrestRecienNacido, etc. Cada una corresponde a una sección de los formularios SNIS.

2 Arquitectura de sincronización

El flujo es unidireccional: registro individual → Observer → Prest* (totales):

┌─────────────────────┐     Observer      ┌──────────────────┐
│  VacunaNino::create  │ ──────────────→  │  PrestVacuna     │
│  (registro individual)│   +1 / -1       │  (total mensual) │
└─────────────────────┘                    └──────────────────┘
                         SincronizaPrestacion
                      (calcula grupo_etareo, campo, etc.)

El trait SincronizaPrestacion vive en app/Observers/Concerns/ y provee 5 métodos protected que los observers utilizan para clasificar:

3 grupoEtareoVacuna() — 11 rangos por edad en meses

Este método traduce la edad en meses del paciente al grupo etáreo de PrestVacuna. Cubre tanto $gruposMenores5 como $gruposOtrasVac del FormularioPai:

protected function grupoEtareoVacuna(int $edadMeses): string
{
    if ($edadMeses < 12)  return 'menor_1';
    if ($edadMeses < 24)  return '12_23m';

    $anios = (int) floor($edadMeses / 12);

    if ($anios === 2)   return '2_anios';
    if ($anios === 3)   return '3_anios';
    if ($anios === 4)   return '4_anios';
    if ($anios < 10)   return '5_9';
    if ($anios === 10)  return '10_anios';
    if ($anios === 11)  return '11_anios';
    if ($anios <= 20)  return '12_20';
    if ($anios < 60)   return '21_59';

    return '60_mas';
}
Correspondencia directa: Los strings devueltos coinciden exactamente con las claves de los arrays $gruposMenores5 y $gruposOtrasVac del FormularioPai (lección 38). Así el observer escribe en la misma fila que el formulario lee.

Las primeras 5 claves (menor_1 a 4_anios) corresponden a menores de 5 años; las 6 restantes a "Otras vacunaciones".

4 grupoEtareoMaterno() — 6 rangos maternos

Para prenatales, partos y anticoncepción, los grupos se basan en la edad de la madre:

protected function grupoEtareoMaterno(
    Carbon $fechaNac,
    Carbon $referencia
): string
{
    $anios = $fechaNac->diffInYears($referencia);

    if ($anios < 10)   return 'menor_10';
    if ($anios <= 14)  return '10_14';
    if ($anios <= 19)  return '15_19';
    if ($anios <= 34)  return '20_34';
    if ($anios <= 49)  return '35_49';

    return '50_mas';
}
Carbon typed parameters: El método exige objetos Carbon, no strings. Los observers son responsables de parsear las fechas antes de llamar al trait. La fecha de referencia es el evento (parto, control, etc.), no "hoy".

5 grupoCrecimiento() — edad + dentro/fuera

Para crecimiento infantil, la clave combina rango de edad y ubicación:

protected function grupoCrecimiento(
    int $edadMeses,
    string $dentroDuera
): string
{
    $base = match (true) {
        $edadMeses < 12 => 'menor_1',
        $edadMeses < 24 => '1_menor_2',
        default         => '2_menor_5',
    };

    return $base . '_' . $dentroDuera;
    // Ejemplo: 'menor_1_dentro', '2_menor_5_fuera'
}
match(true): PHP 8.0+ permite usar match(true) como alternativa elegante a if/elseif. Cada rama evalúa una expresión booleana y devuelve un valor. Es más compacto que la cadena de if/return del método de vacunas.

Las 6 claves resultantes corresponden exactamente a las filas de $labelsCrec en la vista de crecimiento (lección 38).

6 tipoPrenatal() — clasificación por control y trimestre

Traduce el número de control prenatal y las semanas de gestación al tipo del formulario:

protected function tipoPrenatal(
    int $numeroControl,
    int $semanasGestacion
): string
{
    if ($numeroControl >= 4) return 'con_4to_control';
    if ($numeroControl > 1)  return 'repetida';

    // Primer control → clasificar por trimestre
    if ($semanasGestacion <= 13) return 'nueva_1er_trim';
    if ($semanasGestacion <= 27) return 'nueva_2do_trim';

    return 'nueva_3er_trim';
}
Lógica de negocio SNIS:
  • ≥ 4to control: Se cuenta como "con 4to control" (indicador clave)
  • 2do o 3er control: Se cuenta como "repetida"
  • 1er control: Se clasifica por trimestre de captación (1er trimestre es mejor que 3er — detección temprana)
Son las mismas claves que usa $tiposPrenatal en FormularioMaterna (lección 37).

7 campoDentroFueraSexo() — combinador de campos

El método más simple — concatena dentro/fuera con el sexo para obtener el nombre del campo de la tabla Prest*:

protected function campoDentroFueraSexo(
    string $dentroDuera,
    string $sexo
): string
{
    return $dentroDuera . '_' . strtolower($sexo);
    // 'dentro' + 'M' → 'dentro_m'
    // 'fuera'  + 'F' → 'fuera_f'
}

Los 4 campos resultantes (dentro_m, dentro_f, fuera_m, fuera_f) son exactamente las columnas de prest_vacunas y el modelo PrestVacuna.

8 Resumen

Lo que aprendimos en esta lección

  • Trait en Observers/Concerns: Centraliza la lógica de clasificación etárea que 5 observers comparten.
  • Puente individual → consolidado: Los métodos traducen datos del registro individual (edad, fecha, ubicación) a las claves de las tablas Prest*.
  • Correspondencia exacta: Las claves devueltas coinciden con los arrays static de los formularios (Fase 7), garantizando consistencia.
  • match(true): Alternativa elegante a if/elseif para clasificación por rangos (PHP 8.0+).
  • Carbon como tipo: El trait exige objetos Carbon en los parámetros, delegando el parsing a los observers.
  • 5 métodos / 5 observers: Cada observer usa 1-2 métodos del trait según la sección que necesita actualizar.
