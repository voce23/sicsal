1 Qué vamos a construir

SICSAL (Sistema de Información de Centros de Salud) es una aplicación web real usada en centros de salud de 1er nivel en Bolivia. Maneja:

  • Registro de personas — padrón de la población asignada
  • Prestaciones mensuales — captura de datos tipo formularios SNIS
  • Informe CAI — coberturas de vacunación, maternidad, etc.
  • Reportes — exportación a Excel, PDF y PowerPoint
  • Importación — lectura de archivos .ves del SNIS
  • Panel admin — CRUD completo con Filament 5
  • Roles y permisos — superadmin, admin, registrador
ℹ️
Sobre este tutorial
Son 58 lecciones paso a paso. Al terminar, tendrás el proyecto completo con 278+ archivos y ~23,000 líneas de código. Cada lección incluye el código completo a escribir — no fragmentos sueltos.

2 Requisitos previos

Solo necesitas:

  • Un computador con Windows 10/11 (64 bits)
  • Mínimo 4 GB de RAM (recomendado 8 GB)
  • Conexión a internet para descargar herramientas
  • Ganas de aprender — no se necesita experiencia previa en Laravel
💡
Consejo
Si ya tienes XAMPP o WAMP instalado, te recomiendo usar Laragon en su lugar. Es más ligero, tiene Apache + MySQL + PHP preconfigurado, y gestiona virtual hosts automáticamente.

3 Instalar Laragon

Laragon es un entorno de desarrollo local para Windows que incluye Apache, MySQL, PHP y más.

1
Descargar Laragon
Ve a https://laragon.org/download/ y descarga la versión Full (incluye PHP 8.x, MySQL 8, Apache, Node.js, etc.).
2
Instalar
Ejecuta el instalador. La ruta por defecto es C:\laragon. Déjala así — todo el tutorial usa esa ruta.
3
Iniciar servicios
Abre Laragon y haz clic en "Start All". Verás que Apache y MySQL se encienden (indicadores verdes).
4
Verificar
Abre tu navegador y visita http://localhost. Deberías ver la página de bienvenida de Laragon.
⚠️
Importante
Si el puerto 80 está ocupado (por Skype, IIS, etc.), Laragon te avisará. Ve a Menú → Preferencias → Apache y cambia el puerto, o cierra el programa que lo usa.

Estructura de Laragon

C:\laragon\
  • bin/ — ejecutables (PHP, MySQL, Apache, Node, etc.)
  • data/ — datos de MySQL
  • etc/ — archivos de configuración
  • www/ — aquí van tus proyectos ← importante

Todo proyecto que crees dentro de C:\laragon\www\ automáticamente tendrá un virtual host. Por ejemplo, C:\laragon\www\sicsal será accesible como http://sicsal.test.

4 Verificar PHP 8.3

Laravel 13 requiere PHP 8.2 o superior. Laragon Full ya trae PHP 8.x. Verifiquemos la versión.

Abre una terminal (clic derecho en Laragon → Terminal, o usa la terminal de VS Code después):

Terminal bash 
$ php -v
PHP 8.3.26 (cli) (built: ...)

Si la versión es 8.2+, estás bien. Si necesitas actualizar PHP en Laragon:

1
Descarga PHP 8.3 desde https://windows.php.net/download/ (Thread Safe, x64).
2
Extrae en C:\laragon\bin\php\php-8.3.26-Win32-vs16-x64\.
3
En Laragon, ve a Menú → PHP → Versión y selecciona php-8.3.x.

Verifica también las extensiones necesarias para Laravel:

Terminal bash 
$ php -m | findstr /i "pdo_mysql mbstring openssl tokenizer xml curl gd zip"
curl
gd
mbstring
openssl
PDO
pdo_mysql
tokenizer
xml
zip
💡
Extensiones faltantes
Si falta alguna extensión, edita C:\laragon\bin\php\php-8.3.x\php.ini y descomenta la línea correspondiente (quita el ; del inicio). Luego reinicia Laragon.

6 Instalar Composer

Composer es el gestor de dependencias de PHP. Es como npm pero para PHP. Lo usaremos para instalar Laravel y todos los paquetes.

Laragon Full ya incluye Composer. Verifica:

Terminal bash 
$ composer --version
Composer version 2.8.x 2026-xx-xx

Si no está instalado, descárgalo de https://getcomposer.org/download/ y sigue el instalador para Windows. Asegúrate de que usa el PHP de Laragon.

⚠️
PATH del sistema
Si al escribir composer o php en una terminal externa (CMD/PowerShell) no los reconoce, necesitas agregar Laragon al PATH. En Laragon: Menú → Herramientas → PATH → añadir Laragon al PATH del sistema.

7 Instalar Node.js

Node.js se necesita para Vite (el bundler que compila Tailwind CSS y JavaScript). npm viene incluido con Node.

Laragon Full trae Node.js pero puede ser una versión vieja. Verifica:

Terminal bash 
$ node -v
v20.x.x

$ npm -v
10.x.x

Necesitas Node.js 18+. Si la versión es menor, descarga la LTS desde https://nodejs.org/ e instálala. El instalador actualiza el PATH automáticamente.

💡
Alternativa: nvm-windows
Si quieres manejar varias versiones de Node, instala nvm-windows desde GitHub. Luego: nvm install 20 y nvm use 20.

8 Instalar VS Code

Visual Studio Code es el editor que usaremos durante todo el tutorial. Es gratuito, ligero y tiene excelente soporte para PHP, Laravel y Blade.

1
Descargar
Ve a https://code.visualstudio.com/ y descarga el instalador para Windows.
2
Instalar
Ejecuta el instalador. Marca las opciones:
✅ Agregar a PATH
✅ Registrar como editor predeterminado
3
Verificar
Abre una terminal y escribe code . — debería abrir VS Code en la carpeta actual.

10 Verificación final

Ejecuta todos los comandos de una vez para confirmar que todo está correctamente instalado:

Terminal bash 
$ php -v
$ composer --version
$ node -v
$ npm -v
$ mysql --version
$ git --version

Deberías ver algo así:

PHP 8.3.26 (cli) Composer version 2.8.x v20.x.x 10.x.x mysql Ver 8.0.30 git version 2.4x.x.windows.1
🚨
¿Algún comando no funciona?
php no reconocido: Agrega Laragon al PATH del sistema.
composer no reconocido: Reinstala Composer o verifica el PATH.
node no reconocido: Reinstala Node.js desde nodejs.org.
git no reconocido: Instala Git desde https://git-scm.com/download/win.
mysql no reconocido: Verifica que Laragon esté iniciado (Start All).
💡
Git
Si no tienes Git instalado, descárgalo de https://git-scm.com/download/win. Lo necesitarás en la Lección 05 para control de versiones. Laragon Full también incluye Git.
