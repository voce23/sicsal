1 Visión general — 4 widgets en el dashboard

Filament muestra widgets en el dashboard automáticamente. Cada widget define un $sort para su orden:

sort = 1 → EstadisticasPoblacion
4 tarjetas: Padrón, Migrantes, Defunciones, Meta INE
↓
sort = 2 → AlertasWidget
Meses sin cerrar, padrón vacío, o "sin alertas"
↓
sort = 3 → MigracionContexto
MEF activas, MEF migradas, % migrantes
↓
sort = 4 → CoberturaMensual
Barras de progreso: BCG, Prenatal, Crecimiento
Dos tipos de widget
  • StatsOverviewWidget: widget built-in de Filament — solo defines getStats() retornando Stat objects
  • Widget (base): widget libre — defines tu vista Blade con $view y expones datos via computed properties

2 EstadisticasPoblacion — StatsOverviewWidget

app/Filament/Widgets/EstadisticasPoblacion.php
namespace App\Filament\Widgets;

use App\Models\Defuncion;
use App\Models\MetaIne;
use App\Models\Persona;
use Filament\Widgets\StatsOverviewWidget;
use Filament\Widgets\StatsOverviewWidget\Stat;

class EstadisticasPoblacion extends StatsOverviewWidget
{
    protected static ?int $sort = 1;

    protected function getStats(): array
    {
        $centroId = auth()->user()->centro_salud_id;

        if (!$centroId) {
            return [
                Stat::make('Sin centro asignado', '—')
                    ->description('Asigne un centro de salud a su usuario')
                    ->color('danger'),
            ];
        }

        $anio = (int) date('Y');

        $activos = Persona::where('centro_salud_id', $centroId)
            ->where('activo', true)
            ->where('estado', '!=', 'migrado')->count();

        $migrantes = Persona::where('centro_salud_id', $centroId)
            ->where('activo', true)
            ->where('estado', 'migrado')->count();

        $defunciones = Defuncion::where('centro_salud_id', $centroId)
            ->whereYear('fecha_defuncion', $anio)->count();

        $metaIne = MetaIne::where('centro_salud_id', $centroId)
            ->where('anio', $anio)->sum('cantidad');

        return [
            Stat::make('Padrón activo', number_format($activos))
                ->description('Residentes activos')
                ->descriptionIcon('heroicon-m-users')
                ->color('primary'),
            Stat::make('Migrantes', number_format($migrantes))
                ->description('Migración registrada')
                ->descriptionIcon('heroicon-m-arrow-right-start-on-rectangle')
                ->color('warning'),
            Stat::make('Defunciones ' . $anio, number_format($defunciones))
                ->description('Año en curso')
                ->descriptionIcon('heroicon-m-document-minus')
                ->color('danger'),
            Stat::make('Meta INE', number_format($metaIne))
                ->description('Población proyectada ' . $anio)
                ->descriptionIcon('heroicon-m-chart-bar')
                ->color('info'),
        ];
    }
}
StatsOverviewWidget — solo PHP, sin Blade

Extiende StatsOverviewWidget y solo implementa getStats(). Filament genera las tarjetas automáticamente. Cada Stat tiene: título, valor, description, descriptionIcon y color.

Siempre validar centro_salud_id

Si el usuario no tiene centro asignado (superadmin sin selección), retornamos un único Stat de aviso en lugar de queries que devolverían todos los registros. Patrón defensivo que se repite en los 4 widgets.

3 AlertasWidget — widget con vista Blade

El PHP — computed property

app/Filament/Widgets/AlertasWidget.php
class AlertasWidget extends Widget
{
    protected string $view = 'filament.widgets.alertas-widget';
    protected int|string|array $columnSpan = 'full';
    protected static ?int $sort = 2;

    public function getDatosProperty(): array
    {
        $centroId = auth()->user()->centro_salud_id;
        if (!$centroId) {
            return ['mesesSinCerrar' => [], 'totalPersonas' => 0];
        }

        $anio = (int) date('Y');
        $mesActual = (int) date('n');

        $mesesCerrados = MesCerrado::where('centro_salud_id', $centroId)
            ->where('anio', $anio)->pluck('mes')->toArray();

        $nombresMeses = [1=>'Enero', ... 12=>'Diciembre'];
        $mesesSinCerrar = [];
        for ($m = 1; $m < $mesActual; $m++) {
            if (!in_array($m, $mesesCerrados)) {
                $mesesSinCerrar[] = $nombresMeses[$m];
            }
        }

        return [
            'mesesSinCerrar' => $mesesSinCerrar,
            'totalPersonas' => Persona::where(...)->count(),
            'anio' => $anio,
        ];
    }
}
getDatosProperty() — Computed Property de Livewire

El método getDatosProperty() crea una computed property accesible como $this->datos en Blade. El nombre del método sigue la convención Livewire: get{Nombre}Property → $this->nombre.

La vista Blade — 3 estados

<x-filament-widgets::widget>

      icon-color="danger">

      <div class="... border-blue-200 bg-blue-50 ...">

    @endif

    <!-- Estado 3: Todo bien -->
    @if (count($d['mesesSinCerrar']) === 0 && $d['totalPersonas'] > 0)
      <div class="... border-green-200 bg-green-50 ...">
        <x-heroicon-s-check-circle ... />
        Sin alertas pendientes
      </div>
    @endif

</x-filament-widgets::widget>
Componentes Blade de Filament
  • <x-filament-widgets::widget> — wrapper obligatorio para widgets custom
  • <x-filament::section> — caja con heading, icon e icon-color
  • <x-heroicon-s-*> — iconos Heroicon como componentes Blade (s = solid, o = outline)

4 MigracionContexto — indicadores MEF y migración

app/Filament/Widgets/MigracionContexto.php
class MigracionContexto extends Widget
{
    protected string $view = 'filament.widgets.migracion-contexto';
    protected int|string|array $columnSpan = 'full';
    protected static ?int $sort = 3;

    public function getDatosProperty(): array
    {
        $centroId = auth()->user()->centro_salud_id;

        // MEF activas — mujeres 15-49 NO migradas
        $mefActivas = Persona::where('centro_salud_id', $centroId)
            ->where('activo', true)
            ->where('sexo', 'F')
            ->where('estado', '!=', 'migrado')
            ->whereRaw('TIMESTAMPDIFF(YEAR, fecha_nacimiento, CURDATE())
                BETWEEN 15 AND 49')
            ->count();

        // MEF migradas — mujeres 15-49 migradas
        $mefMigradas = Persona::where(...'estado', 'migrado')
            ->whereRaw('TIMESTAMPDIFF(YEAR, ...) BETWEEN 15 AND 49')
            ->count();

        return [
            'mefActivas' => $mefActivas,
            'mefMigradas' => $mefMigradas,
            'migrantes' => $migrantes,
            'pctMigrantes' => $totalPadron > 0
                ? round($migrantes / $totalPadron * 100, 1) : 0,
            'pctMefMigradas' => ($mefActivas + $mefMigradas) > 0
                ? round($mefMigradas / ($mefActivas + $mefMigradas) * 100, 1) : 0,
        ];
    }
}
whereRaw para cálculos de edad

TIMESTAMPDIFF(YEAR, fecha_nacimiento, CURDATE()) calcula la edad exacta en MySQL. No es portable a otros motores, pero es preciso y eficiente. Las MEF (Mujeres en Edad Fértil) son mujeres de 15 a 49 años — indicador clave del SNIS.

Alerta contextual al > 10% migrantes

Si más del 10% del padrón está migrado, aparece un aviso explicando que las coberturas de programas serán más bajas de lo esperado. Este es un requerimiento del SNIS para justificar indicadores bajos.

5 CoberturaMensual — barras de progreso

app/Filament/Widgets/CoberturaMensual.php
class CoberturaMensual extends Widget
{
    protected string $view = 'filament.widgets.cobertura-mensual';
    protected int|string|array $columnSpan = 'full';
    protected static ?int $sort = 4;

    public function getDatosProperty(): array
    {
        $centroId = auth()->user()->centro_salud_id;
        $mes = (int) date('n');
        $anio = (int) date('Y');

        // Metas INE del centro para el año
        $metasIne = MetaIne::where('centro_salud_id', $centroId)
            ->where('anio', $anio)->get();

        // Meta mensual = meta anual ÷ 12
        $metaBcg = $metasIne
            ->where('grupo_etareo', 'nacimientos_esperados')
            ->sum('cantidad') / 12;

        $metaPrenatal = $metasIne
            ->where('grupo_etareo', 'embarazos_esperados')
            ->sum('cantidad') / 12;

        $metaCrecimiento = $metasIne
            ->whereIn('grupo_etareo',
                ['menor_1', '1_anio', '2_anios', '3_anios', '4_anios'])
            ->sum('cantidad') / 12;

        // Datos reales del mes desde tablas prest_*
        $vacunasBcg = PrestVacuna::where(...)
            ->where('tipo_vacuna', 'BCG')
            ->selectRaw('SUM(dentro_m + dentro_f + fuera_m + fuera_f)')
            ->value('total') ?? 0;

        return [
            ['nombre' => 'BCG', 'meta' => $metaBcg,
             'atendidos' => $vacunasBcg],
            ['nombre' => 'Prenatal', 'meta' => $metaPrenatal,
             'atendidos' => $prenatales],
            ['nombre' => 'Crecimiento', 'meta' => $metaCrecimiento,
             'atendidos' => $crecimiento],
        ];
    }
}

Vista Blade — barras de progreso con colores

6 Resumen

Lo que aprendimos

  • StatsOverviewWidget — solo PHP, Filament genera las tarjetas (Stat::make)
  • Widget base + $view — define tu propia vista Blade para control total
  • getDatosProperty() — computed property de Livewire (get{Name}Property → $this->name)
  • $columnSpan = 'full' — widget ocupa todo el ancho del dashboard
  • $sort — orden de aparición en el dashboard
  • x-filament-widgets::widget — wrapper obligatorio para widgets custom
  • x-filament::section — componente de sección con heading, icon, icon-color
  • whereRaw + TIMESTAMPDIFF — cálculos de edad en MySQL
  • selectRaw + SUM — agregaciones complejas para coberturas
  • Barras de progreso — Tailwind CSS con colores condicionales (verde/amarillo/rojo)
  • Patrón defensivo — siempre validar centro_salud_id antes de queries
