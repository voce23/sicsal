1 · Modelo Comentario — 10 líneas

El modelo más simple del proyecto:

// app/Models/Comentario.php — 10 líneas
class Comentario extends Model
{
    protected $fillable = ['pagina', 'nombre', 'contenido'];
}

Sin relaciones, sin casts, sin scopes. La columna pagina actúa como discriminador que permite usar una sola tabla para comentarios de todas las vistas: "blog-mi-slug", "poblacion", "cai", "causas-consulta", "comunidades".

Polimorfismo por convención: En vez de crear tablas separadas (comentarios_blog, comentarios_cai…), una sola tabla con discriminador pagina cubre todos los casos.

2 · Componente Comentarios — captcha y publicación

// app/Livewire/Comentarios.php — 70 líneas
class Comentarios extends Component
{
    public string $pagina;

    #[Validate('required|string|max:100')]
    public string $nombre = '';

    #[Validate('required|string|min:3|max:2000')]
    public string $contenido = '';

    #[Validate('required|integer')]
    public ?int $captchaRespuesta = null;

    public int $captchaA = 0;
    public int $captchaB = 0;
    public bool $exito = false;
}

Captcha matemático

public function generarCaptcha(): void
{
    $this->captchaA = random_int(1, 15);
    $this->captchaB = random_int(1, 15);
    $this->captchaRespuesta = null;
}

Publicación con sanitización

public function publicar(): void
{
    $this->validate();

    // Verificar captcha manualmente
    if ((int) $this->captchaRespuesta !== ($this->captchaA + $this->captchaB)) {
        $this->addError('captchaRespuesta', 'La respuesta del captcha es incorrecta.');
        $this->generarCaptcha();  // Regenerar para evitar brute-force
        return;
    }

    Comentario::create([
        'pagina'    => $this->pagina,
        'nombre'    => strip_tags($this->nombre),     // XSS prevention
        'contenido' => strip_tags($this->contenido),  // XSS prevention
    ]);

    $this->reset(['nombre', 'contenido']);
    $this->generarCaptcha();
    $this->exito = true;
}
strip_tags(): Defensa contra XSS. Elimina todo HTML del nombre y contenido antes de persistir. Como estos comentarios son públicos (sin autenticación), la sanitización es esencial.

Computed — últimos 50 comentarios

public function getComentariosProperty()
{
    return Comentario::where('pagina', $this->pagina)
        ->orderByDesc('created_at')
        ->limit(50)
        ->get();
}

4 · Avatares determinísticos por inicial

Cada comentario muestra un avatar circular con la primera letra del nombre y un color determinístico:

@php
    $colores = [
        'bg-teal-500', 'bg-cyan-500', 'bg-blue-500',
        'bg-violet-500', 'bg-rose-500', 'bg-amber-500',
        'bg-emerald-500',
    ];
    // ord() del primer carácter mod 7 → mismo nombre = mismo color
    $color = $colores[
        ord(mb_strtolower(mb_substr($c->nombre, 0, 1))) % count($colores)
    ];
@endphp

<div class="size-10 rounded-full {{ $color }} text-white font-bold">
    {{ mb_strtoupper(mb_substr($c->nombre, 0, 1)) }}
</div>

El truco: ord() convierte el primer carácter Unicode a su valor numérico, y % 7 lo mapea a uno de los 7 colores. Así, "Ana" siempre tendrá el mismo color en toda la aplicación, sin necesidad de almacenar preferencias.

Bubble de chat asimétrico

<!-- rounded-2xl + rounded-tl-md = esquina superior-izquierda recta -->
<div class="rounded-2xl rounded-tl-md border bg-white px-5 py-4
            transition group-hover:shadow-md">
    <span class="font-semibold">{{ $c->nombre }}</span>
    <span class="text-xs text-gray-400">{{ $c->created_at->diffForHumans() }}</span>
    <p>{{ $c->contenido }}</p>
</div>

5 · Integración en todas las páginas

El componente se incrusta como hijo Livewire en 6 páginas públicas:

<!-- Cada página pública incluye: -->
<livewire:comentarios pagina="blog-{{ $post->slug }}" />
<livewire:comentarios pagina="poblacion" />
<livewire:comentarios pagina="cai" />
<livewire:comentarios pagina="causas-consulta" />
<livewire:comentarios pagina="comunidades" />

El atributo pagina se pasa como parámetro del mount() y filtra los comentarios de esa vista específica. Un solo componente, múltiples instancias aisladas.

IDs únicos: Los inputs usan id="nombre-{{ $pagina }}" para evitar conflictos si hubiera múltiples instancias en una misma página.

6 · Acción Logout — invocable con seguridad

El cierre de sesión es una clase invocable (Action) de 23 líneas:

// app/Livewire/Actions/Logout.php
class Logout
{
    public function __invoke()
    {
        // 1. Cerrar sesión del guard 'web'
        Auth::guard('web')->logout();

        // 2. Invalidar la sesión (destruir todos los datos)
        Session::invalidate();

        // 3. Regenerar token CSRF (prevenir session fixation)
        Session::regenerateToken();

        return redirect('/');
    }
}

Los tres pasos son obligatorios para seguridad:

PasoSin él, el riesgo es...
Auth::guard('web')->logout()El usuario queda autenticado aunque crea que cerró sesión
Session::invalidate()Los datos de sesión (carrito, preferencias) persisten y podrían ser reutilizados
Session::regenerateToken()Session fixation attack: un atacante podría reutilizar el token CSRF de la sesión anterior
¿Por qué clase invocable? Patrón de Jetstream/Fortify. Se usa con app(Logout::class)() desde componentes Livewire sin necesidad de inyectar en un controlador. El __invoke() permite llamarla como función.

Resumen de la lección

  • Comentario es el modelo más simple: 3 campos fillable, sin relaciones. La columna pagina es un discriminador polimórfico.
  • El componente Comentarios (70 líneas) usa captcha matemático (random_int 1-15), validación con #[Validate], y strip_tags() contra XSS.
  • Los avatares son determinísticos: ord() de la inicial % 7 mapea a un color fijo, sin persistencia.
  • Las burbujas usan rounded-2xl + rounded-tl-md para el efecto de chat asimétrico.
  • Se integra en 6 páginas públicas con un simple <livewire:comentarios pagina="x" />.
  • Logout es una Action invocable de 23 líneas que ejecuta los 3 pasos de seguridad: logout, invalidate, regenerateToken.
