1 ¿Qué es Filament?

Filament es un framework de paneles de administración para Laravel. En SICSAL lo usamos para todo el backend: CRUD de personas, centros de salud, metas, usuarios, y las páginas de prestaciones mensuales.

2 Instalación

Terminal — instalar Filament 5
# Instalar el paquete
composer require filament/filament "~5.0"

# Ejecutar el instalador (crea AdminPanelProvider + assets)
php artisan filament:install --panels

El instalador crea:

Archivos creados por filament:install
app/Providers/Filament/
AdminPanelProvider.php
app/Filament/
Resources/
Pages/
Widgets/
resources/css/filament/admin/
theme.css

En composer.json, Filament añade un hook post-autoload:

composer.json — script de actualización
"scripts": {
    "post-autoload-dump": [
        // ... otros scripts
        "@php artisan filament:upgrade"
    ]
}

3 AdminPanelProvider — configuración completa

El corazón de Filament. Todo el panel se configura aquí con una API fluent (method chaining):

app/Providers/Filament/AdminPanelProvider.php
<?php

namespace App\Providers\Filament;

use Filament\Http\Middleware\Authenticate;
use Filament\Http\Middleware\AuthenticateSession;
use Filament\Http\Middleware\DisableBladeIconComponents;
use Filament\Http\Middleware\DispatchServingFilamentEvent;
use Filament\Navigation\NavigationGroup;
use Filament\Pages\Dashboard;
use Filament\Panel;
use Filament\PanelProvider;
use Filament\Support\Colors\Color;
use Filament\Support\Icons\Heroicon;
use Filament\View\PanelsRenderHook;
use Filament\Widgets\AccountWidget;
use Illuminate\Cookie\Middleware\AddQueuedCookiesToResponse;
use Illuminate\Cookie\Middleware\EncryptCookies;
use Illuminate\Foundation\Http\Middleware\PreventRequestForgery;
use Illuminate\Routing\Middleware\SubstituteBindings;
use Illuminate\Session\Middleware\StartSession;
use Illuminate\Support\Facades\Vite;
use Illuminate\Support\HtmlString;
use Illuminate\View\Middleware\ShareErrorsFromSession;

class AdminPanelProvider extends PanelProvider
{
    public function panel(Panel $panel): Panel
    {
        return $panel
            ->default()
            ->id('admin')
            ->path('admin')
            ->viteTheme('resources/css/filament/admin/theme.css')
            ->login()
            ->colors(['primary' => Color::Green])
            ->brandName('SIMUES')
            ->sidebarCollapsibleOnDesktop()
            // ... auto-descubrimiento, grupos nav, hooks, middleware
    }
}

4 Colores y marca

Configuración de identidad visual
->default()                     // Panel principal (solo puede haber uno)
->id('admin')                   // Identificador interno
->path('admin')                 // URL: sicsal.test/admin
->login()                        // Habilita /admin/login
->colors(['primary' => Color::Green]) // Todo verde — salud
->brandName('SIMUES')            // Nombre en el sidebar
->sidebarCollapsibleOnDesktop()  // Sidebar plegable
¿Por qué "SIMUES" y no "SICSAL"?

SIMUES es el nombre de la marca visible: Sistema Municipal de Salud. SICSAL es el nombre técnico del proyecto. Los usuarios del sistema ven "SIMUES" en el panel.

6 Auto-descubrimiento de Resources, Pages y Widgets

Discover pattern
->discoverResources(
    in:  app_path('Filament/Resources'),
    for: 'App\Filament\Resources'
)
->discoverPages(
    in:  app_path('Filament/Pages'),
    for: 'App\Filament\Pages'
)
->discoverWidgets(
    in:  app_path('Filament/Widgets'),
    for: 'App\Filament\Widgets'
)
No necesitas registrar nada manualmente

Con discoverResources(), Filament escanea automáticamente la carpeta app/Filament/Resources/ y registra cualquier clase Resource que encuentre. Lo mismo para Pages y Widgets. Solo creas el archivo y Filament lo detecta.

Adicionalmente, se registran explícitamente el Dashboard y el AccountWidget:

Registros explícitos
->pages([
    Dashboard::class,  // Página principal del panel
])
->widgets([
    AccountWidget::class,  // Widget "Mi cuenta" en dashboard
])

7 Render hooks: Vite + sidebar accordion

Los render hooks permiten inyectar HTML/JS en puntos específicos del panel.

Hook 1: Inyectar app.js con Vite

HEAD_END — cargar JS del proyecto
->renderHook(
    PanelsRenderHook::HEAD_END,
    fn () => app(\Illuminate\Foundation\Vite::class)('resources/js/app.js'),
)

Esto carga app.js (que incluye Chart.js y Alpine.js) dentro del panel admin.

Hook 2: Sidebar accordion (solo un grupo abierto)

SCRIPTS_AFTER — comportamiento accordion
->renderHook(
    PanelsRenderHook::SCRIPTS_AFTER,
    fn () => new HtmlString('
        <script>
            document.addEventListener("alpine:initialized", () => {
                const store = Alpine.store("sidebar");
                if (!store) return;

                store.toggleCollapsedGroup = function(group) {
                    const groups = Array.isArray(this.collapsedGroups)
                        ? this.collapsedGroups : [];
                    const isCollapsed = groups.includes(group);

                    if (isCollapsed) {
                        // Al expandir: colapsar todos los demás
                        const allGroups = [...document.querySelectorAll(
                            "[data-group-label]"
                        )].map(el => el.dataset.groupLabel);
                        this.collapsedGroups = allGroups
                            .filter(g => g !== group);
                    } else {
                        this.collapsedGroups = groups.concat(group);
                    }
                };
            });
        </script>
    ')
)
Comportamiento accordion

Este script modifica el Alpine.js store del sidebar para que al abrir un grupo de navegación, todos los demás se colapsen. Sin esto, puedes tener múltiples grupos abiertos a la vez, lo que en un sidebar de 7 grupos quedaría muy largo.

8 Middleware del panel

Dos capas de middleware
// Middleware para TODAS las rutas del panel
->middleware([
    EncryptCookies::class,
    AddQueuedCookiesToResponse::class,
    StartSession::class,
    AuthenticateSession::class,
    ShareErrorsFromSession::class,
    PreventRequestForgery::class,       // CSRF
    SubstituteBindings::class,
    DisableBladeIconComponents::class, // Filament
    DispatchServingFilamentEvent::class,
])

// Middleware adicional para rutas que requieren autenticación
->authMiddleware([
    Authenticate::class,  // Redirige a login si no está autenticado
])
¿Y RequireAprobado?

El middleware RequireAprobado (lección 18) se registra a nivel global en el kernel, no aquí. Actúa antes que los middleware del panel — si el usuario no tiene activo = true, se redirige a /pendiente antes de llegar al panel.

/* SICSAL — Sidebar personalizado para personal de salud */

   - Configuración: slate
   - Padrón Poblacional: blue
   - Salud Infantil: amber
   - Salud Materna: pink
   - Prestaciones Mensuales: emerald
   - Estadísticas: violet
   - Informes: cyan
*/

/* Items activos: green bg + shadow */
/* Hover: green-50 */
/* Brand/header area: green-50 */
@source — para que Tailwind escanee Filament

Las directivas @source le dicen a Tailwind CSS que escanee las carpetas de Filament al generar clases CSS. Sin esto, las clases personalizadas en tus Resources/Pages no serían incluidas en el build.

El tema se conecta con ->viteTheme('resources/css/filament/admin/theme.css') en el provider. Vite lo compila junto con el resto de assets.

10 Registro del provider

El AdminPanelProvider se registra en bootstrap/providers.php:

bootstrap/providers.php
<?php

return [
    App\Providers\AppServiceProvider::class,
    App\Providers\Filament\AdminPanelProvider::class,
    App\Providers\FortifyServiceProvider::class,
];

Laravel carga estos providers al arrancar. El orden importa poco aquí porque cada provider se resuelve de forma independiente.

11 Resumen

Lo que aprendimos

  • Filament 5 se instala con composer require + filament:install --panels
  • AdminPanelProvider configura todo: ruta (/admin), colores (Green), marca (SIMUES)
  • 7 NavigationGroups con iconos Heroicon organizan el sidebar
  • discoverResources/Pages/Widgets detecta automáticamente clases en las carpetas
  • Render hooks: HEAD_END para Vite JS, SCRIPTS_AFTER para sidebar accordion
  • Middleware: 9 para todas las rutas + Authenticate para rutas protegidas
  • theme.css: importa el tema base de Filament + @source para Tailwind
  • El provider se registra en bootstrap/providers.php
