    return view('public.pendiente');

2 · La ruta inicio: datos de la landing page

La ruta / usa un closure extenso (~130 líneas) que calcula todo in-place:

Route::get('/', function () {
    $anio = (int) date('Y');
    $centrosIds = [38, 39, 40, 41, 42, 1]; // orden de presentación

    // 1️⃣ Población total (1 query)
    $poblacionTotal = DB::table('metas_ine')
        ->whereIn('centro_salud_id', $centrosIds)
        ->where('anio', $anio)
        ->whereIn('grupo_etareo', ['menor_5', 'mayor_5'])
        ->sum('cantidad');

    // 2️⃣ Metas INE en bulk (1 query)
    $todasMetas = DB::table('metas_ine')->...->groupBy('centro_salud_id');

    // 3️⃣ Personas reales en bulk (1 query)
    $todasPersonas = DB::table('personas')
        ->whereIn('centro_salud_id', $centrosIds)
        ->where('activo', true)
        ->select('centro_salud_id', 'sexo',
            DB::raw('TIMESTAMPDIFF(YEAR, fecha_nacimiento, CURDATE()) AS edad'))
        ->groupBy('centro_salud_id');

    // 4️⃣ Construir pirámides por centro
    foreach ($centrosIds as $cid) {
        // Calcular ineM, ineF, realM, realF para cada grupo etáreo
        $piramides[] = [
            'id' => $cid, 'nombre' => ..., 'labels' => $labels,
            'ineM' => $ineM, 'ineF' => $ineF,
            'tieneReal' => $tieneReal,
            'realM' => $realM, 'realF' => $realF,
        ];
    }

    // 5️⃣ Centros con coordenadas para Leaflet
    $centrosMapas = $centrosDb
        ->filter(fn($c) => $c->latitud && $c->longitud)
        ->map(fn($c) => [
            'id' => $c->id, 'nombre' => $c->nombre,
            'lat' => (float) $c->latitud,
            'lng' => (float) $c->longitud,
        ])->values();

    return view('public.inicio', compact(...));
})->name('home');
Optimización N+0 — La ruta inicio solo ejecuta 3 queries SQL (metas_ine, personas, centros_salud) para todos los centros. Los datos se agrupan y procesan en PHP con collections.

3 · settings.php — Rutas de perfil y seguridad

use Laravel\Fortify\Features;

Route::middleware(['auth'])->group(function () {
    Route::redirect('settings', 'settings/profile');
    Route::livewire('settings/profile', 'pages::settings.profile')
        ->name('profile.edit');
});

Route::middleware(['auth', 'verified'])->group(function () {
    Route::livewire('settings/appearance', 'pages::settings.appearance')
        ->name('appearance.edit');

    Route::livewire('settings/security', 'pages::settings.security')
        ->middleware(
            when(
                Features::canManageTwoFactorAuthentication()
                    && Features::optionEnabled(Features::twoFactorAuthentication(), 'confirmPassword'),
                ['password.confirm'],
            ),
        )
        ->name('security.edit');
});

Tres páginas settings: perfil (auth), apariencia (auth+verified), seguridad (auth+verified, posiblemente password.confirm).

4 · bootstrap/app.php — Middleware y enrutamiento

use Illuminate\Foundation\Application;
use Illuminate\Foundation\Configuration\Middleware;

return Application::configure(basePath: dirname(__DIR__))
    ->withRouting(
        web: __DIR__.'/../routes/web.php',
        commands: __DIR__.'/../routes/console.php',
        health: '/up',
    )
    ->withMiddleware(function (Middleware $middleware): void {
        $middleware->alias([
            'aprobado' => \App\Http\Middleware\RequireAprobado::class,
        ]);
    })
    ->withExceptions(function (Exceptions $exceptions): void {
        //
    })->create();
Laravel 12+ bootstrap — Todo se configura fluentemente en bootstrap/app.php usando la API Application::configure(). No hay Http/Kernel.php. Los alias de middleware se registran inline.

5 · RequireAprobado — Middleware de activación

namespace App\Http\Middleware;

class RequireAprobado
{
    public function handle(Request $request, Closure $next): Response
    {
        if ($request->user() && ! $request->user()->activo) {
            return redirect()->route('pendiente');
        }

        return $next($request);
    }
}

Solo 15 líneas. Si el usuario no tiene activo = true, se redirige a la página de espera. El admin activa al usuario desde el panel Filament.

7 · AppServiceProvider — Observers y defaults

class AppServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        $this->configureDefaults();
        $this->registrarObservers();
    }

    protected function registrarObservers(): void
    {
        VacunaNino::observe(VacunaNinoObserver::class);
        Parto::observe(PartoObserver::class);
        ControlPrenatal::observe(ControlPrenatalObserver::class);
        CrecimientoInfantil::observe(CrecimientoInfantilObserver::class);
        MicronutrienteNino::observe(MicronutrienteNinoObserver::class);
        Anticoncepcion::observe(AnticoncepcionObserver::class);
    }

    protected function configureDefaults(): void
    {
        // Fechas inmutables por defecto
        Date::use(CarbonImmutable::class);

        // Prohibir migraciones destructivas en producción
        DB::prohibitDestructiveCommands(app()->isProduction());

        // Contraseñas fuertes en producción
        Password::defaults(fn() =>
            app()->isProduction()
                ? Password::min(12)
                    ->mixedCase()->letters()
                    ->numbers()->symbols()
                    ->uncompromised()
                : null
        );
    }
}
3 protecciones de producción:
  • CarbonImmutable — Previene mutaciones accidentales de fechas.
  • prohibitDestructiveCommands — Bloquea migrate:fresh, db:wipe, etc.
  • Password::defaults — 12 caracteres con mixedCase, números, símbolos y verificación HaveIBeenPwned.
