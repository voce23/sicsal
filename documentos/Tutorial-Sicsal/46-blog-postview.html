1 · El modelo Post — datos, constantes y helpers

El modelo Post (97 líneas) es la base del blog. Tiene campos para contenido, categorización y métricas:

// app/Models/Post.php
class Post extends Model
{
    protected $fillable = [
        'titulo', 'slug', 'extracto', 'contenido',
        'imagen_portada', 'categoria', 'etiquetas',
        'publicado', 'publicado_at', 'autor_nombre', 'vistas',
    ];

    protected $casts = [
        'etiquetas'    => 'array',      // JSON → array PHP
        'publicado'    => 'boolean',
        'publicado_at' => 'datetime',

}

Helpers del modelo

// Scope para posts publicados, ordenados por fecha
public function scopePublicados($query)
{
    return $query->where('publicado', true)
                 ->orderByDesc('publicado_at');
}

// Slug único con incremento automático
public static function generarSlug(string $titulo, ?int $excludeId = null): string
{
    $base = Str::slug($titulo);
    $slug = $base;
    $i    = 1;
    while (static::where('slug', $slug)
            ->when($excludeId, fn ($q) => $q->where('id', '!=', $excludeId))
            ->exists()) {
        $slug = "{$base}-{$i}";
        $i++;
    }
    return $slug;
}

// Tiempo estimado de lectura (200 palabras/min)
public function tiempoLectura(): int
{
    $palabras = str_word_count(strip_tags($this->contenido ?? ''));
    return max(1, (int) ceil($palabras / 200));
}
Patrón constantes-en-modelo: Las categorías, colores y gradientes están en el modelo (no en config ni en base de datos). Esto simplifica el acceso desde Blade sin queries extra y centraliza la configuración visual.

2 · Blog.php — el componente Livewire del listado

El componente Blog (96 líneas) es un Livewire full-page con paginación, búsqueda y filtros por categoría/etiqueta:

// app/Livewire/Blog.php
class Blog extends Component
{
    use WithPagination;

    // Propiedades reactivas sincronizadas con la URL
    #[Url(as: 'q')]
    public string $busqueda = '';

    #[Url(as: 'cat')]
    public string $categoria = '';

    #[Url(as: 'tag')]
    public string $etiqueta = '';

    public string $vista = 'rejilla'; // 'rejilla' | 'lista'
}

URL binding con #[Url]

El atributo #[Url(as: 'q')] sincroniza la propiedad $busqueda con el parámetro ?q= de la URL. Esto permite:

  • URLs compartibles: /blog?q=vacunas&cat=vacunacion&tag=PAI reproduce los filtros exactos.
  • Navegación con historial: El usuario puede usar atrás/adelante del navegador.
  • SEO: Los buscadores indexan las URLs con filtros.

Reset de paginación

// Cada vez que cambia un filtro, volver a página 1
public function updatingBusqueda(): void
{
    $this->resetPage();
}

public function updatingCategoria(): void
{
    $this->resetPage();
}

// Toggle: click en categoría activa → la desactiva
public function setCategoria(string $cat): void
{
    $this->categoria = $this->categoria === $cat ? '' : $cat;
    $this->resetPage();
}

3 · Filtros reactivos con URL binding

El método render() construye la query dinámicamente según los filtros activos:

public function render()
{
    // 1. Query base: solo publicados
    $query = Post::publicados();

    // 2. Filtro de búsqueda por título o extracto
    if ($this->busqueda) {
        $query->where(function ($q) {
            $q->where('titulo',   'like', "%{$this->busqueda}%")
              ->orWhere('extracto', 'like', "%{$this->busqueda}%");
        });
    }

    // 3. Filtro por categoría exacta
    if ($this->categoria) {
        $query->where('categoria', $this->categoria);
    }

    // 4. Filtro por etiqueta (JSON contains)
    if ($this->etiqueta) {
        $query->whereJsonContains('etiquetas', $this->etiqueta);
    }

    // 5. Paginar 9 posts por página
    $posts = $query->paginate(9);

Datos de la barra lateral

    // Categorías con conteo
    $categorias = Post::publicados()
        ->selectRaw('categoria, count(*) as total')
        ->groupBy('categoria')
        ->reorder()     // Quita el orderByDesc del scope
        ->pluck('total', 'categoria');

    // 5 artículos recientes (mini-preview)
    $recientes = Post::publicados()
        ->limit(5)
        ->get(['id', 'titulo', 'slug', 'publicado_at', 'imagen_portada']);

    // Nube de etiquetas — top 20 más usadas
    $todasEtiquetas = Post::publicados()
        ->whereNotNull('etiquetas')
        ->pluck('etiquetas')    // Collection de arrays
        ->flatten()           // Un solo array plano
        ->filter()            // Quita nulls/vacíos
        ->countBy()           // ['PAI' => 5, 'nutrición' => 3]
        ->sortDesc()
        ->take(20);

    return view('livewire.blog', [...])
        ->layout('layouts.public', ['title' => 'Blog de Salud — SIMUES']);
}
whereJsonContains: MySQL busca dentro del array JSON de etiquetas. Si etiquetas = ["PAI", "vacunas"], whereJsonContains('etiquetas', 'PAI') lo encuentra. No necesitas JSON_EXTRACT manual.

6 · PostView.php — detalle del artículo

El componente PostView (36 líneas) es extremadamente elegante:

// app/Livewire/PostView.php
class PostView extends Component
{
    public Post $post;

    public function mount(string $slug): void
    {
        // Buscar post publicado por slug, o 404
        $this->post = Post::where('slug', $slug)
            ->where('publicado', true)
            ->firstOrFail();

        // Incrementar vistas SIN tocar timestamps
        Post::withoutTimestamps(function () {
            $this->post->increment('vistas');
        });
    }

    public function render()
    {
        // 3 posts relacionados (misma categoría, diferente ID)
        $relacionados = Post::publicados()
            ->where('categoria', $this->post->categoria)
            ->where('id', '!=', $this->post->id)
            ->limit(3)
            ->get(['id', 'titulo', 'slug', 'imagen_portada',
                  'publicado_at', 'extracto']);

        return view('livewire.post-view', [...])
            ->layout('layouts.public', ['title' => $this->post->titulo . ' — SIMUES']);
    }
}

Técnica: withoutTimestamps

Post::withoutTimestamps() incrementa el contador de vistas sin modificar updated_at. Sin esto, cada visita actualizaría la fecha de modificación del post, alterando el orden de "recientes".

Resumen de la lección

  • El modelo Post centraliza categorías, colores y gradientes como constantes, con helpers (generarSlug, tiempoLectura, scopePublicados).
  • Blog.php usa #[Url] para filtros en la URL, WithPagination con 9 posts/página, y construye 4 queries paralelas (posts, categorías, recientes, etiquetas).
  • La vista del blog tiene dos modos (rejilla/lista), barra lateral con búsqueda debounce, categorías con conteo y nube de etiquetas top 20.
  • PostView.php carga por slug, incrementa vistas con withoutTimestamps, y muestra 3 relacionados de la misma categoría.
  • El hero del post se adapta: overlay oscuro sobre imagen, o gradiente suave como fallback. El contenido usa prose prose-teal para tipografía automática.
  • Los comentarios se embeben como <livewire:comentarios> con prefijo blog- único por artículo.
