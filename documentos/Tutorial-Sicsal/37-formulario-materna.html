Array de arrays vs array asociativo

$partosConfig es un array de arrays (no asociativo). Cada fila tiene 4 elementos en posición fija: [0]=tipo, [1]=lugar, [2]=atendido_por, [3]=label. Se accede por desestructuración: [$tipo, $lugar, $atendido, $label]. Es útil cuando la "clave" del registro es compuesta (3 campos) y no se puede representar como un simple string.

2 cargarDatos() — claves compuestas de hasta 4 niveles

FormularioMaterna.php — cargar partos (clave de 4 partes)
// Partos: clave compuesta = tipo__lugar__atendido_por__grupo_etareo
$rowsPartos = PrestParto::where(...)
    ->get()->keyBy(
        fn($r) => "{$r->tipo}__{$r->lugar}__{$r->atendido_por}__{$r->grupo_etareo}"
    );

foreach (self::$partosConfig as [$tipo, $lugar, $atendido, $label]) {
    foreach (array_keys(self::$gruposParto) as $ge) {
        $key = "{$tipo}__{$lugar}__{$atendido}__{$ge}";
        $row = $rowsPartos->get($key);
        $this->partos[$key] = $row->cantidad ?? 0;
    }
}
Desestructuración de arrays en PHP 8

foreach ($partosConfig as [$tipo, $lugar, $atendido, $label]) usa desestructuración de arrays de PHP 7.1+. Equivale a $tipo = $item[0]; $lugar = $item[1]; etc. Leer bien la posición: la clave compuesta del modelo tiene 4 campos, pero el label (posición 3) es solo para la UI.

3 guardarX() y verificarCero — dispatch de evento

FormularioMaterna.php — guardarPrenatales + verificarCero
public function guardarPrenatales(): void
{
    if ($this->mesCerrado) return;
    $centroId = auth()->user()->centro_salud_id;

    foreach ($this->prenatales as $key => $datos) {
        [$tc, $ge] = explode('__', $key);  // Reconstruir campos
        PrestPrenatal::updateOrCreate(
            ['centro_salud_id' => $centroId, 'mes' => $this->mes,
             'anio' => $this->anio,
             'tipo_control' => $tc, 'grupo_etareo' => $ge],
            ['dentro' => (int)$datos['dentro'],
             'fuera'  => (int)$datos['fuera']]
        );
    }

    $this->verificarCeroPrenatales($centroId);
    Notification::make()->title('Prenatales guardados')->success()->send();
}

private function verificarCeroPrenatales(int $centroId): void
{
    $total = PrestPrenatal::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)->where('anio', $this->anio)
        ->sum(\DB::raw('dentro + fuera'));

    if ($total == 0) {
        $this->dispatch('abrir-justificacion-cero',
            indicador: 'control_prenatal');
    }
}
Cero justificado — regla de negocio clave

Si el total de prenatales o puerperio es cero, se dispara un evento abrir-justificacion-cero que abre un modal pidiendo justificación. Es una regla del SNIS Bolivia: todo indicador clave en cero requiere explicación obligatoria. El modal se incluye con @include('filament.pages.partials.justificacion-cero-modal') al final del Blade.

public function guardarPartosYPuerperio(): void
{
    // 1. Partos — desestructurar $partosConfig
    foreach (self::$partosConfig as [$tipo, $lugar, $atendido, $label]) {

                ['centro_salud_id' => $centroId, 'mes' => ...,
                 'tipo' => $tipo, 'lugar' => $lugar,
                 'atendido_por' => $atendido, 'grupo_etareo' => $ge],
                ['cantidad' => (int)($this->partos[$key] ?? 0)]
            );
        }
    }

    // 2. Puerperio — 4 tipos de control
    foreach ($this->puerperio as $tc => $cantidad) { ... }

    // 3. Recién nacidos — 10 indicadores
    foreach ($this->recienNacidos as $ind => $cantidad) { ... }

    $this->verificarCeroPuerperio($centroId);
}
Un botón, tres tablas

guardarPartosYPuerperio() guarda en 3 tablas a la vez (prest_partos, prest_puerperio, prest_recien_nacidos) porque en la UI están en la misma pestaña. Agrupación lógica: el usuario piensa en "partos y puerperio" como una unidad, no como 3 tablas separadas.

$loop->first + rowspan

El @foreach interno itera 6 grupos etáreos por cada tipo de control. Solo en la primera iteración ($loop->first) se renderiza la celda del tipo con rowspan="{{ count($labelsGE) }}". Esto fusiona las 6 filas del grupo en una sola celda vertical.

wire:model con 4 partes en la clave

wire:model="partos.vaginal__domicilio__partera_capacitada__15_19" enlaza directamente al valor $this->partos['vaginal__domicilio__partera_capacitada__15_19']. Las 4 partes separadas por __ corresponden a los 4 campos de la clave compuesta. 7 tipos × 6 grupos = 42 inputs.

11 mini-tablas = 132 inputs

Se genera una sub-tabla por cada uno de los 11 métodos anticonceptivos. Cada una tiene 2 filas (nueva/continua) × 6 columnas (grupos etáreos) = 12 inputs por tabla, total 132 inputs. Triple @foreach anidado: método → tipo → grupo.

8 Resumen

Lo que aprendimos en esta lección

  • Desestructuración de arrays en foreach: [$tipo, $lugar, $atendido, $label]
  • Claves compuestas de hasta 4 partes separadas por __
  • explode('__', $key) para reconstruir los campos al guardar
  • $loop->first + rowspan para fusionar celdas en tablas Blade
  • Sub-tablas por método — @foreach que genera N tablas completas
  • verificarCero + dispatch() — disparar modal de justificación
  • Un botón → 3 tablas — agrupación lógica por sección del formulario
  • @include para partials — reutilizar el modal de justificación
  • 132 inputs en anticoncepción, 42 inputs en partos — todo wire:model
