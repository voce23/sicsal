1 · Arquitectura del componente InformeCAI

El componente InformeCAI (106 líneas) es el más conciso de la interfaz pública porque delega toda la lógica pesada a CaiHelper (que vimos en la Lección 42). Es un orquestador puro:

// app/Livewire/InformeCAI.php
class InformeCAI extends Component
{
    use HasCentroSelector;

    #[Url]
    public string $periodo = 'cai1';    // 'cai1' | 'cai2' | 'gestion'

    #[Url]
    public int $anio = 0;

    public function mount(): void
    {
        $this->anio = $this->anio ?: (int) date('Y');
        $this->mountHasCentroSelector();
    }
}

Responsabilidades del componente:

2 · Computed properties — datos, centros, periodo

Tres computed properties proporcionan datos a la vista sin re-calcular en cada render innecesario:

// Lista de centros disponibles para el usuario
public function getCentrosProperty(): array
{
    return $this->centrosDisponibles;
}

// Todos los datos del informe — el corazón
public function getDatosProperty(): ?array
{
    if ($this->centroSaludId === 0) {
        return null;
    }
    return CaiHelper::getDatosInforme(
        $this->centroSaludId,
        $this->periodo,
        $this->anio
    );
}

// Nombre legible del periodo
public function getNombrePeriodoProperty(): string
{
    return CaiHelper::getNombrePeriodo($this->periodo, $this->anio);
}

El getDatosProperty retorna null si no hay centro seleccionado, lo que la vista usa para mostrar un mensaje de "seleccione un centro".

Lazy evaluation: $this->datos no se calcula hasta que la vista lo accede. Cuando el usuario cambia centro/periodo/año, Livewire re-renderiza y la computed se re-evalúa con los valores nuevos.

3 · Generación de descargas — Excel, PDF, PPTX

El componente ofrece tres formatos de descarga, cada uno con su propia estrategia:

Excel con Maatwebsite

public function generarExcel()
{
    if ($this->centroSaludId === 0) return;

    $datos  = CaiHelper::getDatosInforme(...);
    $nombre = 'InformeCAI_'
        . str_replace(' ', '_', $datos['encabezado']['centro_nombre'])
        . '_' . $this->periodo . '_' . $this->anio . '.xlsx';

    return Excel::download(
        new InformeCAIExport($datos),
        $nombre
    );
}

PDF con DomPDF

public function generarPdf()
{
    $datos  = CaiHelper::getDatosInforme(...);
    $nombre = 'InformeCAI_..._' . $this->anio . '.pdf';

    $pdf = Pdf::loadView('pdf.informe-cai', ['datos' => $datos])
        ->setPaper('a4', 'portrait')
        ->setOption('margin-top', 15)
        ->setOption('margin-bottom', 15)
        ->setOption('margin-left', 15)
        ->setOption('margin-right', 15);

    return response()->streamDownload(
        fn () => print($pdf->output()),
        $nombre
    );
}

PowerPoint con PhpPresentation

public function generarPptx()
{
    $centro = CentroSalud::find($this->centroSaludId);

    // PresentacionCAI genera TODAS las diapositivas del municipio
    $pptx    = new PresentacionCAI($centro->municipio_id, $this->periodo, $this->anio);
    $tmpFile = $pptx->generate();  // Retorna path temporal

    return response()->streamDownload(
        function () use ($tmpFile) {
            readfile($tmpFile);
            @unlink($tmpFile);  // Limpiar archivo temporal
        },
        $nombre,
        ['Content-Type' => 'application/vnd.openxmlformats-officedocument.presentationml.presentation']
    );
}
streamDownload + @unlink: El PPTX se genera como archivo temporal. readfile() lo envía al navegador y luego @unlink() lo elimina. El @ suprime errores si el archivo ya fue eliminado.

5 · Secciones colapsables con Alpine x-collapse

Las secciones 3, 5 y 6 son colapsables para no abrumar la pantalla. Usan Alpine.js con x-collapse:

<!-- Sección colapsable con Alpine -->
<div x-data="{ open: false }" class="rounded-xl border bg-white p-6">
    <!-- Botón toggle con chevron rotable -->
    <button @click="open = !open"
            class="flex w-full items-center justify-between text-left">
        <h2>3. Prestaciones Acumuladas</h2>
        <svg :class="{ 'rotate-180': open }"
             class="size-5 text-gray-400 transition-transform">
            <!-- Chevron down -->
        </svg>
    </button>

    <!-- Contenido con animación x-collapse -->
    <div x-show="open" x-collapse class="mt-4 space-y-4">
        <!-- Tablas de prestaciones aquí -->
    </div>
</div>

Prestaciones acumuladas (Sección 3)

Dentro del colapsable, hay 5 sub-tablas:

  1. Vacunas: tipo, grupo etáreo, dentro M/F, fuera M/F, total.
  2. Micronutrientes: tipo → total.
  3. Control de Crecimiento: grupo → nuevos M/F, repetidos M/F.
  4. Recién Nacidos: indicador → total.
  5. Puerperio: tipo control → total.
x-collapse vs x-show: x-collapse (plugin de Alpine) anima la altura del elemento con transición suave. x-show solo controla visibilidad. Juntos crean un acordeón fluido.

<!-- Cobertura dual: INE % y Real % -->

Tasas de deserción

<!-- Tasa deserción: rojo >10%, amarillo >0%, verde 0% -->
<td class="{{
    $d['tasa'] > 10 ? 'text-red-500'    :
    ($d['tasa'] > 0  ? 'text-yellow-600' :

La deserción mide el porcentaje de personas que inician un esquema (ej. Pentavalente 1ra) pero no lo completan (Pentavalente 3ra). Una tasa >10% indica un problema serio de cobertura.

Resumen de la lección

  • InformeCAI.php (106 líneas) es un orquestador lean que delega la lógica pesada a CaiHelper.
  • Usa 3 computed properties (datos, centros, nombrePeriodo) para evaluación lazy.
  • Ofrece 3 formatos de descarga: Excel (Maatwebsite), PDF (DomPDF con streamDownload), PPTX (PhpPresentation con tmpfile + unlink).
  • La vista (399 líneas) muestra 7 secciones numeradas del informe CAI, con 3 secciones colapsables (Alpine x-collapse).
  • Los badges de meses muestran el estado de cierre (verde/amarillo), y las tablas de cobertura usan semáforo de 3 colores.
  • El bloque de firma incluye nombre del responsable, centro y fecha — listo para impresión.
