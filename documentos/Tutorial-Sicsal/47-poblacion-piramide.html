    ['label' => '20-39',   'min' => 20, 'max' => 39,  'ine' => ['20_39']],
    ['label' => '40-49',   'min' => 40, 'max' => 49,  'ine' => ['40_49']],
    ['label' => '50-59',   'min' => 50, 'max' => 59,  'ine' => ['50_59']],
    ['label' => '60+',     'min' => 60, 'max' => 200, 'ine' => ['mayor_60']],
];

Cada grupo tiene:

  • label: Texto visible en el gráfico y la tabla.
  • min/max: Rango de edad para queries de personas reales (TIMESTAMPDIFF).
  • ine: Campos del modelo MetaIne para consultar proyecciones INE.
1-4 años desagregados: El INE tiene datos por año individual (1_anio, 2_anios, etc.) para menores de 5, pero agrupados para mayores. La configuración maneja ambos casos con el array ine.

3 · buildPiramide() — INE vs. Real

El método buildPiramide() construye los datos para ambas pirámides en una sola pasada:

private function buildPiramide(int $centroId): array
{
    // 1. Cargar TODAS las metas INE del centro/año en memoria
    $metasIne = MetaIne::where('centro_salud_id', $centroId)
        ->where('anio', $this->anio)->get();

    foreach ($this->gruposConfig as $grupo) {
        // 2. Datos INE — filtrar de la colección en memoria
        $grupoIne    = $metasIne->whereIn('grupo_etareo', $grupo['ine']);
        $imExplicito = $grupoIne->where('sexo', 'M')->sum('cantidad');
        $ifExplicito = $grupoIne->where('sexo', 'F')->sum('cantidad');
        $ambos       = $grupoIne->where('sexo', 'ambos')->sum('cantidad');

        // 3. Fallback ambos → split M/F
        $im = $imExplicito > 0 ? $imExplicito : (int) ceil($ambos / 2);
        $if = $ifExplicito > 0 ? $ifExplicito : (int) floor($ambos / 2);

        // 4. Población real — queries a tabla personas
        $queryBase = Persona::where('centro_salud_id', $centroId)
            ->where('activo', true)
            ->where('estado', '!=', 'migrado');

        if ($grupo['max'] === 0) {
            // Menores de 1 año: fecha_nacimiento > hace 1 año
            $rm = (clone $queryBase)->where('sexo', 'M')
                ->where('fecha_nacimiento', '>', now()->subYear())->count();
        } else {
            // Otros grupos: TIMESTAMPDIFF entre min y max
            $rm = (clone $queryBase)->where('sexo', 'M')
                ->whereRaw('TIMESTAMPDIFF(YEAR, fecha_nacimiento, CURDATE())
                     BETWEEN ? AND ?', [$grupo['min'], $grupo['max']])
                ->count();
        }
    }

clone $queryBase: Sin clone, cada iteración del loop modificaría el mismo builder, acumulando condiciones WHERE. El patrón (clone $queryBase)->where(...)->count() garantiza queries aislados.

4 · La pirámide Chart.js — barras horizontales negativas

La técnica para crear una pirámide poblacional con Chart.js es usar un gráfico de barras horizontales apiladas donde los valores masculinos son negativos:

// Inicialización en Blade con @script + Alpine
window._crearPiramide = function(windowKey, canvasId, labels, dataM, dataF) {
    // Destruir gráfico previo si existe (para re-renders)
    if (window[windowKey]) {
        window[windowKey].destroy();
    }

    window[windowKey] = new Chart(document.getElementById(canvasId), {
        type: 'bar',
        data: {
            labels,  // ['<1 año', '1-4', '5-9', ...]
            datasets: [
                {
                    label: 'Masculino',
                    data: dataM.map(v => -v),  // ← NEGATIVO
                    backgroundColor: 'rgba(59,130,246,0.7)',  // Azul
                },
                {
                    label: 'Femenino',
                    data: dataF,                // ← POSITIVO
                    backgroundColor: 'rgba(236,72,153,0.7)', // Rosa
                },
            ],
        },
        options: {
            indexAxis: 'y',           // Barras horizontales
            responsive: true,
            scales: {
                x: {
                    stacked: true,
                    ticks: { callback: v => Math.abs(v) }  // Mostrar valor absoluto
                },
                y: {
                    stacked: true,
                    reverse: true  // Menores arriba, mayores abajo
                }
            },
            plugins: {
                tooltip: {
                    callbacks: {
                        label: c => c.dataset.label + ': ' + Math.abs(c.parsed.x)
                    }
                }
            }
        }
    });
};

¿Por qué valores negativos?

Chart.js no tiene un tipo "pirámide" nativo. El truco es:

  1. Hombres con valores negativos → barras hacia la izquierda.
  2. Mujeres con valores positivos → barras hacia la derecha.
  3. Math.abs() en ticks y tooltips para mostrar números positivos.
  4. reverse: true en Y para que los grupos más jóvenes estén arriba.

Inicialización con Alpine + data attribute

<!-- Los datos PHP se pasan como JSON en un data-attribute -->
<div data-pir='@json($pir)'
     x-data
     x-init="$nextTick(() => {
         const d = JSON.parse($el.dataset.pir);
         window._crearPiramide('_ine',  'piramideIne',  d.labels, d.ineM, d.ineF);
         window._crearPiramide('_real', 'piramideReal', d.labels, d.realM, d.realF);
     })">

  <canvas id="piramideIne"  height="300"></canvas>
  <canvas id="piramideReal" height="300"></canvas>

</div>
$nextTick: Espera a que Alpine termine de procesar el DOM antes de crear los gráficos. Sin esto, los <canvas> podrían no estar disponibles aún.

// Meses que incluye cada periodo
protected function getMeses(): array
{
    return match ($this->periodo) {
        'cai1'    => range(1, 4),    // Ene-Abr
        'cai2'    => range(1, 8),    // Ene-Ago
        'gestion' => range(1, 12),   // Ene-Dic
        default   => [(int) $this->periodo],  // Mes individual
    };
}

// Factor para ajustar metas anuales al periodo
protected function getFactorMeta(): float
{
    return match ($this->periodo) {
        'cai1'    => 4 / 12,   // 33.3%
        'cai2'    => 8 / 12,   // 66.7%
        'gestion' => 1.0,     // 100%
        default   => 1 / 12,  // 8.3%
    };
}

buildFila() — cálculo de doble cobertura

protected function buildFila(string $nombre, float $metaIne,
                              int $pobReal, int $atendidos): array
{
    return [
        'nombre'   => $nombre,
        'meta_ine' => round($metaIne),
        'pob_real' => $pobReal,
        'atendidos' => $atendidos,
        'cob_ine'  => $metaIne > 0
            ? round($atendidos / $metaIne * 100, 1) : 0,
        'cob_real' => $pobReal > 0
            ? round($atendidos / $pobReal * 100, 1) : 0,
    ];
}
Doble cobertura = la innovación de SICSAL: La cobertura INE puede ser >100% porque la meta no refleja la población real (migración). La cobertura real divide por personas realmente activas, dando una cifra más útil para la gestión local.

8 · Patrones técnicos destacados

Resumen de la lección

  • El componente Poblacion (222 líneas) tiene dos tabs: pirámide poblacional y análisis INE vs Real con doble cobertura.
  • La pirámide usa Chart.js con barras horizontales apiladas, valores masculinos negativos y Math.abs() en labels/tooltips.
  • buildPiramide() carga MetaIne en memoria una vez y filtra en PHP, pero hace queries individuales a Persona por grupo/sexo.
  • El tab INE vs Real calcula 21 indicadores de cobertura dual usando 5 modelos diferentes (vacunas, prenatales, partos, crecimiento, micronutrientes).
  • La migración contextualiza coberturas bajas: si >10% del padrón migró, se muestra una alerta explicativa.
  • El semáforo visual (verde ≥80%, amarillo ≥50%, rojo <50%) da feedback inmediato sobre el estado de cada programa.
