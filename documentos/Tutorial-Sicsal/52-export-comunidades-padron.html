1 · ComunidadesPoblacionExport — orquestador de 3 hojas

// app/Exports/ComunidadesPoblacionExport.php — 22 líneas
class ComunidadesPoblacionExport implements WithMultipleSheets
{
    use Exportable;

    public function __construct(private array $datos) {}

    public function sheets(): array
    {
        return [
            'Resumen'      => new Sheets\ComunidadesResumenSheet($this->datos),
            'Detalle Sexo' => new Sheets\ComunidadesDetalleSheet($this->datos),
            'INE vs Real'  => new Sheets\ComunidadesConsolidadoSheet($this->datos),
        ];

}

2 · ComunidadesResumenSheet — blue-800 con 9 grupos

La hoja Resumen (105 líneas) genera una tabla con columna dinámica basada en los grupos etáreos:

public function array(): array
{
    $grupos = $this->datos['grupos']; // 9 grupos etáreos

    // Header dinámico
    $header = ['Comunidad', 'Km', 'Total', 'H', 'M'];
    foreach ($grupos as $label) {
        $header[] = $label;          // "< 1 año", "1-4", "5-9", etc.
    }
    $header[] = 'Migr.';

    // Filas de datos
    foreach ($this->datos['filas'] as $fila) {
        $row = [$fila['comunidad'], $fila['km'], $fila['total'], $fila['hombres'], $fila['mujeres']];
        foreach ($grupos as $key => $label) { $row[] = $fila[$key]; }
        $row[] = $fila['migrantes'];
        $rows[] = $row;
    }

    // 3 filas de pie: TOTAL REAL, META INE, DIFERENCIA
    $t = $this->datos['totales'];
    $rows[] = ['TOTAL REAL', '', $t['total'], ...];
    $rows[] = ['META INE', '', $this->datos['metaIne']];

    $dif = $this->datos['diferencia'];
    $pct = round($t['total'] / $this->datos['metaIne'] * 100, 1);
    $rows[] = ['DIFERENCIA', '', ($dif >= 0 ? '+' : '') . $dif . ' (' . $pct . '%)'];
}

WithStyles vs WithEvents

A diferencia de los Sheets del InformeCAI (que usan WithEvents + EstiloExcel trait), los sheets de Comunidades usan WithStyles directamente:

public function styles(Worksheet $sheet): array
{
    $headerRow = 3;
    $totalRow  = $headerRow + count($this->datos['filas']) + 1;

    // Blue-800 header
    $sheet->getStyle("A{$headerRow}:O{$headerRow}")->applyFromArray([
        'font' => ['bold' => true, 'color' => ['rgb' => 'FFFFFF']],
        'fill' => ['startColor' => ['rgb' => '1E40AF']],
    ]);

    // Bordes + centrado en datos
    $sheet->getStyle("A{$headerRow}:O{$totalRow}")
        ->getBorders()->getAllBorders()
        ->setBorderStyle(Border::BORDER_THIN);
}

3 · ComunidadesDetalleSheet — H/F fusionado y colLetter()

El sheet más técnico (127 líneas) genera un header de dos filas con celdas fusionadas:

// Fila 3: grupo etáreo (cada grupo ocupa 2 columnas: H + M)
// Fila 4: "H" | "M" alternando bajo cada grupo

// Comunidad | < 1 año    | 1-4 años   | 5-9 años   | ...
//           | H    | M   | H    | M   | H    | M   | ...

colLetter() — conversión índice → columna Excel

private function colLetter(int $index): string
{
    $letter = '';
    while ($index >= 0) {
        $letter = chr(65 + ($index % 26)) . $letter;
        $index  = intdiv($index, 26) - 1;
    }
    return $letter;
}

Con 9 grupos × 2 (H/M) + Migrantes × 2 + Total × 2 = 24 columnas de datos + 1 comunidad = 25 columnas (hasta Y). El helper maneja correctamente indices > 26 (AA, AB, etc.) con el algoritmo de división entera.

Fusión programática de headers

$col = 1; // Después de columna A (Comunidad)
foreach ($grupos as $label) {
    $l1 = $this->colLetter($col);
    $l2 = $this->colLetter($col + 1);
    $sheet->mergeCells("{$l1}3:{$l2}3");  // "< 1 año" ocupa 2 cols
    $col += 2;
}
// + merge para Migrantes (2 cols) y Total (2 cols)

Green-600 (#166534) da identidad visual distinta al sheet de Detalle:

4 · ComunidadesConsolidadoSheet — INE vs Real con semáforo

La hoja Consolidado (100 líneas) compara los 9 grupos etáreos con un semáforo de cobertura:

public function array(): array
{
    $rows[] = ['Grupo', 'INE H', 'INE M', 'INE Total',
              'Real H', 'Real M', 'Real Total',
              'Diferencia', 'Cobertura'];

    foreach ($this->datos['consolidado'] as $row) {
        $rows[] = [
            $row['label'],
            $row['ine_m'], $row['ine_f'], $row['ine_total'],
            $row['real_m'], $row['real_f'], $row['real_total'],
            ($row['diferencia'] >= 0 ? '+' : '') . $row['diferencia'],
            $row['cobertura'] . '%',
        ];
    }
}

Colores de cobertura — 3 niveles

// A diferencia del 4-nivel del InformeCAI, aquí son solo 3
for ($r = $headerRow + 1; $r <= $lastDataRow; $r++) {
    $val   = $sheet->getCell("I{$r}")->getValue();
    $num   = (float) str_replace('%', '', $val);
    $color = $num >= 100 ? '059669'     // Verde ≥ 100%
           : ($num >= 80 ? 'D97706'     // Ámbar ≥ 80%
                          : 'DC2626');  // Rojo < 80%
    $sheet->getStyle("I{$r}")->getFont()->getColor()->setRGB($color);
}
Diseño de umbrales: El InformeCAI usa 4 niveles (≥95/≥80/≥50/<50) porque evalúa metas anuales. El Consolidado usa 3 niveles (≥100/≥80/<80) porque compara censo total con INE, donde ≥100% = cubierto, y ≥80% = aceptable.

5 · PadronComunidadesExport — hojas dinámicas por comunidad

A diferencia de los otros exports que tienen hojas fijas, PadronComunidadesExport genera una hoja por comunidad:

// app/Exports/PadronComunidadesExport.php — 27 líneas
class PadronComunidadesExport implements WithMultipleSheets
{
    public function __construct(private int $centroSaludId) {}

    public function sheets(): array
    {
        $centro      = CentroSalud::find($this->centroSaludId);
        $comunidades = Comunidad::where('centro_salud_id', $this->centroSaludId)
            ->orderBy('nombre')
            ->get();

        $sheets = [];
        foreach ($comunidades as $comunidad) {
            $sheets[] = new PadronComunidadSheet($comunidad, $centro);
        }

        return $sheets;
    }
}

Si un centro tiene 20 comunidades, el Excel resultante tendrá 20 pestañas, cada una con el listado nominal de personas activas.

Constructor recibe ID, no array: A diferencia de ComunidadesPoblacionExport (que recibe datos pre-calculados), este export recibe solo el centroSaludId y hace sus propios queries. Cada PadronComunidadSheet carga personas on-demand.

6 · PadronComunidadSheet — lista nominal con colores por sexo

Cada hoja (133 líneas) representa una comunidad con su lista de personas:

Nombre de pestaña con límite de 31 caracteres

public function title(): string
{
    return mb_substr($this->comunidad->nombre, 0, 31);
}
Límite de Excel: Los nombres de pestañas en Excel no pueden exceder 31 caracteres. mb_substr() trunca correctamente nombres largos con caracteres UTF-8.

Lazy-loading de personas

private ?Collection $personas = null;

private function getPersonas(): Collection
{
    if ($this->personas === null) {
        $this->personas = Persona::where('comunidad_id', $this->comunidad->id)
            ->where('activo', true)
            ->where('estado', '!=', 'migrado')
            ->orderBy('apellidos')
            ->orderBy('nombres')
            ->get();
    }
    return $this->personas;
}

El query se lanza una sola vez — array() y styles() comparten el resultado cacheado.

Color diferenciado por sexo

// Columna D (Sexo): azul para masculino, rosa para femenino
for ($r = $headerRow + 1; $r <= $lastDataRow; $r++) {
    $val = $sheet->getCell("D{$r}")->getValue();
    if ($val === 'Masculino') {
        $sheet->getStyle("D{$r}")->getFont()->getColor()->setRGB('1D4ED8'); // Blue
    } elseif ($val === 'Femenino') {
        $sheet->getStyle("D{$r}")->getFont()->getColor()->setRGB('BE185D'); // Pink
    }
}

Otros detalles de estilización:

  • Header ámbar (#B45309) — identidad visual distinta al resto de exports
  • Filas alternas — FEF3C7 (amber-50) en filas pares
  • freezePane — fija la cabecera al hacer scroll
  • Fila Total al final con conteo de personas

7 · Diferencias con EstiloExcel: WithStyles vs WithEvents

Los exports de Comunidades/Padrón no usan el trait EstiloExcel. Hay dos enfoques de estilización:

AspectoInformeCAI (L51)Comunidades/Padrón (L52)
InterfaceWithEventsWithStyles
TraitEstiloExcel (182 líneas)Ninguno
EstilosMétodos reutilizables del traitInline en cada sheet
AccesoAfterSheet event → $event->sheetMétodo styles(Worksheet $sheet)
VentajaDRY y consistente entre 7 hojasMás simple, sin dependencia
ColoresPaleta teal centralizada (18 constantes)Colores hardcoded por sheet
¿Cuándo usar cada enfoque? Si múltiples sheets comparten la misma identidad visual (como las 7 hojas del InformeCAI), un trait centraliza los estilos. Si los sheets son independientes con colores distintos, WithStyles inline es más directo.

Resumen de la lección

  • ComunidadesPoblacionExport (22 líneas) genera 3 hojas fijas: Resumen (blue-800), Detalle (green-600) y Consolidado (violet-600).
  • PadronComunidadesExport (27 líneas) genera N hojas dinámicas — una por comunidad — con lazy-loading de personas.
  • colLetter() convierte índices numéricos a letras de columna Excel, necesario cuando hay más de 26 columnas.
  • mb_substr($nombre, 0, 31) respeta el límite de 31 caracteres de pestañas Excel con soporte UTF-8.
  • El Consolidado usa 3 niveles de color (≥100/≥80/<80) vs los 4 del InformeCAI (≥95/≥80/≥50/<50).
  • Los sheets de Comunidades/Padrón usan WithStyles inline en lugar del trait EstiloExcel.
