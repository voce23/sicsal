1 FormularioCausasConsulta — panorama

Este formulario registra las 10 primeras causas de consulta externa del mes, desagregadas por 10 grupos de edad y sexo (M/F). Es el formulario más compacto de la familia:

AspectoDetalle
ModeloCausaConsultaExterna
Posiciones10 (filas del 1 al 10)
Grupos etáreos10 (<6m hasta ≥60)
Campos por celda2 (masculino, femenino)
Inputs numéricos10 × 10 × 2 = 200
Inputs texto10 (diagnóstico CIE-10 por posición)
Total inputs210
Tabs AlpineNinguno — tabla única
Estrategia guardadoDelete + Create (no updateOrCreate)
Contraste con los otros formularios: FormularioCausas es el único que usa delete-and-recreate en lugar de updateOrCreate. La razón: cada posición puede cambiar de diagnóstico entre ediciones, lo que invalidaría las claves de búsqueda.

2 cargarDatos() — inicialización manual

A diferencia de los otros formularios que usan keyBy(), este inicializa manualmente las 10 posiciones con valores por defecto y luego sobrescribe con datos existentes:

private function cargarDatos(int $centroId): void
{
    // Paso 1: inicializar 10 posiciones vacías
    $this->causas = [];
    for ($pos = 1; $pos <= 10; $pos++) {
        $grupos = [];
        foreach (array_keys(self::$grupos) as $ge) {
            $grupos[$ge] = ['m' => 0, 'f' => 0];
        }
        $this->causas[$pos] = [
            'diagnostico' => '',
            'grupos'      => $grupos,
        ];
    }

    // Paso 2: cargar datos existentes y sobrescribir
    $rows = CausaConsultaExterna::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)->where('anio', $this->anio)
        ->get();

    foreach ($rows as $row) {
        $pos = $row->posicion ?? 1;
        if ($pos < 1 || $pos > 10) continue;

        // Diagnóstico: mismo para todas las filas de esta posición
        if (!empty($row->diagnostico)) {
            $this->causas[$pos]['diagnostico'] = $row->diagnostico;
        }

        $ge = $row->grupo_etareo;
        if (isset($this->causas[$pos]['grupos'][$ge])) {
            $this->causas[$pos]['grupos'][$ge] = [
                'm' => $row->masculino,
                'f' => $row->femenino,
            ];
        }
    }
}
Estructura del array: $this->causas es un array indexado por posición (1-10), donde cada posición tiene un string diagnostico y un sub-array grupos con 10 entradas M/F. Total: 3 niveles de profundidad.

3 guardar() — delete-and-recreate

El método de guardado usa una estrategia diferente a los otros formularios:

public function guardar(): void
{
    if ($this->mesCerrado) return;
    $centroId = auth()->user()->centro_salud_id;

    // Paso 1: ELIMINAR todo lo del mes
    CausaConsultaExterna::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)->where('anio', $this->anio)
        ->delete();

    // Paso 2: RECREAR solo las que tienen diagnóstico
    foreach ($this->causas as $pos => $causa) {
        $diagnostico = trim($causa['diagnostico'] ?? '');
        if ($diagnostico === '') continue; // omitir vacías

        foreach (self::$grupos as $ge => $_label) {
            CausaConsultaExterna::create([
                'centro_salud_id' => $centroId,
                'mes'             => $this->mes,
                'anio'            => $this->anio,
                'posicion'        => $pos,
                'diagnostico'     => $diagnostico,
                'grupo_etareo'    => $ge,
                'masculino'       => (int) ($causa['grupos'][$ge]['m'] ?? 0),
                'femenino'        => (int) ($causa['grupos'][$ge]['f'] ?? 0),
            ]);
        }
    }
}
¿Por qué delete+create en vez de updateOrCreate?
  • El diagnóstico de cada posición puede cambiar completamente entre ediciones
  • No hay una clave estable tipo vacuna (que siempre es "BCG")
  • Si el usuario borra una posición, las filas huérfanas quedarían en la BD
  • Delete+create garantiza consistencia: lo que está en pantalla = lo que hay en BD

Nótese que las posiciones con diagnóstico vacío se omiten (continue), lo que permite al usuario dejar posiciones sin usar si el centro tiene menos de 10 causas.

Cada fila de datos calcula sus totales con collect()->sum():

@foreach ($causas as $pos => $causa)

        $totalF = collect($causa['grupos'])->sum('f');

@endphp

<!-- Totales por columna de grupo etáreo -->
@foreach ($grupos as $ge => ...)

5 wire:model.lazy y wire:loading

Este formulario introduce dos directivas Livewire que no vimos en los anteriores:

<!-- wire:model.lazy — solo sincroniza al perder foco (blur) -->
<input type="text"
       wire:model.lazy="causas.{{ $pos }}.diagnostico"
       placeholder="Diagnóstico...">

<!-- wire:loading — feedback visual durante guardado -->
<button wire:click="guardar"
        wire:loading.attr="disabled">
    <span wire:loading.remove wire:target="guardar">
        Guardar Causas de Consulta Externa
    </span>
    <span wire:loading wire:target="guardar">
        Guardando...
    </span>
</button>
¿Por qué wire:model.lazy? Con 210+ inputs, el binding en tiempo real (wire:model sin modificador) generaría una petición al servidor en cada tecla. Con .lazy, la sincronización ocurre solo al salir del campo (evento blur), reduciendo drásticamente las peticiones.
wire:loading dirigido: wire:target="guardar" limita la reacción visual al método específico. Sin wire:target, cualquier petición Livewire activaría el estado de carga.

6 FormularioMensualTrait — el trait compartido

Los 4 formularios mensuales (Servicios, Materna, PAI, Causas) comparten un trait que centraliza la lógica de mes/año, cierre y justificación cero:

namespace App\Filament\Pages\Concerns;

trait FormularioMensualTrait
{
    // Propiedades sincronizadas con la URL (#[Url])
    #[Url]
    public int $mes = 1;

    #[Url]
    public int $anio = 0;

    public bool $mesCerrado = false;

    // Propiedades del modal de justificación
    public string $justIndicador = '';
    public string $justMotivo = '';
    public string $justDetalle = '';

    private static array $nombresMeses = [
        1 => 'Enero', 2 => 'Febrero', ...,
        12 => 'Diciembre',
    ];
}
#[Url] es un atributo de Livewire que sincroniza la propiedad con el query string. Así la URL ?mes=3&anio=2025 determina qué mes se edita, y compartir el enlace lleva directamente al mes correcto.

7 mountFormulario() y mesCerrado

El método que todos los formularios llaman en su mount():

public function mountFormulario(): void
{
    // Mes y año vienen del query string, o el actual por defecto
    $this->mes  = (int) request()->query('mes', date('n'));
    $this->anio = (int) request()->query('anio', date('Y'));

    $centroId = auth()->user()->centro_salud_id;

    // Verificar si el mes está cerrado
    $this->mesCerrado = MesCerrado::where('centro_salud_id', $centroId)
        ->where('mes', $this->mes)
        ->where('anio', $this->anio)
        ->exists();
}

// Título dinámico para la página
public function getTitle(): string
{
    $seccion = static::SECCION ?? 'Prestaciones';
    return $seccion . ' — '
         . (self::$nombresMeses[$this->mes] ?? '')
         . ' ' . $this->anio;
}

Cada formulario define su propia constante SECCION que se usa en el título. Por ejemplo, FormularioPai define const SECCION = 'PAI — Salud Infantil', resultando en: "PAI — Salud Infantil — Marzo 2025".

8 cerrarMes() y reabrirMes()

El cierre de mes es una operación privilegiada que bloquea la edición para los registradores:

public function cerrarMes(): void
{
    $user = auth()->user();
    if (! $user->hasAnyRole(['superadmin', 'admin'])) {
        return; // solo admin puede cerrar
    }

    MesCerrado::updateOrCreate(
        ['centro_salud_id' => $user->centro_salud_id,
         'mes' => $this->mes,
         'anio' => $this->anio],
        ['cerrado_por' => $user->id,
         'fecha_cierre' => now()]
    );
    $this->mesCerrado = true;
}

public function reabrirMes(): void
{
    $user = auth()->user();
    if (! $user->hasAnyRole(['superadmin', 'admin'])) {
        return;
    }

    $mc = MesCerrado::where(...)->first();
    if ($mc) {
        $mc->update(['reabierto_por' => $user->id,
                      'fecha_reapertura' => now()]);
        $mc->delete(); // soft delete
    }
    $this->mesCerrado = false;
}
Auditoría de reapertura: Antes de borrar el registro de cierre, se actualiza con reabierto_por y fecha_reapertura. Si el modelo usa SoftDeletes, el registro queda en la BD para auditoría con deleted_at preservando quién lo reabrió.

9 Header actions dinámicas

El trait genera los botones del encabezado de página según el estado y el rol:

protected function getHeaderActions(): array
{
    $actions = [];
    $user = auth()->user();

    if ($user->hasAnyRole(['superadmin', 'admin'])) {
        if ($this->mesCerrado) {
            // Botón: Reabrir mes (warning, candado abierto)
            $actions[] = Action::make('reabrirMes')
                ->label('Reabrir mes')
                ->icon(Heroicon::OutlinedLockOpen)
                ->color('warning')
                ->requiresConfirmation()
                ->modalHeading('¿Reabrir este mes?')
                ->action(fn () => $this->reabrirMes());
        } else {
            // Botón: Cerrar mes (success, candado cerrado)
            $actions[] = Action::make('cerrarMes')
                ->label('Cerrar mes')
                ->icon(Heroicon::OutlinedLockClosed)
                ->color('success')
                ->requiresConfirmation();
        }
    }

    // Siempre visible: Volver al selector
    $actions[] = Action::make('volver')
        ->label('Volver al selector')
        ->url(Prestaciones::getUrl());

    return $actions;
}
requiresConfirmation() es un método de Filament que automáticamente muestra un modal de confirmación con "¿Estás seguro?" antes de ejecutar la acción. modalHeading() personaliza el título del modal.

10 Justificación cero — modal y evento

El partial justificacion-cero-modal.blade.php es incluido con @include en los 4 formularios. Combina Alpine.js para la apertura y Livewire para la persistencia:

<div
    x-data="{ open: false, indicador: '' }"
    x-on:abrir-justificacion-cero.window="
        open = true;
        indicador = $event.detail.indicador;
        $wire.abrirJustificacion($event.detail.indicador)
    "
    x-on:close-modal.window="
        if ($event.detail.id === 'justificacion-cero') open = false
    ">

    <template x-if="open">
        <div class="fixed inset-0 z-50 flex items-center justify-center bg-black/50">
            <div class="max-w-md rounded-xl bg-white p-6 shadow-xl">
                <!-- Radio buttons con opciones predefinidas -->
                @foreach ($motivosOpciones as $value => $label)
                    <label>
                        <input type="radio"
                               wire:model="justMotivo"
                               value="{{ $value }}">
                        {{ $label }}
                    </label>
                @endforeach

                <!-- Campo condicional para "Otro motivo" -->
                <div x-show="$wire.justMotivo === 'otro'">
                    <input wire:model="justDetalle"
                           maxlength="300">
                </div>

                <button wire:click="guardarJustificacion">
                    Confirmar y guardar
                </button>
            </div>
        </div>
    </template>
</div>

El flujo completo de justificación cero:

PasoQuiénQué hace
1PHP (guardarX)Detecta que la suma = 0, ejecuta $this->dispatch('abrir-justificacion-cero', indicador: '...')
2Alpine (modal)Escucha el evento x-on:abrir-justificacion-cero.window, abre el modal
3Alpine → Livewire$wire.abrirJustificacion() sincroniza el indicador con el PHP
4UsuarioSelecciona un motivo (radio) y opcionalmente detalla
5Livewire (trait)guardarJustificacion() persiste en JustificacionCero con updateOrCreate
6Livewire → Alpinedispatch('close-modal', id: 'justificacion-cero') cierra el modal
5 motivos predefinidos: no_hay_poblacion_activa_padron, poblacion_migrada_temporal, atendida_otro_centro, no_se_presento_razon_desconocida, otro. Cuando se elige "otro", x-show="$wire.justMotivo === 'otro'" muestra el campo de detalle libre (maxlength="300").

11 Resumen

Lo que aprendimos en esta lección

  • Delete-and-recreate: Cuando las claves no son estables (diagnóstico libre), es más seguro borrar y recrear que intentar updateOrCreate.
  • Inicialización manual: Un for-loop genera el esqueleto de 10 posiciones × 10 grupos con valores por defecto, luego se sobrescriben con datos de la BD.
  • collect()->sum() en Blade: Totales por fila y por columna calculados directamente en la vista con closures de colección.
  • wire:model.lazy: Reduce peticiones al servidor sincronizando solo al perder foco — crítico con 210+ inputs.
  • wire:loading + wire:target: Feedback visual dirigido durante operaciones asíncronas.
  • FormularioMensualTrait: Centraliza mes/año (#[Url]), mesCerrado, cerrarMes/reabrirMes (solo admin), justificación cero y header actions.
  • Flujo de justificación: PHP dispatch → Alpine escucha evento → abre modal → usuario elige motivo → Livewire persiste → cierra modal.
  • Header actions condicionales: Filament Actions con requiresConfirmation() que mutan según estado y rol del usuario.
