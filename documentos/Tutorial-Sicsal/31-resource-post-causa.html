1 PostResource — blog oculto del sidebar

app/Filament/Resources/Posts/PostResource.php
class PostResource extends Resource
{
    protected static ?string $model = Post::class;
    protected static string|BackedEnum|null $navigationIcon
        = Heroicon::OutlinedNewspaper;
    protected static string|\UnitEnum|null $navigationGroup
        = 'Blog';

    protected static bool $shouldRegisterNavigation = false;

    protected static ?string $recordTitleAttribute = 'titulo';
    // Sin canAccess, sin getEloquentQuery — acceso global
}
shouldRegisterNavigation = false

El Resource existe y funciona, pero no aparece en el menú lateral. Se accede escribiendo directamente la URL /admin/posts. Útil para módulos secundarios que no necesitan visibilidad permanente.

Auto-slug desde el título

PostForm.php — título y auto-slug
TextInput::make('titulo')
    ->required()->maxLength(255)
    ->live(onBlur: true)
    ->afterStateUpdated(function ($state, $set, $get) {
        if (!$get('slug')) {
            $set('slug', Post::generarSlug($state));
        }
    }),
live(onBlur: true) — eficiencia

live(onBlur: true) solo envía el request cuando el usuario sale del campo (no en cada tecla). Más eficiente que live() puro. El slug solo se genera si está vacío — si el usuario lo editó manualmente, no se sobreescribe.

Editor con 3 Tabs: Visual, HTML, Vista Previa

PostForm.php — Tabs del editor
Tabs::make('editor_tabs')
    ->tabs([
        Tab::make('✏️  Visual')
            ->schema([
                RichEditor::make('contenido')
                    ->toolbarButtons([
                        'bold', 'italic', 'underline', 'strike',
                        'h2', 'h3', 'link', 'blockquote',
                        'codeBlock', 'bulletList', 'orderedList',
                        'attachFiles',
                    ])
                    ->fileAttachmentsDirectory('blog/adjuntos'),
            ]),
        Tab::make('◇  HTML')
            ->schema([
                Textarea::make('contenido')
                    ->rows(20)
                    ->extraAttributes([
                        'class' => 'font-mono text-xs',
                        'spellcheck' => 'false',
                    ]),
            ]),
        Tab::make('👁  Vista Previa')
            ->schema([
                Placeholder::make('vista_previa')
                    ->content(fn ($get) => new HtmlString(
                        '<div class="prose">'
                        . $get('contenido')
                        . '</div>'
                    )),
            ]),
    ]),
Mismo campo en dos tabs — truco de edición dual

El campo contenido aparece dos veces: como RichEditor (visual) y como Textarea (HTML). Ambas editan el mismo campo del modelo. Al cambiar entre tabs, el contenido se sincroniza. La tercera tab usa Placeholder con HtmlString para renderizar HTML en vivo.

Componentes nuevos
  • FileUpload: sube imágenes con crop automático 16:9, resize a 1200×675, almacena en storage/app/public/blog/portadas

ImageColumn — miniatura en la tabla

ImageColumn::make('imagen_portada')->disk('public')->width(80)->height(45) muestra la imagen de portada como miniatura de 80×45 px directamente en la tabla. Primera vez que usamos este tipo de columna.

4 CausaConsultaExternaResource — todo en uno

A diferencia de los otros Resources, este tiene form() y table() directamente en el Resource, sin archivos separados Schemas/ ni Tables/.

CausaConsultaExternaResource.php — encabezado
class CausaConsultaExternaResource extends Resource
{
    protected static ?string $model = CausaConsultaExterna::class;
    protected static string|BackedEnum|null $navigationIcon
        = Heroicon::OutlinedDocumentMagnifyingGlass;
    protected static string|\UnitEnum|null $navigationGroup
        = 'Prestaciones Mensuales';
    protected static bool $shouldRegisterNavigation = false;

    // Filtrado estándar por centro
    public static function getEloquentQuery(): Builder { ... }

    // form() y table() definidos aquí mismo
    public static function form(Schema $schema): Schema { ... }
    public static function table(Table $table): Table { ... }
}
¿Cuándo separar Form/Table y cuándo no?

En SICSAL, los Resources complejos (Persona, Defuncion, User) usan archivos separados. Los más simples o especializados como CausaConsultaExterna ponen todo en el Resource. Ambos enfoques son válidos — es cuestión de tamaño y mantenimiento.

5 Formulario inline con propiedades estáticas del modelo

CausaConsultaExternaResource.php — form()
public static function form(Schema $schema): Schema
{
    $meses = CausaConsultaExterna::$meses;

    return $schema->components([
        Section::make('Período y Centro')
            ->schema([
                Select::make('centro_salud_id')
                    ->options(fn () =>
                        CentroSalud::orderBy('nombre')->pluck('nombre', 'id'))
                    ->disabled(fn () => !auth()->user()->hasRole('superadmin'))
                    ->dehydrated(true),
                Select::make('mes')
                    ->options($meses)
                    ->default((int) date('n'))->required(),
                TextInput::make('anio')
                    ->numeric()->default((int) date('Y'))->required(),
            ])->columns(3),

        Section::make('Diagnóstico y Grupo Etáreo')
            ->schema([
                TextInput::make('diagnostico')
                    ->required()->maxLength(255)->columnSpanFull(),
                Select::make('grupo_etareo')
                    ->options(CausaConsultaExterna::$grupos)
                    ->required(),
            ])->columns(2),

        Section::make('Cantidad por Sexo')
            ->schema([
                TextInput::make('masculino')
                    ->numeric()->minValue(0)->default(0)->required(),
                TextInput::make('femenino')
                    ->numeric()->minValue(0)->default(0)->required(),
            ])->columns(2),
    ]);
}
Propiedades estáticas del modelo → opciones del formulario

CausaConsultaExterna::$meses y ::$grupos son arrays definidos en el modelo. Centralizar las opciones en el modelo (en lugar de repetirlas en cada formulario/tabla) es un patrón limpio — un cambio en el modelo se propaga a todo.

dehydrated(true) con disabled

Un campo disabled por defecto NO envía su valor al servidor (dehydrated = false). Pero aquí necesitamos que centro_salud_id se guarde, así que ->dehydrated(true) fuerza su inclusión. Alternativa: usar mutateFormDataBeforeCreate() como en PersonaResource.

6 Tabla con columna calculada (total)

CausaConsultaExternaResource.php — table()
->columns([
    TextColumn::make('centroSalud.nombre')->toggleable(),
    TextColumn::make('mes')
        ->formatStateUsing(fn ($state) =>
            CausaConsultaExterna::$meses[$state] ?? $state),
    TextColumn::make('anio')->sortable(),
    TextColumn::make('diagnostico')->searchable()->limit(50),
    TextColumn::make('grupo_etareo')
        ->formatStateUsing(fn ($state) =>
            CausaConsultaExterna::$grupos[$state]),
    TextColumn::make('masculino')->numeric()->alignCenter(),
    TextColumn::make('femenino')->numeric()->alignCenter(),
    // Columna calculada — no existe en BD
    TextColumn::make('total')
        ->state(fn ($record) =>
            $record->masculino + $record->femenino)
        ->numeric()->alignCenter()
        ->weight('bold'),
])
->defaultSort('anio', 'desc')
->filters([
    SelectFilter::make('mes')
        ->options(CausaConsultaExterna::$meses),
    SelectFilter::make('anio')
        ->options(
            collect(range((int) date('Y'), 2020))
                ->mapWithKeys(fn ($y) => [$y => $y])
                ->toArray()
        ),
])
state() vs getStateUsing()

->state(fn ($record) => ...) y ->getStateUsing(fn ($record) => ...) son equivalentes — ambos calculan el valor de la columna desde el registro. La columna "Total" suma masculino + femenino en tiempo real. ->weight('bold') la muestra en negrita.

7 Resumen

Lo que aprendimos

  • shouldRegisterNavigation = false — Resource oculto del menú
  • Tabs::make() — tabs de contenido dentro del formulario
  • RichEditor — editor WYSIWYG con toolbar personalizado y adjuntos
  • FileUpload — subida de imágenes con crop, resize y configuración de disco
  • TagsInput — input de etiquetas como chips
  • live(onBlur: true) — reactivo solo al salir del campo
  • auto-slug — generado desde el título con $set + $get
  • ImageColumn — miniatura de imagen en la tabla
  • Modelo::$propiedades estáticas — centralizar opciones para Form y Table
  • dehydrated(true) — forzar envío de campos disabled
  • state(fn) — columna calculada (total = masculino + femenino)
  • collect(range())->mapWithKeys() — generar opciones de filtro con Collection
