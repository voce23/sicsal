1 ¿Qué es Vite?

Vite es un bundler — una herramienta que toma tus archivos CSS y JavaScript "fuente" y los convierte en archivos optimizados para producción.

Sin Vite Con Vite
Escribes CSS manualmente en archivos separados Escribes Tailwind CSS y Vite lo compila
Import de Chart.js no funciona en el navegador Vite empaqueta Chart.js con tu código
Cambias un archivo y refrescas manualmente Vite detecta cambios y recarga automáticamente (HMR)
ℹ️
HMR = Hot Module Replacement
Cuando npm run dev está corriendo, cada vez que guardas un archivo CSS, JS o Blade, el navegador se actualiza sin recargar la página. Es la función más útil de Vite para desarrollo.

2 Archivo vite.config.js

Este archivo en la raíz del proyecto indica a Vite qué archivos compilar y qué plugins usar. Crea o reemplaza el contenido de vite.config.js:

vite.config.js javascript 
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
    plugins: [
        laravel({
            input: [
                'resources/css/app.css',
                'resources/js/app.js',
                'resources/css/filament/admin/theme.css',
            ],
            refresh: true,
        }),
        tailwindcss(),
    ],
    server: {
        cors: true,
        watch: {
            ignored: ['**/storage/framework/views/**'],
        },
    },
});

Explicación línea por línea

Propiedad Qué hace
input Los 3 archivos fuente que Vite compilará: CSS app, JS app, y tema Filament
refresh: true Recarga el navegador cuando cambias archivos Blade
tailwindcss() Plugin que integra Tailwind CSS 4 con Vite
server.cors Permite que el servidor de Vite sirva assets cross-origin
watch.ignored Ignora la caché de vistas compiladas para no recargar innecesariamente

3 ¿Qué es Tailwind CSS?

Tailwind CSS es un framework CSS "utility-first". En vez de escribir clases como .btn-primary, usas clases pequeñas directamente:

Comparación html 
<!-- CSS tradicional -->
<button class="btn btn-primary btn-lg">Guardar</button>

<!-- Tailwind CSS -->
<button class="bg-blue-600 text-white px-6 py-3 rounded-lg hover:bg-blue-700">
    Guardar
</button>

SICSAL usa Tailwind CSS 4.2, que no necesita archivo tailwind.config.js — la configuración va dentro de app.css con directivas @theme.

💡
Tailwind 4 vs Tailwind 3
En Tailwind 3 necesitabas tailwind.config.js. En Tailwind 4, la configuración se define con @theme { } dentro del CSS. Es más simple y rápido.

4 Archivo resources/css/app.css

Este es el archivo CSS principal de la aplicación. Crea o reemplaza resources/css/app.css:

resources/css/app.css css 
@import 'tailwindcss';
@import '../../vendor/livewire/flux/dist/flux.css';

@source '../views';
@source '../../vendor/laravel/framework/src/Illuminate/Pagination/resources/views/*.blade.php';
@source '../../vendor/livewire/flux-pro/stubs/**/*.blade.php';
@source '../../vendor/livewire/flux/stubs/**/*.blade.php';

@custom-variant dark (&:where(.dark, .dark *));

@theme {
    --font-sans: 'Instrument Sans', ui-sans-serif, system-ui, sans-serif,
        'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol',
        'Noto Color Emoji';

    --color-zinc-50: #fafafa;
    --color-zinc-100: #f5f5f5;
    --color-zinc-200: #e5e5e5;
    --color-zinc-300: #d4d4d4;
    --color-zinc-400: #a3a3a3;
    --color-zinc-500: #737373;
    --color-zinc-600: #525252;
    --color-zinc-700: #404040;
    --color-zinc-800: #262626;
    --color-zinc-900: #171717;
    --color-zinc-950: #0a0a0a;

    --color-accent: var(--color-neutral-800);
    --color-accent-content: var(--color-neutral-800);
    --color-accent-foreground: var(--color-white);
}

Explicación

Línea Qué hace
@import 'tailwindcss' Importa todas las utilidades de Tailwind CSS 4
@import '...flux.css' Importa estilos de Flux (componentes UI de Livewire)
@source '../views' Escanea archivos Blade para detectar clases Tailwind usadas
@custom-variant dark Define cómo funciona el modo oscuro
@theme { } Configura fuentes y colores personalizados. Reemplaza al tailwind.config.js de v3

5 Archivo resources/js/app.js

El archivo JavaScript principal. Lo usamos para registrar Chart.js globalmente (para pirámides poblacionales y gráficos de coberturas):

resources/js/app.js javascript 
import { Chart, registerables } from 'chart.js';
Chart.register(...registerables);
window.Chart = Chart;

Solo 3 líneas. Esto:

  1. Importa Chart.js y todos sus componentes registrables
  2. Registra todos los tipos de gráfico (bar, line, pie, etc.)
  3. Expone Chart en window para que los componentes Livewire puedan usarlo
💡
¿Por qué window.Chart?
Los componentes Livewire ejecutan JavaScript en el navegador. Necesitan acceso a Chart globalmente. Sin window.Chart = Chart, Livewire no podría crear gráficos porque import solo funciona dentro de módulos Vite.

6 Tema personalizado para Filament

Filament tiene su propio sistema de temas. Creamos un archivo CSS que personaliza el sidebar del panel admin con colores médicos:

1
Crea la carpeta resources/css/filament/admin/
2
Crea el archivo theme.css dentro
Terminal bash 
$ mkdir -p resources/css/filament/admin
resources/css/filament/admin/theme.css css 
@import '../../../../vendor/filament/filament/resources/css/theme.css';

@source '../../../../app/Filament/**/*';
@source '../../../../resources/views/filament/**/*';

/* =========================================================
   SICSAL — Sidebar personalizado para personal de salud
   ========================================================= */

/* --- Sidebar fondo suave --- */
.fi-sidebar {
    @apply bg-gradient-to-b from-green-50 to-white
           dark:from-gray-900 dark:to-gray-950;
}

/* --- Grupos de navegación: etiquetas con color --- */
.fi-sidebar-group-btn {
    @apply rounded-lg px-3 py-1.5 mt-1 transition-all duration-200;
}

.fi-sidebar-group-label {
    @apply text-[13px] font-bold tracking-wide uppercase;
}
ℹ️
¿Qué es @apply?
@apply es una directiva de Tailwind que permite usar clases utility dentro de CSS. Así personalizamos componentes de Filament sin escribir CSS puro.

Este tema le da al panel admin un estilo médico sutil: sidebar con degradado verde claro y etiquetas de grupos con colores diferenciados. Lo completaremos en la Lección 24 cuando configuremos el panel.

7 Compilar assets

Hay dos modos de compilación:

Modo desarrollo (con HMR)

Terminal bash 
$ npm run dev
  VITE v8.0.0  ready in 285 ms
  ➜  Local:   http://localhost:5173/
  ➜  press h + enter to show help

Esto inicia el servidor de Vite en el puerto 5173. Cada vez que guardas un archivo CSS, JS o Blade, el navegador se actualiza automáticamente.

⚠️
Importante
npm run dev debe estar corriendo mientras desarrollas. Si lo cierras, los estilos de Tailwind no se cargarán porque Laravel espera que Vite los sirva.

Modo producción

Terminal bash 
$ npm run build
vite v8.0.0 building for production...
✓ 50 modules transformed.
public/build/manifest.json          0.50 kB
public/build/assets/app-AbCdEf.css   28.32 kB │ gzip: 6.12 kB
public/build/assets/app-GhIjKl.js   198.45 kB │ gzip: 62.30 kB
✓ built in 3.20s

Esto genera archivos minificados en public/build/ con nombres hasheados para cache-busting.

8 Usar assets en Blade

Para incluir los CSS y JS compilados en tus vistas Blade, usa la directiva @vite:

resources/views/layout.blade.php (ejemplo) blade 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>SICSAL</title>

    {{-- Esto incluye app.css y app.js compilados por Vite --}}
    @vite(['resources/css/app.css', 'resources/js/app.js'])
</head>
<body>
    {{ $slot }}
</body>
</html>

La directiva @vite genera automáticamente las etiquetas <link> y <script> correctas. En desarrollo apunta al servidor de Vite (puerto 5173); en producción apunta a los archivos compilados en public/build/.
