1 DefuncionResource — misma estructura

app/Filament/Resources/Defuncions/DefuncionResource.php
class DefuncionResource extends Resource
{
    protected static ?string $model = Defuncion::class;
    protected static string|BackedEnum|null $navigationIcon
        = Heroicon::OutlinedDocumentText;
    protected static string|\UnitEnum|null $navigationGroup
        = 'Padrón Poblacional';
    protected static ?string $navigationLabel = 'Defunciones';
    protected static ?string $modelLabel      = 'Defunción';
    protected static ?string $pluralModelLabel = 'Defunciones';
    protected static ?int    $navigationSort   = 2;

    // Mismo patrón: filtrado por centro
    public static function getEloquentQuery(): Builder
    {
        $query = parent::getEloquentQuery();
        $user  = auth()->user();
        if ($user && !$user->hasRole('superadmin')) {
            $query->where('centro_salud_id', $user->centro_salud_id);
        }
        return $query;
    }

    // Delega a Form y Table separados
    public static function form(Schema $schema): Schema
    { return DefuncionForm::configure($schema); }

    public static function table(Table $table): Table
    { return DefuncionsTable::configure($table); }
}

Mismo grupo de navegación (Padrón Poblacional) con sort 2. Defunciones aparece justo debajo de Personas en el sidebar.

2 Formulario: vinculación al padrón (auto-fill)

La gran novedad de este Resource: un Select de búsqueda que al elegir una persona del padrón, auto-llena 5 campos del formulario.

Usuario escribe nombre en "Buscar en el padrón"
↓
getSearchResultsUsing() → busca por nombres/apellidos/ci
↓
afterStateUpdated() → auto-llena nombres, apellidos, fecha_nacimiento, sexo, comunidad_id
↓
Usuario solo complementa: fecha_defuncion, lugar, causa
DefuncionForm.php — Sección vinculación
Section::make('Vinculación al padrón')
    ->schema([
        Select::make('persona_id')
            ->label('Buscar en el padrón (opcional)')
            ->searchable()
            ->getSearchResultsUsing(fn (string $search) =>
                Persona::query()
                    ->where('centro_salud_id', auth()->user()?->centro_salud_id)
                    ->where(function ($q) use ($search) {
                        $q->where('nombres', 'like', "%{$search}%")
                          ->orWhere('apellidos', 'like', "%{$search}%")
                          ->orWhere('ci', 'like', "%{$search}%");
                    })
                    ->limit(20)
                    ->get()
                    ->mapWithKeys(fn (Persona $p) =>
                        [$p->id => "{$p->nombres} {$p->apellidos} ({$p->ci})"]
                    )->toArray()
            )
            ->getOptionLabelUsing(fn ($value) =>
                Persona::find($value)?->nombre_completo ?? $value
            )
            ->live()
            ->afterStateUpdated(function ($state, Set $set) {
                if (!$state) return;
                $persona = Persona::find($state);
                if (!$persona) return;
                $set('nombres', $persona->nombres);
                $set('apellidos', $persona->apellidos);
                $set('fecha_nacimiento', $persona->fecha_nacimiento?->format('Y-m-d'));
                $set('sexo', $persona->sexo);
                $set('comunidad_id', $persona->comunidad_id);
            }),
    ]),
getSearchResultsUsing() — búsqueda personalizada

En lugar de relationship(), este Select usa getSearchResultsUsing() para implementar una búsqueda totalmente personalizada:

  • Filtra por centro_salud_id del usuario (seguridad)
  • Busca en 3 campos: nombres, apellidos, CI
  • Limita a 20 resultados para rendimiento
  • Formatea la opción: "Juan Mamani (1234567)"

getOptionLabelUsing() se usa para mostrar el nombre cuando se edita un registro existente (donde el valor ya está seleccionado).

Set $set — injection de Filament

El parámetro Set $set es una clase de Filament (no un callable como en PersonaForm). Ambas formas funcionan igual: $set('campo', valor). En este caso usa la clase tipada Filament\Forms\Set.

3 Formulario: datos del fallecido + defunción

Section::make('Datos del fallecido')
    ->schema([
        TextInput::make('nombres')->required()->maxLength(100),

        Textarea::make('causa_defuncion')
            ->rows(3)->columnSpanFull(),
    ])->columns(2),
Diseño del formulario

Los campos de la sección "Datos del fallecido" se pueden llenar de dos formas:

  • Automática: buscando la persona en el padrón → se auto-llenan los 5 campos
  • Manual: escribiendo directamente (persona no registrada en el padrón)

Esto permite registrar defunciones de personas que no estaban en el sistema.

4 Tabla: 8 columnas, grupo etáreo, defaultSort

app/Filament/Resources/Defuncions/Tables/DefuncionsTable.php
->columns([
    TextColumn::make('nombres')->searchable()->sortable(),
    TextColumn::make('apellidos')->searchable()->sortable(),
    TextColumn::make('comunidad.nombre')->searchable(),
    TextColumn::make('fecha_defuncion')
        ->date('d/m/Y')->sortable(),
    TextColumn::make('sexo')->badge()
        ->formatStateUsing(fn (string $state) =>

        }),
    // Accessor del modelo — grupo etáreo calculado
    TextColumn::make('grupo_etareo_defuncion')
        ->badge()->label('Grupo etáreo'),
    TextColumn::make('registradoPor.name')
        ->toggleable(isToggledHiddenByDefault: true),
])
->filters([
    SelectFilter::make('lugar')
        ->options([...]),
    SelectFilter::make('grupo_etareo_defuncion')
        ->options([
            'neonatal'         => 'Neonatal',
            'infantil_menor_1' => 'Infantil <1 año',
            'menor_5'          => 'Menor de 5 años',
            '5_a_59'           => '5 a 59 años',
            'adulto_mayor'     => 'Adulto mayor (≥60)',
        ]),

->defaultSort('fecha_defuncion', 'desc')
defaultSort() — orden por defecto

->defaultSort('fecha_defuncion', 'desc') ordena la tabla mostrando las defunciones más recientes primero. Es la primera tabla de SICSAL que usa un sort por defecto diferente al ID.

grupo_etareo_defuncion — accessor del modelo

Este campo no existe en la base de datos — es un accessor calculado del modelo Defuncion que determina el grupo etáreo (neonatal, infantil, etc.) a partir de la edad al fallecer. Los SelectFilter pueden usarlo porque Filament lo resuelve vía el modelo.

5 Pages estándar

A diferencia de PersonaResource, CreateDefuncion no tiene mutateFormDataBeforeCreate. Las 3 Pages siguen el patrón básico:

  • CreateDefuncion — extends CreateRecord (vacía)
  • EditDefuncion — extends EditRecord + DeleteAction
  • ListDefuncions — extends ListRecords + CreateAction

6 Resumen

Lo que aprendimos

  • getSearchResultsUsing() — búsqueda personalizada con query custom
  • getOptionLabelUsing() — mostrar label cuando se edita un registro existente
  • afterStateUpdated + $set() — auto-llenar 5 campos desde otra entidad
  • defaultSort() — orden por defecto de la tabla
  • grupo_etareo_defuncion — accessor calculado como columna de tabla
  • Formulario dual — auto-fill desde padrón o captura manual
