1 — ¿Qué es el informe CAI?

El CAI (Comité de Análisis de Información) es la reunión periódica que cada centro de salud realiza para evaluar el avance de sus programas de salud. SICSAL genera este informe en tres períodos:

PeríodoMeses incluidosNombre
cai1Enero – AbrilCAI 1
cai2Enero – AgostoCAI 2
gestionEnero – DiciembreCierre de Gestión

Los tres períodos son acumulativos (siempre empiezan desde enero). La cobertura se mide contra la meta INE anual completa — no se prorratea.

Regla clave
La meta INE es siempre el dato anual. Si en CAI 1 tienes 30 vacunados de una meta de 100, tu cobertura es 30% — no 90% (30/33).

3 — Períodos y factor de meta

Tres métodos utilitarios controlan qué meses y cómo se muestra el período:

app/Helpers/CaiHelper.php — Métodos de período
class CaiHelper
{
    private static array $nombresMeses = [
        1 => 'Enero', 2 => 'Febrero', 3 => 'Marzo', 4 => 'Abril',
        5 => 'Mayo', 6 => 'Junio', 7 => 'Julio', 8 => 'Agosto',
        9 => 'Septiembre', 10 => 'Octubre', 11 => 'Noviembre', 12 => 'Diciembre',
    ];

    public static function getMeses(string $periodo): array
    {
        return match ($periodo) {
            'cai1'    => range(1, 4),
            'cai2'    => range(1, 8),
            'gestion' => range(1, 12),
            default   => range(1, 12),
        };
    }

    public static function getFactorMeta(string $periodo): float
    {
        return match ($periodo) {
            'cai1'    => 4 / 12,  // 0.333
            'cai2'    => 8 / 12,  // 0.667
            'gestion' => 1.0,
            default   => 1.0,
        };
    }

    public static function getNombrePeriodo(string $periodo, int $anio): string
    {
        return match ($periodo) {
            'cai1'    => "CAI 1 (Ene–Abr {$anio})",
            'cai2'    => "CAI 2 (Ene–Ago {$anio})",
            'gestion' => "Cierre de Gestión {$anio}",
            default   => "Gestión {$anio}",
        };
    }
getFactorMeta()
Existe pero no se usa en el cálculo de cobertura actual — la cobertura se mide atendidos/meta_anual directamente. Se reserva para informes que necesiten proyectar o prorratear.

public static function getEncabezado(int $centroId, string $periodo, int $anio): array

5 — Sección 1: Contexto de Migración

Calcula indicadores de migración del padrón activo:

getMigracion() — indicadores migratorios
public static function getMigracion(int $centroId): array
{
    $base = Persona::where('centro_salud_id', $centroId)
        ->where('activo', true);

    $totalPadron = (clone $base)->count();
    $migrantes   = (clone $base)->where('estado', 'migrado')->count();
    $residentes  = $totalPadron - $migrantes;

    $mefActivas = (clone $base)
        ->where('sexo', 'F')
        ->where('estado', '!=', 'migrado')
        ->whereRaw('TIMESTAMPDIFF(YEAR, fecha_nacimiento, CURDATE()) BETWEEN 15 AND 49')
        ->count();

    $mefMigradas = (clone $base)
        ->where('sexo', 'F')
        ->where('estado', 'migrado')
        ->whereRaw('TIMESTAMPDIFF(YEAR, fecha_nacimiento, CURDATE()) BETWEEN 15 AND 49')
        ->count();

    return [
        'total_padron'     => $totalPadron,
        'residentes'       => $residentes,
        'migrantes'        => $migrantes,
        'pct_migrantes'    => $totalPadron > 0
            ? round($migrantes / $totalPadron * 100, 1) : 0,
        'mef_activas'      => $mefActivas,
        'mef_migradas'     => $mefMigradas,
        'pct_mef_migradas' => ($mefActivas + $mefMigradas) > 0
            ? round($mefMigradas / ($mefActivas + $mefMigradas) * 100, 1) : 0,
        'hombres_migrados' => $hombresMigrados,
    ];
}
Patrón clone $base
Se usa (clone $base) repetidamente porque cada ->where() muta el query builder. Sin clone, cada filtro se acumularía sobre la misma instancia y el segundo conteo daría 0. Este patrón se repite en todo CaiHelper.

MEF = Mujer en Edad Fértil (15-49 años). El informe distingue MEF activas (residentes) de MEF migradas porque la meta de anticoncepción solo aplica a residentes.

6.2 — Pirámide poblacional

Compara la proyección INE con el padrón real por grupo etáreo y sexo:

getCenso() — pirámide (10 grupos etáreos)
$gruposPiramide = [
    ['label' => '< 1 año',    'min' => 0,  'max' => 0,   'ine' => 'menor_1'],
    ['label' => '1 año',      'min' => 1,  'max' => 1,   'ine' => '1_anio'],
    ['label' => '2-4 años',   'min' => 2,  'max' => 4,   'ine' => '2_4'],
    // ... 5_9, 10_14, 15_19, 20_39, 40_49, 50_59 ...
    ['label' => '≥ 60 años', 'min' => 60, 'max' => 120, 'ine' => 'mayor_60'],
];

foreach ($gruposPiramide as $g) {
    // Meta INE del Excel importado
    $ineM = $metasIne->where('grupo_etareo', $g['ine'])
        ->where('sexo', 'M')->sum('cantidad');
    $ineF = $metasIne->where('grupo_etareo', $g['ine'])
        ->where('sexo', 'F')->sum('cantidad');
    $ineAmbos = $metasIne->where('grupo_etareo', $g['ine'])
        ->where('sexo', 'ambos')->sum('cantidad');

    // Fallback: si solo hay dato "ambos", dividir 50/50
    if ($ineM === 0 && $ineF === 0 && $ineAmbos > 0) {
        $ineM = intdiv($ineAmbos, 2);
        $ineF = $ineAmbos - $ineM;
    }

    // Padrón real: activos no migrados ni fallecidos
    $base = Persona::where('centro_salud_id', $centroId)
        ->where('activo', true)
        ->whereNotIn('estado', ['migrado', 'fallecido']);

    $realM = (clone $base)->where('sexo', 'M')
        ->whereRaw('TIMESTAMPDIFF(YEAR, fecha_nacimiento, CURDATE()) BETWEEN ? AND ?',
            [$g['min'], $g['max']])
        ->count();

    $piramide[] = [
        'label'  => $g['label'],
        'ine_m'  => $ineM,  'ine_f'  => $ineF,
        'real_m' => $realM, 'real_f' => $realF,
    ];
}
Fallback ambos → M/F
Cuando el Excel INE solo trae un valor para «ambos sexos» sin desglose, se divide con intdiv() — el residuo va a femenino para no perder una unidad.

7 — Sección 3: Prestaciones Acumuladas

getPrestaciones() es el método más extenso — consulta 9 tablas prest_* y devuelve los totales acumulados por período:

getPrestaciones() — 9 consultas paralelas
public static function getPrestaciones(
    int $centroId, int $anio, array $meses
): array {
    // 1. Consulta externa — groupBy grupo_etareo
    $consultaExterna = PrestConsultaExterna::where('centro_salud_id', $centroId)
        ->where('anio', $anio)->whereIn('mes', $meses)
        ->selectRaw('grupo_etareo,
            SUM(primera_m) as primera_m, SUM(primera_f) as primera_f,
            SUM(nueva_m) as nueva_m, SUM(nueva_f) as nueva_f,
            SUM(repetida_m) as repetida_m, SUM(repetida_f) as repetida_f')
        ->groupBy('grupo_etareo')->get()
        ->keyBy('grupo_etareo')->toArray();

    // 2. Vacunas — groupBy tipo_vacuna + grupo_etareo
    $vacunas = PrestVacuna::where(...)->selectRaw('tipo_vacuna, grupo_etareo,
        SUM(dentro_m + dentro_f + fuera_m + fuera_f) as total,
        SUM(dentro_m) as dm, SUM(dentro_f) as df,
        SUM(fuera_m) as fm, SUM(fuera_f) as ff')
        ->groupBy('tipo_vacuna', 'grupo_etareo')->get();

    // 3. Micronutrientes — groupBy tipo, pluck total,tipo
    // 4. Prenatales — groupBy tipo_control + grupo_etareo
    // 5. Partos — groupBy tipo + lugar + atendido_por + grupo_etareo
    // 6. Puerperio — groupBy tipo_control
    // 7. Crecimiento — groupBy grupo_etareo
    // 8. Recién nacidos — groupBy indicador
    // 9. Anticoncepción — groupBy metodo + tipo_usuaria

    return [
        'consulta_externa' => $consultaExterna,
        'vacunas'          => $vacunas,
        'micronutrientes'  => $micronutrientes,
        'prenatales'       => $prenatales,
        'partos'           => $partos,
        'puerperio'        => $puerperio,
        'crecimiento'      => $crecimiento,
        'recien_nacidos'   => $recienNacidos,
        'anticoncepcion'   => $anticoncepcion,
    ];
}

Cada consulta sigue el mismo patrón: filtrar por centro_salud_id, anio y whereIn('mes', $meses), agrupar con selectRaw + groupBy, y devolver como array.

¿Por qué selectRaw?
Se usa selectRaw() con SUM porque las tablas prest_* tienen un registro por mes. Para obtener el acumulado del período hay que sumar. Eloquent no tiene un ->sum() múltiple con groupBy.

8 — Sección 4: Cobertura de Programas

El corazón del CAI. getCobertura() calcula para cada programa de salud:

  • Meta INE — dato anual del INE (nacimientos esperados, embarazos, etc.)
  • Población real — conteo del padrón actual
  • Atendidos — suma de prestaciones en el período
  • % Cobertura INE — atendidos / meta_ine × 100
  • % Cobertura Real — atendidos / pob_real × 100

getCobertura() — programas mensuales
$nMeses = count($meses);

Promedio mensual
Para Nutribebé y Carmelo el «atendidos» es total_periodo / nMeses porque la cobertura mensual mide «cuántos recibieron este mes». Si entregaste 120 paquetes en 4 meses, el promedio es 30/mes — comparado contra la meta de niños que deberían recibirlo cada mes.

10 — Deserción de vacunas

La tasa de deserción mide cuántos niños iniciaron el esquema pero no lo completaron:

getDesercion() — indicadores de abandono
public static function getDesercion(
    int $centroId, int $anio, array $meses
): array {
    $sumVacuna = fn (string $tipo) =>
        PrestVacuna::where('centro_salud_id', $centroId)
        ->where('anio', $anio)->whereIn('mes', $meses)
        ->where('tipo_vacuna', $tipo)
        ->selectRaw('SUM(dentro_m+dentro_f+fuera_m+fuera_f) as t')
        ->value('t') ?? 0;

    $penta1 = $sumVacuna('Pentavalente_1');
    $penta3 = $sumVacuna('Pentavalente_3');
    $srp1   = $sumVacuna('SRP_1');
    $srp2   = $sumVacuna('SRP_2');

    return [
        [
            'indicador' => 'Pentavalente 1ra → 3ra',
            'primera'   => $penta1,
            'ultima'    => $penta3,
            'tasa'      => $penta1 > 0
                ? round(($penta1 - $penta3) / $penta1 * 100, 1) : 0,
        ],
        [
            'indicador' => 'SRP 1ra → 2da',
            'primera'   => $srp1,
            'ultima'    => $srp2,
            'tasa'      => $srp1 > 0
                ? round(($srp1 - $srp2) / $srp1 * 100, 1) : 0,
        ],
    ];
}

La fórmula: (primera_dosis − última_dosis) / primera_dosis × 100. Una tasa de deserción de 20% significa que 2 de cada 10 niños abandonaron el esquema.

11 — Ceros justificados y observaciones

Dos métodos simples que extraen datos de las tablas auxiliares:

getCerosJustificados() + getObservaciones()
public static function getCerosJustificados(
    int $centroId, int $anio, array $meses
): array {
    return JustificacionCero::where('centro_salud_id', $centroId)
        ->where('anio', $anio)->whereIn('mes', $meses)
        ->orderBy('mes')->orderBy('indicador')
        ->get()
        ->map(fn ($j) => [
            'mes'       => self::$nombresMeses[$j->mes] ?? $j->mes,
            'indicador' => $j->indicador,
            'motivo'    => $j->motivo,
            'detalle'   => $j->detalle,
        ])->toArray();
}

public static function getObservaciones(
    int $centroId, int $anio, array $meses
): array {
    return ObservacionMensual::where('centro_salud_id', $centroId)
        ->where('anio', $anio)->whereIn('mes', $meses)
        ->whereNotNull('texto')->where('texto', '!=', '')
        ->orderBy('mes')
        ->get()
        ->map(fn ($o) => [
            'mes'   => self::$nombresMeses[$o->mes] ?? $o->mes,
            'texto' => $o->texto,
        ])->toArray();
}

Las justificaciones de cero se ingresan en el formulario mensual (lección 39) y se presentan aquí para que el CAI las revise. Las observaciones narrativas son notas libres del registrador.

11.1 — Estado de meses cerrados

getMesesCerrados()
public static function getMesesCerrados(
    int $centroId, int $anio, array $meses
): array {
    $cerrados = MesCerrado::where('centro_salud_id', $centroId)
        ->where('anio', $anio)->whereIn('mes', $meses)
        ->pluck('mes')->toArray();

    return collect($meses)->map(fn ($m) => [
        'mes'     => self::$nombresMeses[$m] ?? $m,
        'cerrado' => in_array($m, $cerrados),
    ])->toArray();
}

El informe indica qué meses están cerrados (datos finales) y cuáles abiertos (datos provisionales).

13 — buildFilaCobertura — helper privado

Todas las filas de cobertura se construyen con este método:

buildFilaCobertura() — cálculo estandarizado
private static function buildFilaCobertura(
    string $nombre,
    int $meta,
    int $real,
    int $atendidos
): array {
    return [
        'nombre'   => $nombre,
        'meta'     => $meta,
        'real'     => $real,
        'atendidos' => $atendidos,
        'cob_ine'  => $meta > 0
            ? round($atendidos / $meta * 100, 1) : 0,
        'cob_real' => $real > 0
            ? round($atendidos / $real * 100, 1) : 0,
    ];
}
CampoSignificadoEjemplo
metaProyección INE anual15 nacimientos
realPoblación del padrón12 menores de 1 año
atendidosPrestaciones en el período10 BCG aplicadas
cob_ine10/15 × 100 = 66.7%Respecto a meta INE
cob_real10/12 × 100 = 83.3%Respecto a padrón real
Dos coberturas
cob_ine es la cobertura oficial que pide el SNIS. cob_real es la cobertura real del centro — generalmente más alta porque el padrón suele ser menor que la proyección INE.
