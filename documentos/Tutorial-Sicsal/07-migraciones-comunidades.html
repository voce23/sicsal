1 Tabla comunidades

Cada centro de salud atiende a varias comunidades. La tabla comunidades registra estas localidades junto con su distancia y ubicación GPS.

Terminal bash 
$ php artisan make:migration create_comunidades_table

Diseño de la tabla

Columna Tipo Descripción
PK id bigint Identificador
FK centro_salud_id bigint Centro de salud al que pertenece
nombre varchar(100) Nombre de la comunidad
distancia_km decimal(6,2) Distancia al centro en km
latitud decimal(10,7) Coordenada GPS
longitud decimal(10,7) Coordenada GPS
activo boolean Borrado lógico

Código completo

database/migrations/XXXX_create_comunidades_table.php php 
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('comunidades', function (Blueprint $table) {
            $table->id();
            $table->foreignId('centro_salud_id')
                  ->constrained('centros_salud');
            $table->string('nombre', 100);
            $table->decimal('distancia_km', 6, 2)
                  ->nullable();                  // 9999.99 km máx
            $table->decimal('latitud', 10, 7)
                  ->nullable();                  // -17.5489321
            $table->decimal('longitud', 10, 7)
                  ->nullable();                  // -66.2539876
            $table->boolean('activo')->default(true);
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('comunidades');
    }
};
ℹ️
¿Por qué decimal para coordenadas?
decimal(10, 7) permite 3 dígitos enteros y 7 decimales, lo que da una precisión de ~1 cm. Suficiente para ubicar una comunidad en un mapa. Las coordenadas en Bolivia están alrededor de lat -17° lon -66°.

2 La tabla users de Laravel

Laravel viene con una migración predeterminada que crea la tabla users, password_reset_tokens y sessions. Ya existe en tu proyecto:

database/migrations/0001_01_01_000000_create_users_table.php (viene con Laravel) php 
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('users', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->string('email')->unique();
            $table->timestamp('email_verified_at')->nullable();
            $table->string('password');
            $table->rememberToken();
            $table->timestamps();
        });

        Schema::create('password_reset_tokens', function (Blueprint $table) {
            $table->string('email')->primary();
            $table->string('token');
            $table->timestamp('created_at')->nullable();
        });

        Schema::create('sessions', function (Blueprint $table) {
            $table->string('id')->primary();
            $table->foreignId('user_id')->nullable()->index();
            $table->string('ip_address', 45)->nullable();
            $table->text('user_agent')->nullable();
            $table->longText('payload');
            $table->integer('last_activity')->index();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('users');
        Schema::dropIfExists('password_reset_tokens');
        Schema::dropIfExists('sessions');
    }
};
⚠️
No modifiques este archivo
Esta migración viene con Laravel y ya se ejecutó. Para agregar columnas a users, creamos una nueva migración (en la siguiente sección). Así el historial queda limpio.

3 Ampliar users con campos SICSAL

SICSAL necesita campos adicionales en la tabla users: apellidos, nombre de usuario, centro de salud asignado, estado activo y último acceso.

Terminal bash 
$ php artisan make:migration add_sicsal_fields_to_users_table

Campos que agregaremos

Columna Tipo Descripción
apellidos varchar(100) Apellidos del usuario
usuario varchar(50) UNIQUE Username para login
FK centro_salud_id bigint nullable Centro asignado (null para superadmin)
activo boolean Si el usuario puede acceder
ultimo_acceso timestamp nullable Fecha/hora del último login
ℹ️
¿Por qué centro_salud_id es nullable?
El superadmin no pertenece a ningún centro — puede ver todos los centros. El admin y registrador sí están asignados a un centro específico. Por eso es nullable.

Código completo

Nota: esta migración usa Schema::table (ALTER) en vez de Schema::create:

database/migrations/XXXX_add_sicsal_fields_to_users_table.php php 
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::table('users', function (Blueprint $table) {
            $table->string('apellidos', 100)
                  ->after('name');
            $table->string('usuario', 50)
                  ->unique()
                  ->after('apellidos');
            $table->foreignId('centro_salud_id')
                  ->nullable()
                  ->after('usuario')
                  ->constrained('centros_salud');
            $table->boolean('activo')
                  ->default(true)
                  ->after('centro_salud_id');
            $table->timestamp('ultimo_acceso')
                  ->nullable()
                  ->after('activo');
        });
    }

    public function down(): void
    {
        Schema::table('users', function (Blueprint $table) {
            $table->dropForeign(['centro_salud_id']);
            $table->dropColumn([
                'apellidos', 'usuario', 'centro_salud_id',
                'activo', 'ultimo_acceso'
            ]);
        });
    }
};

Diferencia: Schema::create vs Schema::table

Método SQL generado Cuándo usarlo
Schema::create('tabla', ...) CREATE TABLE tabla (...) Tabla nueva
Schema::table('tabla', ...) ALTER TABLE tabla ADD ... Modificar tabla existente
💡
El modificador ->after()
->after('columna') indica dónde insertar la nueva columna en MySQL. No es obligatorio pero mantiene el orden lógico de columnas. Solo funciona en MySQL, no en SQLite.

4 Relaciones hasta ahora

Con estas 4 tablas ya tenemos la estructura base de SICSAL:

municipios
1 ──── N
centros_salud
1 ──── N
comunidades
centros_salud
1 ──── N
users

Lectura:

  • Un municipio tiene muchos centros de salud
  • Un centro de salud tiene muchas comunidades
  • Un centro de salud tiene muchos usuarios
  • Un usuario pertenece a un centro de salud (o a ninguno si es superadmin)
