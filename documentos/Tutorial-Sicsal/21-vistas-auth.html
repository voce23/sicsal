2 Layout auth y componentes comunes

Todas las vistas auth comparten esta estructura:

Patrón común de todas las vistas auth blade 
<x-layouts::auth>
    <!-- Header con título y descripción -->
    <x-auth-header>
        :title="Iniciar sesión"
        :description="Ingrese sus credenciales"
    </x-auth-header>

    <!-- Mensajes de sesión (éxito, error) -->
    <x-auth-session-status :status="session('status')" />

    <!-- Formulario con Flux UI -->
    <form method="POST" action="{{ route('login') }}">
        @csrf
        <flux:input name="email" label="Correo electrónico" />
        <flux:input name="password" type="password" />
        <flux:button type="submit">Ingresar</flux:button>
    </form>
</x-layouts::auth>
Componente Origen Función
<x-layouts::auth> Layout Blade Wrapper: centrado, fondo, meta tags
<x-auth-header> Componente Blade Logo + título + descripción
<x-auth-session-status> Componente Blade Mensajes flash (éxito/error)
<flux:input> Flux UI (paquete) Input estilizado con label y error
<flux:button> Flux UI Botón con variantes (primary, danger)
<flux:otp> Flux UI Input de código OTP (para 2FA)

3 Login — formulario principal

resources/views/pages/auth/login.blade.php (estructura) blade 
<form method="POST" action="{{ route('login') }}">
    @csrf
    <flux:input name="email" type="email"
        label="Correo electrónico" required />

    <flux:input name="password" type="password"
        label="Contraseña" required />

    <flux:checkbox name="remember"
        label="Recordarme" />

    <flux:button type="submit" variant="primary" class="w-full">
        Ingresar
    </flux:button>

    <flux:link href="{{ route('password.request') }}">
        ¿Olvidaste tu contraseña?
    </flux:link>
</form>
ℹ️
Rutas de Fortify — automáticas
route('login'), route('register'), route('password.request') — todas registradas automáticamente por Fortify. No necesitas definirlas en routes/web.php.

4 Register — con centro de salud

resources/views/pages/auth/register.blade.php (extracto) blade 
<flux:input name="name"
    label="Nombres" required />

<flux:input name="apellidos"
    label="Apellidos" required />

<flux:input name="email" type="email"
    label="Correo electrónico" required />

<!-- Select de Centro de Salud — se valida contra DB -->
<select name="centro_salud_id" required>
    @foreach(\App\Models\CentroSalud::where('activo', true)
        ->orderBy('nombre')->get() as $centro)
        <option value="{{ $centro->id }}">
            {{ $centro->nombre }}
        </option>
    @endforeach
</select>

<flux:input name="password" type="password"
    label="Contraseña" required />
<flux:input name="password_confirmation" type="password"
    label="Confirmar contraseña" required />
⚠️
centro_salud_id obligatorio al registrarse
El formulario de registro incluye un select de centros de salud. CreateNewUser valida que el ID exista en la tabla centros_salud. El usuario queda ligado a ese centro desde el momento del registro.
