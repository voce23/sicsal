1 User — Authenticatable, FilamentUser, HasRoles

User es el único modelo que NO extiende Model — extiende Authenticatable e implementa FilamentUser:

app/Models/User.php php 
use Filament\Models\Contracts\FilamentUser;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Laravel\Fortify\TwoFactorAuthenticatable;
use Spatie\Permission\Traits\HasRoles;

class User extends Authenticatable implements FilamentUser
{
    use HasFactory, HasRoles, Notifiable, TwoFactorAuthenticatable;
}
💡
4 traits en acción
  • HasFactory → permite User::factory()->create()
  • HasRoles → agrega hasRole(), assignRole(), hasPermissionTo() (Spatie)
  • Notifiable → $user->notify() para emails/SMS
  • TwoFactorAuthenticatable → autenticación de 2 factores (Fortify)

$fillable y $hidden

app/Models/User.php — propiedades php 
protected $fillable = [
    'name', 'apellidos', 'usuario', 'email',
    'password', 'centro_salud_id', 'activo', 'ultimo_acceso',
];

protected $hidden = [
    'password',
    'two_factor_secret',
    'two_factor_recovery_codes',
    'remember_token',
];

protected function casts(): array
{
    return [
        'email_verified_at' => 'datetime',
        'ultimo_acceso'     => 'datetime',
        'password'          => 'hashed',   // ← ¡Laravel hashea automáticamente!
        'activo'            => 'boolean',
    ];
}
⚠️
Cast 'hashed' — seguridad automática
Con 'password' => 'hashed', Laravel aplica bcrypt automáticamente al asignar la contraseña. Nunca necesitas llamar Hash::make() manualmente.

2 User — canAccessPanel, accessor, initials

app/Models/User.php — métodos php 
// Filament exige este método. Solo entran usuarios activos con rol válido.
public function canAccessPanel(Panel $panel): bool
{
    return $this->activo
        && $this->hasAnyRole(['superadmin', 'admin', 'registrador']);
}

// Relación: cada usuario pertenece a un centro de salud
public function centroSalud(): BelongsTo
{
    return $this->belongsTo(CentroSalud::class);
}

// Accessor: $user->nombre_completo
public function getNombreCompletoAttribute(): string
{
    return "{$this->name} {$this->apellidos}";
}

// Para avatares: $user->initials() → "JM"
public function initials(): string
{
    $parts = array_filter([
        trim($this->name ?? ''),
        trim($this->apellidos ?? ''),
    ]);
    return collect($parts)
        ->map(fn (string $part) => mb_strtoupper(mb_substr($part, 0, 1)))
        ->implode('');
}
ℹ️
canAccessPanel — el guardián de Filament
FilamentUser requiere este método. Si retorna false, el usuario es redirigido fuera del panel aunque tenga sesión válida. SICSAL verifica: (1) usuario activo y (2) tiene alguno de los 3 roles del sistema.

3 Post — Blog con categorías y slugs

app/Models/Post.php php 
class Post extends Model
{
    protected $fillable = [
        'titulo', 'slug', 'extracto', 'contenido',
        'imagen_portada', 'categoria', 'etiquetas',
        'publicado', 'publicado_at', 'autor_nombre', 'vistas',
    ];

    protected $casts = [
        'etiquetas'    => 'array',      // JSON → array de PHP
        'publicado'    => 'boolean',
        'publicado_at' => 'datetime',
    ];
}

Slug único + helpers

app/Models/Post.php — métodos php 
// Scope: solo posts publicados, ordenados por fecha
public function scopePublicados($query)
{
    return $query->where('publicado', true)
                 ->orderByDesc('publicado_at');
}

// Genera slug único, evitando colisiones
public static function generarSlug(string $titulo, ?int $excludeId = null): string
{
    $base = Str::slug($titulo);
    $slug = $base;
    $i    = 1;
    while (
        static::where('slug', $slug)
            ->when($excludeId, fn($q) => $q->where('id', '!=', $excludeId))
            ->exists()
    ) {
        $slug = "{$base}-{$i}";
        $i++;
    }
    return $slug;
}

// Tiempo de lectura estimado
public function tiempoLectura(): int
{
    $palabras = str_word_count(strip_tags($this->contenido ?? ''));
    return max(1, (int) ceil($palabras / 200));
}
💡
Cast 'array' para JSON
'etiquetas' => 'array' convierte automáticamente entre JSON en la base de datos y arrays en PHP. Puedes hacer $post->etiquetas = ['vacunas', 'menores'] y Eloquent lo guarda como ["vacunas","menores"].

4 Comentario — el modelo más simple

app/Models/Comentario.php php 
class Comentario extends Model
{
    protected $fillable = ['pagina', 'nombre', 'contenido'];
}

Tres campos, sin relaciones, sin casts, sin scopes. Este es el modelo mínimo posible en Eloquent. Almacena comentarios del blog indexados por nombre de página.

5 CausaConsultaExterna — top 10 de morbilidad

app/Models/CausaConsultaExterna.php php 
class CausaConsultaExterna extends Model
{
    protected $table = 'causas_consulta_externa';

    protected $fillable = [
        'centro_salud_id', 'mes', 'anio',
        'posicion',       // 1 a 10 (orden de morbilidad)
        'diagnostico',    // texto libre CIE-10
        'grupo_etareo',   // clave del array $grupos
        'masculino', 'femenino',
    ];

    protected $casts = [
        'mes' => 'integer', 'anio' => 'integer',
        'masculino' => 'integer', 'femenino' => 'integer',
    ];
}

Arrays estáticos como catálogos

app/Models/CausaConsultaExterna.php — arrays php 
public static array $grupos = [
    'menor_6m'   => '< 6 meses',
    '6m_menor_1' => '6m – 1 año',
    '1_4'        => '1 – 4 años',
    '5_9'        => '5 – 9 años',
    '10_14'      => '10 – 14 años',
    '15_19'      => '15 – 19 años',
    '20_39'      => '20 – 39 años',
    '40_49'      => '40 – 49 años',
    '50_59'      => '50 – 59 años',
    'mayor_60'   => '≥ 60 años',
];

public static array $meses = [
    1 => 'Enero', 2 => 'Febrero', ... 12 => 'Diciembre',
];
ℹ️
¿Por qué arrays estáticos y no una tabla?
Los grupos etáreos y meses nunca cambian. Tenerlos como public static array en el modelo los hace accesibles desde Filament (Select::make('grupo_etareo')->options(CausaConsultaExterna::$grupos)) sin necesidad de una tabla extra ni queries adicionales.
