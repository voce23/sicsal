1 Patrón común de los 9 modelos

Todos los modelos clínicos comparten estas características:

Característica Detalle
$tableNombre explícito (la convención no aplica para plurales en español)
BelongsTo PersonaTodo registro clínico está ligado a una persona
BelongsTo UserAuditoría: quién registró el dato (FK personalizada)
Cast 'date'Todas las fechas se castean a Carbon
mes + anioPara agrupar datos por período mensual
ℹ️
¿Por qué cada modelo tiene registrado_por / aplicado_por?
Es un requisito de auditoría: siempre saber quién ingresó cada dato. La FK apunta a users con un nombre personalizado (belongsTo(User::class, 'registrado_por')).

2 VacunaNino

app/Models/VacunaNino.php php 
class VacunaNino extends Model
{
    protected $table = 'vacunas_ninos';

    protected $fillable = [
        'persona_id', 'tipo_vacuna', 'fecha_aplicacion',
        'dentro_fuera', 'lote', 'aplicado_por', 'mes', 'anio',
    ];

    protected function casts(): array
    {
        return ['fecha_aplicacion' => 'date'];
    }

    public function persona(): BelongsTo
    {
        return $this->belongsTo(Persona::class);
    }

    public function aplicadoPor(): BelongsTo
    {
        return $this->belongsTo(User::class, 'aplicado_por');
    }
}

dentro_fuera indica si la vacuna se aplicó dentro del establecimiento o en la comunidad. lote es el número de lote de la vacuna (para trazabilidad). Las 31 vacunas disponibles se definen como enum en la migración.

3 CrecimientoInfantil

app/Models/CrecimientoInfantil.php php 
class CrecimientoInfantil extends Model
{
    protected $table = 'crecimiento_infantil';

    protected $fillable = [
        'persona_id', 'fecha', 'peso_kg', 'talla_cm',
        'perimetro_cefalico_cm', 'clasificacion', 'tipo_control',
        'dentro_fuera', 'mes', 'anio', 'registrado_por',
    ];

    protected function casts(): array
    {
        return [
            'fecha'                => 'date',
            'peso_kg'              => 'decimal:2',
            'talla_cm'             => 'decimal:2',
            'perimetro_cefalico_cm' => 'decimal:2',
        ];
    }

    public function persona(): BelongsTo { return $this->belongsTo(Persona::class); }
    public function registradoPor(): BelongsTo { return $this->belongsTo(User::class, 'registrado_por'); }
}

Nótese el uso de 'decimal:2' para las medidas antropométricas. Un bebé puede pesar 3.45 kg o medir 52.30 cm — necesitamos exactamente 2 decimales.

5 Embarazo — cadena de relaciones

El modelo Embarazo es especial porque inicia una cadena de 3 niveles:

Persona
→ hasMany →
Embarazo
→ hasMany →
ControlPrenatal
Persona
→
Embarazo
→ hasOne →
Parto
→ hasOne →
Puerperio
app/Models/Embarazo.php php 
class Embarazo extends Model
{
    protected $fillable = [
        'persona_id', 'fecha_inicio', 'fecha_probable_parto',
        'semanas_gestacion_ingreso', 'estado', 'observaciones', 'activo',
    ];

    protected function casts(): array
    {
        return [
            'fecha_inicio'         => 'date',
            'fecha_probable_parto' => 'date',
            'activo'               => 'boolean',
        ];
    }

    public function persona(): BelongsTo
    {
        return $this->belongsTo(Persona::class);
    }

    public function controles(): HasMany
    {
        return $this->hasMany(ControlPrenatal::class);
    }

    public function parto(): HasOne
    {
        return $this->hasOne(Parto::class);
    }

    public function scopeActivas($query)
    {
        return $query->where('estado', 'activa');
    }
}
💡
HasMany vs HasOne
Un embarazo tiene muchos controles prenatales (HasMany), pero solo un parto (HasOne). El scope activas() filtra embarazos en curso.

ℹ️
4 controles puerperales
El puerperio tiene 4 fechas de control obligatorio: 48 horas, 7 días, 28 días y 42 días después del parto. Si una fecha es null, significa que la paciente no asistió a ese control.

9 Defuncion — el modelo más sofisticado

Defuncion es el único modelo clínico que usa lifecycle hooks (booted()) para ejecutar lógica automática al guardar.

app/Models/Defuncion.php — lifecycle hooks php 
protected static function booted(): void
{
    // ANTES de guardar —————————————————————
    static::saving(function (Defuncion $defuncion) {
        // 1. Calcular grupo etáreo automáticamente
        if ($defuncion->fecha_nacimiento && $defuncion->fecha_defuncion) {
            $defuncion->grupo_etareo_defuncion = self::calcularGrupoEtareo(
                $defuncion->fecha_nacimiento,
                $defuncion->fecha_defuncion
            );
        }

        // 2. Asignar centro_salud_id del usuario
        if (! $defuncion->centro_salud_id) {
            $defuncion->centro_salud_id = auth()->user()?->centro_salud_id;
        }

        // 3. Asignar registrado_por
        if (! $defuncion->registrado_por) {
            $defuncion->registrado_por = auth()->id();
        }
    });

    // DESPUÉS de crear —————————————————————
    static::created(function (Defuncion $defuncion) {
        // Marcar la persona como inactiva en el padrón
        if ($defuncion->persona_id) {
            Persona::where('id', $defuncion->persona_id)
                ->update(['activo' => false]);
        }
    });
}
⚠️
Efecto en cascada: defunción → persona inactiva
Cuando se registra una defunción, el hook created desactiva automáticamente a la persona en el padrón. Esto es un borrado lógico — la persona sigue en la BD pero ya no aparece en conteos activos.

Cálculo automático de grupo etáreo

app/Models/Defuncion.php — calcularGrupoEtareo() php 
public static function calcularGrupoEtareo(
    mixed $fechaNacimiento,
    mixed $fechaDefuncion
): string {
    $nacimiento = Carbon::parse($fechaNacimiento);
    $defuncion  = Carbon::parse($fechaDefuncion);
    $dias       = $nacimiento->diffInDays($defuncion);
    $anios      = $nacimiento->diffInYears($defuncion);

    if ($dias < 28)  return 'neonatal';
    if ($dias < 365) return 'infantil_menor_1';
    if ($anios < 5)  return 'menor_5';
    if ($anios < 60) return '5_a_59';
    return 'adulto_mayor';
}

A diferencia de Persona que calcula el grupo respecto a now(), las defunciones calculan la edad al momento de la muerte, usando la diferencia entre fecha de nacimiento y fecha de defunción.

10 Cadena completa: Embarazo → Parto → Puerperio

Con estos 9 modelos, podemos navegar toda la historia clínica de una persona:

Ejemplo — navegación por relaciones php 
// Obtener una persona con toda su historia clínica
$persona = Persona::with([
    'vacunas',
    'crecimiento',
    'micronutrientes',
    'embarazos.controles',       // eager load anidado
    'embarazos.parto.puerperio', // 3 niveles de profundidad
    'anticoncepciones',
    'defuncion',
])->find(1);

// ¿Cuántos controles prenatales tiene el embarazo activo?
$controles = $persona->embarazos
    ->where('estado', 'activa')
    ->first()
    ?->controles
    ->count();

// ¿El puerperio fue a las 48h?
$puerperio48 = $persona->embarazos->last()
    ?->parto?->puerperio?->control_48h;
💡
Eager Loading con with()
with(['embarazos.controles']) carga los embarazos y sus controles en solo 3 queries SQL (en vez de N+1). La notación con punto (.) navega relaciones anidadas.

✅ Resumen de la Lección 14

  • 9 modelos clínicos: VacunaNino, CrecimientoInfantil, MicronutrienteNino, Embarazo, ControlPrenatal, Parto, Puerperio, Anticoncepcion, Defuncion
