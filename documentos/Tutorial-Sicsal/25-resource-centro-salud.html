1 Estructura de un Resource Filament

Un Resource en Filament es la unidad CRUD. Agrupa formulario, tabla y páginas para un modelo Eloquent.

app/Filament/Resources/CentroSaluds/
CentroSaludResource.php
Pages/
CreateCentroSalud.php
EditCentroSalud.php
ListCentroSaluds.php
Schemas/
CentroSaludForm.php
Tables/
CentroSaludsTable.php
Convención SICSAL

En SICSAL, los formularios y tablas están en clases separadas (Schemas/ y Tables/), no inline en el Resource. Esto mantiene cada archivo pequeño y enfocado. El Resource solo conecta las piezas.

2 CentroSaludResource — clase principal

app/Filament/Resources/CentroSaluds/CentroSaludResource.php
<?php

namespace App\Filament\Resources\CentroSaluds;

use App\Filament\Resources\CentroSaluds\Pages\CreateCentroSalud;
use App\Filament\Resources\CentroSaluds\Pages\EditCentroSalud;
use App\Filament\Resources\CentroSaluds\Pages\ListCentroSaluds;
use App\Filament\Resources\CentroSaluds\Schemas\CentroSaludForm;
use App\Filament\Resources\CentroSaluds\Tables\CentroSaludsTable;
use App\Models\CentroSalud;
use BackedEnum;
use Filament\Resources\Resource;
use Filament\Schemas\Schema;
use Filament\Support\Icons\Heroicon;
use Filament\Tables\Table;

class CentroSaludResource extends Resource
{
    protected static ?string $model = CentroSalud::class;

    // ─── Navegación ───
    protected static string|BackedEnum|null $navigationIcon
        = Heroicon::OutlinedBuildingOffice2;
    protected static string|\UnitEnum|null $navigationGroup
        = 'Configuración';
    protected static ?string $navigationLabel = 'Centros de Salud';
    protected static ?string $modelLabel = 'Centro de Salud';
    protected static ?string $pluralModelLabel = 'Centros de Salud';
    protected static ?int $navigationSort = 1;
    protected static ?string $recordTitleAttribute = 'nombre';

    // ─── Acceso ───
    public static function canAccess(): bool
    {
        return auth()->user()?->hasRole('superadmin') ?? false;
    }

    // ─── Form y Table delegados ───
    public static function form(Schema $schema): Schema
    {
        return CentroSaludForm::configure($schema);
    }

    public static function table(Table $table): Table
    {
        return CentroSaludsTable::configure($table);
    }

    public static function getRelations(): array
    {
        return [];
    }

    public static function getPages(): array
    {
        return [
            'index'  => ListCentroSaluds::route('/'),
            'create' => CreateCentroSalud::route('/create'),
            'edit'   => EditCentroSalud::route('/{record}/edit'),
        ];
    }
}

canAccess() — solo superadmin

Solo el superadmin puede gestionar centros de salud. Los admin y registrador no ven este Resource en el sidebar. Si intentan acceder por URL, Filament devuelve 403.

3 CentroSaludForm — formulario con 2 secciones

app/Filament/Resources/CentroSaluds/Schemas/CentroSaludForm.php
<?php

namespace App\Filament\Resources\CentroSaluds\Schemas;

use Filament\Forms\Components\Placeholder;
use Filament\Schemas\Components\Section;
use Filament\Forms\Components\Select;
use Filament\Forms\Components\TextInput;
use Filament\Forms\Components\Toggle;
use Filament\Schemas\Schema;

class CentroSaludForm
{
    public static function configure(Schema $schema): Schema
    {
        return $schema
            ->components([

                // ─── Sección 1: Información General ───
                Section::make('Información General')
                    ->columns(2)
                    ->schema([
                        Select::make('municipio_id')
                            ->relationship('municipio', 'nombre')
                            ->searchable()->preload()->required()
                            ->label('Municipio'),
                        TextInput::make('nombre')
                            ->required()->maxLength(150)
                            ->label('Nombre del centro'),
                        TextInput::make('codigo_snis')
                            ->maxLength(20)
                            ->label('Código SNIS'),
                        Select::make('subsector')
                            ->options([
                                'Público'       => 'Público',
                                'Seguro Social' => 'Seguro Social',
                                'Privado'       => 'Privado',
                                'ONG'           => 'ONG',
                            ])
                            ->default('Público')->required()
                            ->label('Subsector'),
                        TextInput::make('red_salud')
                            ->maxLength(100)
                            ->label('Red de Salud'),
                        TextInput::make('poblacion_ine')
                            ->numeric()->default(0)
                            ->label('Población INE asignada'),
                        Toggle::make('activo')
                            ->default(true)
                            ->label('Activo')->columnSpan(2),
                    ]),

                // ─── Sección 2: Ubicación Geográfica ───
                Section::make('Ubicación Geográfica')
                    ->description('Coordenadas GPS del establecimiento...')
                    ->columns(2)
                    ->schema([
                        TextInput::make('latitud')
                            ->numeric()->label('Latitud')
                            ->placeholder('-17.886227')
                            ->rules(['nullable', 'numeric', 'between:-90,90']),
                        TextInput::make('longitud')
                            ->numeric()->label('Longitud')
                            ->placeholder('-66.201318')
                            ->rules(['nullable', 'numeric', 'between:-180,180']),
                        Placeholder::make('mapa_ayuda')
                            ->label('')
                            ->content('💡 Cómo obtener coordenadas: ...')
                            ->columnSpan(2),
                    ]),
            ]);
    }
}
Componentes Filament usados
  • Select::make('municipio_id')->relationship() — genera dropdown desde la relación Eloquent con búsqueda
  • TextInput::make()->numeric() — input con validación numérica
  • Toggle::make('activo') — switch on/off
  • Placeholder::make() — texto informativo sin input
  • Section::make()->columns(2) — agrupación visual en 2 columnas

4 CentroSaludsTable — listado con 7 columnas

app/Filament/Resources/CentroSaluds/Tables/CentroSaludsTable.php
<?php

namespace App\Filament\Resources\CentroSaluds\Tables;

use Filament\Actions\BulkActionGroup;
use Filament\Actions\DeleteBulkAction;
use Filament\Actions\EditAction;
use Filament\Tables\Columns\IconColumn;
use Filament\Tables\Columns\TextColumn;
use Filament\Tables\Table;

class CentroSaludsTable
{
    public static function configure(Table $table): Table
    {
        return $table
            ->columns([
                TextColumn::make('nombre')
                    ->searchable()->sortable()->label('Nombre'),
                TextColumn::make('codigo_snis')
                    ->searchable()->label('Código SNIS'),
                TextColumn::make('municipio.nombre')
                    ->searchable()->sortable()->label('Municipio'),
                TextColumn::make('red_salud')
                    ->searchable()->label('Red de Salud'),
                TextColumn::make('subsector')
                    ->badge()->label('Subsector'),
                TextColumn::make('poblacion_ine')
                    ->numeric()->sortable()->label('Pob. INE'),
                IconColumn::make('activo')
                    ->boolean()->label('Activo'),
            ])
            ->filters([])
            ->recordActions([EditAction::make()])
            ->toolbarActions([
                BulkActionGroup::make([
                    DeleteBulkAction::make(),
                ]),
            ]);
    }
}

municipio.nombre — notación punto

TextColumn::make('municipio.nombre') accede al campo nombre de la relación municipio. Filament hace el JOIN automáticamente. Lo mismo funciona con searchable() y sortable().

5 Pages: List, Create, Edit

Las 3 páginas son casi idénticas — herencia directa de clases base de Filament:

Pages/ListCentroSaluds.php
<?php

namespace App\Filament\Resources\CentroSaluds\Pages;

use App\Filament\Resources\CentroSaluds\CentroSaludResource;
use Filament\Actions\CreateAction;
use Filament\Resources\Pages\ListRecords;

class ListCentroSaluds extends ListRecords
{
    protected static string $resource = CentroSaludResource::class;

    protected function getHeaderActions(): array
    {
        return [CreateAction::make()];
    }
}
Pages/CreateCentroSalud.php
class CreateCentroSalud extends CreateRecord
{
    protected static string $resource = CentroSaludResource::class;
}
Pages/EditCentroSalud.php
class EditCentroSalud extends EditRecord
{
    protected static string $resource = CentroSaludResource::class;

    protected function getHeaderActions(): array
    {
        return [DeleteAction::make()];
    }
}
Minimal boilerplate

Las páginas solo necesitan declarar $resource y opcionalmente getHeaderActions(). Filament se encarga de renderizar el formulario (Create/Edit) o la tabla (List) automáticamente.

6 Patrón SICSAL: Schemas/ y Tables/ separados

Resource
Conecta piezas, acceso, navegación
↓ delega
Schemas/Form
Campos del formulario
Tables/Table
Columnas, filtros, acciones
Pages/*
List, Create, Edit

Este patrón se repite en los 8 Resources de SICSAL. Las ventajas:

  • Cada archivo tiene una sola responsabilidad
  • Los formularios complejos (como PersonaForm, 121 líneas) no ensucian el Resource
  • Fácil de encontrar: "necesito cambiar una columna" → ve a Tables/

7 Resumen

Lo que aprendimos

  • Un Resource Filament agrupa modelo, formulario, tabla y páginas CRUD
  • CentroSaludResource — solo accesible para superadmin, grupo "Configuración"
  • CentroSaludForm — 2 secciones: info general (7 campos) + GPS (2 coordenadas)
  • CentroSaludsTable — 7 columnas con búsqueda, ordenamiento y badge
  • 3 Pages minimal: List (+ CreateAction), Create (vacía), Edit (+ DeleteAction)
  • Patrón SICSAL: Form en Schemas/, Table en Tables/, separados del Resource
  • Componentes clave: Select->relationship(), Toggle, IconColumn->boolean(), badge()
