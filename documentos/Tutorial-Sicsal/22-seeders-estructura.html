1 ¿Qué son los seeders?

Los seeders son clases Laravel que insertan datos iniciales en la base de datos. Se ubican en database/seeders/ y se ejecutan con php artisan db:seed.

Patrón clave: firstOrCreate

Todos los seeders de SICSAL usan firstOrCreate en vez de create. Esto significa que puedes ejecutarlos múltiples veces sin duplicar datos — se crea el registro solo si no existe ya.

SICSAL tiene 8 seeders en total:

2 DatabaseSeeder — orquestador

El DatabaseSeeder es el punto de entrada. Laravel lo llama cuando ejecutas php artisan db:seed. Su trabajo es invocar los demás seeders en el orden correcto.

database/seeders/DatabaseSeeder.php
<?php

namespace Database\Seeders;

use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    public function run(): void
    {
        $this->call([
            RolesSeeder::class,
            MunicipioSeeder::class,
            CentroSaludSeeder::class,
            ComunidadSeeder::class,
            MetaIneSeeder::class,
            UserSeeder::class,
        ]);
    }
}
Orden importa

El orden de la lista no es arbitrario. Cada seeder depende de datos que creó el anterior:

  1. Roles — primero, porque UserSeeder necesita roles para assignRole()
  2. Municipio — necesario antes de CentroSalud (FK municipio_id)
  3. CentroSalud — necesario antes de Comunidad y User (FK centro_salud_id)
  4. Comunidad — necesita centro_salud_id
  5. MetaIne — necesita centro_salud_id
  6. User — necesita roles + centro_salud para asignar
DatosPruebaSeeder NO está incluido

DatosPruebaSeeder no forma parte de $this->call(). Es un seeder separado que se ejecuta manualmente solo cuando necesitas datos de prueba para desarrollo. Lo veremos en la lección 23.

3 RolesSeeder — tres roles del sistema

Crea los 3 roles que necesita Spatie Permission. Es el seeder más sencillo.

database/seeders/RolesSeeder.php
<?php

namespace Database\Seeders;

use Illuminate\Database\Seeder;
use Spatie\Permission\Models\Role;

class RolesSeeder extends Seeder
{
    public function run(): void
    {
        Role::firstOrCreate(['name' => 'superadmin',   'guard_name' => 'web']);
        Role::firstOrCreate(['name' => 'admin',        'guard_name' => 'web']);
        Role::firstOrCreate(['name' => 'registrador',  'guard_name' => 'web']);
    }
}
Tabla roles (resultado)
nameguard_nameAcceso
superadminwebTodo el sistema, todos los centros de salud
adminwebPanel admin, filtrado por su centro de salud
registradorwebSolo formularios de captura del mes

Recuerda: los permisos granulares no se usan en SICSAL. El control se hace por rol con hasRole() y filtrado por centro_salud_id (lección 19).

4 MunicipioSeeder

Crea el municipio base: Capinota, Cochabamba.

database/seeders/MunicipioSeeder.php
<?php

namespace Database\Seeders;

use App\Models\Municipio;
use Illuminate\Database\Seeder;

class MunicipioSeeder extends Seeder
{
    public function run(): void
    {
        Municipio::firstOrCreate(
            ['nombre' => 'Capinota'],
            ['departamento' => 'Cochabamba']
        );
    }
}
firstOrCreate — dos argumentos

El primer array son las columnas de búsqueda (WHERE). El segundo array son las columnas adicionales para INSERT si no existe. Si ya hay un municipio llamado "Capinota", no hace nada.

5 CentroSaludSeeder

Crea el centro de salud principal: C.S.A. HORNOMA.

database/seeders/CentroSaludSeeder.php
<?php

namespace Database\Seeders;

use App\Models\CentroSalud;
use App\Models\Municipio;
use Illuminate\Database\Seeder;

class CentroSaludSeeder extends Seeder
{
    public function run(): void
    {
        $municipio = Municipio::where('nombre', 'Capinota')->firstOrFail();

        CentroSalud::firstOrCreate(
            ['codigo_snis' => '300183'],
            [
                'municipio_id'  => $municipio->id,
                'nombre'        => 'C.S.A. HORNOMA',
                'subsector'     => 'Público',
                'red_salud'     => 'Capinota',
                'poblacion_ine' => 641,
                'activo'        => true,
            ]
        );
    }
}
Dato clave del centro
CampoValorPropósito
codigo_snis300183Código SNIS oficial — clave de búsqueda
nombreC.S.A. HORNOMACentro de Salud de Área
poblacion_ine641Población INE — base para calcular metas
red_saludCapinotaRed de salud a la que pertenece

Nótese que usa firstOrFail() para el municipio — si no existe, lanza excepción en vez de continuar con null. Esto garantiza que MunicipioSeeder se ejecutó antes.

6 ComunidadSeeder — 8 comunidades

Inserta las 8 comunidades del área de influencia de HORNOMA.

database/seeders/ComunidadSeeder.php
<?php

namespace Database\Seeders;

use App\Models\CentroSalud;
use App\Models\Comunidad;
use Illuminate\Database\Seeder;

class ComunidadSeeder extends Seeder
{
    public function run(): void
    {
        $centro = CentroSalud::where('codigo_snis', '300183')->firstOrFail();

        $comunidades = [
            ['nombre' => 'Hornoma',       'distancia_km' => 0],
            ['nombre' => 'Huaychoma',     'distancia_km' => 5],
            ['nombre' => 'Villcabamba',   'distancia_km' => 6],
            ['nombre' => 'Cocoma',        'distancia_km' => 24],
            ['nombre' => 'Tocohalla',     'distancia_km' => 48],
            ['nombre' => 'Challavilque',  'distancia_km' => 48],
            ['nombre' => 'Calacaja',      'distancia_km' => 72],
            ['nombre' => 'Siquimirani',   'distancia_km' => null],
        ];

        foreach ($comunidades as $c) {
            Comunidad::firstOrCreate(
                [
                    'centro_salud_id' => $centro->id,
                    'nombre'          => $c['nombre'],
                ],
                ['distancia_km' => $c['distancia_km']]
            );
        }
    }
}

¿Por qué 25 entradas?

Son 15 grupos etarios simples + 10 grupos especiales (embarazos esperados, partos, MEF, dT, etc.). Estos números vienen de las proyecciones oficiales del INE para la población de Hornoma (641 habitantes) en 2026.

Ejemplo de entradas — grupos simples vs. especiales
grupo_etareosexocantidadTipo

Nota que 7_49 aparece dos veces — una vez para M y otra para F. Lo mismo con dt_7_49. La clave de búsqueda es [centro_salud_id, anio, grupo_etareo, sexo].

8 UserSeeder — 3 usuarios con roles

Crea los 3 usuarios iniciales del sistema, cada uno con un rol diferente.

database/seeders/UserSeeder.php
<?php

namespace Database\Seeders;

use App\Models\CentroSalud;
use App\Models\User;
use Illuminate\Database\Seeder;
use Illuminate\Support\Facades\Hash;

class UserSeeder extends Seeder
{
    public function run(): void
    {
        $centro = CentroSalud::where('codigo_snis', '300183')->first();

        // ─── Superadmin ───
        $superadmin = User::firstOrCreate(
            ['usuario' => 'superadmin'],
            [
                'name'            => 'Administrador',
                'apellidos'       => 'General del Sistema',
                'email'           => 'admin@sicsal.bo',
                'password'        => Hash::make(env('SEED_SUPERADMIN_PASS', 'Cambiar123!')),
                'centro_salud_id' => null,
                'activo'          => true,
            ]
        );
        $superadmin->assignRole('superadmin');

        // ─── Admin (médico) ───
        $admin = User::firstOrCreate(
            ['usuario' => 'medico.hornoma'],
            [
                'name'            => 'Eusebio',
                'apellidos'       => 'Panozo Franco',
                'email'           => 'medico@hornoma.bo',
                'password'        => Hash::make(env('SEED_ADMIN_PASS', 'Cambiar123!')),
                'centro_salud_id' => $centro?->id,
                'activo'          => true,
            ]
        );
        $admin->assignRole('admin');

        // ─── Registrador ───
        $registrador = User::firstOrCreate(
            ['usuario' => 'registrador.hornoma'],
            [
                'name'            => 'María',
                'apellidos'       => 'Quispe Mamani',
                'email'           => 'registrador@hornoma.bo',
                'password'        => Hash::make(env('SEED_REGISTRADOR_PASS', 'Cambiar123!')),
                'centro_salud_id' => $centro?->id,
                'activo'          => true,
            ]
        );
        $registrador->assignRole('registrador');
    }
}
Seguridad: contraseñas desde .env

Las contraseñas se leen con env('SEED_SUPERADMIN_PASS', 'Cambiar123!'). El valor por defecto 'Cambiar123!' es solo para desarrollo. En producción, configura las variables en .env:

.env (variables de seeder)
SEED_SUPERADMIN_PASS=TuContraseñaSegura1!
SEED_ADMIN_PASS=OtraContraseña2!
SEED_REGISTRADOR_PASS=OtraContraseña3!
Los 3 usuarios iniciales
usuarioRolcentro_salud_idPropósito
superadminsuperadminnullAcceso total, sin centro fijo
medico.hornomaadminHORNOMAMédico responsable, ve solo su centro

¿Por qué superadmin tiene centro_salud_id = null?

Porque el superadmin puede ver todos los centros de salud. Los filtros Eloquent (getEloquentQuery) omiten el WHERE centro_salud_id cuando el usuario es superadmin (lección 19). Los otros roles tienen un centro asignado y solo ven datos de ese centro.

9 Ejecutar los seeders

Hay varias formas de ejecutar los seeders:

Terminal — ejecutar todos los seeders base
# Ejecutar DatabaseSeeder (los 6 seeders de la cadena)
php artisan db:seed

# Ejecutar un seeder específico
php artisan db:seed --class=RolesSeeder

# Borrar todo y recrear (migrate:fresh + seed)
php artisan migrate:fresh --seed

# Ejecutar DatosPruebaSeeder (separado, manual)
php artisan db:seed --class=DatosPruebaSeeder
migrate:fresh borra todo

migrate:fresh --seed elimina todas las tablas y las recrea. Úsalo solo en desarrollo. En producción, nunca ejecutes migrate:fresh.

10 Resumen

Lo que aprendimos

  • DatabaseSeeder orquesta 6 seeders en orden de dependencia
  • Todos usan firstOrCreate — ejecución idempotente (seguro repetir)
  • RolesSeeder → 3 roles: superadmin, admin, registrador
  • MunicipioSeeder → Capinota, Cochabamba
  • CentroSaludSeeder → HORNOMA (código SNIS 300183, población 641)
  • ComunidadSeeder → 8 comunidades con distancias (0-72 km)
  • MetaIneSeeder → 25 entradas poblacionales para cálculos de cobertura
  • UserSeeder → 3 usuarios con roles y contraseñas desde .env
  • DatosPruebaSeeder queda aparte — se ejecuta manualmente (lección 23)
