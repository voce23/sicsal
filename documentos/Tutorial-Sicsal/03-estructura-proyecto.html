1 Visión general de las carpetas

Abre VS Code en C:\laragon\www\sicsal y observa la estructura. Estas son las carpetas principales de un proyecto Laravel:

sicsal/
  • app/ — Tu código PHP (modelos, controladores, livewire, etc.)
  • bootstrap/ — Arranque de la aplicación y caché
  • config/ — Archivos de configuración
  • database/ — Migraciones, seeders y factories
  • public/ — Punto de entrada web (index.php, assets compilados)
  • resources/ — Vistas Blade, CSS, JavaScript sin compilar
  • routes/ — Definición de rutas (URLs)
  • storage/ — Logs, caché, archivos subidos
  • tests/ — Tests automatizados
  • vendor/ — Dependencias de Composer (no tocar)
  • artisan — Herramienta CLI de Laravel
  • composer.json — Manifiesto de dependencias PHP
  • package.json — Manifiesto de dependencias npm
  • .env — Variables de entorno (no se sube a Git)
  • vite.config.js — Configuración del bundler
ℹ️
Regla de oro
app/ = tu código · vendor/ = código de terceros (nunca lo edites, se sobreescribe con cada composer install).

2 La carpeta app/ — tu código

Aquí es donde vivirá el 90% del código de SICSAL. Veamos qué contiene ahora (recién creada) y qué agregaremos durante el tutorial:

app/
  • Actions/ — Acciones de Fortify (crear usuario, reset password)
  • Console/ — Comandos artisan personalizados
  • Concerns/ — Traits compartidos (HasCentroSelector, etc.)
  • Exports/ — Clases de exportación Excel/PPT
  • Filament/ — Recursos, páginas y widgets del panel admin
  • Helpers/ — Funciones utilitarias (CaiHelper, OmsHelper)
  • Http/ — Controladores y middleware
  • Livewire/ — Componentes Livewire (Blog, InformeCAI, etc.)
  • Models/ — Modelos Eloquent (37 en total)
  • Observers/ — Observers que sincronizan datos automáticamente
  • Providers/ — Service providers
  • Services/ — Servicios complejos (VesMdbReader)
💡
Carpetas que Laravel no crea por defecto
Las carpetas Concerns/, Exports/, Helpers/, Livewire/, Observers/ y Services/ las crearemos nosotros a medida que avancemos. Laravel solo crea Http/, Models/, Providers/ y Console/ automáticamente.

Subcarpetas de app/Filament/ (panel admin)

Filament organiza su código así:

app/Filament/
  • Pages/ — Páginas personalizadas (Prestaciones, Reportes, Importar)
  • Resources/ — CRUD de entidades (Persona, CentroSalud, etc.)
  • Widgets/ — Widgets del dashboard (alertas, coberturas)

3 La carpeta database/

Aquí definimos la estructura de la base de datos con código PHP, sin escribir SQL directamente:

database/
  • factories/ — Factories para generar datos de prueba
  • migrations/ — 50 archivos que crean/modifican tablas
  • seeders/ — 8 archivos que insertan datos iniciales
Concepto Qué hace Ejemplo
Migración Crea o modifica una tabla create_personas_table.php
Seeder Inserta datos iniciales MunicipioSeeder.php (190 municipios)
Factory Genera datos falsos para tests UserFactory.php
ℹ️
¿Por qué migraciones y no SQL directo?
Las migraciones son versionables (van en Git), reversibles (se pueden deshacer) y portables (funcionan en cualquier motor de BD). Todo el equipo comparte la misma estructura solo con php artisan migrate.

4 La carpeta resources/

Contiene todo lo visual que no es PHP: vistas, CSS, JavaScript.

resources/
  • css/ — Hojas de estilo (app.css, theme para Filament)
  • js/ — JavaScript (app.js, chart.js imports)
  • views/ — 59 archivos Blade (.blade.php)

¿Qué es Blade?

Blade es el motor de plantillas de Laravel. Los archivos .blade.php mezclan HTML con directivas especiales:

Ejemplo Blade blade 
{{-- Esto es un comentario Blade --}}
<h1>Bienvenido, {{ $usuario->nombre }}</h1>

@if($usuario->hasRole('admin'))
    <p>Tienes acceso al panel.</p>
@endif

@foreach($personas as $p)
    <li>{{ $p->nombre_completo }}</li>
@endforeach

Las directivas principales son:

Directiva Qué hace
{{ $var }}Imprime con escape HTML (seguro)
{!! $html !!}Imprime sin escape (cuidado: solo para HTML confiable)
@if / @else / @endifCondicionales
@foreach / @endforeachBucles
@extends('layout')Hereda de un layout
@section / @yieldSecciones de contenido
@livewire('componente')Inserta un componente Livewire

5 La carpeta routes/

Define las URLs de la aplicación. SICSAL tiene 3 archivos de rutas:

routes/
  • web.php — Rutas principales (páginas públicas, descargas, importar)
  • console.php — Comandos artisan personalizados
  • settings.php — Rutas de configuración de usuario (perfil, contraseña)

Ejemplo de una ruta:

routes/web.php (ejemplo simplificado) php 
use Illuminate\Support\Facades\Route;

// Cuando el usuario visita http://sicsal.test/
Route::get('/', function () {
    return view('inicio');  // Muestra resources/views/inicio.blade.php
});
💡
Rutas de Filament
Filament registra sus propias rutas automáticamente bajo /admin. No necesitas definirlas en web.php. Las páginas como CentroSalud, Persona, etc. se acceden en http://sicsal.test/admin/centros-salud.

6 La carpeta config/

Archivos que controlan el comportamiento de Laravel y sus paquetes. Los valores se leen del .env:

config/
  • app.php — Nombre, timezone, locale
  • auth.php — Configuración de autenticación
  • database.php — Conexiones de BD (MySQL, SQLite, etc.)
  • filesystems.php — Discos de almacenamiento
  • fortify.php — Configuración de Fortify (2FA, email verification)
  • mail.php — Configuración de correo
  • permission.php — Configuración de Spatie Permission
  • session.php — Sesiones

Patrón típico de un config: lee del .env con un valor por defecto:

config/app.php (fragmento) php 
// Lee APP_NAME del .env; si no existe, usa 'Laravel'
'name' => env('APP_NAME', 'Laravel'),

// Lee APP_DEBUG del .env; si no existe, usa false
'debug' => (bool) env('APP_DEBUG', false),

7 La carpeta bootstrap/

Contiene el código de arranque de la aplicación. Casi nunca la editaremos directamente.

bootstrap/
  • app.php — Configura rutas, middleware, excepciones
  • providers.php — Lista de Service Providers
  • cache/ — Caché de configuración y rutas (auto-generada)

El archivo bootstrap/providers.php es donde se registran los providers de SICSAL:

bootstrap/providers.php php 
return [
    App\Providers\AppServiceProvider::class,
    App\Providers\Filament\AdminPanelProvider::class,
    App\Providers\FortifyServiceProvider::class,
];

8 La carpeta public/

Es el document root del servidor web. Apache apunta aquí. Todo lo que debe ser accesible por URL va en public/.

public/
  • index.php — Punto de entrada (recibe TODAS las peticiones)
  • robots.txt — Instrucciones para buscadores
  • build/ — CSS y JS compilados por Vite
  • css/ — CSS estáticos
  • fonts/ — Fuentes tipográficas
  • js/ — JS estáticos
  • storage → symlink a storage/app/public
⚠️
Seguridad
Nunca pongas .env, modelos, o código PHP en public/. Solo index.php y assets estáticos. Todo lo demás queda inaccesible al navegador gracias a que Apache apunta a public/, no a la raíz del proyecto.

9 Artisan: la herramienta de línea de comandos

artisan es el CLI integrado de Laravel. Lo usaremos constantemente. Algunos comandos clave:

Terminal bash 
# Ver todos los comandos disponibles
$ php artisan list

# Ejecutar migraciones (crear tablas en la BD)
$ php artisan migrate

# Revertir la última migración
$ php artisan migrate:rollback

# Ejecutar seeders (insertar datos iniciales)
$ php artisan db:seed

# Crear un modelo con migración
$ php artisan make:model Persona -m

# Limpiar toda la caché
$ php artisan optimize:clear

# Consola interactiva (tinker)
$ php artisan tinker
Comando Uso en SICSAL Lección
migrateCrear las 50 tablas06-11
db:seedInsertar municipios, centros, roles22-23
make:modelCrear los 37 modelos12-16
make:filament-resourceCrear recursos del panel admin25-31
make:livewireCrear componentes Livewire45-50
optimize:clearLimpiar caché tras cambiosSiempre

10 Ciclo de vida de una petición

Cuando un usuario visita http://sicsal.test/admin/personas, esto es lo que sucede:

1
Navegador envía petición HTTP
↓
2
Apache recibe → public/index.php
↓
3
Laravel bootstrap → carga .env, config, providers
↓
4
Router busca la ruta → Filament la resuelve
↓
5
Middleware: auth, permisos, centro_salud
↓
6
PersonaResource → consulta Persona::query()
↓
7
MySQL devuelve datos → modelo Persona
↓
8
Blade renderiza tabla HTML
↓
9
Respuesta HTTP → navegador muestra la página
