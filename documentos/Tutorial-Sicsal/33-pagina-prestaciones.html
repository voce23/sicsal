2 Prestaciones.php — Filament Page con getViewData()

app/Filament/Pages/Prestaciones.php
class Prestaciones extends Page
{
    protected string $view = 'filament.pages.prestaciones';
    protected static string|\UnitEnum|null $navigationGroup
        = 'Prestaciones Mensuales';
    protected static ?string $title = 'Prestaciones Mensuales';
    protected static ?int $navigationSort = 1;

    public int $anio;

    public function mount(): void
    {
        $this->anio = (int) date('Y');
    }

    public function getViewData(): array
    {
        $centroId = auth()->user()->centro_salud_id;

        // ① Meses que ya se cerraron
        $mesesCerrados = MesCerrado::where(...)
            ->pluck('mes')->toArray();

        // ② Meses que tienen ALGÚN dato en 16 tablas prest_*
        $mesesConDato = collect();
        foreach ([
            PrestConsultaExterna::class,
            PrestVacuna::class,
            PrestPrenatal::class,
            // ... 13 modelos más ...
            CausaConsultaExterna::class,
        ] as $model) {
            $mesesConDato = $mesesConDato->merge(
                $model::where('centro_salud_id', $centroId)
                    ->where('anio', $this->anio)
                    ->distinct()->pluck('mes')
            );
        }
        $mesesConDato = $mesesConDato->unique()->toArray();

        // ③ Construir array de 12 meses
        for ($m = 1; $m <= 12; $m++) {
            $cerrado = in_array($m, $mesesCerrados);
            $tieneAlgunDato = in_array($m, $mesesConDato);

            $estado = $cerrado
                ? 'cerrado'
                : ($tieneAlgunDato ? 'parcial' : 'vacio');

            $params = ['mes' => $m, 'anio' => $this->anio];

            $meses[$m] = [
                'numero' => $m,
                'nombre' => $nombresMeses[$m],
                'estado' => $estado,
                'cerrado' => $cerrado,
                'urls' => [
                    ['label' => 'PAI',
                     'url' => FormularioPai::getUrl($params),
                     'icon' => 'heroicon-o-shield-check'],
                    ['label' => 'Materna', ...],
                    ['label' => 'Servicios', ...],
                    ['label' => 'Causas Consulta', ...],
                ],
            ];
        }

        return ['meses' => $meses, 'anio' => $this->anio];
    }
}
getViewData() — pasar datos a Blade

En Filament Pages, getViewData() retorna un array que se pasa automáticamente a la vista. Equivale a return view('...', [...]); en controladores. Las variables quedan disponibles como $meses y $anio en el Blade.

Detección de estado en 16 tablas

El loop itera sobre 16 modelos Prest* y hace un distinct()->pluck('mes') en cada uno. Si cualquier tabla tiene datos para un mes, ese mes se marca como "parcial". Si está en MesCerrado, es "cerrado".

4 FormularioMensualTrait — trait compartido

Los 4 formularios (PAI, Materna, Servicios, CausasConsulta) usan este trait. Define funcionalidad común:

app/Filament/Pages/Concerns/FormularioMensualTrait.php — propiedades
trait FormularioMensualTrait
{
    #[Url]
    public int $mes = 1;

    #[Url]
    public int $anio = 0;

    public bool $mesCerrado = false;

    // Para el modal de justificación de cero
    public string $justIndicador = '';
    public string $justMotivo = '';
    public string $justDetalle = '';

    public function getTitle(): string
    {
        $seccion = static::SECCION ?? 'Prestaciones';
        return $seccion . ' — ' . self::$nombresMeses[$this->mes]
            . ' ' . $this->anio;
    }

    public function mountFormulario(): void
    {
        $this->mes = (int) request()->query('mes', date('n'));
        $this->anio = (int) request()->query('anio', date('Y'));

        $this->mesCerrado = MesCerrado::where(...)
            ->exists();
    }
}
#[Url] — Livewire URL binding

El atributo #[Url] sincroniza la propiedad con la URL. Al acceder a /formulario-pai?mes=3&anio=2025, las propiedades $mes y $anio se hidratan automáticamente. Así el enlace desde las tarjetas de Prestaciones pasa los parámetros.

5 Justificación de cero — modal y persistencia

FormularioMensualTrait.php — justificación
public function abrirJustificacion(string $indicador): void
{
    $this->justIndicador = $indicador;
    $this->justMotivo = '';
    $this->justDetalle = '';
    $this->dispatch('open-modal', id: 'justificacion-cero');
}

public function guardarJustificacion(): void
{
    $this->validate([
        'justMotivo' => 'required',
    ]);

    JustificacionCero::updateOrCreate(
        [
            'centro_salud_id' => auth()->user()->centro_salud_id,
            'mes' => $this->mes,
            'anio' => $this->anio,
            'indicador' => $this->justIndicador,
        ],
        [
            'motivo' => $this->justMotivo,
            'detalle' => $this->justMotivo === 'otro'
                ? $this->justDetalle : null,
            'registrado_por' => auth()->id(),
        ]
    );

    $this->dispatch('close-modal', id: 'justificacion-cero');
    Notification::make()->title('Justificación de cero registrada')
        ->success()->send();
}
Regla de negocio: cero justificado

Cuando un indicador tiene valor 0, el formulario muestra un botón "Justificar" que abre un modal. El usuario selecciona un motivo (migración, no hubo demanda, etc.) y se guarda en justificaciones_cero. Es obligatorio antes de cerrar el mes — requerimiento del SNIS.

6 Cierre y reapertura de mes

FormularioMensualTrait.php — cerrar/reabrir
public function cerrarMes(): void
{
    $user = auth()->user();
    if (!$user->hasAnyRole(['superadmin', 'admin'])) {
        return;
    }

    MesCerrado::updateOrCreate(
        ['centro_salud_id' => $user->centro_salud_id,
         'mes' => $this->mes, 'anio' => $this->anio],
        ['cerrado_por' => $user->id, 'fecha_cierre' => now()]
    );
    $this->mesCerrado = true;
}

public function reabrirMes(): void
{
    $mc = MesCerrado::where(...)->first();
    if ($mc) {
        $mc->update(['reabierto_por' => $user->id,
                      'fecha_reapertura' => now()]);
        $mc->delete();
    }
    $this->mesCerrado = false;
}

Header Actions — botones condicionales

FormularioMensualTrait.php — getHeaderActions()
protected function getHeaderActions(): array
{
    $actions = [];

    if ($user->hasAnyRole(['superadmin', 'admin'])) {
        if ($this->mesCerrado) {
            $actions[] = Action::make('reabrirMes')
                ->label('Reabrir mes')
                ->icon(Heroicon::OutlinedLockOpen)
                ->color('warning')
                ->requiresConfirmation()
                ->modalHeading('¿Reabrir este mes?')
                ->modalDescription('Esto permitirá editar los datos.')
                ->action(fn () => $this->reabrirMes());
        } else {
            $actions[] = Action::make('cerrarMes')
                ->label('Cerrar mes')
                ->icon(Heroicon::OutlinedLockClosed)
                ->color('success')
                ->requiresConfirmation();
        }
    }

    $actions[] = Action::make('volver')
        ->label('Volver al selector')
        ->url(Prestaciones::getUrl());

    return $actions;
}
requiresConfirmation() — modal de confirmación

Filament muestra automáticamente un modal "¿Estás seguro?" con los textos de modalHeading y modalDescription. Solo los admins ven el botón de cerrar/reabrir — los registradores solo ven "Volver al selector".
