1 ¿Qué es Fortify y por qué lo usamos?

Laravel Fortify es un backend de autenticación headless: registra rutas, maneja lógica de login/registro/reset, pero no incluye vistas. Tú agregas tu propio frontend (en nuestro caso, Blade + Tailwind).

💡
¿Por qué Fortify y no Breeze o Jetstream?
SICSAL usa Filament como panel admin. Fortify ofrece máximo control sin imponer un diseño — ideal cuando la UI de autenticación es 100% personalizada.

2 config/fortify.php — todas las opciones

config/fortify.php php 
return [
    'guard'     => 'web',        // guard de autenticación
    'passwords' => 'users',      // broker de passwords
    'username'  => 'email',      // campo para login
    'email'     => 'email',      // campo para reset

    'lowercase_usernames' => true,  // normaliza emails a minúsculas

    'home'       => '/dashboard', // redirección post-login
    'prefix'     => '',           // prefijo de rutas (vacío = /login, /register)
    'domain'     => null,         // subdominio (no usado)
    'middleware' => ['web'],       // middleware group
    'views'      => true,         // registra rutas de vistas

    'limiters' => [
        'login'      => 'login',      // 5 intentos/minuto
        'two-factor' => 'two-factor', // 5 intentos/minuto
    ],
];
ℹ️
lowercase_usernames
Cuando true, Fortify convierte Juan@Gmail.COM a juan@gmail.com antes de buscar en la DB. Evita problemas de case-sensitivity.

3 Features habilitadas

config/fortify.php — features php 
'features' => [
    Features::registration(),
    Features::resetPasswords(),
    Features::emailVerification(),
    Features::twoFactorAuthentication([
        'confirm'         => true,  // confirma código al activar
        'confirmPassword' => true,  // pide password antes de activar 2FA
    ]),
],
Feature Rutas que registra Propósito en SICSAL
registration() GET/POST /register Registro público, pero cuenta inactiva hasta aprobación
resetPasswords() GET/POST /forgot-password, /reset-password Recuperación vía email
emailVerification() GET /email/verify, /email/verify/{id}/{hash} Verificación del email del usuario
twoFactorAuthentication() POST /two-factor-*, GET /two-factor-challenge 2FA con TOTP (Google Authenticator, etc.)
⚠️
registration() habilitado ≠ libre acceso
Aunque registration() está activo, SICSAL crea usuarios con activo = false. El admin debe aprobar cada cuenta manualmente. El usuario registrado ve una pantalla de "pendiente de aprobación".

4 FortifyServiceProvider — el cerebro

El provider se registra en bootstrap/providers.php:

bootstrap/providers.php php 
return [
    App\Providers\AppServiceProvider::class,
    App\Providers\Filament\AdminPanelProvider::class,
    App\Providers\FortifyServiceProvider::class,
];

Y su método boot() delega en 3 métodos privados:

app/Providers/FortifyServiceProvider.php — boot php 
public function boot(): void
{
    $this->configureActions();      // CreateNewUser, ResetUserPassword
    $this->configureViews();       // 7 vistas personalizadas
    $this->configureRateLimiting(); // login + 2FA throttling
}

private function configureActions(): void
{
    Fortify::createUsersUsing(CreateNewUser::class);
    Fortify::resetUserPasswordsUsing(ResetUserPassword::class);
}

5 Responses personalizados (Login y Register)

El método register() del provider personaliza las respuestas de Fortify:

app/Providers/FortifyServiceProvider.php — register() php 
public function register(): void
{
    // Login: si usuario NO está activo → /pendiente
    $this->app->singleton(
        LoginResponse::class,
        function () {
            return new class implements LoginResponse {
                public function toResponse($request)
                {
                    $redirect = $request->user()->activo
                        ? config('fortify.home')    // → /dashboard
                        : route('pendiente');     // → /pendiente

                    return $request->wantsJson()
                        ? response()->json(['two_factor' => false])
                        : redirect()->intended($redirect);
                }
            };
        }
    );

    // Register: siempre → /pendiente (la cuenta nace inactiva)
    $this->app->singleton(
        RegisterResponse::class,
        function () {
            return new class implements RegisterResponse {
                public function toResponse($request)
                {
                    return redirect()->route('pendiente');
                }
            };
        }
    );
}
Usuario se registra
→
CreateNewUser (activo=false)
→
RegisterResponse → /pendiente
→
Admin aprueba (activo=true)
→
Login → LoginResponse → /dashboard

6 Vistas personalizadas

FortifyServiceProvider — configureViews() php 
private function configureViews(): void
{
    Fortify::loginView(
        fn() => view('pages::auth.login'));
    Fortify::registerView(
        fn() => view('pages::auth.register'));
    Fortify::verifyEmailView(
        fn() => view('pages::auth.verify-email'));
    Fortify::twoFactorChallengeView(
        fn() => view('pages::auth.two-factor-challenge'));
    Fortify::confirmPasswordView(
        fn() => view('pages::auth.confirm-password'));
    Fortify::resetPasswordView(
        fn() => view('pages::auth.reset-password'));
    Fortify::requestPasswordResetLinkView(
        fn() => view('pages::auth.forgot-password'));
}
ℹ️
Notación pages::
pages::auth.login usa el namespace pages registrado en el AdminPanelProvider de Filament. Son las mismas vistas Blade del directorio resources/views/ pero bajo el namespace del panel.

7 Rate limiting

FortifyServiceProvider — configureRateLimiting() php 
private function configureRateLimiting(): void
{
    // Login: 5 intentos por minuto, por email+IP
    RateLimiter::for('login', function (Request $request) {
        $throttleKey = Str::transliterate(
            Str::lower($request->input(Fortify::username())) 
            . '|' . $request->ip()
        );
        return Limit::perMinute(5)->by($throttleKey);
    });

    // 2FA: 5 intentos por minuto, por sesión
    RateLimiter::for('two-factor', function (Request $request) {
        return Limit::perMinute(5)
            ->by($request->session()->get('login.id'));
    });
}
⚠️
Str::transliterate — seguridad con caracteres especiales
Str::transliterate() convierte caracteres acentuados a ASCII antes de hacer la key del throttle. Esto evita que alguien evada el rate limit usando variantes Unicode del mismo email.

✅ Resumen de la Lección 17

  • Fortify = backend headless de autenticación (sin vistas propias)
  • 4 features habilitadas: registration, resetPasswords, emailVerification, 2FA
  • LoginResponse personalizado: usuarios inactivos → /pendiente
  • RegisterResponse: siempre → /pendiente (cuenta nace inactiva)
  • 7 vistas Blade personalizadas mapeadas vía Fortify::*View()
  • Rate limiting: 5 intentos/min para login y 2FA
  • Todo se conecta desde FortifyServiceProvider registrado en bootstrap/providers.php
