1 Páginas custom vs Resources

Resource (Lección 08)

Para operaciones CRUD estándar: listar, crear, editar, eliminar.

Ejemplo: PersonaResource, CentroSaludResource.

Página Custom (esta lección)

Para lógica especializada que no encaja en un CRUD.

Ejemplo: Prestaciones mensuales, importar .ves, reportes.

2 Prestaciones: el selector de meses

Esta página muestra los 12 meses del año, cada uno con su estado y enlaces a los formularios:

app/Filament/Pages/Prestaciones.php — Estructura
class Prestaciones extends Page
{
    // Vista Blade personalizada (no una tabla de Filament)
    protected string $view = 'filament.pages.prestaciones';

    protected static $navigationIcon  = Heroicon::OutlinedClipboardDocumentList;
    protected static $navigationGroup = 'Prestaciones Mensuales';
    protected static $navigationLabel = 'Prestaciones';

    public int $anio;

    public function mount(): void
    {
        $this->anio = (int) date('Y');
    }
}
Prestaciones.php — getViewData() determina el estado de cada mes
public function getViewData(): array
{
    $centroId = auth()->user()->centro_salud_id;

    // ¿Qué meses están cerrados?
    $mesesCerrados = MesCerrado::where('centro_salud_id', $centroId)
        ->where('anio', $this->anio)->pluck('mes')->toArray();

    // ¿Qué meses tienen algún dato? (consultar TODAS las tablas prest_*)
    $mesesConDato = collect();
    foreach ([PrestVacuna::class, PrestPrenatal::class, ...] as $model) {
        $mesesConDato = $mesesConDato->merge(
            $model::where('centro_salud_id', $centroId)
                ->where('anio', $this->anio)
                ->distinct()->pluck('mes')
        );
    }

    // Construir datos para la vista
    for ($m = 1; $m <= 12; $m++) {
        $meses[$m] = [
            'nombre'  => $nombresMeses[$m],
            'estado'  => in_array($m, $mesesCerrados)
                ? 'cerrado'       // 🔒 Verde: datos completos
                : (in_array($m, $mesesConDato)
                    ? 'parcial'    // ⚠️ Amarillo: tiene datos
                    : 'vacio'),     // ⬜ Gris: sin datos
            'urls' => [
                ['label' => 'PAI',      'url' => FormularioPai::getUrl([...])],
                ['label' => 'Materna',  'url' => FormularioMaterna::getUrl([...])],
                ['label' => 'Servicios','url' => FormularioServicios::getUrl([...])],
            ],
        ];
    }

    return ['meses' => $meses];
}

3 FormularioMensualTrait: lógica compartida

Las 4 páginas de formularios mensuales comparten un trait con la lógica común:

app/Filament/Pages/Concerns/FormularioMensualTrait.php
trait FormularioMensualTrait
{
    #[Url]
    public int $mes = 1;     // Viene de ?mes=3

    #[Url]
    public int $anio = 0;    // Viene de ?anio=2026

    public bool $mesCerrado = false;

    // Props para la modal de justificación
    public string $justIndicador = '';
    public string $justMotivo = '';
    public string $justDetalle = '';

    public function getTitle(): string
    {
        return static::SECCION . ' — '
            . self::$nombresMeses[$this->mes]
            . ' ' . $this->anio;
        // "PAI — Marzo 2026"
    }

    public function mountFormulario(): void
    {
        $this->mes  = (int) request()->query('mes', date('n'));
        $this->anio = (int) request()->query('anio', date('Y'));

        $this->mesCerrado = MesCerrado::where(...)
            ->exists();
    }
}
¿Qué es un trait?

Un trait es una colección de métodos reutilizables. En vez de copiar las mismas funciones en 4 páginas diferentes, las pones en un trait y cada página hace use FormularioMensualTrait. Es como un "plugin" de funcionalidad.

4 Cero justificado

Regla de negocio del SNIS: si un indicador reporta cero, se debe justificar el motivo. El trait maneja esto:

FormularioMensualTrait.php — Justificación de cero
public function abrirJustificacion(string $indicador): void
{
    $this->justIndicador = $indicador;  // Ej: "bcg_dentro_m"
    $this->dispatch('open-modal', id: 'justificacion-cero');
}

public function guardarJustificacion(): void
{
    $this->validate(['justMotivo' => 'required']);

    JustificacionCero::updateOrCreate(
        [
            'centro_salud_id' => auth()->user()->centro_salud_id,
            'mes'             => $this->mes,
            'anio'            => $this->anio,
            'indicador'       => $this->justIndicador,
        ],
        [
            'motivo'          => $this->justMotivo,
            'registrado_por'  => auth()->id(),
        ]
    );

    Notification::make()->title('Justificación registrada')
        ->success()->send();
}
Regla SNIS: cero justificado

Si un centro de salud reporta 0 vacunas BCG en marzo, debe explicar por qué. Opciones: no hubo pacientes, falta de insumos, personal ausente, etc. Esto es obligatorio para el cierre del mes.

5 Cierre de mes

FormularioMensualTrait.php — Solo admin/superadmin
public function cerrarMes(): void
{
    $user = auth()->user();

    // Solo admin y superadmin pueden cerrar
    if (!$user->hasAnyRole(['superadmin', 'admin'])) {
        return;
    }

    MesCerrado::updateOrCreate(
        ['centro_salud_id' => $user->centro_salud_id,
         'mes' => $this->mes, 'anio' => $this->anio],
        ['cerrado_por' => $user->id, 'fecha_cierre' => now()]
    );

    $this->mesCerrado = true;
}

public function reabrirMes(): void
{
    // Misma verificación de roles...
    $mc = MesCerrado::where(...)
        ->first();

    if ($mc) {
        $mc->update(['reabierto_por' => $user->id]);
        $mc->delete();
    }
    $this->mesCerrado = false;
}

6 Widgets del dashboard

Los widgets son componentes que aparecen en el dashboard de Filament. SICSAL tiene 4:

app/Filament/Widgets/AlertasWidget.php
class AlertasWidget extends Widget
{
    protected string $view = 'filament.widgets.alertas-widget';

    protected int|string|array $columnSpan = 'full';  // Ancho completo

    protected static ?int $sort = 2;  // Orden en el dashboard

    public function getDatosProperty(): array
    {
        $centroId = auth()->user()->centro_salud_id;
        $mesActual = (int) date('n');

        // Buscar meses que deberían estar cerrados pero no lo están
        $mesesCerrados = MesCerrado::where(...)
            ->pluck('mes')->toArray();

        $mesesSinCerrar = [];
        for ($m = 1; $m < $mesActual; $m++) {
            if (!in_array($m, $mesesCerrados)) {
                $mesesSinCerrar[] = $nombresMeses[$m];
            }
        }

        return [
            'mesesSinCerrar' => $mesesSinCerrar,
            'totalPersonas'  => Persona::where(...)->count(),
        ];
    }
}

7 Widget de coberturas

Calcula coberturas comparando datos reportados vs metas INE:

app/Filament/Widgets/CoberturaMensual.php — Cálculo
public function getDatosProperty(): array
{
    // Meta mensual = meta anual / 12
    $metaBcg = $metasIne
        ->where('grupo_etareo', 'nacimientos_esperados')
        ->sum('cantidad') / 12;

    // Datos reales del mes
    $vacunasBcg = PrestVacuna::where(...)
        ->where('tipo_vacuna', 'BCG')
        ->selectRaw('SUM(dentro_m + dentro_f + fuera_m + fuera_f) as total')
        ->value('total') ?? 0;

    // Cobertura = (atendidos / meta) × 100
    return [
        ['nombre' => 'BCG',       'meta' => $metaBcg,      'atendidos' => $vacunasBcg],
        ['nombre' => 'Prenatal',  'meta' => $metaPrenatal, 'atendidos' => $prenatales],
        ['nombre' => 'Crecimiento','meta' => $metaCrecimiento,'atendidos' => $crecimiento],
    ];
}
Fórmula de cobertura

Cobertura = (atendidos / meta INE mensual) × 100. Si la meta anual de BCG es 12 y en marzo se vacunaron 2, la cobertura es 2 / (12/12) × 100 = 200%. Esto indica que ese mes se vacunó el doble de lo esperado.

Resumen de la Lección 10

  • Páginas custom se usan para lógica que no encaja en un CRUD estándar.
  • Prestaciones es un selector de meses que muestra estado: vacío, parcial o cerrado.
  • FormularioMensualTrait comparte lógica de cierre de mes y cero justificado entre 4 páginas.
  • El cero justificado es una regla del SNIS: todo indicador en cero necesita explicación.
  • Solo admin/superadmin pueden cerrar y reabrir meses.
  • Widgets muestran alertas y coberturas en el dashboard: $sort controla el orden, $columnSpan el ancho.
