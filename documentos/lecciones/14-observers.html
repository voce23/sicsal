1 ¿Qué es un Observer?

Un Observer es una clase que "observa" los eventos de un modelo Eloquent. Cada vez que se crea, actualiza o elimina un registro, Laravel llama automáticamente al método correspondiente del Observer.

Eventos disponibles en un Observer
class MiObserver
{
    public function created(MiModelo $modelo)  {} // Después de INSERT
    public function updated(MiModelo $modelo)  {} // Después de UPDATE
    public function deleted(MiModelo $modelo)  {} // Después de DELETE
    public function creating(MiModelo $modelo) {} // Antes de INSERT
    public function updating(MiModelo $modelo) {} // Antes de UPDATE
}
¿Cuándo se ejecutan?

Los Observers se ejecutan automáticamente. No necesitas llamarlos manualmente. Si creas un registro con VacunaNino::create([...]), Laravel ejecutará VacunaNinoObserver::created() sin que lo pidas.

2 El problema: datos individuales vs. agregados

→ Observer →
Tabla agregada

Centro 5 • Marzo 2025 • BCG = 14

SICSAL maneja dos niveles de datos:

  1. Individual: Cada vacuna, control prenatal, parto, etc. de cada persona.
  2. Agregado (SNIS): Totales mensuales por centro de salud — lo que se reporta al ministerio.

El Observer es el puente: cada vez que se registra un dato individual, el Observer actualiza automáticamente el total agregado.

3 El trait SincronizaPrestacion

Los 6 Observers comparten lógica de clasificación a través de un trait:

app/Observers/SincronizaPrestacion.php
trait SincronizaPrestacion
{
    /**
     * Clasifica la edad en un grupo etáreo para vacunas.
     * Ejemplo: 3 meses → "menor_1", 18 meses → "12_23m"
     */
    protected function grupoEtareoVacuna(int $edadMeses): ?string
    {
        return match (true) {
            $edadMeses < 12  => 'menor_1',
            $edadMeses < 24  => '12_23m',
            $edadMeses < 60  => '24_59m',
            default          => null,
        };
    }

    /**
     * Clasifica la edad materna: 10–14, 15–19, 20+
     */
    protected function grupoEtareoMaterno(int $edad): string
    {
        return match (true) {
            $edad < 15 => '10_14',
            $edad < 20 => '15_19',
            default    => '20_mas',
        };
    }

    /**
     * Tipo de prenatal: nuevo, repetido, 4to+
     */
    protected function tipoPrenatal(int $numControl): string { ... }

    /**
     * Grupo de crecimiento infantil: RN, 1m, 2-5m, etc.
     */
    protected function grupoCrecimiento(int $edadMeses): string { ... }

    /**
     * Determina campo: dentro/fuera del área + sexo.
     * Ejemplo: dentro_m, fuera_f
     */
    protected function campoDentroFueraSexo(Persona $persona): string
    {
        $ubicacion = $persona->dentro_area ? 'dentro' : 'fuera';
        $sexo      = strtolower($persona->sexo);
        return "{$ubicacion}_{$sexo}";
    }
}
¿Por qué un trait?

Los 6 Observers necesitan las mismas funciones de clasificación. Un trait permite compartir código sin herencia. Cada Observer hace use SincronizaPrestacion; para tener todos los métodos disponibles.

4 VacunaNinoObserver: el patrón base

app/Observers/VacunaNinoObserver.php
class VacunaNinoObserver
{
    use SincronizaPrestacion;

    // Cuando se crea una vacuna → sumar +1
    public function created(VacunaNino $vacuna): void
    {
        $this->sincronizar($vacuna, +1);
    }

    // Cuando se elimina una vacuna → restar -1
    public function deleted(VacunaNino $vacuna): void
    {
        $this->sincronizar($vacuna, -1);
    }

    private function sincronizar(VacunaNino $vacuna, int $delta): void
    {
        // 1. Obtener datos de la persona (edad, sexo, área)
        $persona = $vacuna->persona;
        $grupo   = $this->grupoEtareoVacuna($vacuna->edad_meses);

        if (!$grupo) return; // Fuera de rango SNIS

        // 2. Buscar o crear el registro agregado del mes
        $prest = PrestVacuna::firstOrCreate([
            'centro_salud_id' => $persona->centro_salud_id,
            'anio'            => $vacuna->fecha->year,
            'mes'             => $vacuna->fecha->month,
            'tipo_vacuna'     => $vacuna->tipo,
            'grupo_etareo'    => $grupo,
        ]);

        // 3. Sumar o restar en el campo correcto
        $campo = $this->campoDentroFueraSexo($persona);
        $prest->increment($campo, $delta);
    }
}

5 PartoObserver: el más complejo

app/Observers/PartoObserver.php (simplificado)
class PartoObserver
{
    use SincronizaPrestacion;

    public function created(Parto $parto): void
    {
        // 1. Sincronizar prest_partos (tipo parto, grupo materno)
        $this->sincronizarParto($parto, +1);

        // 2. Sincronizar prest_recien_nacidos (peso, talla, Apgar)
        $this->sincronizarRecienNacido($parto, +1);

        // 3. ¡Crear BCG automáticamente!
        if ($parto->recien_nacido_vivo) {
            VacunaNino::create([
                'persona_id' => $parto->hijo_persona_id,
                'tipo'       => 'bcg',
                'fecha'      => $parto->fecha,
                'dosis'      => 1,
            ]);
            // Esto a su vez dispara VacunaNinoObserver::created() → ¡cadena!
        }
    }
}
Cascada de Observers

Cuando PartoObserver crea una VacunaNino, esto dispara VacunaNinoObserver::created(). Los Observers pueden formar cadenas. Esto es poderoso, pero hay que tener cuidado de no crear ciclos infinitos.

Resumen de la Lección 14

  • Los Observers escuchan eventos de modelos Eloquent (created, deleted) y ejecutan lógica automáticamente.
  • El trait SincronizaPrestacion provee clasificadores compartidos: grupo etáreo, tipo prenatal, dentro/fuera + sexo.
  • El patrón es: firstOrCreate (buscar o crear registro agregado) + increment/decrement (sumar/restar).
  • PartoObserver es el más complejo: sincroniza 2 tablas + crea BCG automáticamente (cascada de Observers).
  • SICSAL tiene 6 Observers que sincronizan datos individuales → 7+ tablas prest_* agregadas.
