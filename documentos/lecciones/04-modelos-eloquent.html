1 ¿Qué es un modelo Eloquent?

Un modelo es una clase PHP que representa una tabla de la base de datos. En vez de escribir consultas SQL, le hablas al modelo como si fuera un objeto de la vida real:

❌ SQL directo (difícil de mantener)
$result = DB::select(
  "SELECT * FROM personas WHERE activo = 1
   AND centro_salud_id = ?",
  [3]
);
✅ Eloquent (limpio y seguro)
$personas = Persona::activos()
  ->where('centro_salud_id', 3)
  ->get();
Analogía

Piensa en un modelo como una plantilla inteligente que sabe cómo hablar con su tabla. Si la tabla personas tiene 20 columnas, el modelo Persona sabe leerlas, escribirlas y conectar con otras tablas relacionadas.

2 Crear un modelo

Terminal
# Crear solo el modelo
$ php artisan make:model CentroSalud

# Crear modelo + migración (combo más usado)
$ php artisan make:model Persona -m

Laravel usa una convención de nombres para conectar modelos con tablas automáticamente:

Modelo (singular, PascalCase)Tabla (plural, snake_case)Automático?
Personapersonas✅ Sí
Userusers✅ Sí
VacunaNinovacuna_ninos❌ Necesita $table
CentroSaludcentro_saluds❌ Necesita $table
Cuando la convención no funciona

Laravel pluraliza en inglés. Los nombres en español a veces fallan: CentroSalud → centro_saluds (mal). Por eso definimos la tabla explícitamente con protected $table = 'centros_salud';

3 Propiedades fundamentales: fillable, casts, table

Anatomía de un modelo Eloquent
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class MiModelo extends Model
{
    // 1. Nombre de la tabla (si no sigue la convención)
    protected $table = 'mi_tabla_personalizada';

    // 2. Campos que se pueden llenar masivamente
    protected $fillable = [
        'nombre', 'email', 'activo',
    ];

    // 3. Conversiones automáticas de tipos
    protected function casts(): array
    {
        return [
            'fecha_nacimiento' => 'date',       // String → Carbon
            'activo'           => 'boolean',    // 0/1 → true/false
            'poblacion_ine'    => 'integer',    // String → int
        ];
    }
}
PropiedadQué hace¿Obligatorio?
$table Nombre exacto de la tabla en MySQL Solo si el nombre no sigue la convención
$fillable Lista de columnas que se pueden asignar masivamente (seguridad) Sí — siempre definirlo
casts() Convierte datos de la BD a tipos PHP automáticamente Recomendado para dates, booleans
¿Por qué $fillable es importante?

Sin $fillable, un atacante podría enviar campos extra (como is_admin=true) y modificar datos que no debería. $fillable actúa como una lista blanca — solo los campos listados se pueden asignar masivamente.

4 Relaciones: belongsTo y hasMany

Las relaciones conectan modelos entre sí. Las dos más comunes son:

belongsTo — "Pertenece a"

Una persona pertenece a un centro de salud. La tabla personas tiene la columna centro_salud_id.

Persona.php
public function centroSalud(): BelongsTo
{
    return $this->belongsTo(CentroSalud::class);
}

// Uso: $persona->centroSalud->nombre
hasMany — "Tiene muchos"

Un centro de salud tiene muchas personas. La relación inversa.

CentroSalud.php
public function personas(): HasMany
{
    return $this->hasMany(Persona::class);
}

// Uso: $centro->personas (colección)

Convención de nombres para la FK

Si el nombre de la relación es centroSalud(), Laravel busca automáticamente la columna centro_salud_id en la tabla. Si la columna tiene otro nombre, lo especificas como segundo parámetro:
$this->belongsTo(User::class, 'created_by')

5 Ejemplo real: CentroSalud

app/Models/CentroSalud.php — Modelo completo
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;

class CentroSalud extends Model
{
    protected $table = 'centros_salud';             // Nombre real de la tabla

    protected $fillable = [
        'municipio_id', 'nombre', 'codigo_snis',
        'subsector', 'red_salud', 'poblacion_ine',
        'activo', 'latitud', 'longitud',
    ];

    protected function casts(): array
    {
        return [
            'activo'        => 'boolean',
            'poblacion_ine' => 'integer',
            'latitud'       => 'float',
            'longitud'      => 'float',
        ];
    }

    // ─── Relaciones ───

    public function municipio(): BelongsTo          // Pertenece a un municipio
    {
        return $this->belongsTo(Municipio::class);
    }

    public function comunidades(): HasMany         // Tiene muchas comunidades
    {
        return $this->hasMany(Comunidad::class);
    }

    public function personas(): HasMany            // Tiene muchas personas
    {
        return $this->hasMany(Persona::class);
    }

    public function metasIne(): HasMany            // Tiene metas INE
    {
        return $this->hasMany(MetaIne::class);
    }

    public function usuarios(): HasMany            // Tiene usuarios asignados
    {
        return $this->hasMany(User::class);
    }
}

6 Ejemplo real: Persona (modelo completo)

El modelo Persona es el más completo de SICSAL. Tiene relaciones, scopes, accessors y mutators:

app/Models/Persona.php — Versión simplificada para enseñar
class Persona extends Model
{
    protected $fillable = [
        'centro_salud_id', 'comunidad_id', 'nombres', 'apellidos',
        'fecha_nacimiento', 'sexo', 'ci', 'tipo_seguro', 'estado',
        'activo', 'created_by', 'verificado',
        // ... más campos
    ];

    protected function casts(): array
    {
        return [
            'fecha_nacimiento' => 'date',      // Convierte a Carbon
            'activo'           => 'boolean',   // Convierte a true/false
            'verificado'       => 'boolean',
        ];
    }

    // ─── RELACIONES ───

    public function centroSalud(): BelongsTo
    {
        return $this->belongsTo(CentroSalud::class);
    }

    public function vacunas(): HasMany
    {
        return $this->hasMany(VacunaNino::class);
    }

    public function embarazos(): HasMany
    {
        return $this->hasMany(Embarazo::class);
    }

    public function defuncion(): HasOne             // Solo UNA defunción
    {
        return $this->hasOne(Defuncion::class);
    }
}
hasOne vs hasMany

hasOne significa que la relación es 1:1 — una persona solo puede tener una defunción. hasMany es 1:N — una persona puede tener muchas vacunas.

7 Scopes: filtros reutilizables

Un scope es un método que agrega condiciones WHERE a una consulta. En vez de repetir ->where('activo', true) en 20 lugares, defines un scope una sola vez:

app/Models/Persona.php — Scopes
// ─── DEFINIR el scope (en el modelo) ───

public function scopeActivos($query)
{
    return $query->where('activo', true);
}

public function scopeResidentes($query)
{
    return $query->where('estado', 'residente');
}

public function scopeMenoresDe5($query)
{
    return $query->where('fecha_nacimiento', '>=', now()->subYears(5));
}

public function scopePadronActivo($query)
{
    return $query->where('activo', true)
                 ->whereIn('estado', ['residente', 'temporal'])
                 ->where('verificado', true);
}
Usando scopes (en cualquier parte del código)
// Sin scope (verbose, repetitivo)
$personas = Persona::where('activo', true)
    ->where('estado', 'residente')
    ->get();

// Con scopes (limpio, reutilizable) ✅
$personas = Persona::activos()->residentes()->get();

// Los scopes se pueden encadenar
$ninos = Persona::padronActivo()->menoresDe5()->count();
// → "¿Cuántos niños menores de 5 hay en el padrón activo?"
Convención de nombres

El método se llama scopeNombreScope (con prefijo "scope"), pero se usa sin él: Persona::nombreScope() (minúscula).

8 Accessors y Mutators

Los accessors calculan valores al leer un dato. Los mutators transforman valores al guardar.

app/Models/Persona.php — Accessors (lectura)
use Carbon\Carbon;

// Accessor: calcula la edad a partir de fecha_nacimiento
public function getEdadAttribute(): int
{
    return Carbon::parse($this->fecha_nacimiento)->age;
}

// Uso: $persona->edad → 25 (calculado automáticamente)

// Accessor: nombre completo
public function getNombreCompletoAttribute(): string
{
    return "{$this->nombres} {$this->apellidos}";
}

// Uso: $persona->nombre_completo → "María López"
app/Models/Persona.php — Mutators (escritura)
use Illuminate\Database\Eloquent\Casts\Attribute;

// Mutator: convierte nombres a Title Case al guardar
protected function nombres(): Attribute
{
    return Attribute::make(
        set: fn (string $value) =>
            mb_convert_case(mb_strtolower($value), MB_CASE_TITLE, 'UTF-8'),
    );
}

// Efecto: si guardan "MARÍA" → se almacena como "María"
// Si guardan "juan carlos" → se almacena como "Juan Carlos"
¿Cuándo usar cada uno?
  • Accessor → para datos calculados que no existen en la tabla: edad, nombre completo, grupo etáreo calculado.
  • Mutator → para normalizar datos antes de guardar: capitalizar nombres, limpiar espacios extra.

Tinker es la consola interactiva de Laravel. Te permite probar modelos en tiempo real:

Terminal — php artisan tinker
$ php artisan tinker

>>> Persona::activos()->count()

# Buscar un centro de salud
>>> $centro = CentroSalud::find(1);
>>> $centro->nombre
=> "Centro de Salud Tacopaya"

# Ver las personas de un centro

=> "María López"
>>> $persona->edad
=> 3
>>> $persona->vacunas->count()

Resumen de la Lección 04

  • Un modelo es una clase PHP que representa una tabla de la BD.
  • $fillable protege contra asignación masiva, casts() convierte tipos automáticamente.
  • belongsTo = "pertenece a" (la tabla hija tiene el _id). hasMany = "tiene muchos".
  • Scopes son filtros reutilizables: Persona::activos()->residentes().
  • Accessors calculan datos al leer (edad), mutators transforman al guardar (nombres).
  • SICSAL tiene 37 modelos con un sistema dual: registros individuales + tablas agregadas SNIS.
