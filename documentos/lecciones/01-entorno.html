2 Instalar Laragon

Laragon es un entorno de desarrollo local para Windows que incluye Apache, PHP, MySQL y otras herramientas en un solo paquete. Es como un "servidor web portátil" que corre en tu computadora.

¿Por qué Laragon y no XAMPP?

Laragon es más ligero, crea automáticamente URLs bonitas (como sicsal.test en vez de localhost/sicsal/public), y facilita cambiar entre versiones de PHP.

1
Descargar Laragon

Ve a https://laragon.org/download/ y descarga la versión Full (incluye más herramientas).

2
Instalar

Ejecuta el instalador. La ubicación por defecto es C:\laragon. Puedes dejar todo por defecto.

3
Iniciar Laragon

Abre Laragon y haz clic en "Start All". Esto inicia Apache y MySQL.

4
Verificar PHP

Abre la terminal de Laragon (botón "Terminal") y escribe:

Terminal de Laragon
$ php -v

PHP 8.3.26 (cli) (built: ...)
Copyright (c) The PHP Group
¿Tu versión de PHP es menor a 8.3?

Laravel 13 requiere PHP 8.3 mínimo. Si Laragon trae una versión anterior, descarga PHP 8.3 desde https://windows.php.net/download/ y cópialo a C:\laragon\bin\php\php-8.3.xx. Luego en Laragon: clic derecho → PHP → versión 8.3.

4 Instalar Composer

Composer es el gestor de paquetes de PHP. Así como en tu teléfono usas la "Play Store" para instalar apps, en PHP usamos Composer para instalar librerías.

¿Qué es un paquete?

Un paquete es código que alguien más escribió y que puedes reutilizar. Por ejemplo, Laravel es un paquete. Filament es otro paquete. En vez de programar todo desde cero, usamos paquetes que resuelven problemas comunes.

1
Verificar si ya está instalado

Laragon normalmente incluye Composer. Abre la terminal de Laragon y escribe:

Terminal
$ composer --version

Composer version 2.8.x 2025-xx-xx
2
Si NO está instalado

Descarga el instalador desde https://getcomposer.org/download/ y ejecútalo. Asegúrate de que apunte al PHP de Laragon.

Cuando instalamos paquetes con Composer, se guardan en la carpeta vendor/ y se registran en composer.json. Nuestro proyecto SICSAL usa estos paquetes principales:

composer.json — Dependencias principales
{
    "require": {
        "php":                       "^8.3",         // PHP 8.3 mínimo
        "laravel/framework":          "^13.0",        // Laravel 13
        "filament/filament":          "~5.0",         // Panel de administración
        "livewire/livewire":          "^4.1",         // Componentes reactivos
        "spatie/laravel-permission":  "*",            // Roles y permisos
        "maatwebsite/excel":          "*",            // Exportar a Excel
        "barryvdh/laravel-dompdf":    "*",            // Generar PDFs
        "phpoffice/phppresentation":  "*"             // Presentaciones PowerPoint
    }
}
Leyendo las versiones
  • ^8.3 → "versión 8.3 o superior, pero menor a 9.0"
  • ~5.0 → "versión 5.0.x (solo parches)"
  • * → "cualquier versión disponible"

5 Instalar Node.js y NPM

Node.js es un entorno para ejecutar JavaScript fuera del navegador. NPM (Node Package Manager) viene incluido con Node y sirve para instalar paquetes de JavaScript, como Tailwind CSS y Vite.

1
Descargar Node.js

Ve a https://nodejs.org y descarga la versión LTS (Long Term Support). Al momento de escribir esta lección, es la versión 22.x.

2
Instalar y verificar
Terminal
$ node -v
v22.x.x

$ npm -v
10.x.x

Los paquetes JavaScript del proyecto se listan en package.json:

package.json — Dependencias principales
{
    "dependencies": {
        "vite":                 "^8.0.0",        // Empaquetador rápido
        "laravel-vite-plugin":  "^3.0.0",        // Integración Vite + Laravel
        "chart.js":             "^4.5.1"         // Gráficos en el dashboard
    },
    "devDependencies": {
        "tailwindcss":          "^4.2.2",        // Framework CSS utilitario
        "@tailwindcss/vite":    "^4.2.2"         // Plugin de Tailwind para Vite
    }
}
¿Qué es Vite?

Vite es una herramienta que toma tus archivos CSS y JavaScript, los procesa (compila Tailwind, minimiza código) y genera archivos optimizados para el navegador. Cuando desarrollas, Vite detecta cambios automáticamente y recarga el navegador al instante.

6 Crear el Proyecto Laravel

Ahora que tenemos todas las herramientas instaladas, vamos a crear el proyecto. Laravel ofrece un instalador que configura todo automáticamente.

1
Instalar el instalador de Laravel
Terminal de Laragon
$ composer global require laravel/installer
2
Crear el proyecto

Navega a la carpeta donde Laragon busca proyectos y crea uno nuevo:

Terminal
$ cd C:\laragon\www

$ laravel new sicsal

El instalador te preguntará qué starter kit usar. SICSAL usa Livewire Starter Kit.

3
Instalar dependencias

El instalador ejecuta composer install automáticamente. Pero si clonas un proyecto existente, necesitas hacerlo manualmente:

Terminal — Dentro de C:\laragon\www\sicsal
# Instalar paquetes PHP
$ composer install

# Instalar paquetes JavaScript
$ npm install
Si clonas el proyecto desde Git

Cuando clonas un repositorio, la carpeta vendor/ y node_modules/ NO vienen incluidas (están en .gitignore). Debes ejecutar composer install y npm install para descargar todas las dependencias.

7 Configurar la Base de Datos

Laravel guarda toda la configuración sensible en un archivo llamado .env que está en la raíz del proyecto. Este archivo NUNCA se sube al repositorio (está en .gitignore).

1
Crear la base de datos en MySQL

Abre HeidiSQL (viene con Laragon, botón "Database") o la terminal MySQL:

Terminal MySQL
mysql> CREATE DATABASE sicsal CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
2
Editar el archivo .env

Abre .env en VS Code y busca las líneas de la base de datos:

.env — Configuración de base de datos
APP_NAME=SICSAL
APP_ENV=local
APP_DEBUG=true
APP_URL=http://sicsal.test

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=sicsal
DB_USERNAME=root
DB_PASSWORD= (vacío en Laragon por defecto)
Regla de seguridad fundamental

NUNCA pongas contraseñas, tokens o credenciales directamente en archivos de código PHP. Siempre usa el archivo .env y accede a ellos con env('NOMBRE_VARIABLE'). Veremos esto en detalle en la Lección 16.

URL automática con Laragon

Laragon crea automáticamente la URL http://sicsal.test para cualquier carpeta en C:\laragon\www\sicsal. No necesitas configurar nada extra. Solo asegúrate de que Apache esté corriendo.

9 ¿Qué es cada herramienta? (Resumen)

HerramientaQué haceAnalogía

PHP 8.3 Lenguaje de programación del backend El idioma en que escribes las instrucciones
MySQL Base de datos relacional El archivero donde se guardan los datos

Resumen de la Lección 01

  • Instalamos Laragon como servidor local (Apache + PHP 8.3 + MySQL).
  • Instalamos Composer para gestionar paquetes PHP y Node.js/NPM para paquetes JavaScript.
  • Creamos el proyecto con laravel new sicsal usando el Livewire Starter Kit.
  • Configuramos la base de datos en .env (contraseñas NUNCA en código).
  • Verificamos con php artisan migrate, npm run dev y el navegador.
  • Aprendimos qué hace cada herramienta del stack.
