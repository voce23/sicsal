1 ¿Qué es un archivo .ves?

Un archivo comprimido disfrazado
→ expand.exe →

Access 97 Database
Base de datos con 56+ tablas
→ Import →
prest_*
MySQL / SICSAL
Datos organizados por centro/mes

El SNIS (Sistema Nacional de Información en Salud) de Bolivia usa archivos .ves para transmitir datos entre establecimientos de salud. Internamente, un .ves es un archivo Microsoft Cabinet (.cab) que contiene una base de datos Access 97 (.mdb).

¿Por qué Access 97?

El sistema SNIS fue creado hace décadas y usa tecnología antigua. Los archivos .mdb contienen todas las prestaciones de un municipio: vacunas, consultas externas, partos, controles prenatales, etc. Cada tabla usa códigos crípticos como G01, G02 que representan diferentes formularios SNIS.

2 Extraer el .ves → obtener .mdb

Terminal — Extraer archivo .ves
# Windows: usar expand.exe (viene con el sistema)
$ expand archivo.ves -F:* carpeta_destino/

# Linux: usar cabextract
$ cabextract -d carpeta_destino/ archivo.ves

Después de extraer, obtienes uno o más archivos .mdb. Estos archivos contienen tablas con datos codificados:

3 VesMdbReader: leer Access 97

La clase VesMdbReader resuelve un problema cross-platform: leer archivos .mdb tanto en Windows como en Linux.

app/Services/VesMdbReader.php — Estrategia dual
class VesMdbReader
{
    // Detectar sistema operativo y usar la estrategia correcta
    public function leerTabla(string $mdbPath, string $tabla): array
    {
        // Validar que la tabla esté en la whitelist
        if (!$this->tablaPermitida($tabla)) {
            throw new \InvalidArgumentException("Tabla no permitida: $tabla");
        }

        return PHP_OS_FAMILY === 'Windows'
            ? $this->leerConVbscript($mdbPath, $tabla)
            : $this->leerConMdbtools($mdbPath, $tabla);
    }
}
🪟 Windows: VBScript + DAO

Genera un script VBScript temporal que usa DAO.DBEngine.36 (motor nativo de Access) para abrir el .mdb y leer registros.

Ejecuta con cscript.exe (32-bit) y captura la salida JSON.

🐧 Linux: mdbtools

Usa el paquete mdbtools que incluye mdb-json para exportar tablas Access a JSON.

Es más limitado: WHERE y filtros se emulan en PHP con filtrarEnPhp().

4 Seguridad: whitelist de tablas

VesMdbReader.php — Validación de tablas
// Solo estas tablas pueden ser leídas
private const TABLAS_PERMITIDAS = [
    'G01_CE_M5',       // Consulta externa menores 5
    'G01_CE_5MAS',     // Consulta externa 5+
    'G02_REFERENCIA',  // Referencias
    'G03_PAI',         // Vacunas PAI
    'G04_PRENATAL',    // Control prenatal
    'G04_PARTO',       // Partos
    'G05_ANTICON',     // Anticoncepción
    // ... 56 entradas en total
];

public function tablaPermitida(string $tabla): bool
{
    // Primero: búsqueda exacta en la whitelist
    if (in_array($tabla, self::TABLAS_PERMITIDAS, true)) {
        return true;
    }

    // Fallback: patrón regex para variantes (G01_CE_M5_2024, etc.)
    return (bool) preg_match(
        '/^(G\d{2}_[A-Z0-9_]+)(_\d{4})?$/', $tabla
    );
}
¿Por qué una whitelist?

Porque el nombre de la tabla se pasa a shell_exec() (en el VBScript) o a un comando externo. Sin validación, un atacante podría inyectar código malicioso a través del nombre de tabla. La whitelist previene inyección de comandos.

5 ImportarSnis: el comando maestro

Terminal — Ejecutar importación SNIS
# Importar datos SNIS de archivos .mdb
$ php artisan snis:importar \
    --ruta="Punto-ves/extraido/" \
    --anio=2024 \
    --meses=1,2,3,4,5,6 \
    --limpiar   # Opcional: borrar datos existentes del período
app/Console/Commands/ImportarSnis.php — Estructura
class ImportarSnis extends Command
{
    protected $signature = 'snis:importar
        {--ruta= : Carpeta con archivos .mdb}
        {--anio= : Año de los datos}
        {--anio-destino= : Año destino (si difiere)}
        {--meses= : Meses a importar (1,2,3...)}
        {--limpiar : Borrar datos existentes}';

    public function handle(): int
    {
        // 1. Abrir conexión PDO ODBC al .mdb
        $dsn = "odbc:Driver={Microsoft Access Driver (*.mdb)};Dbq=$ruta";
        $pdo = new PDO($dsn);

        // 2. Mapear centros de salud por codigo_snis
        $centros = CentroSalud::pluck('id', 'codigo_snis');

        // 3. Por cada centro y cada mes: importar en transacción
        foreach ($centros as $codigo => $centroId) {
            foreach ($meses as $mes) {
                DB::transaction(function () use (...) {
                    $this->importarG01($pdo, $centroId, $mes); // Consulta ext.
                    $this->importarG03($pdo, $centroId, $mes); // Vacunas
                    $this->importarG04($pdo, $centroId, $mes); // Prenatal
                    $this->importarG05($pdo, $centroId, $mes); // Anticon.
                    // ... G06 a G09, G37, G38
                });
            }
        }
    }
}

6 Mapeo de códigos SNIS

Los archivos MDB usan códigos crípticos (codsubvar) para identificar cada indicador. ImportarSnis tiene ~20 constantes que traducen estos códigos a campos de SICSAL:

ImportarSnis.php — Constantes de mapeo (ejemplo: vacunas)
// codsubvar → campo en prest_vacunas
private const VAC_MENORES5_TIPO = [
    '030101' => 'bcg',
    '030201' => 'penta_1',
    '030202' => 'penta_2',
    '030203' => 'penta_3',
    '030301' => 'neumococo_1',
    '030302' => 'neumococo_2',
    '030303' => 'neumococo_3',
    '030401' => 'rotavirus_1',
    '030402' => 'rotavirus_2',
    // ... más códigos
];

// codsubvar → qué columnas leer del MDB
private const VAC_MENORES5_COL = [
    'dentro_m' => 'COL3',   // Columna 3 = dentro + masculino
    'dentro_f' => 'COL4',   // Columna 4 = dentro + femenino
    'fuera_m'  => 'COL5',   // Columna 5 = fuera + masculino
    'fuera_f'  => 'COL6',   // Columna 6 = fuera + femenino
];
¿Cómo funciona el mapeo?

El .mdb tiene filas con un codsubvar (ej: 030101) y columnas genéricas (COL3, COL4...). Las constantes traducen: 030101 = BCG, COL3 = varones dentro del área. Así se convierte el formato SNIS al esquema limpio de SICSAL.

Resumen de la Lección 15

  • Los archivos .ves son Microsoft Cabinet (.cab) que contienen bases de datos Access 97 (.mdb).
  • VesMdbReader lee .mdb en Windows (VBScript + DAO) o Linux (mdbtools), con whitelist de 56 tablas para seguridad.
  • ImportarSnis es un comando Artisan que mapea códigos SNIS (codsubvar) a campos SICSAL usando ~20 constantes.
  • La importación es transaccional: por cada centro y mes, todo se importa o nada.
  • Las tablas G01-G09, G37, G38 cubren consultas, vacunas, prenatal, partos, anticoncepción, odontología, crecimiento, micronutrientes, enfermedades y actividades comunitarias.
