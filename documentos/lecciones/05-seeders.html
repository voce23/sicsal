1 ¿Qué es un seeder?

Un seeder (del inglés "sembrar") es una clase PHP que inserta datos iniciales en la base de datos. Después de crear las tablas con migraciones, los seeders las "llenan" con datos que el sistema necesita para funcionar.

Analogía

Si las migraciones son los planos de la casa, los seeders son los muebles que pones adentro. Sin muebles, la casa existe pero no es funcional.

¿Cuándo usar seeders?
  • Datos obligatorios — Roles (superadmin, admin, registrador), que sin ellos el sistema no funciona.
  • Datos de referencia — Municipios, centros de salud, comunidades.
  • Usuarios iniciales — Al menos un superadmin para poder acceder por primera vez.
  • Datos de prueba — Datos ficticios para desarrollo y testing.

2 El orquestador: DatabaseSeeder

DatabaseSeeder es el seeder principal. No inserta datos por sí mismo — llama a otros seeders en el orden correcto:

database/seeders/DatabaseSeeder.php
<?php

namespace Database\Seeders;

use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    public function run(): void
    {
        $this->call([
            RolesSeeder::class,         // 1° Roles (no depende de nada)
            MunicipioSeeder::class,     // 2° Municipios
            CentroSaludSeeder::class,   // 3° Centros (depende de municipio)
            ComunidadSeeder::class,     // 4° Comunidades (depende de centro)
            MetaIneSeeder::class,       // 5° Metas INE (depende de centro)
            UserSeeder::class,          // 6° Usuarios (depende de centro + roles)
        ]);
    }
}

El orden importa

Los seeders se ejecutan de arriba a abajo. Si UserSeeder necesita asignar un rol y un centro de salud, entonces RolesSeeder y CentroSaludSeeder deben ejecutarse antes.

3 Primer seeder: RolesSeeder

SICSAL tiene 3 roles. Sin ellos, nadie puede hacer nada en el sistema:

database/seeders/RolesSeeder.php
<?php

namespace Database\Seeders;

use Illuminate\Database\Seeder;
use Spatie\Permission\Models\Role;

class RolesSeeder extends Seeder
{
    public function run(): void
    {
        Role::firstOrCreate(
            ['name' => 'superadmin', 'guard_name' => 'web']
        );
        Role::firstOrCreate(
            ['name' => 'admin',      'guard_name' => 'web']
        );
        Role::firstOrCreate(
            ['name' => 'registrador','guard_name' => 'web']
        );
    }
}
Rol¿Quién es?¿Qué puede hacer?
superadmin Administrador del sistema Todo: crear centros, usuarios, ver todos los datos
admin Médico/responsable del centro Gestionar datos de SU centro de salud
registrador Personal de estadística Registrar datos clínicos de SU centro

4 Seeders encadenados: Municipio → Centro → Comunidad

Estos tres seeders crean la estructura geográfica en orden de dependencia:

database/seeders/MunicipioSeeder.php
class MunicipioSeeder extends Seeder
{
    public function run(): void
    {
        Municipio::firstOrCreate(
            ['nombre' => 'Capinota'],                    // Buscar por nombre
            ['departamento' => 'Cochabamba', 'activo' => true]  // Crear si no existe
        );
    }
}
database/seeders/CentroSaludSeeder.php
class CentroSaludSeeder extends Seeder
{
    public function run(): void
    {
        // Primero buscar el municipio (ya fue creado por MunicipioSeeder)
        $municipio = Municipio::where('nombre', 'Capinota')->first();

        CentroSalud::firstOrCreate(
            ['codigo_snis' => '300183'],              // Buscar por código SNIS
            [
                'municipio_id'  => $municipio->id,     // FK al municipio
                'nombre'        => 'C.S.A. HORNOMA',
                'subsector'     => 'Público',
                'red_salud'     => 'Capinota',
                'poblacion_ine' => 641,
                'activo'        => true,
            ]
        );
    }
}
database/seeders/ComunidadSeeder.php — Usando un array con foreach
class ComunidadSeeder extends Seeder
{
    public function run(): void
    {
        $centro = CentroSalud::where('codigo_snis', '300183')->first();

        // Array con datos de todas las comunidades
        $comunidades = [
            ['nombre' => 'Hornoma',      'distancia_km' => 0],
            ['nombre' => 'Huaychoma',    'distancia_km' => 5],
            ['nombre' => 'Villcabamba',  'distancia_km' => 6],
            ['nombre' => 'Cocoma',       'distancia_km' => 24],
            ['nombre' => 'Tocohalla',    'distancia_km' => 48],
            ['nombre' => 'Challavilque', 'distancia_km' => 48],
            ['nombre' => 'Calacaja',     'distancia_km' => 72],
        ];

        foreach ($comunidades as $datos) {
            Comunidad::firstOrCreate(
                ['centro_salud_id' => $centro->id, 'nombre' => $datos['nombre']],
                ['distancia_km' => $datos['distancia_km'], 'activo' => true]
            );
        }
    }
}

5 UserSeeder: usuarios con roles y contraseñas seguras

database/seeders/UserSeeder.php
use Illuminate\Support\Facades\Hash;

class UserSeeder extends Seeder
{
    public function run(): void
    {
        $centro = CentroSalud::where('codigo_snis', '300183')->first();

        // ─── Superadmin (sin centro asignado, ve todo) ───
        $superadmin = User::firstOrCreate(
            ['usuario' => 'superadmin'],
            [
                'name'            => 'Administrador',
                'apellidos'       => 'Sistema',
                'email'           => 'admin@sicsal.bo',
                'password'        => Hash::make(
                    env('SEED_SUPERADMIN_PASS', 'Cambiar123!')
                ),                                     // ← Contraseña del .env
                'centro_salud_id' => null,             // ← Sin centro = ve todo
                'activo'          => true,
            ]
        );
        $superadmin->assignRole('superadmin');       // Spatie: asignar rol

        // ─── Admin del centro (médico responsable) ───
        $admin = User::firstOrCreate(
            ['usuario' => 'medico.hornoma'],
            [
                'name'            => 'Eusebio',
                'apellidos'       => 'Panozo Franco',
                'password'        => Hash::make(
                    env('SEED_ADMIN_PASS', 'Cambiar123!')
                ),
                'centro_salud_id' => $centro->id,       // ← Atado a SU centro
                'activo'          => true,
            ]
        );
        $admin->assignRole('admin');

        // ─── Registrador (personal de estadística) ───
        $registrador = User::firstOrCreate(
            ['usuario' => 'registrador.hornoma'],
            [
                'name'            => 'María',
                'apellidos'       => 'Quispe Mamani',
                'password'        => Hash::make(
                    env('SEED_REGISTRADOR_PASS', 'Cambiar123!')
                ),
                'centro_salud_id' => $centro->id,
                'activo'          => true,
            ]
        );
        $registrador->assignRole('registrador');
    }
}
Seguridad: contraseñas del .env

Las contraseñas NO están en el código. Usamos env('SEED_SUPERADMIN_PASS', 'Cambiar123!'): lee la contraseña del .env, y usa 'Cambiar123!' como respaldo. En producción, el .env debe tener contraseñas fuertes y únicas.

¿Qué es Hash::make()?

Hash::make() convierte la contraseña en texto a un hash irreversible usando bcrypt. En la base de datos se guarda algo como $2y$10$xk3..., no la contraseña real. Incluso si alguien roba la BD, no puede ver las contraseñas.

6 MetaIneSeeder: datos de población

Las metas INE son datos de población por grupo etáreo que proporciona el Instituto Nacional de Estadística. Son la base para calcular coberturas:

database/seeders/MetaIneSeeder.php — Fragmento
$centro = CentroSalud::where('codigo_snis', '300183')->first();

$metas = [
    ['grupo_etareo' => 'menor_1',             'sexo' => 'ambos', 'cantidad' => 12],
    ['grupo_etareo' => '1_4',                  'sexo' => 'ambos', 'cantidad' => 50],
    ['grupo_etareo' => 'menor_5',             'sexo' => 'ambos', 'cantidad' => 62],
    ['grupo_etareo' => 'embarazos_esperados', 'sexo' => 'F',     'cantidad' => 15],
    ['grupo_etareo' => 'partos_esperados',    'sexo' => 'F',     'cantidad' => 12],
    ['grupo_etareo' => 'mef_15_40',           'sexo' => 'F',     'cantidad' => 153],
    // ... 25 grupos en total
];

foreach ($metas as $meta) {
    MetaIne::firstOrCreate(
        [
            'centro_salud_id' => $centro->id,
            'anio'            => 2026,
            'grupo_etareo'    => $meta['grupo_etareo'],
            'sexo'            => $meta['sexo'],
        ],
        ['cantidad' => $meta['cantidad']]
    );
}
¿Para qué sirven las metas INE?

Se usan para calcular coberturas. Por ejemplo: si el INE dice que hay 12 menores de 1 año y vacunamos a 10, la cobertura de BCG es 10/12 × 100 = 83%. Sin metas INE, no podemos medir el rendimiento del centro de salud.

7 firstOrCreate: la técnica idempotente

Todos los seeders de SICSAL usan firstOrCreate() en vez de create(). ¿Por qué?

❌ create() — falla si ya existe
Role::create(['name' => 'superadmin']);
// Si ejecutas 2 veces → ERROR de duplicado
✅ firstOrCreate() — seguro
Role::firstOrCreate(['name' => 'superadmin']);
// Si ya existe → lo devuelve sin error
// Si no existe → lo crea
Cómo funciona firstOrCreate()
Modelo::firstOrCreate(
    ['campo_busqueda' => 'valor'],    // 1° BUSCAR por estos campos
    ['campo_extra'    => 'valor2']    // 2° CREAR con estos datos si no se encontró
);

// Equivale a:
// "Busca un registro donde campo_busqueda = valor.
//  Si lo encuentras, devuélvelo.
//  Si NO, créalo con ambos arrays combinados."
Idempotente

Un proceso es idempotente cuando puedes ejecutarlo muchas veces y el resultado es siempre el mismo. Con firstOrCreate, puedes correr php artisan db:seed 10 veces sin problemas — no duplicará datos.

8 Comandos artisan para seeders

Terminal
# Ejecutar todos los seeders (DatabaseSeeder)
$ php artisan db:seed

# Ejecutar un seeder específico
$ php artisan db:seed --class=UserSeeder

# Borrar todo + migrar + seedear (desarrollo)
$ php artisan migrate:fresh --seed

# Crear un nuevo seeder
$ php artisan make:seeder NombreSeeder

Resumen de la Lección 05

  • Los seeders insertan datos iniciales que el sistema necesita para funcionar.
  • DatabaseSeeder orquesta el orden de ejecución de los demás seeders.
  • SICSAL tiene 6 seeders: Roles → Municipio → Centro → Comunidad → MetasINE → Users.
  • Las contraseñas se leen del .env con env('VAR', 'fallback') — nunca hardcodeadas.
  • firstOrCreate() hace los seeders idempotentes — seguros de ejecutar múltiples veces.
  • assignRole() de Spatie Permission asigna roles a usuarios — lo veremos en la Lección 07.
