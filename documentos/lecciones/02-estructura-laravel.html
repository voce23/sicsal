Regla de oro

El 90% de tu tiempo lo pasarás en estas carpetas: app/, database/, resources/ y routes/. Las demás carpetas rara vez necesitas modificarlas.

3 Rutas: ¿Cómo llega una petición?

Cuando escribes http://sicsal.test/poblacion en el navegador, Laravel busca en routes/web.php qué código debe ejecutar para esa URL. Las rutas son como un mapa que conecta URLs con código PHP.

routes/web.php — Ejemplos de rutas en SICSAL
use App\Livewire\Poblacion;
use App\Livewire\InformeCAI;
use Illuminate\Support\Facades\Route;

// Ruta pública — cualquiera puede acceder
Route::get('/', function () {
    return view('welcome');
});

// Rutas protegidas — solo usuarios autenticados y aprobados
Route::middleware(['auth', 'aprobado'])->group(function () {
    Route::get('/poblacion',     Poblacion::class);
    Route::get('/informe-cai',   InformeCAI::class);
});
Leyendo una ruta

Route::get('/poblacion', Poblacion::class) se lee así:
"Cuando alguien visite /poblacion con el método GET, ejecuta el componente Poblacion".

4 El archivo bootstrap/app.php

Este es el corazón de la aplicación. Es donde Laravel se enciende y configura todo. En Laravel 13 este archivo es muy corto y elegante:

bootstrap/app.php — Configuración de arranque de SICSAL
<?php

use Illuminate\Foundation\Application;
use Illuminate\Foundation\Configuration\Exceptions;
use Illuminate\Foundation\Configuration\Middleware;

return Application::configure(basePath: dirname(__DIR__))
    ->withRouting(
        web:      __DIR__.'/../routes/web.php',        // ← Rutas web
        commands: __DIR__.'/../routes/console.php',    // ← Comandos artisan
        health:   '/up',                              // ← Ruta de salud
    )
    ->withMiddleware(function (Middleware $middleware): void {
        $middleware->alias([
            'aprobado' => \App\Http\Middleware\RequireAprobado::class,
        ]);
    })
    ->withExceptions(function (Exceptions $exceptions): void {
        // Personalizar manejo de errores aquí
    })->create();
¿Qué hace cada parte?
  • withRouting() — Le dice a Laravel dónde están los archivos de rutas.
  • withMiddleware() — Registra middleware personalizados. SICSAL usa aprobado para verificar que el admin aprobó al usuario.
  • withExceptions() — Permite personalizar cómo se muestran los errores.

Laravel 13 también usa bootstrap/providers.php para registrar los Service Providers (clases que configuran servicios):

bootstrap/providers.php — Providers de SICSAL
<?php

return [
    App\Providers\AppServiceProvider::class,         // Tu provider general
    App\Providers\Filament\AdminPanelProvider::class, // Configura el panel Filament
    App\Providers\FortifyServiceProvider::class,     // Configura la autenticación
];

5 Configuración y .env

Laravel separa la configuración en dos capas:

📄 Archivos config/*.php

Archivos PHP en la carpeta config/. Definen opciones con valores por defecto que leen del .env.

config/app.php — Ejemplo
'name' => env('APP_NAME', 'Laravel'),
// Lee APP_NAME del .env
// Si no existe, usa 'Laravel'
🔒 Archivo .env

Archivo de texto plano con valores que cambian según el servidor (contraseñas, URLs, modo debug). NUNCA se sube al repositorio.

.env — Ejemplo
APP_NAME=SICSAL
APP_ENV=local
DB_DATABASE=sicsal

Archivos de configuración importantes en SICSAL:

ArchivoQué configura
config/app.phpNombre, URL, zona horaria, idioma
config/database.phpConexión a MySQL (lee del .env)
config/auth.phpAutenticación: guards y providers
config/permission.phpConfiguración de Spatie Permission
config/fortify.phpOpciones de login y registro
config/filesystems.phpDónde guardar archivos subidos
El archivo .env.example

El repositorio incluye .env.example como plantilla. Cuando clonas un proyecto, copias este archivo a .env y llenas los valores reales. Nunca pongas datos sensibles en .env.example.

6 Assets: Vite, CSS y JavaScript

Vite es la herramienta que compila tus archivos CSS y JavaScript. En SICSAL, Vite procesa Tailwind CSS y los scripts de Chart.js.

vite.config.js — Configuración de Vite en SICSAL
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
    plugins: [
        laravel({
            input: [
                'resources/css/app.css',               // CSS principal
                'resources/js/app.js',                 // JS principal
                'resources/css/filament/admin/theme.css' // Tema Filament
            ],
            refresh: true,  // Recarga al cambiar Blade
        }),
        tailwindcss(),  // Compila Tailwind CSS
    ],
});

Dos modos de Vite
  • npm run dev — Modo desarrollo. Vigila cambios y recarga el navegador automáticamente. Usa un servidor temporal en localhost:5173.
  • npm run build — Modo producción. Crea archivos optimizados y minificados en public/build/.

7 El comando artisan

artisan es la herramienta de línea de comandos de Laravel. Es como un cuchillo suizo — tiene un comando para casi todo. Se ejecuta desde la raíz del proyecto.

Terminal — Comandos artisan más usados
# Ver todos los comandos disponibles
$ php artisan list

# Información de la aplicación
$ php artisan about

# Crear tablas en la base de datos
$ php artisan migrate

# Poblar con datos de prueba
$ php artisan db:seed

# Crear un modelo nuevo
$ php artisan make:model NombreModelo -m

# Limpiar toda la caché
$ php artisan optimize:clear

# Consola interactiva (probar código PHP)
$ php artisan tinker

SICSAL también tiene comandos personalizados para importar datos:

Terminal — Comandos personalizados de SICSAL
# Importar datos del SNIS
$ php artisan importar:snis

# Importar censo
$ php artisan importar:censo

# Importar causas de consulta
$ php artisan importar:causas-consulta

# Transferir datos de SOAPS
$ php artisan transferir:soaps
¿Cómo se crean esos comandos?

Cada comando personalizado es una clase PHP en app/Console/Commands/. Lo veremos en detalle en la Lección 15 cuando hagamos la importación de archivos .ves.

8 Ciclo de vida de una petición

¿Qué pasa exactamente cuando escribes http://sicsal.test/poblacion y presionas Enter? Este es el viaje completo:

🌐
Navegador envía petición GET /poblacion
▼
🌐
Apache (Laragon) recibe → envía a public/index.php
▼
⚙️
bootstrap/app.php se ejecuta → Laravel arranca
▼
🛤️
routes/web.php → ¿Cuál ruta coincide con /poblacion?
▼
🛡️
Middleware 'auth' + 'aprobado' → ¿Tiene permisos?
▼
⚡
Componente Livewire Poblacion::class se ejecuta
▼
🗃️
Eloquent consulta la base de datos MySQL
▼
🖥️
Blade renderiza el HTML → respuesta al navegador
Para recordar

El flujo siempre es: Petición → Ruta → Middleware → Controlador/Livewire → Modelo → Vista → Respuesta. Con el tiempo este flujo se volverá natural.

Resumen de la Lección 02

  • La carpeta app/ contiene tu código: Models, Controllers, Livewire, Filament, Observers, etc.
  • Las rutas (routes/web.php) conectan URLs con código PHP.
  • bootstrap/app.php arranca Laravel y registra middleware y rutas.
  • La configuración usa dos capas: archivos config/*.php + variables en .env.
  • Vite compila CSS (Tailwind) y JS para el navegador.
  • artisan es tu herramienta de terminal para todo: migraciones, modelos, seeders, limpieza.
  • Una petición web recorre: Request → Route → Middleware → Controller → Model → View → Response.
