1 ¿Qué es Filament?

Filament es un framework para construir paneles de administración en Laravel. Con él, descríbes tu interfaz en PHP puro — sin escribir HTML, CSS ni JavaScript. Filament genera automáticamente:

Tablas
Columnas, búsqueda, filtros, ordenamiento, paginación, acciones masivas
Formularios
Campos de texto, selects, datepickers, toggles, validación automática
Navegación
Menú lateral con grupos, iconos, orden personalizado
CRUD
Create, Read, Update, Delete — todo conectado al modelo Eloquent
SICSAL usa Filament 5

La versión más reciente, construida sobre Livewire. Todo es reactivo sin recargar la página. Los formularios se validan en tiempo real.

3 El Resource principal

app/Filament/Resources/CentroSaluds/CentroSaludResource.php
class CentroSaludResource extends Resource
{
    // ─── 1. Modelo Eloquent asociado ───
    protected static ?string $model = CentroSalud::class;

    // ─── 2. Navegación del panel lateral ───
    protected static $navigationIcon  = Heroicon::OutlinedBuildingOffice2;
    protected static $navigationGroup = 'Configuración';
    protected static $navigationLabel = 'Centros de Salud';
    protected static $navigationSort  = 1;

    // ─── 3. Etiquetas en español ───
    protected static ?string $modelLabel       = 'Centro de Salud';
    protected static ?string $pluralModelLabel = 'Centros de Salud';
    protected static ?string $recordTitleAttribute = 'nombre';

    // ─── 4. Solo superadmin puede ver este Resource ───
    public static function canAccess(): bool
    {
        return auth()->user()?->hasRole('superadmin') ?? false;
    }

    // ─── 5. Conectar formulario, tabla y páginas ───
    public static function form(Schema $schema): Schema
    {
        return CentroSaludForm::configure($schema);
    }

    public static function table(Table $table): Table
    {
        return CentroSaludsTable::configure($table);
    }

    public static function getPages(): array
    {
        return [
            'index'  => ListCentroSaluds::route('/'),
            'create' => CreateCentroSalud::route('/create'),
            'edit'   => EditCentroSalud::route('/{record}/edit'),
        ];
    }
}
Todo es declarativo

No escribes HTML. Declaras: "mi modelo es CentroSalud", "va en el grupo Configuración", "usa este icono". Filament construye toda la interfaz por ti.

4 Las páginas: List, Create, Edit

Cada Resource tiene 3 páginas estándar. La mayoría son simples:

Pages/ListCentroSaluds.php — Listado con botón "Nuevo"
class ListCentroSaluds extends ListRecords
{
    protected static string $resource = CentroSaludResource::class;

    protected function getHeaderActions(): array
    {
        return [CreateAction::make()];
    }
}

En el caso de Personas, la página de creación hace algo especial — asigna automáticamente el centro del usuario:

Pages/CreatePersona.php — Asignar centro automáticamente
class CreatePersona extends CreateRecord
{
    protected static string $resource = PersonaResource::class;

    // Antes de guardar, agregar el centro del usuario
    protected function mutateFormDataBeforeCreate(array $data): array
    {
        $data['centro_salud_id'] = auth()->user()->centro_salud_id;
        return $data;
    }
}
mutateFormDataBeforeCreate

Este hook se ejecuta justo antes de guardar el registro nuevo. Es ideal para agregar datos que el usuario no debe llenar manualmente (como centro_salud_id o user_id).

5 La tabla: columnas, filtros y acciones

Tables/CentroSaludsTable.php — Ejemplo simple
class CentroSaludsTable
{
    public static function configure(Table $table): Table
    {
        return $table
            ->columns([
                TextColumn::make('nombre')
                    ->searchable()->sortable()
                    ->label('Nombre'),

                TextColumn::make('codigo_snis')
                    ->searchable()
                    ->label('Código SNIS'),

                TextColumn::make('municipio.nombre')  // ← relación
                    ->searchable()->sortable()
                    ->label('Municipio'),

                TextColumn::make('subsector')
                    ->badge()                  // ← Muestra como etiqueta
                    ->label('Subsector'),

                TextColumn::make('poblacion_ine')
                    ->numeric()->sortable()
                    ->label('Pob. INE'),

                IconColumn::make('activo')
                    ->boolean()                // ← ✅ o ❌ automático
                    ->label('Activo'),
            ])
            ->recordActions([
                EditAction::make(),
            ]);
    }
}

La tabla de Personas es más compleja — incluye filtros, colores por estado y acciones masivas:

Action::make('verificar_rapido')
    ->label('Verificar')
    ->icon('heroicon-m-check-badge')
    ->color('success')
    ->visible(fn ($record) => !$record->verificado)

            'verificado'         => true,

6 El formulario: campos y secciones

Schemas/CentroSaludForm.php — Formulario con secciones
class CentroSaludForm
{
    public static function configure(Schema $schema): Schema
    {
        return $schema->components([

            Section::make('Información General')->columns(2)->schema([

                Select::make('municipio_id')
                    ->relationship('municipio', 'nombre')
                    ->searchable()->preload()->required()
                    ->label('Municipio'),

                TextInput::make('nombre')
                    ->required()->maxLength(150)
                    ->label('Nombre del centro'),

                TextInput::make('codigo_snis')
                    ->maxLength(20)
                    ->label('Código SNIS'),

                Toggle::make('activo')
                    ->default(true)
                    ->label('Activo')->columnSpan(2),
            ]),

            Section::make('Ubicación Geográfica')->columns(2)->schema([
                TextInput::make('latitud')
                    ->numeric()
                    ->rules(['nullable', 'between:-90,90'])
                    ->label('Latitud'),

                TextInput::make('longitud')
                    ->numeric()
                    ->rules(['nullable', 'between:-180,180'])
                    ->label('Longitud'),
            ]),
        ]);
    }
}

7 Control de acceso por rol

Hay dos patrones para controlar quién ve qué en Filament:

Patrón 1: canAccess() — ocultar todo el Resource
// CentroSaludResource: SOLO superadmin ve el menú "Centros de Salud"
public static function canAccess(): bool
{
    return auth()->user()?->hasRole('superadmin') ?? false;
}
Patrón 2: getEloquentQuery() — filtrar datos por centro
// PersonaResource: todos ven "Personas", pero admin/registrador
// solo ven las de SU centro
public static function getEloquentQuery(): Builder
{
    $query = parent::getEloquentQuery();
    $user  = auth()->user();

    if ($user && !$user->hasRole('superadmin')) {
        $query->where('centro_salud_id', $user->centro_salud_id);
    }
    return $query;
}

Resumen de la Lección 08

  • Un Filament Resource genera un CRUD completo en PHP puro — sin HTML.
  • Estructura: Resource (principal) + Pages/ (listado, crear, editar) + Schemas/ (formulario) + Tables/ (tabla).
  • Propiedades como $model, $navigationGroup, $navigationIcon configuran la navegación.
  • canAccess() oculta Resources completos según el rol.
  • getEloquentQuery() filtra datos para que cada usuario solo vea su centro.
  • mutateFormDataBeforeCreate() inyecta datos automáticos antes de guardar.
