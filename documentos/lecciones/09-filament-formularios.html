1 Anatomía de un formulario

En Filament, un formulario se construye con Section y componentes de campo. Todo es PHP — sin HTML:

Estructura básica
return $schema->components([

    Section::make('Nombre del grupo')
        ->columns(2)                // 2 columnas lado a lado
        ->schema([
            TextInput::make('campo1')
                ->required()
                ->label('Etiqueta'),

            Select::make('campo2')
                ->options([...])
                ->label('Otro campo'),
        ]),

    Section::make('Otro grupo')->schema([...]),

]);
Section = agrupación visual

Cada Section crea una caja con título. Los campos dentro pueden estar en 1, 2 o 3 columnas con ->columns(N). Un campo puede ocupar toda la fila con ->columnSpanFull().

2 Campos básicos: TextInput, Select, DatePicker

Schemas/PersonaForm.php — Datos personales
Section::make('Datos personales')->columns(2)->schema([

    TextInput::make('nombres')
        ->required()              // No puede estar vacío
        ->maxLength(100)          // Máximo 100 caracteres
        ->label('Nombres'),

    DatePicker::make('fecha_nacimiento')
        ->required()
        ->maxDate(now())          // No puede ser fecha futura
        ->label('Fecha de nacimiento'),

    Radio::make('sexo')
        ->options(['M' => 'Masculino', 'F' => 'Femenino'])
        ->required()
        ->inline()               // Opciones en línea horizontal
        ->label('Sexo'),

    TextInput::make('ci')
        ->maxLength(20)           // Opcional (sin required)
        ->label('Cédula de identidad'),
]),
MétodoQué hace
->required()Campo obligatorio — muestra asterisco rojo
->maxLength(N)Límite de caracteres
->maxDate(now())Fecha máxima seleccionable
->default('valor')Valor predeterminado
->placeholder('...')Texto guía dentro del campo
->label('...')Etiqueta visible del campo
->helperText('...')Texto de ayuda debajo del campo

3 Campos condicionales con live()

Un campo puede mostrar u ocultar otros campos cuando cambia su valor. Esto se hace con ->live() y ->hidden():

Schemas/PersonaForm.php — Mostrar campos de migración solo si estado = migrado
Section::make('Estado migratorio')->columns(2)->schema([

    Select::make('estado')
        ->options([
            'residente' => 'Residente',
            'temporal'  => 'Temporal',
            'migrado'   => 'Migrado',
            'fallecido' => 'Fallecido',
        ])
        ->default('residente')
        ->required()
        ->live()                    // ← Reactivo: notifica cambios al servidor
        ->label('Estado'),

    // Este campo solo aparece cuando estado = 'migrado'
    TextInput::make('destino_migracion')
        ->maxLength(150)
        ->label('Destino de migración')
        ->hidden(fn ($get) => $get('estado') !== 'migrado'),
                // ↑ $get() lee el valor actual de otro campo

    DatePicker::make('fecha_migracion')
        ->label('Fecha de migración')
        ->hidden(fn ($get) => $get('estado') !== 'migrado'),
]),

Usuario selecciona "Estado: Migrado"

Otro ejemplo: el toggle de verificación auto-rellena la fecha:

Schemas/PersonaForm.php — Toggle que auto-rellena fecha
Toggle::make('verificado')
    ->label('Persona verificada')
    ->live()
    ->afterStateUpdated(function ($state, callable $set) {
        if ($state) {
            $set('fecha_verificacion', today()->format('Y-m-d'));
            $set('verificado_por', auth()->id());
        }
    }),
// $set() modifica el valor de otro campo del formulario

4 Select con relación y búsqueda

Select simple con relationship()
// Carga todas las comunidades del centro del usuario
Select::make('comunidad_id')
    ->relationship(
        'comunidad',              // Nombre de la relación en el modelo
        'nombre',                 // Campo a mostrar
        fn ($query) => $query->where(
            'centro_salud_id',
            auth()->user()?->centro_salud_id
        )                            // ← Filtrar por centro del usuario
    )
    ->searchable()                // El usuario puede escribir para buscar
    ->preload()                   // Cargar opciones al abrir
    ->required()
    ->label('Comunidad'),

5 Auto-rellenar campos desde otro modelo

En el formulario de Defunciones, al seleccionar una persona del padrón, sus datos se copian automáticamente:

Schemas/DefuncionForm.php — Búsqueda y auto-fill
Select::make('persona_id')
    ->label('Buscar en el padrón (opcional)')
    ->searchable()
    ->getSearchResultsUsing(
        fn (string $search) =>
            Persona::query()
                ->where('centro_salud_id', auth()->user()?->centro_salud_id)
                ->where(function ($q) use ($search) {
                    $q->where('nombres', 'like', "%{$search}%")
                      ->orWhere('apellidos', 'like', "%{$search}%");
                })
                ->limit(20)->get()
                ->mapWithKeys(fn ($p) =>
                    [$p->id => "{$p->nombres} {$p->apellidos}"]
                )->toArray()
    )
    ->live()
    ->afterStateUpdated(function ($state, Set $set) {
        if (!$state) return;
        $persona = Persona::find($state);
        if (!$persona) return;

        // Auto-rellenar campos del formulario
        $set('nombres',          $persona->nombres);
        $set('apellidos',         $persona->apellidos);
        $set('fecha_nacimiento',  $persona->fecha_nacimiento);
        $set('sexo',              $persona->sexo);
        $set('comunidad_id',      $persona->comunidad_id);
    }),
Patrón: Select → afterStateUpdated → $set

Este patrón se repite en SICSAL: cuando el usuario selecciona algo en un Select, afterStateUpdated se ejecuta y usa $set() para llenar otros campos del formulario automáticamente.

6 Contraseña condicional (crear vs editar)

En el formulario de usuarios, la contraseña es obligatoria al crear pero opcional al editar:

Schemas/UserForm.php — Contraseña inteligente
TextInput::make('password')
    ->password()->revealable()

    // Obligatorio solo al crear, no al editar
    ->required(fn (string $operation) =>
        $operation === 'create'
    )

    // Solo guardar si el usuario escribió algo
    ->dehydrated(fn (?string $state) =>
        filled($state)
    )

    ->minLength(8)
    ->label(fn (string $operation) =>
        $operation === 'create'
            ? 'Contraseña'
            : 'Nueva contraseña (vacío = no cambiar)'
    ),
MétodoQué hace
->password()Muestra asteriscos (campo de contraseña)
->revealable()Botón de ojo para ver la contraseña
->required(fn ($operation) => ...)Requerido condicionalmente
->dehydrated(fn ($state) => ...)Solo envía al servidor si tiene valor
->unique(ignoreRecord: true)Único en la tabla, excepto el registro actual

7 Validación integrada

Filament valida automáticamente según los métodos encadenados. También puedes agregar reglas personalizadas:

Validación automática + personalizada
// Validación automática por encadenamiento
TextInput::make('email')
    ->email()             // Valida formato email
    ->required()          // No puede estar vacío
    ->maxLength(255)       // Máximo 255 caracteres
    ->unique(ignoreRecord: true),  // Único en la tabla

// Validación personalizada con rules()
TextInput::make('latitud')
    ->numeric()
    ->rules(['nullable', 'between:-90,90']),  // Rango de latitud GPS
Validación en tiempo real

Gracias a Livewire, los errores de validación aparecen sin recargar la página. Si escribes un email inválido, el mensaje de error aparece instantáneamente debajo del campo.

Resumen de la Lección 09

  • Los formularios Filament se construyen encadenando métodos en PHP — sin HTML.
  • Section agrupa campos visualmente; ->columns(2) los pone lado a lado.
  • ->live() hace un campo reactivo para mostrar/ocultar otros con ->hidden(fn ($get) => ...).
  • afterStateUpdated + $set() auto-rellena campos desde otro modelo.
  • La contraseña puede ser condicional con ->required(fn ($operation) => ...).
  • La validación es automática y en tiempo real gracias a Livewire.
